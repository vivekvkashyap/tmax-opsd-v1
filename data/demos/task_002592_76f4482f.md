Build and test each layer in isolation before wiring them into `audit_tool.c`. First set up the PKI with standard OpenSSL CA workflow commands (`openssl genrsa`/`req -x509` for the root, `req` + `x509 -req -CA rootCA... ` for the intermediate, then another leaf cert signed by the intermediate for `client_valid`, and a completely separate throwaway root+leaf for `client_invalid`), concatenating root+intermediate into `ca-chain.pem` in that order. In C, use OpenSSL's `X509_STORE`/`X509_STORE_CTX` verify API (load the chain into a store, verify the client cert against it) rather than hand-rolling signature checks — test this against both the valid and invalid client certs standalone first. For the JWT check, split the Authorization header token on `.`, base64url-decode (careful: JWT base64url omits padding, so pad it before feeding to a standard base64 decoder) the header segment, and parse/search the resulting JSON text for `"alg":"none"`. XSS detection is a simple substring search for `<script>` across the whole request file. Order matters: check cert first, then algorithm, then XSS, then pass — get this ordering exactly right since the log format depends on which check fails first. Compile with `-lssl -lcrypto`, then build `run_audit.sh` to generate the four request files and invoke `audit_tool` with the right cert for each test, appending output in the specified format.
