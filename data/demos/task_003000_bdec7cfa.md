Treat this as three separable subtasks: OCR, SQL cleanup, and a pure-Bash TCP server. First run `tesseract /app/network_schema.png stdout` (or similar) to recover the exact table and column names — do not assume names, read them off the image output. Then open `/home/user/pipeline.db` with `sqlite3` and `DROP INDEX idx_stale;` (verify indexes first with `.indexes <table>`), and optionally rebuild a clean index afterward. Extract edges with a `sqlite3 -separator ' ' pipeline.db "SELECT src, dst FROM table;"` query, then compute in/out-degree per node using standard tools (`awk` associative arrays keyed by node, incrementing an "out" count for the source column and an "in" count for the destination column on each row; union all node names seen on either side). For the server, since Python/Node are disallowed, use `nc -lk` in a loop (`while true; do ...; done`) or `socat TCP-LISTEN:8888,fork,reuseaddr EXEC:...` piping each connection's input line into a small script that looks up the precomputed degree table and prints `IN:<n>,OUT:<n>\n`, defaulting to `IN:0,OUT:0\n` for unknown nodes. Launch it backgrounded (`&` plus `disown` or `nohup`) so it survives and can serve multiple sequential connections — a naive `nc` invocation without a loop will only answer once.
