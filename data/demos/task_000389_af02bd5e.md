Build this as a strict linear pipeline, checking intermediate outputs at each stage. First, validate: load `schema.json`, iterate `dataset.jsonl` line by line, and use `jsonschema.validate` in a try/except per record, appending only records that pass to `valid_data.jsonl` in original order (a record missing a required field or with wrong types for `features`/`recovery_time` should be silently dropped, not crash the script). Second, retrieval: fit `TfidfVectorizer(stop_words='english')` on the valid records' `message` fields, transform the fixed query string with the *same fitted* vectorizer (do not refit on the query), compute cosine similarity (`sklearn.metrics.pairwise.cosine_similarity`) between the query vector and every message vector, then take the top 50 by similarity using a stable sort so ties preserve original order (e.g. `sorted(..., key=lambda i: -sim[i])` is stable in Python, or use `np.argsort` with a stable kind). Third, on exactly those 50 records extract `features` as X (2 columns) and `recovery_time` as y, fit `BayesianRidge()` with defaults, predict on the same X, and compute MSE with `sklearn.metrics.mean_squared_error`. Finally write `metrics.json` and `weights.txt` (coefficients only, no intercept, 4-decimal comma-separated) into a fresh `artifact/` directory, then `tar -czf /home/user/artifact.tar.gz -C /home/user/mlops artifact` so extraction yields `artifact/` at the top level.
