Once you have a fuzzed input that reliably triggers the failure, wrap it in a bisection test script whose exit code `git bisect run` can use: it must `exit 0` when `./build_report.sh <that-input>` succeeds and a nonzero code (but not 125, which means "skip") when it fails or errors, and must also handle commits where the script itself might not exist/be executable yet. Run `git bisect start`, mark `HEAD` bad and tag `v1.0` good, then `git bisect run <your-wrapper-script>`. Capture the reported first-bad commit with `git rev-parse` (or note it from bisect's output) and write the full 40-character hash to `/home/user/bad_commit.txt`, then `git bisect reset` to restore the working tree without altering history.
