This is a multi-stage systems task — build and validate each stage independently before wiring them together. Stage 1: use `ffmpeg` to extract frames (e.g. `-vf select` or dump all frames to PNGs), then script a check per frame for average pixel color meeting R>200, G<50, B<50 (OpenCV/PIL, computing the mean of each channel across the frame, not per-pixel) to count red-flash frames; derive the congestion multiplier from that count rather than assuming a value. Stage 2: stand up Postgres (Docker or local service), create the `logistics` DB and `routes` table matching the exact column names/types given, `\copy` or `COPY` the CSV in, and add indexes on `source_node` and `target_node` (composite or separate) before writing Rust — a missing index is a directly checked requirement, not just a performance nicety. Use `sqlx`/`postgres` crate with parameterized queries (`$1`) for updates, never string-interpolated SQL. Stage 3: build the graph in Rust (e.g. `petgraph`) from the updated costs, run Dijkstra for the shortest path and compute degree centrality (count of edges per node), then serialize to the exact JSON shape given. Stage 4: the sanitizer must reject any query_filter containing SQL injection markers (`DROP TABLE`, `1=1`) or path traversal (`../`) with exit 1, and accept everything else with exit 0 — test it against both provided corpora before finishing, since it's graded on 100% classification accuracy on held-out clean/evil sets.
