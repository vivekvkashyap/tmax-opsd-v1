Structure this as a linear pipeline in Rust with clearly separated stages so each is verifiable on its own. Tokenize by lowercasing, stripping non-alphabetic characters (keep whitespace as a separator), splitting on whitespace, and dropping empty tokens. Count global token frequency, then select the top 10 by descending count with ties broken alphabetically — implement this as a stable sort on `(-count, word)` rather than relying on hashmap iteration order, which is nondeterministic. For the Bayesian step, treat presence-per-document as Bernoulli, count how many of the N documents contain each top word at least once, and apply the closed-form Beta(2,2) posterior mean `(2 + successes) / (2 + 2 + N)` — no numerical sampling needed there. For bootstrap reproducibility, this is the part naive attempts get wrong: use exactly `rand::rngs::StdRng::seed_from_u64(42)`, and for each of the 1000 resamples call `rng.gen_range(0..N)` exactly N times in strict sequence to build the resampled document index list — any deviation in call order or count changes every downstream number. Recompute posterior means per resample with the same top-10 words fixed from the original run, average across the 1000 resamples, and round all output floats to 4 decimals before writing the JSON.
