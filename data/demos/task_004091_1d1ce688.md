Automate the whole bisection rather than checking commits by hand. First write a self-contained test script (e.g. `test_tz.sh`) that: runs `make` (build failures should count as "skip", exit 125), then runs `strace -f -e trace=open,openat,newfstatat ./event_logger` and greps the trace for whether it actually opens `/etc/localtime` — exit 0 if it does, exit 1 if it opens some other/misspelled path or fails to open it at all. The program's fallback is silent (no stdout/exit-code signal), so tracing syscalls is the only reliable signal — don't rely on exit code or stdout. Find the root commit with `git rev-list --max-parents=0 HEAD`, then run `git bisect start`, mark `HEAD` bad and the root commit good, and hand off to `git bisect run ./test_tz.sh`. Between checkouts, make sure stale build artifacts don't fool your test (`make clean` first if needed, or rebuild unconditionally). After bisect finishes, read the first-bad commit hash and write the full 40-char SHA to the output file — do not modify history (no rebase/reset), and always end with `git bisect reset`.
