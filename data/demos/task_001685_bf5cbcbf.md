The repo at `/app/log_repo` is a red herring you can skim or skip — focus on the two real deliverables: recovering the threshold from the image and writing a pure-bash classifier. Use `tesseract /app/specs.png stdout` (or similar OCR invocation) to read the engineering spec text out of the image and locate the numeric "max safe value" it states — don't guess a round number, extract the actual printed digits including any decimal precision. Build `/home/user/filter.sh` as two independent checks that both must pass: (1) an encoding check using `iconv -f UTF-8 -t UTF-8 "$FILE" >/dev/null 2>&1` (or `file` reporting non-ASCII/binary) to catch invalid byte sequences, exiting 1 immediately on failure; (2) a numeric check using `awk`/`bc` comparing every numeric reading in the file against the OCR'd threshold with a strict `>` (not `>=`) since the spec says "strictly less than or equal to" is valid — get this boundary condition exactly right or you'll misclassify values equal to the threshold. `chmod +x` the script, then validate it yourself against all 50 files in `/app/clean/` (expect exit 0 for every one) and all 50 in `/app/evil/` (expect exit 1 for every one) before considering it done, since the grader requires 100% on both sets.
