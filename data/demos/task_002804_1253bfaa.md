Design `runner.c` as a straightforward line-by-line interpreter with a single mutable `can_execute` flag (start at 1/true), reset by every `REQUIRE` line — not accumulated with AND across multiple REQUIRE lines, since the spec says each `REQUIRE` independently sets the flag based on its own comparison. Parse each line's first token to dispatch: for `REQUIRE <VAR> <OP> <VALUE>`, use `getenv(VAR)` and treat a NULL/unset var as integer 0 (use `atoi` on both the env value and `<VALUE>`), then evaluate `==`, `>`, or `<` accordingly and update `can_execute`. For `EXEC <rest-of-line>`, only call `system()` on the remainder of the line (everything after the `EXEC ` token, preserving it verbatim including any redirection like `>>`) when `can_execute` is currently 1; otherwise skip without executing. Ignore blank lines or lines not starting with those two keywords. Use `fgets` with a buffer sized for the stated 256-char line limit. For the Makefile, define `all: ci_runner` compiling with exactly `gcc -Wall -Wextra -O2 -o ci_runner runner.c`, plus a `clean: rm -f ci_runner` target.
