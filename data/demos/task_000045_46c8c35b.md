(1) Use `ffprobe -i evidence.mp4` to list streams, find the subtitle stream index, then `ffmpeg -i evidence.mp4 -map 0:s:0 out.srt` to dump it and grep the `C2_SECRET:` line for the key. (2) In `/home/user/suspicious/`, use `find . -perm -4000 -type f` to locate the single SUID binary, then `sha256sum` it. For the webhook, use `socat TCP-LISTEN:8080,bind=127.0.0.1,reuseaddr,fork SYSTEM:...` (or a read loop with `nc`) to parse the raw HTTP request text, extract the `X-Incident-Auth` header case-insensitively, and recompute the expected HMAC yourself with `openssl dgst -sha256 -hmac "$SECRET"` over the attacker IP — strip the `HMAC-SHA256(stdin)=` prefix some openssl versions print, keep only the hex digest, and compare strings exactly. Emit the literal `HTTP/1.1 200 OK\r\n\r\n<hash>` or `403 Forbidden\r\n\r\n` bytes with `printf`, not `echo`, to get exact CRLFs. Launch it backgrounded (`&`) and verify with `curl -v` before finishing.
