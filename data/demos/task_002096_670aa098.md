Implement `IntervalCache` using the classic sort-and-sweep merge algorithm to guarantee O(N log N): store added intervals in a list, and in `get_merged_intervals()` sort by start time, then iterate once maintaining a running `(cur_start, cur_end)` — if the next interval's start is `<= cur_end` (note: touching intervals like (1,3) and (3,5) must merge too, so use `<=` not `<`), extend `cur_end = max(cur_end, next_end)`; otherwise push the current merged pair and start a new one. Avoid the naive O(N^2) trap of repeatedly re-scanning/re-merging the whole list on every `add_interval` call — do the merge lazily inside `get_merged_intervals` instead.
