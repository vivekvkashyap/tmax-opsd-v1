Build and smoke-test each endpoint independently with `curl` before assuming the whole server works end to end, and make sure the process is left running in the background (e.g. via `nohup ... &` or a detached goroutine loop) so the verifier can reach it later. For `/archives/clean`: parse `config.yaml`'s `prefixes` list (a small hand-rolled YAML list parser is fine if you avoid pulling in a YAML library, or use `gopkg.in/yaml.v3` if available), then for each `.tar.gz` in the archives directory attempt to actually decompress and list its contents (e.g. via Go's `archive/tar`+`compress/gzip` readers, reading through to EOF) to detect corruption rather than just checking the gzip magic bytes — a truncated or bit-flipped archive can pass a magic-byte check but fail mid-read. Delete files that error out, then filter survivors by prefix match before returning. For `/logs/errors`: parse `storage.wal` entry-by-entry (split on timestamp lines), and count *unique* `ErrID` values with a set, not just occurrences — a duplicate ErrID appearing in two ERROR entries should count once. For `/video/extract`: use `ffmpeg -ss 00:00:05 -i input -frames:v 1 output.jpg` (seek before input for accuracy), zip only the bare `frame.jpg` (no directory entries/paths), and compute the SHA256 of the resulting zip file, not the frame.
