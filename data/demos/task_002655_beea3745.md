A `pre-receive` hook reads lines of `<old-sha> <new-sha> <ref>` on stdin before the push is finalized, so use `git diff --name-status <old-sha> <new-sha>` (or `git diff-tree -r`) to list changed/added files, filtering to `.json`. To read a file's incoming content without it being checked out yet, use `git cat-file -p <new-sha>:<path>` (or `git show <new-sha>:<path>`) rather than reading from the working tree. Parse each JSON's `image_path`, then get *actual* (allocated) disk usage — not virtual capacity — via `qemu-img info --output=json <path>` (look at the `actual-size` field) or `stat` block count (`st_blocks * 512`); this distinction is the main trap since a sparse/thin-provisioned qcow2 can have a huge virtual size but small actual usage. If any actual size exceeds the byte threshold, print the required violation message, append the exact log line (using the `<new-value>` hash) to the alerts log, and exit non-zero to reject the whole push; otherwise exit 0. Make the file executable (`chmod +x`) and test end-to-end by cloning the bare repo elsewhere, committing both a small and an oversized image reference, and pushing each.
