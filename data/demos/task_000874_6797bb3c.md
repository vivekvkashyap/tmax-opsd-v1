Build this as a log-parsing-and-decryption pipeline, testing each stage in isolation. First, grep the access log for `/login?redirect=` requests and extract the redirect target host with a regex; filter to only those whose host is not `app.local` (careful with substring matches — parse the host properly, e.g. via a URL-aware regex or `sed`, not a naive `grep -v app.local` which could false-positive/negative on subdomains). From the matched lines, extract the `token=` query parameter value up to the next `&` or end of string. For each token, URL-decode if needed, then decrypt using the exact openssl command given, piping the token as input. Parse the decrypted `username:epoch` payload, and keep only entries where epoch as an integer is strictly greater than the given threshold — do a numeric comparison, not string. Collect usernames into a set (to dedupe), sort alphabetically, and write one per line to the output file. Finally chmod the output file to 600 — do this as the last step so earlier failed runs don't leave it locked from your own editor/tools.
