Work in three clean stages so each is independently testable. First, list the `.ini` files in the source directory sorted alphabetically (`sorted(glob(...))` or `ls | sort`), and for each one decode its bytes from UTF-16LE to UTF-8 (e.g. Python `open(f, encoding='utf-16-le').read()` or `iconv -f UTF-16LE -t UTF-8`). Build the merged string by concatenating, for each file in order, the exact header `---FILE:<filename>---\n` followed immediately by the decoded content — do not add or strip any newlines beyond what the header specifies and what's already in the file content; a common mistake is accidentally adding a trailing newline between files or normalizing line endings during decode. Second, chunk the merged UTF-8-encoded bytes (not characters — work on the byte string) into consecutive 50-byte pieces, writing each to `chunk_NN` with zero-padded two-digit sequential numbers starting at `chunk_00`; the last chunk holds the remainder and may be shorter. Third, compute the SHA-256 hex digest of each chunk's raw bytes and write `manifest.txt` with one `<chunk_filename> <hash>` line per chunk in sequential order. Verify by manually reconstructing the merged string length and checking chunk count/sizes before trusting the hashes.
