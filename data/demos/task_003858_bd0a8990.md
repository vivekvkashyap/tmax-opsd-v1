This is a two-part task: recover a hidden signal from a video, then build a robust classifier. For Part 1, extract all 64 frames from the mp4 (e.g. with `ffmpeg -i incident.mp4 frame_%03d.png` or `opencv`/`PIL`), then for each frame sample the top-left 50x50 pixel region and check if it is pure white vs pure black — average the region's pixel values rather than checking a single pixel, since compression artifacts can introduce noise near-white/near-black. Map white->1, black->0 in frame order, group bits into 8 bytes, and convert each byte to its ASCII character to reconstruct the 8-character prefix — do this programmatically, don't guess the string. For Part 2, write `detector.py` so it checks, in this order: (a) absolute path (starts with `/`), (b) null byte (`%00` or literal `\x00`), (c) traversal patterns including URL-encoded variants (`../`, `..%2f`, `%2e%2e%2f`, etc., case-insensitively — consider decoding percent-encoding before matching so encoded variants aren't missed), and (d) presence of the recovered 8-char prefix anywhere in the payload. Print exactly `EVIL` or `CLEAN` with no extra text, and test against sample files covering each malicious category plus benign relative paths before trusting it against the hidden corpus.
