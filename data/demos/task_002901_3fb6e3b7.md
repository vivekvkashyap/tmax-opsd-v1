Build this as a real Unix pipeline, not a single script that does everything. In `filter.c`, read stdin line by line (e.g. `fgets` or `getline`) and for each line test two conditions before deciding to print: does it start with `/`, and does it contain the exact substring `../` anywhere (use `strstr`); if either is true, skip the line, otherwise print it unchanged including its newline. Compile with `gcc -O2 -o /home/user/filter /home/user/filter.c`. For the tar listing, use `tar -tf /home/user/artifacts/update.tar` (list-only, do not extract) — note that entries stored with a literal leading `/` or containing `../` inside the tar are exactly the traversal risks your filter must catch, and paths tar strips leading slashes from on extraction still appear as-is in the listing. Chain everything with pipes: `tar -tf ... | ./filter | sed -E 's/\.o$/.obj/' > /home/user/verified_artifacts.log`. Double check your `sed`/`awk` rule only rewrites a `.o` that's at the very end of the line (anchor with `$`), not `.o` occurring mid-path.
