Treat `/app/embedder` as a black-box CLI you shell out to per text record — don't try to reverse-engineer its internals, just call `/app/embedder "<text>"` from Go (`os/exec`) and parse the comma-separated float output into a `[]float64` per record; since there are 500 records this means 500 subprocess calls, which is fine but consider running them concurrently (goroutines + a worker pool) if it's slow. Implement Euclidean distance as a straightforward function over two 16-dim vectors, then for each record compute distances to all others, sort them ascending, and take the 5th smallest as required (index 4 when index 0 is the record itself at distance 0) — this is an O(n^2) all-pairs computation which is fine at n=500. Filter out any record whose 5th-nearest distance exceeds 2.5. Compute the centroid as the elementwise mean of the remaining vectors' 16 dimensions. Write it as one comma-separated line to `/app/output_centroid.txt`. Since the grader checks MSE against a reference centroid, make sure your CSV parsing, distance/sort logic, and filtering threshold (strictly greater than 2.5) exactly match the spec — off-by-one errors in "5th nearest neighbor including self" are the likely failure mode.
