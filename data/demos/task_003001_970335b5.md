Build this bottom-up: TLS receiver first, then the checker, then automation. Generate the cert/key with `openssl req -x509 -newkey rsa:2048 -keyout key.pem -out cert.pem -days 365 -nodes -subj "/CN=localhost"` in `/home/user/webhook/`. Write `server.py` using Python's `http.server` wrapped with `ssl.SSLContext` (`wrap_socket` or `context.wrap_socket`) loading those two files, handling `POST /alert` by reading `Content-Length` bytes from the request body, appending the raw JSON text plus a newline to `alerts.log`, and responding 200 — start it backgrounded with `nohup python3 server.py &` so it survives your shell exiting. For `monitor.sh`, loop over ports 8001-8003, use `curl -s -o /dev/null -w "%{http_code}" http://127.0.0.1:<port>/health`, and when the code isn't 200 POST the exact JSON payload to `https://127.0.0.1:8443/alert` with `curl -k` (to skip the self-signed cert's validation) — build the payload string precisely matching `{"service": "service_<port>", "status": "down"}`, no extra whitespace. Make the script executable. For cron, write `* * * * * /home/user/monitor.sh` to `cron.conf` and load it with `crontab cron.conf`, then run `monitor.sh` once manually to populate `alerts.log` immediately rather than waiting for the cron tick.
