Get the plumbing correct before worrying about parser edge cases, since the grader drives the whole pipeline end to end via an HTTP upload. Edit `config.ini` to set `queue_type = redis` and `redis_url = redis://127.0.0.1:6379/0`, then start `redis-server`, the Flask app, and the worker via `start_services.sh`, and manually verify with `redis-cli` that an uploaded file actually produces a queue entry before touching the parser. For `elf_parser.py`, use `pyelftools` (`ELFFile` from `elftools.elf.elffile`) to read the machine type (`elffile['e_machine']`), entry point (`elffile['e_entry']`, formatted as hex with `0x` prefix), and the `.text` section's `sh_size` (via `get_section_by_name('.text')`, defaulting to 0 if absent) — print them pipe-joined in exactly the specified order, and `chmod +x` the script. Since correctness is judged against many real system binaries by fuzz-equivalence, test your parser against several files in `/usr/bin`/`/usr/lib` yourself and sanity-check the fields against `readelf -h` output rather than trusting a single example. Finally, update `worker.py` to parse the `<Machine>|<hex>|<size>` line from your script's stdout, split on `|`, make the `curated/<Architecture>/` directory if missing, and `os.symlink` the uploaded file's absolute path into it under the original filename.
