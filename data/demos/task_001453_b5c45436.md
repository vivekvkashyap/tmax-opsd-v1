Chain the extraction/inspection steps carefully, verifying at each stage rather than assuming structure. First `tar -xzf` the outer archive to get `bundle.zip`. Then list the zip's contents without extracting everything — use `zipfile.ZipFile.namelist()` in Python (or `unzip -l`) and scan for an entry whose path starts with `../`; write that exact entry name (as it literally appears in the listing) to the malicious-path file — do not guess it, read it from the archive. Extract only `changelog.log` with `zip.extract('changelog.log', ...)` or `unzip bundle.zip changelog.log`, never `extractall` on an archive known to contain traversal entries. Parse `changelog.log` by splitting on lines that are exactly `---`, then search each resulting block for the target phrase and write that block's lines (minus the separator) to the record file — be careful to strip only the separator lines, not surrounding whitespace that belongs to the record. Finally, read the record bytes, base64-encode them as one continuous string (`base64.b64encode`), then gzip that string's bytes (`gzip.open(..., 'wb')`) to produce the `.gz` file — verify by `zcat`-ing it back and confirming it matches the base64 of the record file.
