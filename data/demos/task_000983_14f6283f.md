Work service-by-service. For the Rust parser, run `cargo build --release` first to see the actual borrow-checker error in `main.rs`: `parse_log` borrows `&sev_part[1..]` and `msg_part.trim()` from `log_line`/`sev_part`/`parts` which are locals dropped when the function returns, but `LogEvent<'a>` tries to hold onto those borrows past serialization — fix by owning the strings (use `String` instead of `&str` in `LogEvent`, e.g. `severity: String`, `message: String`) or by serializing before the borrowed locals go out of scope; do not change the parsing logic, only ownership. For the Python gateway, implement `/ingest` (Flask or stdlib `http.server`) that validates the `Authorization: Bearer <token>` header, uses a Redis INCR+EXPIRE (or sorted-set) sliding-window pattern keyed per token to enforce exactly 5 req/sec (return 429 past that), then opens a raw TCP socket to `127.0.0.1:9090`, sends the body, reads the newline-terminated JSON response, and relays it back with HTTP 200. In `start_all.sh`, start redis-server (or confirm it's running), build the Rust release binary, launch both services backgrounded with stdout/stderr redirected to `app.log`, and exit 0 while leaving them running — verify by curling `/ingest` yourself before finishing.
