Set up a venv and install numpy/scipy/opencv-python/flask. Read `/app/sequencing_droplets.mp4` frame by frame with OpenCV (`cv2.VideoCapture`), and for each grayscale frame compute a normalized 256-bin histogram (`np.histogram(frame, bins=256, range=(0,256), density=False)` then divide by pixel count so it sums to 1). Maintain a running average histogram `A_N` incrementally as you process frames (accumulate sum, divide by N) rather than recomputing from scratch each time, for efficiency. Load `/app/ref_dist.json` and for each N compute `scipy.stats.wasserstein_distance(u_values=np.arange(256), v_values=np.arange(256), u_weights=A_N, v_weights=ref_dist)`; track the previous `W_{N-1}` and stop at the first N≥2 where `abs(W_N - W_{N-1}) < 1e-4`, recording that N as `converged_N` and that distance as `converged_W`. Then in one script, start a Flask/http.server app on `127.0.0.1:8080` serving `GET /convergence` → `{"converged_N": <int>, "wasserstein_distance": <float rounded to 6 dp>}`, and concurrently (via a thread) run a raw `socket` TCP server on `127.0.0.1:9090` that on receiving exactly `PING\n` replies `PONG <converged_N>\n` then closes the connection — keep both listeners running in the background/foreground so the grader can hit them after your analysis finishes.
