Structure the daemon as: load `rules.json` once at startup, then loop (polling `os.listdir` every ~1s is simpler and more reliable in this environment than `watchdog`), processing any file present in `/home/user/incoming/` including ones already there at launch. For `.json` files, load the list-of-dicts, project only the keys listed in `rules.json`'s `json_extract_keys` (in that order) for each row, and write a CSV with those keys as the header. For `.log` files, parse with a small state machine: match the `[YYYY-MM-DD HH:MM:SS] LEVEL` header line to start a new record, then accumulate subsequent lines (joined with a single space, not newlines) into the message until you hit a line that is strictly `===`, at which point the record is complete; emit rows with columns `timestamp,level,message`. For every output, write to a `*.tmp` file in the output directory first, flush/close it, then use `os.rename` (atomic on the same filesystem) to move it to the final `{original_name}.processed.csv` name — never write directly to the final path. Only after the atomic rename succeeds, delete the original file from `/home/user/incoming/`. Test manually by dropping the two example files, waiting a few seconds, and confirming both the CSVs and file deletions happened, then start the script in the background as instructed.
