Model the backups as a forest: parse the CSV, build a map from `id` to its record, and a map from `id` to its `parent_id`. For each backup, walk up the `parent_id` chain to the root `FULL` backup, collecting the size and status of every ancestor along the way (including the node itself). A node is a valid restore point only if every node visited on that walk (self plus all ancestors) has `status == "OK"`; if any is `CORRUPT`, exclude it and skip descendants inheriting from it too — but you don't need special-case propagation logic, since re-walking ancestors for each node naturally invalidates any descendant of a corrupt one. The materialized size is the sum of `size_bytes` over that same walk. Use only `encoding/csv`, `encoding/json`, and `strconv` from the standard library — no third-party modules. After computing all valid points, sort the resulting slice by `id` using `sort.Slice` before marshaling to JSON, and write with `os.WriteFile`/`json.MarshalIndent` to the exact output path. Watch for: parent lookups failing silently on typos (return an error instead), and the empty-string `parent_id` marking a `FULL` root — treat empty string as the stopping condition, not a lookup key. Compile and run with `go run` to produce the file, then inspect it before declaring done.
