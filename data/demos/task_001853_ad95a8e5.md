Get the service topology working end-to-end manually before writing any C++. Run start_services.sh, then edit `/app/coordinator/config.json` to point redis_host/redis_port at the TCP endpoint (127.0.0.1:6379) instead of a unix socket, and storage_url at the Nginx files path — kill and restart the Flask process so it picks up the new config, then confirm with `curl http://127.0.0.1:5000/api/stale_ids` that it actually returns data before writing any archiver code. In archiver.cpp, chain the calls in order: libcurl GET the stale_ids endpoint and split the comma-separated response; for each ID, query hiredis for `file:<ID>`, parse the JSON value with nlohmann::json to pull out filepath and is_text; libcurl GET `<storage_url><filepath>` for the raw bytes. The detail most likely to be missed: the pre-processing (stripping `\r` and collapsing runs of 2+ spaces to one) must happen only on files where is_text is true, and must happen before concatenation, not after — the verifier explicitly checks the decompressed output contains no `\r` or double-spaces. Concatenate all processed buffers, then zlib-deflate with gzip windowBits (e.g. `deflateInit2` with `15+16`) and write to stale_archive.gz. Check the decompressed size and gzipped size yourself against the threshold before finishing.
