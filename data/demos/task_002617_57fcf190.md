Build this as a pipeline of clear stages rather than one monolithic transform. First load each CSV separately and normalize timestamps to a common UTC `datetime` type (parse ISO8601 for one file, convert Unix epoch seconds for the other). Deduplicate each source independently by its own ID column, keeping the first occurrence in file order (pandas `drop_duplicates(subset=..., keep='first')` or an awk associative array works). Assign each row to a 15-minute bucket using floor division on epoch seconds (`epoch - (epoch % 900)`), not string matching on timestamps — this avoids off-by-one errors at bucket boundaries. Compute per-bucket means for temperature and humidity separately, then build the full fixed grid of 8 buckets from 10:00 to 11:45 and left-join your aggregates onto it so missing buckets appear as NaN. Apply forward-fill (`ffill`) separately per column after the join, leaving genuinely leading NaNs (before any real reading) as blank strings rather than 0 or NaN literal. Round to 2 decimals only at the very end, right before writing, so intermediate ffill values stay exact. Verify manually by checking a couple of bucket boundary timestamps against your bucketing math before trusting the whole output.
