Design the SQL first in the sqlite3 CLI before wiring it into Go, since window-function correctness is easy to verify interactively. Create the covering index with `CREATE INDEX idx_sales_perf ON sales(region, salesperson)` (covering the grouping columns used in aggregation/ranking). For the report query, aggregate per salesperson first (`SUM(amount) AS total_sales GROUP BY region, salesperson`), then wrap that in an outer query applying `RANK() OVER (PARTITION BY region ORDER BY total_sales DESC) AS rank` and `SUM(total_sales) OVER (PARTITION BY region) AS region_total` — the region total must be a window aggregate over the per-salesperson sums, not a naive subquery re-summing raw rows (though for this data they'd match, the point of the task is using an analytical window function). Filter to `rank <= 2` in an outer-most query since window functions can't be filtered directly in the same SELECT they're defined in (SQLite has no QUALIFY, so nest one more level or use a CTE). Order final rows by region ASC, rank ASC. In the Go program, connect via `github.com/mattn/go-sqlite3`, run this single query, scan into a struct, and marshal to JSON matching the exact key names and nesting shown in the spec. Run `go mod init` and `go get github.com/mattn/go-sqlite3` in `/home/user` first.
