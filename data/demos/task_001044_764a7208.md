Triage the three bugs independently before touching integration. For encoding: wrap the ingest service's decode in `try: payload.decode('utf-8') except UnicodeDecodeError: payload.decode('windows-1252')` so legacy sensor bytes don't crash the connection. For the race condition: a plain Redis GET-then-SET from Python is not atomic under concurrent workers — use a Redis transaction (`WATCH` the key, `MULTI`/`EXEC` the read-modify-write, retry on `WatchError`) or push the logic into a Lua script executed with `EVAL` so the read+add+write happens atomically server-side. For precision: don't accumulate as a Redis float/INCRBYFLOAT — store the accumulator as a string representing a `decimal.Decimal`, and on each update parse it as Decimal, add the new delta as Decimal, and write the string back, so tiny deltas (~1e-12) aren't lost to floating-point rounding against a large running total.
