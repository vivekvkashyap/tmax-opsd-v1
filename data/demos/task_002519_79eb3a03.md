Split this into two clear stages and verify each independently before moving on.

Stage 1 (schema filter): process the file line by line with `awk -F,`. Always pass through the header unchanged. For every other line, reject it unless `NF==3` (catches missing or extra columns) AND field 1 is non-empty/alphanumeric AND fields 2 and 3 each match a strict numeric regex covering both integers and decimals, e.g. `^-?[0-9]+([.][0-9]+)?$`. This regex approach is what correctly rejects rows with words like "bad" or "error" in a numeric column, rows with too few/too many comma-separated fields, and rows missing a value — a naive `grep -c ','` count or unquoted `$2+0` numeric test will silently accept garbage. Write only the surviving rows (header + valid data rows, in original order) to the clean file.

Do the arithmetic in `awk` (or pipe to `bc -l`) to preserve floating-point precision — don't round intermediate sums. Format the final result to exactly 4 decimal places with `printf "%.4f"` and write only that bare number (no trailing newline issues, no extra text) to the output file.
