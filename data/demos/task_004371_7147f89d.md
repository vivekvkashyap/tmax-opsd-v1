For each line, split off the `[timestamp]`, the log level, and the remaining JSON text (e.g. with a regex or manual bracket-matching on the first `{`), skip the line entirely if the level is `ERROR`, then parse the JSON and verify all three required keys exist. Counting Unicode code points correctly is the crux: do not use `std::string::size()` (byte count) or naive char iteration — decode the UTF-8 string codepoint-by-codepoint (nlohmann::json strings are stored as UTF-8) counting one codepoint per lead byte (bytes that are not `10xxxxxx` continuation bytes), so multi-byte characters like accented letters, Japanese characters, and emoji each count as one, not per-byte. Compare that count to `declared_len` for validity. For the rolling SMA over valid messages, keep a sliding window of at most the last 5 *valid* declared_len values; when checking the current message for anomaly, compute the average of the window *before* appending the current value (excluding it, per spec), then append the current value to the window afterward regardless of anomaly status.
