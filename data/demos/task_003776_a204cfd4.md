For the C filter, read stdin line-by-line with `fgets`/`getline` in a loop (use `getline` with a growable buffer so long lines don't get truncated or misparsed), and for each line check that it contains BOTH the literal substring `[error]` AND the literal substring `connect() to unix:/app/run/upstream.sock failed (111: Connection refused)` (e.g. via `strstr` for each, requiring both to be non-NULL) — checking for either alone is what lets spoofed access-log lines or wrong-socket errors slip through, since attackers embed the connection-refused text inside quoted User-Agent/URL fields of otherwise non-`[error]` lines. Only print lines passing both checks, verbatim, to stdout; handle EOF cleanly. Build a small corpus of your own genuine vs. spoofed/wrong-level/wrong-socket log lines to test against before trusting it. For `supervise.sh`: before the loop, `mkdir -p /app/run`, create the symlink with `ln -sf /app/run/upstream.sock /home/user/active_socket`, and launch the SSH tunnel exactly as specified (it's backgrounded via `-f` so it won't block your script). Then use a `while true; do /app/upstream_daemon; done` loop so a crash (non-zero exit) is followed by an immediate restart.
