Do not rely on a string check like `".." in name` to detect Zip Slip — it's easy to bypass and misses cases like `plugins/../../x`. Instead, for each `ZipInfo` in the archive, compute the fully resolved absolute path with `os.path.normpath(os.path.join(target_dir, member.filename))`, resolve the target dir itself to an absolute path once, and confirm the resolved member path is equal to or nested under it (e.g. via `os.path.commonpath([resolved, target_abs]) == target_abs`, with a trailing-separator-safe comparison). Anything that fails this check goes straight to `/home/user/rejected_updates.log` using the *original* archive path string (not the resolved one), one per line, and is skipped entirely — never extracted, never opened. For safe entries, create parent directories as needed and extract preserving the archive's relative structure under the target dir. After extraction, read each safe file's text, split non-empty lines on the first `=`, and append one ledger line per key=value pair in the exact `[UPDATE] <relative_archive_path> | <key>=<value>` format, processing files in the order they appear in the archive.
