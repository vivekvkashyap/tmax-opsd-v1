Build the pipeline in stages and test each independently before combining. First write a `detect_and_decode(path)` helper that tries `bytes.decode('utf-8')`, then `'iso-8859-1'`, then `'utf-16le'` inside a try/except chain — ISO-8859-1 never raises `UnicodeDecodeError` (it maps every byte), so it must come after UTF-8 in the fallback order, and UTF-16LE should be tried before ISO-8859-1 if you want it to actually get selected for UTF-16 files (or detect BOM/null-byte patterns to route correctly), otherwise every UTF-16LE file will wrongly decode as ISO-8859-1 garbage. For parallelism, use `concurrent.futures.ProcessPoolExecutor` or `ThreadPoolExecutor` to map the decode-and-split-lines step over the list of `.txt` files found via `glob`/`os.walk`. For each line: `.strip()` then `unicodedata.normalize('NFC', line)` — this is essential because NFD vs NFC forms of the same visible character (e.g. accented letters) must hash identically for dedup to work. Hash with `hashlib.sha256(text.encode('utf-8')).hexdigest()`, collect into a dict keyed by hash to dedup across files, then write JSONL with `os.makedirs(..., exist_ok=True)`. Finally add the crontab line with `crontab -l` piped through a script that appends the exact `0 2 * * * /usr/bin/python3 /home/user/clean_pipeline.py` entry and reinstalls via `crontab -`, then run the script once manually to produce the output file.
