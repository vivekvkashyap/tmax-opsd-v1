Download and extract Jena first, then design the SPARQL query around the specific deadlock pattern before worrying about output formatting. A 2-cycle deadlock is: `?t1 ex:waitsFor ?r1 . ?r1 ex:heldBy ?t2 . ?t2 ex:waitsFor ?r2 . ?r2 ex:heldBy ?t1 .` with `?t1 != ?t2` — write this as a SPARQL SELECT over `/home/user/transactions.ttl` using Jena's `sparql --data=transactions.ttl --query=query.rq` (or the ARQ command). Because the cycle is symmetric, each deadlocked pair will surface twice (T1/T2 and T2/T1) — dedupe by requiring the alphabetically-smaller local name to bind to `?t1` (e.g. add a `FILTER(STR(?t1) < STR(?t2))` on the local names, not the full URIs) so each pair is reported once with tx1 < tx2. Strip the `http://example.org/` prefix from URIs to get local names (e.g. with `sed`, `awk`, or SPARQL's `STRAFTER`). Pipe the query's tabular/CSV output through `awk`/`jq` to build the JSON array of `{"tx1":...,"tx2":...}` objects sorted by tx1, and make the whole thing one executable, non-interactive `detect_deadlocks.sh` that downloads Jena if not already present.
