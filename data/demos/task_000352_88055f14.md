Build the normalizer as a pure Python pipeline, independent of the C binary — the binary is only for you to sanity-check assumptions, not to call from the script. Steps: (1) read the line, (2) apply `unicodedata.normalize('NFKC', text)` to collapse full-width digits/operators to ASCII; (3) build an explicit mapping for superscript codepoints (², ³, ¹, etc.) to `^` + digit, since NFKC alone turns some superscripts into plain digits without the caret — you must add the caret yourself before/after NFKC as appropriate; (4) strip zero-width/format characters by filtering codepoints whose `unicodedata.category(ch) == 'Cf'` (or an explicit blacklist like U+200B) rather than a fixed list of visible chars; (5) after normalization, validate with a regex allowing only `[0-9a-zA-Z+\-*/^=(). ]` — reject anything else, including homoglyphs that survive NFKC; (6) separately track paren depth char-by-char to confirm it never goes negative and ends at zero. Print `SAFE: <normalized>` or `REJECT` accordingly. Test against both `/app/data/clean/` and `/app/data/evil/` samples before finishing, since edge cases (mixed homoglyphs, injected shell text) are easy to under- or over-reject.
