Build this Go tool as three independent, individually-testable passes rather than one monolithic function. First, write a Combined Log Format line parser (regex or manual split on quoted/bracketed fields) that extracts the request path from each line and the client IP, then filter for GET requests whose path contains `.pem`, `.key`, or `id_rsa` — collect the IPs into a set to dedupe, then sort. Second, walk the web root with `filepath.Walk` (or `filepath.WalkDir`), read each file's contents, and check for the pattern `-----BEGIN ` ... ` PRIVATE KEY-----` — use a regex like `-----BEGIN [A-Z ]*PRIVATE KEY-----` rather than an exact string match since the key type varies (RSA, OPENSSH, etc). For each match, `os.Chmod(path, 0600)` then move it into the quarantine directory with `os.Rename` (same filesystem, so this is atomic and preserves the basename) — collect just the basename into your quarantined list, then sort it too. Third, write the static sshd_config snippet and marshal the JSON report with `json.MarshalIndent` using both sorted slices. Compile with `go build` and actually run the binary — don't just compile-check — since the report and quarantine moves only happen at runtime.
