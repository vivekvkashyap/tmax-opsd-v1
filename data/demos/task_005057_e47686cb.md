Treat this as three loosely-coupled services to stand up in sequence. First, fix the vendored server: open `/app/vendor/metrics_server-1.0.0/server.py`, find the hardcoded `PORT = 80` line, and change it to read `int(os.environ.get('METRICS_PORT', 8000))` (import `os` if not already), then launch it in the background with `METRICS_PORT=9090 python3 server.py &` so it serves whatever is in `/home/user/stats.json` on GET requests. Second, write `gather_stats.py` to count processes via `subprocess` calling `ps -e` (count lines minus the header) and dump `{"process_count": N}` as exact JSON to `/home/user/stats.json`; run it once manually to seed the file, then register it in the user's crontab (`crontab -e` or `crontab -l | { cat; echo "* * * * * python3 /home/user/gather_stats.py"; } | crontab -`) for per-minute execution. Third, write `aggregator.py` as a raw TCP server (`socket.socket(AF_INET, SOCK_STREAM)`, bind to 127.0.0.1:8080, listen/accept loop) that reads incoming bytes, strips whitespace, checks for exact match against `FETCH_CAPACITY`, then makes an HTTP GET to `http://127.0.0.1:9090/` using `urllib.request` (stdlib only), parses the JSON body, and writes back `CAPACITY_OK: <n>\n` before closing the socket — run it in the background too, and smoke-test both ports with `curl` and `nc`/a small Python client before considering it done.
