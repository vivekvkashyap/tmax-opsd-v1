Design for speed from the start — this is a 500k-row pipeline with a hard 10-second budget, so avoid row-by-row scripting anywhere in the hot path. Start services, then create the tables with `psql`. For bulk import, use PostgreSQL's `\copy raw_data FROM '/home/user/data/sensors.csv' WITH (FORMAT csv)` (client-side copy, invoked from your bash script via `psql -c "\copy ..."`), which is dramatically faster than per-row INSERTs. For the rolling average, do it in one SQL statement using a window function — `INSERT INTO sensor_features SELECT ts, sensor_id, AVG(value) OVER (PARTITION BY sensor_id ORDER BY ts ROWS BETWEEN 4 PRECEDING AND CURRENT ROW) FROM raw_data` — rather than looping in bash/Python. For Redis, use a single SQL query to pick each sensor's row with the max `ts` (e.g. `DISTINCT ON (sensor_id) ... ORDER BY sensor_id, ts DESC`), format the output as `SET sensor:<id> <value>` lines with `psql -t -A -c`, and pipe that directly into `redis-cli --pipe` for bulk loading instead of issuing individual `SET` commands. After writing the script, time it yourself (`time bash /home/user/process.sh`) and check row counts in both Postgres tables and Redis key count before considering it done — if you're over budget, the likely culprit is a per-row loop somewhere instead of set-based SQL.
