First run `tesseract /app/network_diagram.png -` (or a stdout variant) to OCR the image and read off the actual PORT, ENDPOINT, and AUTH_KEY values printed there — do not guess or hardcode placeholder values, use exactly what OCR recovers (double-check OCR output for stray characters/whitespace, especially around the endpoint path and key string). Then write a minimal C TCP server using raw sockets (socket/bind/listen/accept) that parses the first line of each incoming request ("GET <path> HTTP/1.x"), compares the path against the extracted endpoint, and replies with a literal HTTP/1.0 response: "HTTP/1.0 200 OK\r\nContent-Type: text/plain\r\n\r\n<AUTH_KEY>\n" for a match, or a 404 status line/body otherwise. Loop accept() in a while(1) so the server keeps serving multiple connections rather than exiting after one request. Compile with gcc, launch it with `&`/nohup so it survives your shell exiting, and write `manage_service.sh` to check (e.g. via pgrep or a pidfile) whether the process is alive and (re)start it only if not — that's what makes it idempotent.
