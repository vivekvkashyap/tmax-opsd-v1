(2) Database recovery: the main `telemetry.db` file is deliberately corrupted (header damage), so a plain `sqlite3 telemetry.db "select..."` will fail — try `sqlite3 telemetry.db ".recover"` or `.dump`, and separately extract rows sitting only in `telemetry.db-wal` (uncheckpointed) by scanning the WAL file's raw bytes/strings for the `timestamp`/`value` record patterns, since the WAL is a different binary format than the main db pages. Combine both sources into one full record set. (3) Write `/home/user/repair_metrics.sh` as pure Bash/awk/sed/bc — no Python/Perl/Ruby — that takes the recovered timestamp,value pairs, multiplies each value by the calibration factor you extracted in step 1 (bc handles floating point; awk can too), and writes a `timestamp,value` CSV header plus rows to `/home/user/recovered_metrics.csv`. Precision matters: keep enough decimal places in bc/awk output to keep MSE low.
