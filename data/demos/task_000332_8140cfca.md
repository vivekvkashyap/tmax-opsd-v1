Use Python's `tarfile` module opened in streaming mode on the outer tar (`tarfile.open(path, mode='r|')` or `mode='r'`), and for each member call `tar.extractfile(member)` to get a file-like object rather than extracting to disk. Wrap that file object's bytes in `io.BytesIO` before handing it to a nested `tarfile.open(fileobj=..., mode='r:gz')` — this keeps everything in memory. Wrap each nested-archive open/read in a `try/except (tarfile.ReadError, tarfile.TarError, OSError, EOFError, zlib.error)` block so a corrupt gzip stream (e.g. random bytes) is caught and that entry is simply skipped rather than crashing the whole run. To find `settings.conf` regardless of its directory, iterate `nested_tar.getmembers()` and match on `os.path.basename(member.name) == 'settings.conf'` instead of assuming a fixed path. Read that member via `extractfile()`, decode it, and parse lines of the form `KEY=VALUE` (e.g. split on `=`, strip whitespace) to pull out the integer for `MAX_CONNECTIONS`. Accumulate results into a dict keyed by nested archive filename and write it with `json.dump` to the required output path, omitting any archive that failed integrity checks or lacked the config file.
