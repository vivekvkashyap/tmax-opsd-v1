Work from the outside in: first get the C program to compile cleanly with `gcc operator.c -o operator.bin`, then wire up the environment, then wrap it in supervision. Write `.env_profile` with a plain `export NETWORK_CONFIG_PATH=/home/user/operator_project/network_config.json` line — no quotes issues, no relative paths. In `ci_cd.sh`, compile first, then `source /home/user/operator_project/.env_profile` (or `. `) so the exported variable is in the current shell before you launch the binary — sourcing after compiling but before running is the key ordering naive attempts get wrong (they often source in a subshell or forget it entirely, so `getenv` sees NULL). For the supervision loop, use a bounded `for`/`while` loop capped at 3 attempts: run the binary, capture `$?` immediately after, `break` if it's 0, otherwise loop and retry, and don't treat a non-zero exit after 3 tries as fatal-crash — just let the loop end. `chmod +x ci_cd.sh` then execute it and let the binary itself write `deploy.log`; don't create or edit that log or `operator.c` yourself, since the check is that the program's own output matches exactly.
