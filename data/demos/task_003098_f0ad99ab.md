Parse the WAL with field-aware tools rather than naive string splitting: use `awk -F: '{print $1, $2, $3}'` (the payload itself is base64 and won't contain colons, so splitting on `:` with a max of 3 fields is safe). To find the latest entry per module, sort numerically by timestamp descending (`sort -t: -k1,1nr`) then use `awk '!seen[$2]++'` (or the reverse: sort ascending and keep the last occurrence per module) to keep only the first/highest-timestamp row per MODULE_NAME. For each surviving row, decode with `base64 -d <<< "$payload" > /home/user/configs/versions/<module>_<ts>.conf` — watch for trailing newlines/padding issues in the base64 string when extracted via awk. Then create directories with `mkdir -p` before writing, and use `ln -sf` (force) to (re)create the symlink at `/home/user/configs/active/<module>.conf` pointing at the exact versioned filename you just wrote — use an absolute path in the symlink target so it resolves correctly regardless of working directory. Double check no stale/older versioned files or symlinks from earlier timestamps get created for the same module.
