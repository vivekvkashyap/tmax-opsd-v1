The core technique is replacing the floating-point comparison `R/32767 < GC/L` with an equivalent integer cross-multiplication so no `bc`/`awk`/`python` or floating point is needed inside the loop: since all values are non-negative, `R/32767 < GC/L` is exactly equivalent to `R*L < GC*32767` (cross-multiply, both denominators positive so the inequality direction is preserved). First compute L (sequence length, strip the header line and newlines) and GC (count of G/C characters) once, outside the loop, using plain Bash string operations (`${#seq}` for length, `grep -o` or `tr -cd` + `wc -c` for the count — these are fine since they run once, not per-step). Then loop over each line in randoms.txt with a `while read` (or `mapfile`), and for each R use pure Bash arithmetic `(( R * L < GC * 32767 ))` to decide whether to increment or decrement X, avoiding any subshell/external-process spawn per iteration for the comparison itself. Watch for integer overflow risk (32-bit systems) — Bash arithmetic is 64-bit so R*L and GC*32767 are safe here. Print final X, make the script executable, run it, and redirect to the result file.
