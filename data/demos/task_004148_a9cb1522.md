Treat this as four independent sub-pipelines and validate each numerically before moving on. (1) Denoising: load `raw_squiggle.wav` with `scipy.io.wavfile.read`, design the filter with `scipy.signal.butter(4, 1500/(0.5*8000), btype='low')`, apply it with `scipy.signal.filtfilt` (zero-phase — using `lfilter` instead will introduce a phase shift that fails the MSE check), clip/round back to int16 range, and write with `wavfile.write` at the same 8000 Hz sample rate — getting the sample rate or bit depth wrong will silently corrupt the MSE comparison. (2) Features: compute `scipy.signal.stft(signal, nperseg=256, noverlap=128)`, take `np.abs()` of the complex result to get the 2D magnitude matrix, save with `np.save`. (3) Alignment: implement Smith-Waterman yourself with a DP scoring matrix (match +3, mismatch -1, linear gap -2), trace back from the max-scoring cell to find where the optimal local alignment begins on the reference, and write that 0-indexed start position as a plain integer to `alignment_pos.txt` — read the candidate and reference sequences from their given files rather than assuming their content. (4) Plot the STFT magnitude as a heatmap (e.g. `plt.pcolormesh` or `imshow`) and save to PNG. Sanity-check the alignment by eyeballing whether the candidate substring actually appears in the reference near your computed offset before finalizing.
