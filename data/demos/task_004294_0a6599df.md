Build `/home/user/jsd_matrix.sh` as an executable bash script that drives `awk` for the numeric work. Read the CSV once to get Row 1 as the reference array, then for each subsequent row: add 0.01 to every cell of both the reference row and the current row (this is what avoids log(0)/division-by-zero on sparse rows), sum each smoothed row to get its normalizer, and divide every cell by that sum to get true probability vectors P and Q of equal length. Compute M_i = 0.5*(P_i+Q_i) elementwise, then JSD = 0.5*sum(P_i*ln(P_i/M_i)) + 0.5*sum(Q_i*ln(Q_i/M_i)) using natural log (awk's `log()`). Format the JSD to exactly 6 decimals with `printf "%.6f"`, classify VALID if JSD<=0.1 else INVALID, and append a line `Row_<k>: <jsd> <VALID/INVALID>` per row (k starts at 2) to `/home/user/jsd_output.txt`. Watch for off-by-one row numbering, CSVs with varying row lengths, and trailing whitespace/newline issues in awk field splitting. Test with your own small CSV before trusting the script, and `chmod +x` it.
