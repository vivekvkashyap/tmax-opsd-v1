Structure this as: hash current files -> diff against baseline -> write manifest -> archive changed files. Parse `/home/user/base_manifest.txt` into a dict of filename->hash first. When hashing files in `/home/user/configs/`, check `os.path.getsize` (or equivalent) against the 1 MiB threshold and branch: for files under the limit read normally in binary mode and hash in chunks or all at once; for files at/over the limit, open the file, `mmap.mmap(fd, 0, access=ACCESS_READ)` and feed the mmap object directly into the hashlib updater — this is the requirement most naive scripts skip, defaulting to plain reads for everything. Write the new manifest sorted alphabetically by filename in the same `<hex> <filename>` format. A file counts as "modified or new" if its filename is absent from the baseline dict or its hash differs. Build the tar with `tarfile.open(path, 'w')` and `tar.add(full_path, arcname=os.path.basename(full_path))` for each changed file so absolute directory prefixes never end up inside the archive — verify with `tar -tf` afterward that only relative basenames appear and unchanged files are excluded.
