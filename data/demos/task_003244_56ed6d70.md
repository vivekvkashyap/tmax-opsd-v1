Treat this as two separate deterministic computations over the same filtered dataset, not one pass. First load `/home/user/audit_data/transfers.jsonl` line by line (or with a JSON-lines-aware tool like `jq`/`pandas.read_json(lines=True)`), and filter to only `status == "COMPLETED"` before doing anything else — a common mistake is including FAILED/other-status transfers in either the byte-sum or the graph edges, which corrupts both the max-sender total and the shortest path (a FAILED edge must not be traversable). For the aggregation, group completed rows by `source` and sum `bytes`, then pick the max — watch for ties broken correctly if they occur. For the shortest path, build a directed adjacency list from only the completed edges and run BFS (not DFS) from `NODE_SUSPECT` to `NODE_VAULT`, since BFS guarantees the minimum hop count on an unweighted graph; count edges traversed, not nodes visited. Finally write the three required lines to `/home/user/audit_report.txt` in the exact given format (`Highest Transfer Node: ...`, `Highest Transfer Bytes: ...`, `Shortest Path Length: ...`) since the verifier greps for these exact strings.
