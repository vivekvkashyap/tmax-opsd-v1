Treat this as three separable stages: read the HDF5 input, solve the ODE + find the root per triplet, write the NetCDF output — validate each stage before chaining them. Read `x0`, `y0`, `alpha` arrays with `h5py`. For each triplet, call `scipy.integrate.solve_ivp` on the system with `dense_output=True` over `[0, 10]` so you get a continuous `sol.sol(t)` callable rather than only discrete samples. Define an objective function `sol.sol(t)[0] - sol.sol(t)[1]` and pass it to `scipy.optimize.root_scalar` (or `brentq`) with a bracket — since t=0 trivially may or may not be a root depending on x0≠y0, bracket away from 0 (e.g. `[1e-6, 10.0]`) so you find the first strictly positive root, not the trivial one. Vectorize the per-sample loop but don't try to batch the ODE solve across all rows at once — solve each triplet independently for correctness.
