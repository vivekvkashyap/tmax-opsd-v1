Write `transform.c` to read CSV lines from stdin with `fgets`/a fixed 1024-byte buffer, split on commas with `strtok` or manual scanning (fields have no embedded commas), then: skip the row if `atoi(id) % 2 == 0`; lowercase every character of the email field in place (`tolower`); for the SSN, locate the first hyphen and replace all digits before it with 'X' and the two digits immediately after the hyphen with 'X' (leave the rest, including the second hyphen, intact) — get the exact masking boundary right since off-by-one here is the classic bug. Compile with `gcc -O2 -o transform transform.c`. In `pipeline.sh`: `head -1` the input into `output.csv`, then `tail -n +2` the rest, split into N chunks (e.g. `split -n l/N` or by line count), launch each chunk through `./transform` in the background (`&`) and `wait` for all, `cat` the chunk outputs together. For deduplication by email keeping the lowest id, sort by email then by numeric id ascending and use `awk '!seen[$3]++'` (after sorting so the lowest id comes first per email) rather than dedup-then-sort, since order of operations changes which row survives. Finally sort the deduped rows numerically by id (`sort -t, -k1,1n`) and append to `output.csv`. Run `pipeline.sh` to confirm the file materializes correctly.
