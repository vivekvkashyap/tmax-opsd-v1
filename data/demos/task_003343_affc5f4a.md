Get the video frame count with `ffprobe -v error -count_frames -select_streams v:0 -show_entries stream=nb_read_frames -of default=nokey=1:noprint_wrappers=1 /app/topology_sim.mp4` (or an equivalent ffprobe/ffmpeg invocation) rather than estimating from duration*fps, since container metadata can be inaccurate — count actual frames. Read that count dynamically in your script (don't hardcode a number you happened to observe once, since the grading environment regenerates the video) and use `F % 10` as your maximum allowed BFS distance. For the graph query itself, this is standard unweighted-shortest-path: parse the JSON graph, run BFS from `--start` recording hop-distance to each first-visited node (distance 0 for the start node itself), filter out nodes whose distance exceeds the threshold, then sort remaining nodes by distance descending and node-id ascending as a secondary key, and print them comma-joined with no trailing newline-affecting extra output. Since the grader diffs your output byte-for-byte against a reference implementation across many random graphs, test with a few hand-constructed graphs (including disconnected components and self-referential edges) to confirm your BFS and sort tie-breaking exactly match expected semantics before trusting it.
