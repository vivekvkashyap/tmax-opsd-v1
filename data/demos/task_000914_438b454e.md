Use a library or ffmpeg that reports/lets you compute exact peak amplitude rather than relying on approximate volume filters. With pydub: load the wav, compute `sound.max_dBFS` (peak relative to full scale), then apply a gain of `(-3.0 - sound.max_dBFS)` dB via `sound.apply_gain(...)` so the resulting peak lands exactly at -3.0 dBFS, and export as wav — check the verifier's dBFS formula (20*log10(peak_amplitude) on normalized float samples) matches your understanding of "peak" (max absolute sample, not RMS). If using ffmpeg instead, two-pass with `volumedetect` to read the current max_volume then apply a `volume=` filter with the computed dB delta. Wrap the whole thing in try/except so any failure doesn't leave a corrupt file, and only write success.eml (with the exact `Subject: Audio Processed` line) after confirming the output file was written successfully. Create the directories and the virtual_fstab file exactly as specified — these are simple deterministic file-content checks, not something to derive.
