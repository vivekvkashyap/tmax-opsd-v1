First get the C extractor building and producing data before touching Rust: `gcc /home/user/src/extractor.c -o /home/user/bin/extractor -lm` (create `bin/` if missing), then run `/home/user/bin/extractor 1024 > /home/user/data/signal.csv` and sanity-check the file has 1024 lines. For the Rust side, `cargo new processor` in `/home/user/processor`, add `rustfft` as a dependency. Read the 1024 real floats and build a `Vec<Complex<f64>>` with each value's imaginary part set to 0.0. Run a forward FFT plan sized 1024 from `rustfft::FftPlanner`. The low-pass filtering step is the part naive attempts get wrong: you must zero out the *middle* band of the spectrum — indices 32 through 992 inclusive (this correctly removes high frequencies in a standard FFT's symmetric layout, keeping only the low positive frequencies near index 0 and their mirrored low-negative-frequency counterparts near the end) — not the edges. Then run the inverse FFT plan on the filtered array, and normalize by dividing every complex value by 1024 (rustfft does not normalize automatically). Take the real part of each entry, format to exactly 4 decimal places, one per line, and write to `/home/user/data/processed.csv`. Run the binary to produce the final file.
