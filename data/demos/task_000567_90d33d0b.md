In `main.rs`, open the connection, execute `REINDEX idx_measurements_ts;` as its own statement before any SELECT. Build the analytical query as a single parameterized SQL statement using a window function: `AVG(reading) OVER (PARTITION BY device_id ORDER BY timestamp ROWS BETWEEN 2 PRECEDING AND CURRENT ROW)` computes the current-plus-previous-2 rolling average per device — get this window frame exactly right, since "strictly previous 2 rows" plus current means `ROWS BETWEEN 2 PRECEDING AND CURRENT ROW`, not `1 PRECEDING` or an unbounded frame. Compute the rolling average in a subquery/CTE first, then filter in an outer query on the ORIGINAL `reading > ?threshold` (a common mistake is filtering before windowing, which corrupts the rolling average by removing rows from the window). Join to `devices` for the name, `ORDER BY device_name ASC, timestamp ASC`, then `LIMIT 10 OFFSET 5`. Use `rusqlite`'s parameter binding (`params![threshold]`) rather than string-formatting the threshold into SQL. When writing the CSV, format `rolling_avg` with `format!("{:.2}", val)` to guarantee exactly two decimals, and read the CLI threshold via `std::env::args()`.
