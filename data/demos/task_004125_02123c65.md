For each archive, list its contents without extracting first (`unzip -l` / `tar -tzf`) and scan every listed path for a leading `/` or a `../` component (check path segments after splitting on `/`, not just a naive substring search, to avoid both false negatives and being fooled by legitimate filenames) — if any entry is unsafe, log the archive's basename to security_alerts.log and delete it, skipping extraction entirely. Only safe archives get extracted into a like-named subdirectory. Aggregate per-owner `size_bytes` sums cumulatively across all processed archives into storage_report.json, recalculating fully or updating incrementally as each CSV lands (using `jq` for JSON read-modify-write from bash is fine).
