Load the CSV into sqlite3 (e.g. `sqlite3 :memory:` or a temp db file with `.mode csv` / `.import`) and do the graph work in SQL rather than hand-rolling Bash loops. For 3-cycles, self-join the transfers table three times (t1.receiver=t2.sender, t2.receiver=t3.sender, t3.receiver=t1.sender) to find A->B->C->A triples, then collect the DISTINCT set of all three accounts across all matching triples — remember an account can appear in multiple cycles, so dedupe with a UNION/DISTINCT over sender+receiver columns, not just one column. For degree centrality, compute total degree per account over the WHOLE transfers table (count as sender + count as receiver), restrict to accounts found in the cycle set, and break ties alphabetically with `ORDER BY degree DESC, account ASC LIMIT 1`. For top transactions, filter rows where BOTH sender and receiver are in the cycle-account set (not necessarily the same cycle), sort by amount DESC, and take the top 2 tx_ids. Finally assemble the JSON by hand (or via `jq`) matching the exact key names/types — sort `cycle_accounts` alphabetically and keep `top_cycle_transactions` as integers, not strings.
