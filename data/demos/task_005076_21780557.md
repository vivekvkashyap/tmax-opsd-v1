Build and verify the C library manually before writing any Python around it, so you know the shared object and symbol are actually correct. In `run.sh`, create `build/`, `cd` into it, run `cmake ..` then `make` to produce `libxss_check.so`, and export `LD_LIBRARY_PATH` to include that build directory (e.g. `export LD_LIBRARY_PATH=$(pwd):$LD_LIBRARY_PATH`) before invoking the Python test — a common failure is exporting the wrong path or exporting it in a subshell that doesn't propagate to the python3 call, so keep both in the same script scope. In `test_xss.py`, mirror the C struct field-for-field with `ctypes.Structure._fields_` (char* as `ctypes.c_char_p`, both ints as `ctypes.c_int`), and set `lib.check_xss.argtypes = [ctypes.POINTER(Payload)]` and `restype = None` before calling it — mismatched argtypes is the usual segfault source here. For the hypothesis test, encode each generated `str` to bytes, keep a reference to the bytes object alive (ctypes does not keep bytes objects alive on its own) while populating `content`, `length = len(bytes)`, and `has_xss = 0`, call `check_xss(ctypes.byref(payload))`, then assert `payload.has_xss in (0, 1)`. Run `run.sh`, and confirm `report.txt` shows a clean pytest/hypothesis run with no traceback.
