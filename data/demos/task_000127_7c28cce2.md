This has two independent parts — a one-time secret rotation and a hardened script — do them in order. First, transcribe /app/intercepted_voice_auth.wav (use a speech-to-text tool available on the box) to recover the leaked token text, then convert whatever is spoken into the actual credential string the config uses (e.g. spelled-out words/letters may map to a short alphanumeric code — read it carefully rather than assuming a fixed word list). Replace ADMIN_TOKEN in /app/auth_config.env with a freshly generated 32-hex-char string (e.g. `openssl rand -hex 16`) and `chmod 600` the file. For upload_processor.sh, rewrite it from scratch in pure bash: read headers line-by-line stripping `\r`, stop at the blank line, then read the remainder of stdin as the body (e.g. with `cat`). Parse `Authorization: Bearer <token>` and compare against the *sourced* ADMIN_TOKEN — auth check must come before path handling. Use `basename` on the X-Upload-Filename header value to strip any `../` or absolute-path components, reject empty basenames (400) and non-`.wav` extensions (403) before ever touching the filesystem, write the body to /app/safe_uploads/<basename>, then `chmod 600` that saved file, and emit exactly the required status lines with matching exit codes.
