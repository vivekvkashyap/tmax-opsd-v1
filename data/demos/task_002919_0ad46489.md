Build this as three independent, individually-testable pieces rather than one big script. First write the C++ extractor to read stdin line by line, split on commas to get timestamp/hostname, then locate the substring `CPU:` in raw_message and parse the digits up to the following `%` (use `strtol`/`sscanf` rather than fixed offsets, since the CPU value may be 1-3 digits) — print `timestamp,hostname,cpu_percentage` per line. Compile with `g++ -O2 extractor.cpp -o extractor` and manually test it against a couple of crafted lines before wiring it into the pipeline. Second, write process.sh as `./extractor < telemetry.csv | sort -t, -k3 -n -r | head -n 5 > top_cpu.csv` — remember `-k3` sorts on the third comma-delimited field and needs `-n` for numeric (not lexicographic) order, and `-r` for descending; verify the output order by eye since ties or missing `-n` are the classic bug. Third, write the cron line exactly as `0 * * * * /bin/bash /home/user/process.sh` into telemetry.cron — no extra whitespace variations that would break a regex check. Finally chmod +x process.sh and run it once so top_cpu.csv exists.
