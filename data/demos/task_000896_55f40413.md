This is a constraint-propagation resolver, not a simple tree walk. For each target, traverse `depends` and every time a package is reached, intersect its new constraint with any constraint already recorded for it (don't pick a version greedily on first visit — a package can be depended on multiple times with different operators). Once bounds are final, pick the highest satisfying version per package using numeric (not lexical) semver comparison — split on `.` and compare integers. Use `jq` to parse `registry.json` into package/version/constraint data, then drive the graph/constraint logic in a helper script (Python/Perl is fine) invoked from Bash.
