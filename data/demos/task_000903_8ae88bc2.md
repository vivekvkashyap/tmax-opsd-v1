Handle the encoding and locking correctly before touching the parsing logic. Open `/home/user/active_service.log` in binary mode, take a *shared* `flock` (not exclusive, since the writer needs to keep writing) around the read, then decode the raw bytes from `UTF-16LE` to a Python/text string (mind an optional BOM at the start of the file). Split the decoded text into `[RECORD]...[/RECORD]` blocks — do this by splitting on the record markers rather than a naive line-count assumption, since the field set is fixed (TIMESTAMP, LEVEL, MSG, DETAILS) but you should still parse by matching each `KEY: value` line explicitly and trimming whitespace, rather than positional indexing. Keep only blocks whose LEVEL field equals exactly `ERROR`, build one JSON object per record with uppercase keys in the given order, and write the resulting array (pretty-printed is fine) to `/home/user/error_archive.json` encoded in UTF-8. The two failure modes to avoid: reading before the writer has released its lock (causing torn reads), and forgetting to re-encode from UTF-16LE which otherwise corrupts the ASCII field values into null-byte-interleaved garbage.
