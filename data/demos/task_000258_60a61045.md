Build this as two clearly separated phases in the same script: an assembler and an emulator. For assembling, iterate the `instructions` list, split each string on whitespace, map the mnemonic to its opcode byte (`PUSH=0x01, ADD=0x02, SUB=0x03, MUL=0x04`), and for `PUSH` append `struct.pack('<BH', 0x01, value)` (opcode then a little-endian unsigned 16-bit operand); other opcodes are just the single opcode byte with `struct.pack('<B', opcode)`. Concatenate all encoded instructions and write the raw bytes to `/home/user/api_payload.bin` in binary mode. For the emulator, re-read that file as bytes, walk it with an index pointer: on `0x01` read the next 2 bytes as little-endian to push onto a Python list-as-stack; on `0x02/0x03/0x04` pop two values (mind operand order for SUB — first-popped is B, second-popped is A, result is A-B) and push the computed result. After execution, write the single remaining stack value as plain text to `/home/user/final_state.txt`. Test round-trip on a small hand-written instruction list before running against the real input.
