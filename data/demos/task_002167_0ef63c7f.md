Split the work into an offline feature-extraction pass and a thin serving layer that just reads precomputed results. Extract frames at 1 fps with `ffmpeg -i /app/experiment_record.mp4 -vf fps=1 frame_%02d.jpg` (or map ffmpeg's 1-indexed output to your required zero-padded 0-indexed naming, e.g. `frame_00.jpg`) — get the indexing convention right up front since off-by-one frame numbering will break the anomaly indices downstream. For each frame, convert to grayscale (`Image.open(path).convert('L')`), resize to 32x32 with bilinear interpolation, flatten to a 1024-length integer vector — do this once and cache all embeddings in memory or on disk rather than recomputing per HTTP request. Compute Pearson correlation between consecutive embeddings (e.g. `scipy.stats.pearsonr` or `numpy.corrcoef`), and flag frame `t` (for `t>=1`) as anomalous when `corr(t, t-1) < 0.85` strictly — build the full anomalies list once at startup. Then serve two endpoints from a lightweight framework (Flask/FastAPI) on 127.0.0.1:8080: `/anomalies` returning the cached list as JSON, and `/correlation?f1=&f2=` computing/looking up the pairwise Pearson correlation between the two requested indices' cached embeddings, rounding to 4 decimals in the JSON response. Leave the server running.
