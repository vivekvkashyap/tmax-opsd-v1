Design `doc_builder.go` as a two-phase pipeline: first build an in-memory map from output-document-name to ordered content (walk `source_dir` with `filepath.Walk`, compute each file's path relative to `source_dir`, check membership against the flattened `merge_groups` config — if the relative path matches an entry in some group's array, append its content under that group's target key in the JSON-array order given, not filesystem order; otherwise use `filepath.Base` of the file as its own standalone document key). Second, for each accumulated document, split its lines into chunks of at most `max_lines_per_chunk`; if only one chunk resulted and total lines <= the limit, use the original document name unchanged, otherwise name each piece `[basename-without-ext]_partN.md` (1-indexed, basename without the `.md` extension plus `_partN.md`). For every chunk written, follow the atomic-write pattern precisely: `os.CreateTemp(output_dir, "...")`, write content, `Close()`, then `os.Rename(tmpPath, finalPath)` — writing directly to the final path defeats the whole point and will fail verification. Ensure `output_dir` is created with `os.MkdirAll` first, then `go build`/`go run` the program against the given `config.json` and confirm `/home/user/dist` contains the expected chunked files.
