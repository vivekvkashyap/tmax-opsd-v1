Work outward-in: extract the outer archive first with `tar -xzf /home/user/raw_docs.tar.gz -C <workdir>`, which should reveal `structure.json` and the nested `content.tar.bz2`; then extract that inner archive too (`tar -xjf content.tar.bz2`) to get the actual draft Markdown files. Parse `structure.json` with `jq -c '.file_mapping[]'` to iterate `original`/`published` pairs, and rename each draft file in place with `mv "$original" "$published"` (or copy into a clean staging directory) rather than trying to string-manipulate the JSON by hand. The main pitfalls: forgetting to `cd` into the extraction directory before renaming (so `tar` picks up stray path prefixes), and building the final tarball from the wrong directory so it includes directory entries or leftover JSON/archive files. Stage only the renamed `.md` files into one flat directory, then run `tar -cf /home/user/release_docs.tar -C <staging_dir> .` (or list the files explicitly) so the archive root contains exactly the three renamed files with no nesting. Verify with `tar -tf /home/user/release_docs.tar`.
