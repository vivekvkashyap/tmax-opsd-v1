Treat this as a relational join-and-group problem, best solved by loading both CSVs into structures you can cross-reference (an associative array keyed by emp_id -> name,department_id works well in `awk` or a short embedded `python3`/`sqlite3` block inside the bash script). Steps: (1) build a lookup from `emp_id` to `(name, department_id)` from `employees.csv`; (2) for each row in `mentorships.csv`, look up mentor and mentee, and keep the edge only if both department_ids match exactly; (3) group the kept edges by that shared department_id; (4) within each department, sort edges alphabetically by mentor_name, breaking ties by mentee_name; (5) emit valid pretty-printed JSON with department_id as string keys mapping to lists of `[mentor_name, mentee_name]` pairs — building the structure programmatically (e.g., in a `python3 -c` heredoc invoked from the bash script) and using `json.dumps(..., indent=2)` is far less error-prone than hand-assembling JSON with awk/sed. Wrap all of this in `/home/user/process_graph.sh`, `chmod +x` it, run it, and diff the resulting JSON's structure (e.g. via `jq -S .`) against what you compute mentally for a couple of departments before trusting the full output.
