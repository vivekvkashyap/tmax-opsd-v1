Build this as two separate scripts sharing one SQLite database. In `ingest.py`, load the CSV with `csv` or `pandas`, create the `transactions` table, and add indexes on `sender_id` and `receiver_id` (and consider a composite index) — without these, a self-join across three copies of the table for cycle detection will be extremely slow on a large CSV. In `detect_cycles.py`, accept `min_amount` via `sys.argv`, and build the SQL as a 3-way self-join (`t1.receiver_id = t2.sender_id`, `t2.receiver_id = t3.sender_id`, `t3.receiver_id = t1.sender_id`) with each `amount > ?` bound as a parameter via `cursor.execute(query, (min_amount, min_amount, min_amount))` — never interpolate `min_amount` into the SQL string. To avoid duplicate cycles (same triangle found from different starting points/directions), normalize by requiring `sender_id` values be sorted lexicographically (`node1 < node2 < node3`) as a WHERE condition, then dedupe on that triple. Compute `bottleneck_amount` as `MIN` of the three transaction amounts per cycle, sort results by bottleneck descending then node1 ascending, and serialize to the exact JSON schema given.
