Before writing the wrapper, understand the binary's runtime needs: `ldd /app/legacy_cert_gen` (or `file` if statically linked) tells you what, if anything, must be visible inside the sandbox. Isolate execution with a lightweight, fast-starting mechanism — `bwrap` with a minimal `--ro-bind` of only the required libs plus `--unshare-net` and `--tmpfs` for its working directory is a good fit since it avoids the setup cost of a full chroot and blocks network access without special privileges; pair it with `ulimit`/ `--die-with-parent` for defense in depth, and keep the isolation setup itself cheap since this runs thousands of times. After generation, parse policy with `openssl x509 -noout -text -in cert.pem`: extract the public key algorithm and bit length (grep `Public-Key:` and confirm RSA), the exact signature algorithm string (must be `sha256WithRSAEncryption` or `sha512WithRSAEncryption`, nothing else), and compute the validity window in days from `notBefore`/`notAfter` (`openssl x509 -noout -dates`, converted via `date -d`). On any failure or a crashed binary, `shred`/`rm` the `.pem` files, print `FAIL: <domain>`, exit 1; otherwise print `PASS: <domain>`, exit 0.
