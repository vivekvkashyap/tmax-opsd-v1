Use delta-debugging rather than eyeballing 100 files. First write a small script that loops over every file, extracts column 4 with `awk -F,`, and tries to sum it numerically; catch failures per-file (e.g. `awk` producing a non-numeric error, or a value whose magnitude is wildly out of range compared to the rest) so you isolate exactly which file breaks parsing (malformed string) versus which file passes parsing but skews the total (a huge negative outlier). Don't just filter out any negative number — negatives could be legitimate; instead compare each file's per-file subtotal against the typical per-file subtotal to spot the true outlier, and separately grep/validate that column 4 matches a numeric regex to catch the unparseable one. Once both bad filenames are confirmed, list their base names alphabetically into `bad_files.txt`, then recompute the sum over the remaining 98 files only (exclude both bad files entirely, don't try to "fix" their values), formatting the final total to exactly two decimal places with something like `printf "%.2f\n"` or `awk`'s printf into `correct_total.txt`.
