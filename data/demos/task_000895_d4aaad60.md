Structure the script as three clearly separated phases so each is independently testable: (1) parse `doc_config.ini` with `configparser`, reading the `[Terms]` section as the replacement map and `[Backup]` `target_dir`; (2) walk `docs/` for `*.md` files, do case-sensitive literal string replacement of every term, and write atomically — write to a `NamedTemporaryFile` in the *same directory* as the original (atomic rename only works within one filesystem) then `os.replace()` over the original, never write-in-place; (3) after all edits are done, recompute MD5 of every `.md` file and diff against `snapshot.json`'s checksums — a file counts as changed if it's absent from the snapshot OR its new hash differs. Build the diff set first, then create the tarball with `tarfile.open(..., "w:gz")` adding each changed file under its path relative to `docs/` (use `arcname=` explicitly, not the full absolute path, or the extracted archive will have the wrong layout). Finally write the sorted relative paths, one per line, to `backup_summary.txt`. Compute checksums only after replacements are applied, not before — the whole point is to compare post-edit content against the pre-edit snapshot.
