Two sub-problems: recover the PIN for the black-box decoder, then build the classifier around it. To find the 4-digit PIN, brute-force it — loop all 0000-9999, calling `/app/decode_payload <pin> <known-obfuscated-sample>` and checking for a response other than "Access Denied"; script this brute force once during development (not inside the final `filter_uploads.sh`, which should just hardcode the discovered PIN) since you must not print the PIN or decoded payload to stdout in the final script. Once you have the correct PIN, in `filter_uploads.sh`: read the obfuscated string from stdin into a variable, call `/app/decode_payload "$PIN" "$input"` capturing its stdout into a variable (redirect any extraneous output away), then grep the decoded text case-appropriately for path-traversal indicators (`../`, `..%2f`, `%2e%2e%2f`, and their URL-encoded/backslash variants) and XSS indicators (`<script>`, `onerror=`, `javascript:`). If any pattern matches, print exactly `REJECT`; otherwise print exactly `ACCEPT`. Test against a handful of your own malicious and benign obfuscated samples (encode plaintext yourself using the same scheme the binary decodes, once you've reverse-engineered it) before trusting it against the hidden corpora.
