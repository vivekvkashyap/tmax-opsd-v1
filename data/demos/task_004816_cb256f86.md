Two independent subsystems: get each working and testable separately before wiring them together. For the OCR step, run `tesseract /app/config.png stdout` (or convert to a higher-res/thresholded image first if OCR is noisy) and read off the aggregation interval and the bearer token from the recognized text yourself — don't guess or hardcode values, extract them from the actual OCR output and use those extracted strings directly in your code/config. For the ETL: load raw_sensors.csv, parse timestamps, sort chronologically, drop duplicate (timestamp, sensor_id) pairs keeping the first occurrence, then per-sensor use pandas `interpolate(method='linear')` on the value column (only between known points, not extrapolating) to fill nulls, then resample into fixed-width buckets aligned to the interval you OCR'd (e.g. `resample('15min', label='left', closed='left')` off the first timestamp, computing bucket means), and bulk-load rows into a SQLite table `sensor_data(time_bucket TEXT, sensor_id TEXT, avg_value REAL)` via `sqlite3`/`pandas.to_sql`. For the API, use Flask/http.server on 127.0.0.1:8055 with a `/query` handler that checks the `Authorization: Bearer <token>` header against the OCR'd token exactly (401 on mismatch/missing), queries SQLite filtered by sensor_id ordered by time_bucket, and returns the JSON array with `time`/`value` keys. Launch the server with nohup/background and confirm it's listening before finishing.
