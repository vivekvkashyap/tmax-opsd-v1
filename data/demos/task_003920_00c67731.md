Structure the Go program in clear stages and verify each independently before wiring concurrency in. (1) Read each .dat file as raw bytes and decode from ISO-8859-1 to UTF-8 using `golang.org/x/text/encoding/charmap.ISO8859_1.NewDecoder()` piped through `transform.NewReader` — do not assume the bytes are already valid UTF-8, since bytes like 0xF6 must map to the correct Unicode code point (ö), not be passed through raw or misinterpreted. (2) Implement RLE on runes (not bytes) so multi-byte UTF-8 characters aren't split: scan for runs of 3+ identical consecutive runes and replace with `char + "*" + count`; runs of 1-2 are left untouched. (3) For atomic writes, write to `<name>.rle.tmp` in the destination dir, then `os.Rename` to the final `.rle` path — Rename is atomic on the same filesystem. (4) Use one goroutine per file with a shared `*os.File` log handle protected by `syscall.Flock` (LOCK_EX) around each append-write-unlock, or serialize log writes through a channel/mutex, to avoid interleaved lines. (5) After all goroutines finish (use a sync.WaitGroup), compute SHA256 of each .rle file's content, sort filenames alphabetically, and write the manifest in `hash  filename` format with two spaces, matching sha256sum's convention.
