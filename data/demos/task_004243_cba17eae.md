Set up the venv first (`python3 -m venv /home/user/venv && source /home/user/venv/bin/activate && pip install numpy scipy matplotlib`) and use that interpreter for every subsequent run so results are reproducible in the intended environment. Then read `wave_sim.py` closely to find where subdomain results are combined into the global "energy" sum each timestep — the nondeterminism comes from iterating over subdomains in a shuffled/unordered sequence before summing; fix it by sorting the subdomain collection by its `id` attribute (e.g. `sorted(blocks, key=lambda b: b.id)`) immediately before the reduction loop, removing any `shuffle` call, so floating-point addition order is fixed run-to-run. Re-run the simulation with the venv's python to regenerate `energy.npy`. For `analyze.py`, load the array, apply `numpy.fft.rfft`/`fft` with the correct frequency axis derived from `dt=0.01` (e.g. `np.fft.fftfreq(N, d=dt)`), exclude the 0 Hz bin explicitly before taking the magnitude argmax, write that frequency as a plain float to `peak.txt`, and plot amplitude vs. frequency to `spectrum.png` with matplotlib (remember matplotlib needs a non-interactive backend like `Agg` in a headless terminal).
