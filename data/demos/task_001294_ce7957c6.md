In bridge.py, run the async gRPC server (grpc.aio) and a `websockets.serve` server concurrently in the same asyncio event loop (e.g. via `asyncio.gather`); maintain a shared set of connected websocket clients, and in the `BroadcastLog` RPC handler, iterate that set and `await ws.send(payload)` on each. Watch out for two common mistakes: forgetting to run both servers in one event loop (they must share state, not be separate processes), and a race where the gRPC call fires before the websocket client has connected — have test_pipeline.sh start bridge.py, sleep briefly, launch the websocket-client-and-writer first (or ensure it's ready) and only then invoke the gRPC client that sends the BroadcastLog call. Use `pkill` or capture the bridge.py PID to cleanly terminate it at the end of the script, and `chmod +x` the script.
