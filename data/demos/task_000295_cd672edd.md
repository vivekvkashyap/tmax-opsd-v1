Treat this as three independently checkable stages rather than one pipeline. (1) Frame extraction: `ffmpeg -i /app/diagnostic_screen.mp4 frame_%03d.png` to get all 100 frames, then for each frame count white pixels (e.g. with Python/OpenCV/PIL: threshold near-white pixels and `np.sum`, or ImageMagick's `convert frame.png -fill black -fuzz 10% -opaque white -format "%[fx:...]"`-style pixel counting) and write one integer per line to `/home/user/baseline_load.csv` in frame order. (2) MCMC: in `mcmc_fit.c` implement a standard Metropolis-Hastings sampler with a symmetric proposal (e.g. Gaussian random walk) for mu and sigma jointly or via Gibbs-like alternation, using the Gaussian log-likelihood of the observed loads under N(mu,sigma^2) as the target, accept/reject via the Metropolis ratio, run 50k+ iterations, discard an initial burn-in fraction (e.g. first 10-20%), and average the retained samples to get point estimates written as `mu sigma` to 2 decimals. (3) Classifier: compute the threshold `mu + 4*sigma` from the fitted params and reject (exit 1) any trace file containing a value exceeding it, else exit 0 — test directly against `/app/traces/clean` and `/app/traces/evil` to confirm 100% separation before finishing, since a poorly-converged mu/sigma will misclassify borderline traces.
