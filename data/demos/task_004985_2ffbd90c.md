Work in dependency order: extract first, fix the library second, generate new keys third, then write the filter last since it depends on all three. To extract the embedded PEM from `/app/bin/legacy_auth_service`, use `strings` (or `binwalk`/manual `grep` over the binary) to locate the `-----BEGIN PUBLIC KEY-----`...`-----END PUBLIC KEY-----` block in `.rodata` and write it out exactly, preserving line breaks, to `old_pub.pem`; validate it parses with `openssl pkey -pubin -in old_pub.pem -text -noout`. For the vendored PyJWT, diff or read `jwt/algorithms.py`'s `RSAAlgorithm.verify` — look for a return path that was short-circuited to always return `False` (or similarly disabled) instead of calling the real cryptographic verification function; restore the real call, then `pip install -e` the vendored package so your filter imports the fixed version. Generate the new RSA keypair and self-signed cert with `openssl req -x509 -newkey rsa:2048 ... -days 365 -subj "/CN=auth.local"`. Build `jwt_filter.py` last: decode the JWT header first to check `alg` case-insensitively for "none", separately attempt verification against the compromised key (reject on success) and against your new key (accept only on success with RS256), and scan decoded claims (`sub`, `role`) for traversal substrings including URL-encoded variants before deciding ACCEPT/REJECT — test against both a known-good and known-bad token you construct yourself before trusting the hidden corpus.
