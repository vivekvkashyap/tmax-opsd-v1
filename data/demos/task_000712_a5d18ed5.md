Build this incrementally and test each stage in isolation before wiring the pipeline. Create the venv and pip install a lightweight web framework (Flask/FastAPI or just `http.server`) plus an ASR tool (`openai-whisper` or `speech_recognition`+a local engine) — pick whichever installs reliably in your environment. Implement `/health` first and curl it to confirm the server binds correctly to 127.0.0.1:8080. For `/build`: run the transcriber on the fixture WAV, inspect the raw transcript text yourself to see how the spoken version phrase actually reads, then write a small parser (regex/word-to-number mapping) that extracts the three number words and converts them to `X.Y.Z` — don't hardcode a version, derive it from what the ASR actually outputs. Compare against `/app/current_version.txt` using tuple/semver comparison (not string comparison) to gate downgrades. For memory profiling, spawn `/app/data_processor.py` under `tracemalloc` or `mprof run` and capture peak MiB, writing just the float to the report file. Launch the server with `nohup`/`&` or a background process manager so it persists after your task completes.
