Nginx config (`/home/user/nginx.conf`): since you can't use system default paths, explicitly set `pid`, `error_log`, and inside `http{}` block set `client_body_temp_path`, `proxy_temp_path`, `fastcgi_temp_path`, etc. to writable subdirectories under `/home/user/` (create them first), and use a non-privileged `listen 127.0.0.1:8080;` inside a `server{}` block with `location /v2/ { proxy_pass http://127.0.0.1:9000/; }` — note the trailing slash on `proxy_pass` is what strips the `/v2/` prefix when forwarding, a common thing to get wrong. Start nginx with `nginx -c /home/user/nginx.conf` (or `-p` prefix as needed) and confirm it's listening before testing. Migration script (`/home/user/migrate.sh`): read `/home/user/legacy_data.csv` skipping the header (`tail -n +2`), split each line on commas, build JSON with `identifier` as a bare integer, `display_name` as a string, and `is_admin` as a JSON boolean `true`/`false` (only `true` when role is exactly `"admin"`), POST it with `curl -s -o /dev/null -w '%{http_code}' -X POST -H 'Content-Type: application/json' -d '<json>' http://127.0.0.1:8080/v2/import`, and append `ID <id>: HTTP <code>` per row to `/home/user/migration_results.log`.
