TLS: generate a self-signed cert with `openssl req -x509 -newkey rsa:2048 -nodes -keyout /home/user/certs/key.pem -out /home/user/certs/cert.pem -days 365 -subj "/CN=localhost"` (the `-nodes` flag avoids an encrypted key that would block automated startup). Server: write `serve_metrics.py` using Python's `http.server` with `os.chdir('/home/user/metrics')` before creating the handler so files are served from the right root, then wrap the raw socket in an `ssl.SSLContext` (`PROTOCOL_TLS_SERVER`) loaded with your cert/key via `load_cert_chain`, binding to `localhost:8443`. Deploy: launch it with `nohup python3 /home/user/serve_metrics.py &` (or similar detachment) so it survives the shell session, then confirm it's actually listening (`ss -ltnp` or a quick `curl -k`) before finishing — a background process that crashed immediately is the most common failure mode here, so check its exit status/log, not just that you launched it.
