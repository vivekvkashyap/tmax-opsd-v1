First create `/home/user/staging/` and `/home/user/repository/`. For `POST /upload`: read the full request body, write it to a uniquely-named temp file in staging (e.g. using `os.CreateTemp` or a random/UUID suffix to avoid races), then run `/app/extractor --analyze <tempfile>` and capture stdout. Parse the `Filepath:` lines out of each `=== ARCHIVE ENTRY ===` block (simple line-scan/regex is fine) and flag the upload malicious if any filepath starts with `/` or contains `../` or `..\`. On malicious: remove the temp file and respond 406. On safe: compute the SHA-256 of the file's contents, rename (or copy+remove) the temp file to `/home/user/repository/<hex_sha256>.pkg`, respond 201. For `GET /audit`: glob `*.pkg` in the repository, re-run the extractor on each, collect all `Filepath` values across all files into a set to dedupe, sort alphabetically, and return as a JSON string array. Keep the HTTP server running (don't let main() return) and bind explicitly to `0.0.0.0:8080`.
