Trigger `POST http://127.0.0.1:5000/generate` first and confirm files landed under /home/user/raw_logs/. In your script, use `find /home/user/raw_logs -name '*.log'` to enumerate files, and for each one query Redis (via `redis-cli GET "processed:$abspath"`) to skip already-done files. For the archive, do NOT compress files individually or use default gzip — the payload is highly repetitive JSON, so you must stream a single tar of all unprocessed files through a solid compressor with aggressive settings (e.g. `tar -cf - <filelist> | xz -9e -o archive.tar.xz`, or zstd --ultra -22) so redundancy across files gets deduplicated in one pass; naive per-file or low-level compression will blow past the 1.5MB budget. After each file is successfully streamed into the archive, mark it in Redis with SET and only then rename it by appending `.done` — do the rename after both the archive write and Redis update succeed, so no `.log` files remain unprocessed at the end. Verify final archive size with `du -sb` before declaring done.
