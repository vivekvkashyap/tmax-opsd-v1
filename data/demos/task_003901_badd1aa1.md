Treat this as scan-then-fix-then-reverify, in that order. Write `/home/user/scan.sh` to `curl -sI http://127.0.0.1:8080/` and inspect headers for `X-Frame-Options`/`X-Content-Type-Options`, grep the `Set-Cookie` line for both `Secure` and `HttpOnly` substrings, and `stat`/`ls -l` the SSH host key to check its permission bits are wider than owner-only; run it once now and save to `pre_report.txt` to establish the baseline. Then edit `nginx.conf` to add `add_header X-Frame-Options DENY;` and `add_header X-Content-Type-Options nosniff;`, and use `proxy_cookie_flags` (or a similar directive) so upstream cookies get `Secure` and `HttpOnly` appended. Edit `sshd_config` to set `PasswordAuthentication no`, `PermitEmptyPasswords no`, and `Ciphers chacha20-poly1305@openssh.com,aes256-gcm@openssh.com`. `chmod 600` the host key. Run `/home/user/app/restart.sh`, then actually re-test — curl the proxy again and attempt an SSH connection — before writing `post_report.txt`, since a config typo will silently fail to apply and only shows up on reverification.
