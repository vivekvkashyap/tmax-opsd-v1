Extract `temp=` and `humidity=` values with a pattern that doesn't assume fixed key ordering (search for `[temp=N]` and `[humidity=N]` independently anywhere in the MSG block); a line missing either key entirely must be discarded. Check temp is in [-50,150] and humidity in [0,100] (inclusive bounds, so boundary values like -50, 150, 0, 100 are valid but one-past is not). For each surviving line compute `risk_level` (`HIGH` if temp>80 or humidity>90, else `LOW`), and append rows to the CSV preserving original line order.
