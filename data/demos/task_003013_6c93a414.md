Do this as three independent numerical sub-analyses feeding one JSON writer, and lean on numpy/scipy rather than hand-rolled math to avoid subtle bugs. (1) FFT: load the CSV, take `np.fft.fft` on the power column and `np.fft.fftfreq(n, 1/fs)` with fs=1024 (n=1024 samples over 1 second); mask to strictly positive frequencies (exclude 0 and negative/mirror bins), sort by magnitude descending, and take the top two frequency values — the DC bin at freq=0 is a common trap, make sure you exclude it. (2) KDE/percentile: use `np.percentile(latency, 99)` for p99, and `scipy.stats.gaussian_kde(latency)` (default Scott's-rule bandwidth — don't override it) evaluated on `np.linspace(0, 200, 2000)`; the peak is the grid x-value where the KDE output array is maximal (`np.argmax`), not the raw data's max. (3) Convergence: iterate N from 2 upward, at each step compute `np.std(latency[:N], ddof=1) / sqrt(N)` (population vs sample std is a common mistake — ddof=1 is required), and record the first N where this cumulative SEM drops strictly below 0.20; do this as a single forward pass rather than recomputing from scratch each time if performance matters. Round every output to exactly 2 decimals before writing the JSON.
