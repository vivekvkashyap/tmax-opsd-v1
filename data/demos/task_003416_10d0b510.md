Build the Go-to-Python FFI bridge in stages and verify each layer before moving on. First compile the Go file with `go build -buildmode=c-shared -o /home/user/libintegrate.so /home/user/integrate.go` — this needs `import "C"`, an `//export FuncName` comment directly above the exported function, and an empty `func main() {}`, otherwise c-shared mode will fail to build. Confirm the output with `file /home/user/libintegrate.so` to check it's a proper ELF shared object. In the Python script, load it via `ctypes.CDLL(path)`, then explicitly set `lib.CalculatePi.argtypes = [ctypes.c_int]` and `lib.CalculatePi.restype = ctypes.c_double` before calling — skipping this is the most common source of silent garbage/crash results since ctypes defaults to assuming `int` return type. Call the function once with the specified step count, round with Python's `round(value, 6)`, and write the JSON with the exact keys and types requested (steps as an int, result as a float) to the exact output path.
