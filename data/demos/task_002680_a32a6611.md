Write the validator as a small explicit state machine, not a regex soup. Read the whole file, split off the request line, verify the method token is exactly `GET` (reject/handle anything else sanely). Extract the raw URI substring between the method and the HTTP version. Write your own percent-decoder that walks the string and converts `%XX` (two hex digits) into the corresponding byte, leaving other characters untouched, and applies to the full URI in one pass before any other check — decoding order matters because traversal sequences and the required prefix can be hidden behind percent-encoding (e.g. `%2E%2E%2F`). After decoding, check the required `/artifacts/` prefix first, then scan the decoded string for `../` to catch traversal. Separately parse headers line by line looking for a case-correct `X-Build-Token:` header, trim leading/trailing whitespace and any `\r`, and validate the value is exactly 16 characters, every one a hex digit (0-9a-fA-F) — reject on length mismatch, non-hex chars, or a missing/empty header. Always print exactly one `VALID ...`/`REJECTED ...` line and exit 0. Build a handful of your own request files (valid, encoded traversal, wrong prefix, short/missing token) to test each branch before trusting the compiled binary.
