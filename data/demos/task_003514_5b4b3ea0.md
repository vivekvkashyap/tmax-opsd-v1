Work in stages, extracting one archive layer at a time: `tar -xf master_dataset.tar` to reveal `logs.tar.gz` and `sensors.tar.bz2`, then `tar -xzf logs.tar.gz` and separately `tar -xjf sensors.tar.bz2` (note the different compression flag) into distinct directories so you don't need to extract binaries you won't use. Parse `experiment.log` by splitting on the `--` separator lines into records (e.g. with `awk` using `RS="--\n"` or a simple Python loop accumulating lines until a `--` marker), then for each record check whether the `Status:` line is exactly `ERROR` and, if so, capture the value after `DataFile:`. Collect the matching filenames into a sorted, deduplicated list — this is what drives both later steps. Use that list to selectively extract only those named members from `sensors.tar.bz2` (e.g. `tar -xjf sensors.tar.bz2 <file1> <file2>`) rather than extracting everything. Build the final archive with `tar -czf anomalies_backup.tar.gz -C <dir> file1 file2` so the paths inside are flat (no directory prefixes) — verify with `tar -tzf`. Write the sorted filenames, one per line, to summary.txt.
