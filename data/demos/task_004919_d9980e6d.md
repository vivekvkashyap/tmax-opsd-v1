Structure `archiver.c` as a standard inotify event loop: `inotify_init1()`, `inotify_add_watch(fd, watch_dir, IN_CLOSE_WRITE)`, then `read()` on the inotify fd into a buffer sized for several `struct inotify_event` plus name, and iterate events checking `event->mask & IN_CLOSE_WRITE` and that `event->len > 0` (ignore directory-only events). For each matching filename: build the tar command with `snprintf` into fixed buffers (archive path = `<archive_dir>/<filename>.tar.gz`), invoke `system()` (or fork/exec for robustness) running `tar -czf <archive> -C <watch_dir> <filename>`, check the return status, then verify by running `tar -tzf <archive>` and checking its exit status too — only on success proceed to `unlink()` the original file under the watch directory and `fopen(audit.log, "a")` to append the exact `ARCHIVED AND VERIFIED: <filename>\n` line, flushing/closing the file each time so entries persist even if the daemon is killed. Compile with plain `gcc archiver.c -o archiver`, launch it backgrounded (`./archiver /home/user/hot_data /home/user/cold_archive &`), then create the three test files sequentially with a short pause between each so inotify can process them one at a time, and confirm the final state (empty hot_data, three valid .tar.gz files, three audit log lines) before leaving the daemon running.
