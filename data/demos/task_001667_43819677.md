Build the ETL as pure text-processing with strict regex matching, not loose splitting: define a single anchored pattern that requires the exact literal punctuation `[TIMESTAMP] SERVER - ACTION: X - KEY: Y - VALUE: Z`, and skip (don't error on) any line that doesn't fully match — this is how malformed lines get silently dropped. For deduplication, treat the entire parsed record (timestamp, server, action, key, value) as the identity key and keep only first occurrences, e.g. by piping through `sort -u` on the raw matched lines or using an associative array keyed on the full tuple, before ever writing output — this makes reruns idempotent since sorting+deduping the same input always yields the same output file. Sort with `sort -t, -k1,1 -k2,2 -k4,4` (timestamp, then server, then key) for the main CSV. For the "latest state" script, group by (server,key) and keep only the row with the max timestamp per group — an easy way is to sort descending by timestamp then take the first match per (server,key) pair with awk, then re-sort the result alphabetically by server,key for final output.
