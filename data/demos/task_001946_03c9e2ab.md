Do not assume you know what the anomaly looks like — derive it empirically from the labeled samples. Load each `.h5` file with `h5py`, extract the `rf_signal` dataset (2000 samples, 2000 Hz), and compute its magnitude spectrum with `numpy.fft.rfft`. Average the spectra separately across all clean files and all evil files, then subtract/compare the two average spectra bin-by-bin to find which frequency bin(s) consistently carry excess energy only in the evil set — that is the injected anomaly. Once you've located the discriminating bin, pick a threshold on the energy (or amplitude) at that bin that cleanly separates the clean and evil training samples (e.g. midpoint between the max clean value and min evil value, with margin). Write `detector.py` to FFT the input file the same way, check that bin against your threshold, and exit 1 if it exceeds it, else 0. Validate against all provided clean/evil files before considering it done, since the hidden holdout requires 100% accuracy — a threshold overfit to only a couple of examples will likely fail edge cases, so build in a safety margin.
