Work in two independent passes and only merge at the end. First, derive screen-active windows from the video: run `ffmpeg -vf fps=1` to dump one frame per second, load each with PIL/numpy, compute mean pixel intensity, and mark a second "active" if brightness > 10; group consecutive active seconds into windows (expect a handful of ~5s windows separated by ~1s blackouts) — do not hardcode window boundaries, derive them from the actual frames. Second, load the JSONL and, for each record, assign it to the video-derived window whose range contains its (jittered) timestamp rather than trusting the raw timestamp directly — this is the key fix for the retry-duplication bug. Within each window, group by `loc_key` and keep only the entry with the longest `text` string, discarding the rest. Build `/stats` from the count of windows and the count of surviving records, and `/data` as those records sorted by timestamp. Finally implement the server with Python's built-in `http.server` (or Flask) bound to 127.0.0.1:8080, launched in the background (e.g. nohup + &) so it stays alive for the verifier, and confirm with a quick curl before finishing.
