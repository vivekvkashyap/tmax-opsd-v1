Build the pipeline as two clearly separated stages. In `prepare_data.py`: read the FASTA file, strip the header line and join remaining lines (removing newlines) into one contiguous reference string; read `candidates.txt` line by line preserving order, and keep only candidates that are exact substrings of the reference (plain `in` operator suffices — no alignment tooling needed, and this naturally filters out things like a homopolymer run that isn't present). For each surviving candidate compute length L and GC-content as count('G'+'C')/L. For the stability index, define `f(x) = math.exp(x/L) + x*math.log(GC+0.1) - 10` and pass it to `scipy.optimize.fsolve` with `x0=1.0`, taking `fsolve`'s returned array's first element as the root — don't just do a manual Newton step, since fsolve handles the nonlinearity robustly. Round GC and x to exactly 4 decimals before writing the CSV with the exact header `sequence,L,GC,x`, preserving input order. Then write `run_pipeline.sh` as a thin wrapper (`#!/bin/bash\npython3 /home/user/prepare_data.py`) and `chmod +x` it, and actually execute it to confirm the CSV is produced correctly before finishing.
