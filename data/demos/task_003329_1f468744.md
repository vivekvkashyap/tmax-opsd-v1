Read `upload_handler.c` first: it builds an output path by directly concatenating an attacker-controlled filename (from an env var acting as an HTTP header) into a fixed directory prefix with `snprintf`, with no check for `..` or absolute paths — that's a classic path traversal, CWE-22; write that ID to the report file. For the mTLS material, generate a key and CSR with the required CN and sign it with the provided CA in one pass, e.g. `openssl req -newkey rsa:2048 -nodes -keyout client.key -out client.csr -subj "/CN=admin_inspector"` then `openssl x509 -req -in client.csr -CA ca.crt -CAkey ca.key -CAcreateserial -out client.crt -days 30`; verify with `openssl verify -CAfile ca.crt client.crt`.
