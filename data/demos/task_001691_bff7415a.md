Break this into three independently testable stages rather than one monolithic script. Stage 1 (active keys): use `jq` to flatten `settings.json` into dotted leaf paths (walk the object recursively, e.g. `jq -r 'paths(scalars) | join(".")' settings.json`), use `xmlstarlet` or a recursive XML walk to get dotted leaf-node paths from `system.xml`, and read the `key` column values from `env.csv` with `awk -F, 'NR>1{print $1}'` — normalize every extracted key the same way: lowercase, dots→underscores, trimmed. Stage 2 (history): filter `history.log` for `[UPDATE]` lines with `grep`/`awk`, extract the bracketed raw key and the `YYYY-MM` prefix of the date, apply the identical normalization function, then aggregate counts per (key, month) — a `awk` associative array or a `sort | uniq -c` pipeline both work. Stage 3: intersect against the active-keys set from Stage 1 before emitting, dropping any month with zero updates, and build the final nested JSON with `jq -n --argjson` or by constructing it programmatically to guarantee valid JSON syntax. Test each stage's intermediate output independently before wiring them together.
