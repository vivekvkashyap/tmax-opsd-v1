Treat this as a multi-service integration: start by launching the pre-existing services with `/app/start_services.sh` and confirming Redis, the ingestion API (8001), and webhook API (8002) each respond directly before touching Nginx, so you can isolate config issues from app issues. Edit the Nginx config (`/etc/nginx/nginx.conf` or the site config under `/app/nginx/`) with two `location` blocks under the `:8080` server — `location /ingest { proxy_pass http://127.0.0.1:8001; }` and `location /results { proxy_pass http://127.0.0.1:8002; }` — then run `nginx -t` to validate syntax and start/reload it as a daemon. Write `/app/worker.sh` as a loop: `redis-cli blpop incoming_matrices 0` to block-pop a JSON matrix, feed it to an inline Python (via heredoc or `python3 -c`) that reconstructs the matrix with `numpy`, computes singular values via `numpy.linalg.svd`, loads the baseline CSV, and computes `scipy.stats.wasserstein_distance` and `scipy.stats.ks_2samp` between the two singular-value sets — reuse scipy's own functions rather than reimplementing the statistics, since the verifier checks against scipy's exact output. Set `drift_detected` true iff the KS p-value < 0.05, then `curl -X POST` the JSON to `http://localhost:8002/report` with header `Authorization: Bearer secret-ml-token`. Launch the worker in the background and leave all services running.
