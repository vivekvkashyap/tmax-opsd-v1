The core technique is to make the deploy script data-driven instead of hardcoded: parse `/home/user/nginx/nginx.conf`, extract just the `upstream backend { ... }` block, and pull out each `server unix:<path>;` line's path (e.g. with `awk`/`grep -oP` or `sed`), preserving the order they appear in the file — don't sort or dedupe them. Loop over the extracted paths in that order; for each one, launch `/home/user/bin/qemu_mock.sh <socket_path> &`, immediately capture `$!` as the PID, and append the log line `DEPLOYED PID <pid> TO <socket_path>` to `/home/user/deployment_final.log` — do this logging step *before* the `sleep 1`, not after, since the requirement is staged/sequential startup with health-checking pauses between launches. Finally actually execute the fixed `/home/user/deploy_vms.sh` so the background processes and log are real, not just written.
