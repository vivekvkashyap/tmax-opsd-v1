First get `/app/loc-processor-1.0.0` compiling: run `make`, read the actual failure (a broken `OBJS`/`CFLAGS` line in the Makefile is a common culprit — check for stray preprocessor defines that silently disable logic, and for object files missing from the link command), fix the Makefile, and confirm `./loc_tool --dedup input.csv` produces sane deduplicated output on a small hand-made test CSV before moving on. For `format_translations.c`, write a strict CSV field parser that treats consecutive commas as an empty field (and skips emitting a row for empty translations), and implement hour-bucketing by parsing the timestamp components and zeroing out minutes/seconds (don't just string-truncate, since that breaks on malformed input) — feed it deliberately malformed/edge-case timestamps and empty fields yourself to see how it behaves before trusting it. Since correctness is judged by exact stdout match against a hidden reference binary across many random inputs, favor faithfully reproducing standard C library formatting (e.g., `strftime`) over inventing your own string logic, and test with a wide variety of self-generated CSV rows, not just the happy path.
