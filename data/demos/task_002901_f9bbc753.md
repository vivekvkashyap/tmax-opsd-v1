Treat this as two independent pieces of work. For config: edit `/app/nginx.conf`'s `listen`/`root` directives to 8080 and `/app/docs/`, add `disable_symlinks off;` inside the relevant server/location block (needed because Nginx by default refuses to follow symlinks outside the root by checking each path component), and update `/app/watcher.conf`'s `PROCESS_SCRIPT` to point at `/home/user/parse_docs.sh`, then make it executable. For the parser: read all of stdin, split into records on blank lines (`awk`'s `RS=""` or a bash loop tracking blank-line boundaries), and for each record extract `PATH:`, `AUTHOR:`, `TAGS:` fields tolerating arbitrary whitespace around `:` and `,` — use `sed -E 's/^\s*KEY:\s*//'` or awk field splitting rather than assuming single spaces. For each tag, `mkdir -p` its directory under `/app/docs/by-tag/`, wrap the symlink-creation critical section with `flock` on `/app/docs/.lock` to avoid races, `ln -sf` the absolute source path to `by-tag/<tag>/<basename>`, and print the exact `[AUTHOR] processed [filename] with [N] tags` line per record before moving to the next.
