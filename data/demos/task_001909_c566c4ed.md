Set up the Rust project with `chrono` and `csv` crates. Parse each `timestamp` value by first trying RFC 3339 (`DateTime::parse_from_rfc3339`), and if that fails, treat the string as an integer and build a `DateTime<Utc>` from Unix seconds — don't assume one format applies to the whole file, since rows are mixed. Truncate to the minute by zeroing seconds/nanoseconds (e.g. `dt.with_second(0).unwrap().with_nanosecond(0).unwrap()`), then group by `(minute_bucket, locale)` and average `render_time_ms`, rounding to 1 decimal. The critical step most naive solutions miss is the resampling/gap-fill: after aggregating, compute the global min and max minute buckets across ALL rows, enumerate every minute in that inclusive range, cross it with the full set of distinct locales seen anywhere in the file, and emit a row for every (bucket, locale) pair — defaulting to `0.0` when no aggregate exists for that pair. Sort the final rows by bucket ascending then locale ascending before writing CSV with the exact header and ISO 8601 `Z`-suffixed timestamp format.
