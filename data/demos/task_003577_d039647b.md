Build this as an explicit multi-pass graph algorithm in memory rather than trying to be clever with one loop. First load all JSONL records and build two structures: a forward map (paper id -> list of ids it references) and its reverse (paper id -> list of ids that cite it, derived by inverting the references). "Influential" papers are those whose reverse/citing-set has size >= 3 AND every paper in that citing set itself has a nonempty reverse/citing-set (size >= 1) — compute this as a straightforward two-level lookup over the reverse map, not a database query. For each influential paper, gather the `topics` lists of its direct citers, flatten them, and count frequency with a dict/Counter; to pick the top 2, sort candidates primarily by descending count and break ties by ascending alphabetical topic name — implement this as an explicit sort key `(-count, topic_name)` rather than relying on insertion order, since naive `Counter.most_common()` doesn't guarantee alphabetical tie-breaking. Serialize the resulting `{paper_id: [topic1, topic2]}` mapping to the required JSON path with `json.dump`, keeping key order as encountered or sorted (either is fine unless the grader is strict about that too — check the exact spec again before finalizing).
