Structure the concurrent pipeline as: a CSV-reader goroutine parsing `loc_views.csv` and sending `{date, lang, views}` structs on a channel; a JSON-reader goroutine decoding the array of wide-format edit objects (unmarshal into `map[string]interface{}` or a struct with pointer fields so you can distinguish `null`/missing from a real 0) and, for each object, fanning out one `{date, lang, edits}` message per language key found, treating `null` or absent as 0; a merger goroutine that ranges over both channels (e.g., with a `sync.WaitGroup` plus a shared `map[string]map[string]*struct{views,edits int}` guarded by a mutex, or a single select loop merging two channels into one aggregation map) keyed by `(date, lang)`, incrementing whichever field arrives; and a writer goroutine that, once both readers finish (close their channels), sorts the map's keys by date then lang and writes the CSV. The key correctness trap: a `(date, lang)` pair must appear in the output with 0 for whichever metric is missing if it exists in *either* source that day — so build the output key set as the union of all `(date,lang)` pairs seen from both files, not just one. Truncate RFC3339 timestamps to `YYYY-MM-DD` with `time.Parse(time.RFC3339, ...).Format("2006-01-02")`.
