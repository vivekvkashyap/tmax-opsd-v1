Start by trying `make` in the vendored library directory and read the compiler error carefully — with `-Werror=implicit-function-declaration` enabled, an implicit declaration of a heap function like `malloc`/`free` means a standard header is missing from the .c file; add the missing `#include` and rebuild until `libwalparse.a` is produced. Study `walparse.h` to understand the exact signature and ownership semantics of `parse_wal_frame` (it allocates `*data`, caller must `free` it) before writing any C++. In your driver, loop calling `parse_wal_frame(stdin, &page_id, &data, &len)` until it returns 0; for each frame check `page_id % 2 == 0`, compute a running `uint8_t` XOR over the `len` bytes of `data`, then `fwrite` the 4 raw bytes of `page_id` (as parsed, already little-endian) followed by the 1-byte XOR sum — do not reformat or re-encode `page_id`. Free `data` every iteration, including on frames you skip, to avoid leaks the fuzz harness may catch via timing/memory. Compile with `g++ archiver.cpp libwalparse.a -o archiver` (static link), and test manually by piping a handcrafted binary stream before trusting it against the hidden fuzz corpus.
