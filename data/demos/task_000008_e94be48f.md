Design the C program as three clear phases: parse-and-validate, join, and serialize. When tokenizing each CSV line (skip header), use `strtoul`/`strtof`/`strtoull` (or `sscanf` with careful field counts) and always check the parse actually consumed valid input and the endptr / return count matches expectations — a field like "bad_val" or "invalid" must cause that whole record to be dropped, not silently become 0. Build an in-memory hash map or sorted array of sensors keyed by `sensor_id` (parsed as `uint32_t`) so you can do an O(1) or O(log n) lookup per reading rather than an O(n*m) nested scan; drop any reading whose sensor_id isn't found. Compute `adjusted_value = raw_value * calibration_factor` and reject if it's `<= 0.0` or fails `isnan()`/`isfinite()` checks. When writing binary records, `memset` the struct to zero first, then `memcpy` the status string in (truncated/padded to 8 bytes) so you don't leak stack garbage into the padding — this matters for exact byte-level verification. Have the bash wrapper compile with `gcc`, run the binary, and count/write the final line count with something like `wc -l` on your own success log or a counter the C program itself prints.
