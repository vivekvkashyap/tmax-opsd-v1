This is a video-OCR-to-graph-service pipeline. First extract frames from `/app/backup_monitor_incident.mp4` with `ffmpeg` (e.g. one frame per second or per detected scene change), then run `tesseract` on each frame to recover text lines matching `Job: <JobID> | DependsOn: <ParentJobID> | Size: <Bytes>`. Parse all recovered lines into (job, parent, size) tuples, then deduplicate identical (job, parent) edges before building the dependency graph — the video intentionally repeats some frames/lines to simulate a "ghost cross-join" glitch, so naive parsing without dedup will inflate dependency counts. Build an adjacency structure, compute in-degree (number of times each job appears as someone's DependsOn target) to rank criticality, and expose it via a small C++ HTTP server (httplib or Boost.Beast) bound to 127.0.0.1:8080 with `/api/backups`, `/api/critical`, and `/api/summary` matching the exact JSON schemas in the task. Sum `size` fields for total_size, and count unique deduplicated edges for total_dependencies. Compile, launch the server as a background process, and confirm it stays up (don't let it exit when the terminal session ends) before finishing.
