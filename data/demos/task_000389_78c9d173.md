This has three independently-testable pieces — verify each in isolation before assuming the whole pipeline works. (1) Port forward: launch `socat TCP-LISTEN:8080,fork,reuseaddr TCP:127.0.0.1:9090 &` (or nohup/disown it) so it survives, then curl 8080 yourself to confirm it reaches the 9090 backend. (2) Deploy pipeline: `git init --bare /home/user/dashboards.git`, write `deploy.sh` to clone/pull into a workspace, find `*.json` files, and `curl -X POST -H "Content-Type: application/json" --data @file url` each one, chmod +x it, and add it to `crontab -e` with a `* * * * *` line — test deploy.sh manually first before trusting cron. (3) The pre-receive hook is the core logic: read `oldrev newrev refname` lines from stdin in a while-loop, for each ref diff the changed files with `git diff --name-status $oldrev $newrev` (treating `oldrev` of all-zeros as "diff against empty tree"), skip deleted files, and for each surviving `.json` path use `git cat-file -p $newrev:$path | jq` to check the two rules — reject (non-zero exit) on any violation. Test the hook locally by pushing crafted clean/evil JSON files to the bare repo yourself before finishing.
