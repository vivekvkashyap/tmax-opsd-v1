Structure process_configs.sh as a pipeline: (1) list `*.jsonl` files and fan them to a bounded worker pool (`xargs -P N` or GNU `parallel`), each worker running `jq` to pull `server_id`/`timestamp`/`config_data` — let `jq` itself emit the decoded UTF-8 (its `\uXXXX` handling is automatic on output), don't hand-roll escape decoding; (2) hash each decoded `config_data` with sha256sum, then for rows sharing a hash keep only the one with the chronologically earliest ISO-8601 timestamp (sort by hash then timestamp, take the first per group); (3) bulk-load the deduped rows into SQLite with `hash` as PRIMARY KEY via `sqlite3 db "INSERT ..."` or a `.import` of a prepared CSV; (4) separately aggregate ALL raw lines (before dedup) by hash to build stats.csv, sorted by count descending then hash ascending; (5) install the cron entry via `(crontab -l -u user 2>/dev/null; echo "0 * * * * /home/user/process_configs.sh") | crontab -u user -`. Run the script once manually afterward so the DB and CSV exist for verification.
