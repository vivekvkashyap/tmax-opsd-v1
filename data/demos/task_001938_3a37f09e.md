For `normalize.py`, read the input WAV with Python's `wave` module, unpack the 16-bit PCM samples (`struct.unpack` or `array('h', ...)`), find the current peak absolute amplitude, compute `scale = 26214.0 / peak` (handle peak==0 as a no-op), multiply every sample by that scale, round to nearest integer, clip to the int16 range, and write out a new WAV with the same channel count/sample width/frame rate using `wave.open(..., 'wb')` — the grading is MSE-based against an independently recomputed scaled reference, so precision in rounding and using the exact peak-based scale factor (not a fixed gain) matters. For the health service, a minimal `http.server` subclass handling only `GET /health` with a 200 and body `MIGRATION_READY` is sufficient; run it backgrounded. For port forwarding, use `socat TCP-LISTEN:8080,fork,reuseaddr TCP:127.0.0.1:9090 &` so 8080 transparently proxies to 9090. Then run `verify.sh` (`curl -s http://127.0.0.1:8080/health > /home/user/health_status.log`) once the forward and service are both confirmed listening (check with e.g. a short retry loop) so the log captures the real response, not a connection-refused error.
