First reverse-engineer the schema with `sqlite3 legacy_telemetry.db ".schema"` (or `.tables` + `.schema <table>`) to learn the real column names for id, device, timestamp, and reading — don't assume generic names. Install `libsqlite3-dev`/`sqlite3` if missing. In the C program, use `sqlite3_prepare_v2` with a `?` placeholder and `sqlite3_bind_double` for the threshold (never string-concatenate it — that's the injection risk being tested). Compute the moving average with a SQL window function: `AVG(reading) OVER (PARTITION BY device ORDER BY timestamp ROWS BETWEEN 2 PRECEDING AND CURRENT ROW)` inside a CTE, then filter the outer SELECT on `moving_average > ?`. Order results deterministically (e.g. by device, timestamp) so output is stable. When writing the CSV, format `reading` and `moving_average` to exactly 2 decimal places with `printf("%.2f", ...)`, and match the header line exactly. Compile with `gcc -o extractor extractor.c -lsqlite3`, then run it with the 50.00 threshold to produce `alerts.csv`.
