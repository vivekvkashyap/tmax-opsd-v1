Model this explicitly as a directed graph rather than trying to do it with ad-hoc CSV scans. Load nodes.csv into a dict keyed by id (with name and budget), and build an adjacency list plus an in-degree count from edges.csv by iterating source/target pairs. A Director is any node with in-degree 0 and out-degree ≥ 1 — compute both degree maps up front rather than inferring them mid-traversal. For the Network Budget, do a bounded BFS/traversal from each director that tracks hop distance and stops including nodes once hop distance exceeds 2 (director itself is hop 0, direct reports are hop 1, their reports are hop 2) — the common bug is either using unbounded DFS (pulling in hop-3+ nodes) or double-counting a node reachable via two different length-2 paths, so dedupe visited nodes by id per director. Sum budgets of unique nodes within the hop limit including the director's own budget. Write the result as `{name: int_total}` to the JSON output, double-checking budgets are summed as integers not strings.
