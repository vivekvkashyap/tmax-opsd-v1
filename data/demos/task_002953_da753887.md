Structure `build.sh` as: (1) parse `manifest.txt` lines into `VERSION` / `PATCH_FILE` pairs, (2) filter to versions where `1.0.0 < version <= target`, (3) sort the filtered set in true semver order — do not rely on plain `sort` or `sort -V` blindly without testing, since two-digit minor/patch numbers (e.g. `1.2.0` vs `1.10.0`) can sort lexicographically wrong; instead split each version on `.` and compare major/minor/patch as integers (a small awk or bash function using `IFS=.` read into three variables, then numeric comparison, works well), (4) copy `base.proto` to `final.proto` and apply each patch in the sorted order with `patch final.proto < patches/<file>`, logging the exact manifest-listed patch path to `applied_patches.log` after each successful apply, (5) set up a venv with `python3 -m venv venv`, activate it, `pip install grpcio-tools`, `mkdir -p out`, then run `python -m grpc_tools.protoc -I. --python_out=out --grpc_python_out=out final.proto`. Test by actually invoking `build.sh 2.0.0` and confirming `applied_patches.log` reflects only patches whose version is in `(1.0.0, 2.0.0]`, in ascending semver order, and that both generated `_pb2.py`/`_pb2_grpc.py` files exist and are non-empty/importable.
