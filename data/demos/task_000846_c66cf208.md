Build the RDF graph with `rdflib`: define your namespace, create a `URIRef` per city as `NS.City/<name>`, and for each connection add a triple `(city_from, NS.connectedTo, city_to)` plus a way to attach the cost — reify the connection or attach `NS.cost` as an `XSD.integer` Literal on an intermediate edge node, whichever lets SPARQL recover both endpoint and cost. Serialize with `graph.serialize(format='turtle')`. In `api.py`, load the graph once at startup; implement `/sparql` via `graph.query()` and serialize results to JSON; implement `/shortest-path` by running an internal SPARQL SELECT over all connections, populating a `networkx.DiGraph` with `cost` as edge weight, then using `nx.dijkstra_path`/`dijkstra_path_length`. For nginx, configure `listen 127.0.0.1:8080`, `proxy_pass http://127.0.0.1:8000`, and `auth_basic`/`auth_basic_user_file /home/user/.htpasswd` — generate the htpasswd entry with `openssl passwd -apr1` (nginx runs as the invoking user, no root needed, so avoid `user`-directive privilege-drop errors). Start uvicorn and nginx in the background, curl both endpoints yourself (with and without auth) to confirm behavior, then write the "SERVICES READY" log line.
