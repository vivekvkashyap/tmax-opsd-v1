For the C archiver, the core technique is content-based deduplication: on `pack`, read each PNG fully into memory (or hash it, e.g. with a simple checksum or full byte comparison), and before storing a new data block, compare against previously stored unique blocks — if identical, just record a metadata entry (filename + pointer/index into the existing block) instead of writing the bytes again; since many adjacent frames from a static camera will be byte-identical, this is what gets you under the 0.25 compression ratio target. Design your archive format with a header/index section (filenames + block index + block offsets/lengths) followed by the deduplicated raw data blocks. `unpack` must reconstruct every original file (including duplicates) by copying the referenced block's bytes to each of its filenames. `verify` should re-read structure/headers (and ideally checksum each block) without writing anything to disk, returning 0 only if everything is structurally consistent. Test round-trip correctness (`pack` then `unpack`, diff against `raw_frames`) before worrying about the ratio.
