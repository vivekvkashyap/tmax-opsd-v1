Start by treating the legacy binary as a black box you probe interactively, not something you reverse-engineer statically. Set up the directory chain first (`mkdir -p /home/user/configs/v2_migrated`, write `secret.txt` with the given string, `ln -s` the `active` symlink to it) since the binary reads its config unconditionally at startup and will misbehave or refuse connections if the file is missing. Launch `/app/legacy_auth` and use `nc 127.0.0.1 9000` or a tiny script to send candidate lines and observe whether you get `VALID`/`INVALID` — this tells you what transformation of the secret it expects (e.g., try the raw secret, then common hashes of it) without needing to disassemble the binary. Once you know the correct token to send, write the Go proxy: use `net/http` with a custom handler that opens a `net.Dial("tcp", "127.0.0.1:9000")` per request, writes the `X-Auth-Token` header value plus `\n`, reads the response line, and either reverse-proxies (via `httputil.NewSingleHostReverseProxy` or a manual client call) to `127.0.0.2:8081` on VALID or returns 401 JSON on INVALID/connection failure. For the metrics port, run a second goroutine with its own `net.Listen` on 8082 that parses `STATS\n` and reports an atomically-incremented success counter. Test the whole chain manually with curl before finalizing `deploy.sh`, which should build, set up symlinks, background both processes, and verify the ports are listening before exiting 0.
