For the C tool, use zlib's `gzopen`/`gzread`/`gzclose` to stream-decompress each `.gz` file in chunks (don't assume it fits in one buffer), feeding each chunk incrementally into OpenSSL's `SHA256_Init`/`SHA256_Update`/`SHA256_Final`, then print the 32-byte digest as lowercase hex (loop over bytes with `%02x`) followed by a newline; compile with `gcc -o zsha256 zsha256.c -lz -lcrypto`. For traversal, the key pitfall is symlink loops in `/home/user/repo/` — use `find /home/user/repo -type f -name '*.gz'` (the default `-type f` without `-L` does NOT follow symlinks, so it naturally skips symlinked directories/loops) rather than manually recursing or using tools that dereference symlinks by default. For each real `.gz` file found, capture its path relative to `/home/user/repo/` (e.g. with `realpath --relative-to`), run `zsha256` on it to get the hash, copy the original compressed file to `/home/user/curated/<hash>.gz`, and append `<hash> <relative_path>` to the manifest. Finally sort the manifest alphabetically in place with `sort -o manifest.txt manifest.txt`. Test explicitly that the symlink loop directory is excluded and doesn't hang or get processed.
