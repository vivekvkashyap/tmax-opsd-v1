Run `file`/`strings`/objdump (or a disassembler) on `/app/tools/payload_decoder.elf` to confirm it performs percent-decoding, and check whether it decodes once or twice (double URL-encoding) — this matters because the evil corpus hides payloads behind nested `%25..` encoding that a naive single-decode filter will miss. Then diff a handful of files from `/app/corpus/evil/` against `/app/corpus/clean/` to see which decoded parameter (e.g. a redirect target) differs and what marks it malicious (e.g. resolving to an absolute external host vs. a relative path). Implement `/home/user/waf_filter.c` so it: reads the raw HTTP request file, extracts the relevant parameter, fully decodes it (looping decode until no more `%XX` sequences change the string, to catch double-encoding), then classifies based on the decoded value (e.g. absolute-URL/external-host redirect = EVIL, relative path = CLEAN). Compile with `gcc -o /home/user/waf_filter /home/user/waf_filter.c` and iterate by running the binary against every file in both corpora, printing mismatches, until you hit 100% accuracy on both sets — don't just eyeball a few samples and assume it generalizes.
