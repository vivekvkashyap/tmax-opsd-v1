Use `jq` for JSON validity filtering since it will hard-fail (non-zero exit) on lines with malformed unicode escapes, letting you cleanly separate good from bad lines: pipe each line through something like `jq -e . >/dev/null 2>&1` in a loop, or `jq -R 'fromjson?'`/`jq -c 'select(.timestamp and .user and .action)'` reading line-by-line with `while IFS= read -r line`, keeping only lines that parse. For valid lines, extract `timestamp`, `user`, `action` with `jq -r`, truncate the timestamp to the `YYYY-MM-DDTHH` substring (first 13 characters), then aggregate counts per (hour,user,action) triple — `sort | uniq -c` or an associative array in awk/python works well. Sort the aggregated rows by hour, then user, then action (`sort -t, -k1,1 -k2,2 -k3,3`), write the header plus rows as CSV to `/home/user/summary.csv`, and in the same sorted order generate one `INSERT INTO hourly_stats (hour, username, action, event_count) VALUES (...);` line per row into `/home/user/insert.sql`, matching the literal SQL syntax and quoting exactly.
