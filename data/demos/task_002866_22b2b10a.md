Start by running `make -C /app/simple-elf-validator-0.1.0` to see the actual linker error — it will show `undefined reference to std::...`, which is the classic symptom of compiling/linking C++ object files with the C compiler (`gcc`) instead of `g++`/`$(CXX)`. Open the Makefile and check how `CC`/`CXX` are defined and used in the compile and link rules; fix it so C++ sources are compiled and linked with a C++ compiler (e.g. change `CC=gcc` to use `g++`, or make the link rule use `$(CXX)` instead of `$(CC)`), then rerun `make` until `libelfval.a` is produced. For the classifier, write a minimal `main` that takes `argv[1]`, calls `is_valid_elf(argv[1])` from `elf_validator.h`, and returns exit code 0 or 1 accordingly — keep it simple, all real logic lives in the library. Compile with `-I` and `-L` pointing at the library directory and `-lelfval`. Before trusting your build, unzip `/app/corpus.zip` and manually run your classifier against several files in `clean/` and `evil/`, checking `$?` after each, to sanity-check both compile correctness and classifier behavior against edge cases like truncated files.
