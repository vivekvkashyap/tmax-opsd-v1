Break this into independent stages and verify each before moving on. (1) Install the Ubuntu HDF5 dev packages (e.g. `libhdf5-dev`, plus `pkg-config`) before touching Cargo, since the HDF5 Rust bindings link against the system library and will fail to build otherwise; pick Rust crates for WAV I/O, FFT/STFT, and HDF5 (e.g. hound, rustfft, hdf5). (2) Implement STFT manually: slide a 1024-sample Hann window with 256-sample hop over the PCM samples (normalize i16 to f32 first), FFT each windowed frame, and keep both magnitude and complex bins per frame. (3) For each frequency bin, compute the standard deviation of magnitude across all time frames, then slide a 50-bin window over that std-dev vector and pick the window position with the highest mean std-dev — this is a simple sliding-window-average search, not a single global max. (4) Zero out every complex STFT bin outside that winning 50-bin range (leave the ones inside untouched), then inverse-FFT each frame and overlap-add with the same hop size, normalizing for window overlap, to reconstruct the time-domain signal; write it out as 16-bit PCM WAV matching the input's sample rate. (5) Separately dump the full unfiltered magnitude spectrogram and the std-dev vector as named HDF5 datasets. Test end-to-end by listening to/inspecting cleaned.wav for burst-like energy in the retained band before considering it done.
