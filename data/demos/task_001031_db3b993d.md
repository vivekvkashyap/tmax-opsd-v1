First run `/app/start_services.sh` to bring up the nginx and backend health endpoints on 127.0.0.1:8080/8081 before testing. Write `/app/monitor.go` using the standard `net/http` client to GET `http://127.0.0.1:8080/health` and `http://127.0.0.1:8081/health`, capturing each response's `StatusCode`, then write them to `/app/status.txt` in the exact format `nginx: <status>, api: <status>`. The key trick for binary size is the build flags, not the Go code itself: compile with `go build -ldflags="-s -w" -o /app/monitor /app/monitor.go` to strip symbol table and DWARF debug info — a plain `go build` will typically produce a binary well over 3MB, so verify the resulting size with `stat -c %s /app/monitor` and iterate (e.g. also consider `-trimpath` or disabling CGO) if still too large. Finally add a crontab entry (preserving idempotency) via `/app/crontab.conf` containing `* * * * * /app/monitor` and apply it with `crontab /app/crontab.conf`, then confirm with `crontab -l`.
