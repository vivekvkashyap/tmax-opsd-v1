Build the pieces in dependency order: monitor.py first, then the supervisord config, then the idempotent wrapper script. In `monitor.py`, use `http.server.BaseHTTPRequestHandler` with a custom `do_GET` that dispatches on `self.path`: for `/health` read `os.environ.get('TZ')`/`os.environ.get('LANG')` and emit exact-format JSON via `json.dumps`; for `/users` read the file directly (return 404 if `os.path.exists` is false), otherwise stream its raw bytes back unmodified. In `deploy_monitor.sh`, use `mkdir -p` for the config dir, and write `users.json` and `supervisord.conf` with `cat > file <<EOF ... EOF` (overwrite is fine, that's what keeps it idempotent) — put `environment=TZ="America/St_Johns",LANG="fr_CA.UTF-8"` and `autorestart=true` in the `[program:health_monitor]` block. For idempotent startup: check if `supervisord.pid` is a live process (e.g. `pgrep -f "supervisord.*supervisord.conf"` or reading the pidfile and testing `kill -0`); if not running, start with `supervisord -c ...`; if running, use `supervisorctl -c ... update` (or `reread && update`) instead of starting a second instance, which is what would break idempotency/port binding. Finish with a `curl` loop/retry until the port answers, then write that body to the log file.
