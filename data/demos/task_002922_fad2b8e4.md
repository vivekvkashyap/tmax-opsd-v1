Work through the C program logic carefully with pen-and-paper on the sample data before coding, since the windowing and dedup rules are easy to get subtly wrong. Parse each line by extracting the bracketed epoch, the KEY=VALUE pair, and the trailing IP with simple string/sscanf parsing. Compute the window as integer division `(epoch / 3600) * 3600` (integer, not floating-point, division — a common bug is using float division and getting an off-by-one bucket). Use a hash map (or nested structure) keyed by `(window, key)` and, for each incoming record, only overwrite the stored entry if the new epoch is strictly greater than the stored one, so you keep the latest per window per key. Mask the IP by finding the last `.` and replacing everything after it with `XXX` (do not assume IPv4 octets are always 1-3 digits — locate the last dot dynamically with `strrchr`). Output CSV lines `window,key,value,masked_ip` in any order from the C program, then let the bash script pipe raw log -> compiled binary -> `sort -t, -k1,1n -k2,2` (numeric on the window field, alphabetic on the key field) into the final file. Manually verify a couple of your own computed windows against the raw timestamps before trusting the full pipeline.
