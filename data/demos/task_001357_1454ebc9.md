Implement `bup_filter.c` as a strict binary-protocol parser reading raw bytes from stdin with `fread`/`read` — never use line-buffered stdio functions like `fgets` for the header/length/checksum since the payload is arbitrary binary. Read exactly 4 magic bytes and compare against `BKA\x01`; read the next 4 bytes as a little-endian `uint32_t` length `L` (assemble byte-by-byte to avoid endianness/alignment bugs, don't cast a pointer); read exactly `L` payload bytes, tracking short reads/EOF explicitly to detect truncation; read the final 4-byte little-endian checksum and compare against the running sum of payload bytes mod 2^32. Only after all three checks pass, scan the payload for `\n`-delimited lines and write to stdout every line except those starting exactly with `SECRET:`, preserving line boundaries. Test the filter standalone with printf/xxd-crafted byte streams for each of the four error paths (bad magic, truncation, bad checksum, valid) and check exit codes before wiring up the proxy. For Phase 2, use `socat TCP-LISTEN:7070,reuseaddr,fork SYSTEM:'/home/user/bup_filter | socat - TCP:localhost:9090'` (or an equivalent nc pipeline) inside `start_proxy.sh`, launch it backgrounded, and confirm with a manual test connection that data flows end-to-end before finishing.
