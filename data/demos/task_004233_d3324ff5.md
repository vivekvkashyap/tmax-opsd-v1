Work through this as a pipeline of small, verifiable steps. First extract `TargetEventType` from the `[Archival]` section of `/app/config/backup.ini` with `awk`/`sed` (locate the `[Archival]` header, then grep the `TargetEventType=` line under it, not just anywhere in the file). Second, parse `/app/logs/system_events.log`'s multi-line records (split on blank lines or `[Event ID:` markers) and find the block whose `Type:` line equals the event type you extracted; pull its `Date:` value from that same block. Third, search `/app/recordings/*.wav` for the file whose `stat -c %Y` (mtime) falls within 60 seconds of that timestamp converted to epoch (`date -d "..." +%s`) — don't just pick the first .wav file. Create `/app/archive/` and hard-link (`ln`, not `cp`) the matched file in with its original name. Finally install a real speech-to-text tool (e.g. `pip install openai-whisper` or build whisper.cpp) and run it against the hard-linked audio, saving only the plain transcribed text (no timestamps/metadata) to `/app/archive/transcript.txt`; verify the output resembles natural spoken English before finishing since accuracy is scored against a hidden reference transcript.
