Do the cleanup and config edit first since they're static: use `find /home/user/project -name '*.tmp' -size +10k -delete` (careful: `-size +10k` in GNU find means strictly greater than 10KB in 1024-byte blocks — verify size semantics match "strictly larger than 10KB", using `-size +10240c` if you need byte precision), and `sed -i 's/^LOG_LEVEL=DEBUG$/LOG_LEVEL=INFO/' config.ini`. For the live log watcher, since `live.log` is being appended to by a background process, open the file and read incrementally (e.g. seek to end then poll for new lines, or just read the whole growing file in a loop) rather than reading it once — records start with a line beginning with `[` and continue until the next line starting with `[`, so buffer lines into the current record and flush/start a new one whenever a line starts with `[`. Only keep records whose first line starts with `[ERROR]`; once you've captured 3 full multi-line ERROR blocks (each including its traceback continuation lines), write them to `errors_found.log` and exit. Pitfall: reading the log file once at a fixed point instead of continuously polling will likely miss enough ERROR records since they arrive over several seconds — poll with short sleeps until 3 are captured.
