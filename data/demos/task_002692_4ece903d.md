Build and smoke-test each polyglot piece in isolation before wiring the integration test. Compile the shared library with `gcc -shared -fPIC -o libdecoder.so decoder.c` and the sorter with `g++ -o sorter sorter.cpp`; test `sorter` directly by piping a few plain numeric lines into it. Start `api_server.py` in the background (`python3 api_server.py &`) and manually `curl http://127.0.0.1:8080/data` to see the raw JSON shape before writing any Python. In `integration_test.py`, fetch and `json.loads` the response, then load the shared lib with `ctypes.CDLL("./libdecoder.so")` — set `.argtypes`/`.restype` for `decode_payload` (input as `ctypes.c_char_p`, output as a writable `ctypes.create_string_buffer(5)`) since ctypes won't infer C signatures automatically. Decode each payload string, collect the 4-char corrected values, and feed them newline-joined into `subprocess.run(["./sorter"], input=..., capture_output=True, text=True)`. Compare the captured stdout against the contents of `expected_output.txt` — use `difflib` or plain string equality after normalizing trailing newlines, since a mismatched final newline is a common false-diff. Write `PASS` only if truly identical, otherwise write the actual diff text, to `test_result.log`.
