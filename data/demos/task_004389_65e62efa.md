Model this as a small state machine over the GCode lines rather than trying to do it in one regex pass. Read `Z_MIN`/`Z_MAX` out of `experiment.conf` (simple `grep`/`cut` or `awk -F=`). Then process `print_run.gcode` line by line, maintaining a running `current_z` variable that starts at 0.0: for each line, first strip everything from `;` onward and trim trailing whitespace, skip if the result is empty; if the line contains a `Z<number>` token, update `current_z` to that value (this happens whether or not the line is a G0 or G1, and regardless of other params on the line); separately, check if the line starts with `G1` and contains an `E<number>` token — if so, and `current_z` (the value active *after* this line's own Z-update, since Z and E can appear together) is within `[Z_MIN, Z_MAX]` inclusive, keep the comment-stripped, whitespace-trimmed line. Awk is well suited to this since it can hold state across lines. Write kept lines to `filtered_extrusion.gcode` and the line count to `summary.txt`.
