Identify the specific weakness that mechanism exhibits and map it to the correct CWE ID for "use of a non-cryptographic/predictable PRNG for security-sensitive values" — write only that ID to `cwe.txt`. Reimplement the exact same key/IV derivation logic in your own decryption script using the `cryptography` library's AES-CBC decryptor, feed in `vault.enc`, strip PKCS7 padding, and write the result to `vault.tar`; extract it into `/home/user/vault/`. For integrity checking, parse `manifest.sha256` (standard `sha256sum` format: hash then filename) and for each extracted file recompute its SHA-256 and compare — delete any file whose extracted hash doesn't match the manifest entry, leaving only the genuinely intact key(s). Finally copy the surviving `.pem` private key to `target_rsa` and `chmod 600` it, since SSH clients reject keys that are group/world-readable.
