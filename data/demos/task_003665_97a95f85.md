Parse the config with Python's `configparser` to pull `source_dir`, `output_file`, and `on_loop_detect` rather than hardcoding them. The log is multi-line records separated by `--`; split on that delimiter (or read the whole file and use a regex/state machine) and within each block look for a line starting with "Reason: File system loop detected" followed by a "Path:" line — only records with that exact reason count, ignore WARN/INFO blocks even if they mention paths. Collect the offending symlink paths, sort them alphabetically, write one per line (with trailing newline) to `removed_links.txt`, then `os.remove()` (or `os.unlink`) each on-disk — verify with `os.path.islink()` before deleting so you never touch a regular file or a valid symlink. For the archive, use Python's `tarfile` module with mode `w:gz`, adding the source directory with `arcname` set to just the base directory name so the tree root is preserved, and be careful that `tarfile.add` does NOT dereference symlinks by default (don't pass `dereference=True`) so remaining valid symlinks stay as symlinks in the tar. Do the symlink cleanup before creating the tarball, not after.
