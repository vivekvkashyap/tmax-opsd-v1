Work through this as a sequence of small, independently-verifiable steps rather than one big script. First `tar -xzvf` the backup archive into the target directory and inspect its contents before assuming structure. Create the symlink with `ln -s <target> <linkname>` — remember the target is the path the link should point to, and verify with `readlink`. For the env setup, parse the `.env` file line by line (skip blank lines/comments) and write `export KEY=VALUE` lines into `setup_env.sh`; append a `source /home/user/setup_env.sh` line to the end of `.bashrc` so new shells pick it up. For identity mapping, parse the passwd-format file (fields separated by `:`, username is field 1, primary GID is field 4) and the group-format file (group name field 1, GID field 3), then build a username->groupname dict by matching GIDs, and dump it as JSON. For the final verification script, since environment variables set via `.bashrc`/sourcing aren't inherited by a script unless explicitly sourced in the same shell invocation, run it with something like `bash -c "source /home/user/setup_env.sh && python3 /home/user/verify_restore.py"` so `os.environ` actually contains `APP_RESTORE_MODE`. Check the symlink validity with `os.path.islink` and `os.readlink`, comparing against the exact expected target path.
