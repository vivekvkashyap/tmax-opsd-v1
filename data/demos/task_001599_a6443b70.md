Build the C ETL program methodically: parse the input file by reading floats line-by-line into a fixed array while counting N, then call `srand(seed)` exactly once before the bootstrap loop. For each of the `num_bootstraps` iterations, draw N indices via `rand() % N`, sum the sampled values, divide by N to get that sample's mean, and accumulate the mean and its square across bootstraps so you can derive the average bootstrap mean and its standard deviation (population or sample stdev — read the formula carefully) in one pass. Plug the resulting `mu_boot` into the closed-form Bayesian posterior-mean formula using the given prior mean/variance and known sensor variance — do this with double precision arithmetic to avoid rounding drift. Write the CSV line with `fprintf` using exactly `%d,%.6f,%.6f\n` for seed, posterior mean, bootstrap SE. Compile with `gcc etl_processor.c -o etl_processor -lm`. For the reproducibility script, the key insight is that identical `srand(seed)`/`rand()` sequences on the same platform are deterministic, so running the compiled binary twice with the same seed and comparing outputs with `diff` or `cmp` should always succeed — write `REPRODUCIBLE: YES` when they match exactly, `REPRODUCIBLE: NO` otherwise, and make both scripts executable.
