Treat this as building a JWT-inspecting proxy, not a full JWT validator — you never need a signing library since you're only decoding the header/payload to detect malicious patterns. Split the token on `.`, base64url-decode (remember to pad or use `base64.urlsafe_b64decode` with padding handling) the first two segments, and parse as JSON to get `alg` and `iss`. Reject when `alg` case-insensitively equals `none` or is empty; reject when `alg` claims `HS256` but the `iss`/token structure matches one of your known asymmetric-key issuers (define this by checking against a small fixture list, since the task implies key-confusion detection is heuristic, not cryptographic); reject when SHA256(iss) is absent from `/home/user/data/valid_issuers.sha256`. Build the WAF as a simple `http.server`-based reverse proxy: read the incoming request fully, run your checks, then either write a 403 with a logged reason line (format matters — match `[TIMESTAMP] REJECTED: IP=... REASON=...` exactly) or forward via `requests`/`http.client` to the Flask backend and relay its response verbatim. Wire Nginx's `/api/` location to `proxy_pass http://127.0.0.1:8080`, and start Redis, Flask, WAF, then Nginx in that dependency order in your startup script. Test with both a clean and an `alg:none` token through the full stack before finishing.
