Build this as a straight C++ HTTP microservice, not a shell wrapper. Vendor `cpp-httplib` and `nlohmann/json` as single headers (curl/wget them into an include dir), then write one .cpp that: opens the WAV with a plain binary reader, verifies it is 16-bit PCM mono (read `NumChannels`, `BitsPerSample`, `SampleRate` from the standard 44-byte header rather than hardcoding offsets blindly — some WAVs have extra chunks before `data`, so search for the `data` chunk ID instead of assuming byte 44 is always the start), then converts each int16 sample to float via `/32768.0`. For `/features/rms`, compute `window_samples = sample_rate * window_ms / 1000`, iterate in non-overlapping blocks, drop the final partial block, and return `sqrt(mean(s_i^2))` per block as a JSON array. `/tune/threshold` reuses the same RMS computation and just counts windows with `rms > threshold` (strict inequality). Compile with `g++ -std=c++17 -O2 -lpthread`, launch the binary with `&`/`nohup`, and confirm with `curl` that both POST endpoints respond before finishing — a common failure mode is floating-point drift from truncating vs rounding window sizes, so keep window_samples as an integer computed once and reuse it consistently.
