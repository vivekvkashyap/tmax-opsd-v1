First fix the Makefile: a shared library needs both `-fPIC` (position-independent code) and `-shared` on the gcc invocation, plus `-o liblegacy_crypto.so`; a naive attempt that just adds `-shared` without `-fPIC` will build but crash on load on some systems. Second, generate the gRPC stubs with `protoc --go_out=. --go-grpc_out=. service.proto` (check the `go_package` option in the .proto matches your module layout) before touching server.go. Third, before running anything, statically read `legacy_crypto.c` for the buffer overflow: trace every `malloc` size against the actual data written into it (e.g. `strcpy`/`sprintf` of a longer string into a smaller allocation) — this is the classic C bug class causing memory corruption on a 32-byte input; fix the allocation size rather than truncating the output. Build with CGO (`CGO_LDFLAGS`/`LD_LIBRARY_PATH` pointing at the .so dir), run the server, then `go run client.go`.
