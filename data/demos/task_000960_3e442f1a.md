Build this as a two-stage awk pipeline inside the bash script: an extraction stage and a stats stage. First, use `grep`/`awk` to pull only lines containing both `load:` and `latency:` metrics, extracting the numeric values that follow those labels (e.g. with `grep -oP 'load:\K[0-9.]+'` or field-splitting on non-digit characters) — discard any line missing either field rather than treating it as zero. Second, in a single awk pass over the (x,y) pairs, accumulate `n`, `sum_x`, `sum_y`, `sum_x2`, `sum_xy` in the body block, then in an `END` block apply the OLS closed-form formulas exactly as given to get `m` and `b`. For the MSE comparison, since you need per-point residuals you'll need either a second pass (store the pairs in arrays during the first pass, then loop over them in END) or compute sum_y2 in the same pass and derive MSE algebraically from the sums — either works, but be careful that mean-of-y for the null model uses `sum_y/n`, not a naive average that drops rows silently. Format every output number to exactly 2 decimal places with `printf "%.2f"` and hand-assemble the JSON (or use `printf` to build valid JSON directly) since no JSON library is available in pure bash/awk.
