Break this into independent, testable stages so fuzzing against varied log lines is robust. First, OCR `/app/secret_key.png` once (e.g. `tesseract`) to recover the AES key text — hardcode that recovered string as a constant in your script, don't re-OCR at runtime. Write a log-line parser (regex against the query string and headers) that extracts `token=` and `X-Client-Serial`, handling malformed lines by raising/catching and printing `ERROR`. For cert validation, iterate PEM files in `/app/certs/`, parse each with `cryptography.x509`, compare `str(cert.serial_number)` in hex against the header value (normalize case/`0x` prefix), and check `cert.not_valid_after` against now; if no match or expired, print `AUTH_FAILED` and stop. Otherwise base64-decode the token, split off the first 16 bytes as the IV, and AES-128-CBC-decrypt the remainder with the recovered key and that IV, stripping PKCS7 padding, then print the plaintext. Wrap every step in try/except so any unexpected failure prints `ERROR` rather than a traceback, since the harness feeds hundreds of both valid and malformed lines and checks exact stdout equivalence.
