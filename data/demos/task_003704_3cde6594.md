Work this as a strict pipeline, not one script. First install pandas, sentence-transformers, scikit-learn. Export `OMP_NUM_THREADS=1` in the shell *before* invoking python (or via `env OMP_NUM_THREADS=1 python script.py`) — setting it inside the script after numpy/torch is already imported is too late and won't affect thread pools. Read the CSV, apply the confidence>=0.80 and description-length>15 filters first, and keep the surviving row set as your Phase-2 baseline (needed later to know what NOT to log). Load `all-MiniLM-L6-v2`, embed the filtered descriptions, and build the full pairwise cosine-similarity matrix with sklearn. For dedup, don't just do one greedy pass over pairs in matrix order — that can miss transitive duplicates (A~B, B~C but A not~C after removals). Instead loop: find any remaining pair with similarity>0.85, drop the loser by the tie-break rule (lower confidence loses; if equal, higher id loses), recompute which pairs still exceed the threshold among survivors, and repeat until no pair exceeds 0.85. Write the survivors to the cleaned CSV with original columns, and write only the ids removed during this dedup loop (not Phase-2 drops) to the log file, one per line, sorted ascending.
