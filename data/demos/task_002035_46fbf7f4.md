Then write `process_configs.py` with argparse for `--input-dir`/`--output-dir`. For each JSON file: first impute any `null` `load_factor` with the transcribed value, then run the corruption check across *all pairs* of records in the file (not just adjacent ones in list order) — for every pair, compute `abs(t1 - t2) <= 2.0` AND Euclidean distance over the 4-dim `metrics` vectors (`sqrt(sum((a-b)**2 for a,b in zip(m1,m2)))`) `< 0.05`; if any single pair satisfies both conditions simultaneously, reject the whole file (don't write output for it). Only if no such pair exists, write the fully imputed array to the output dir using the identical filename. Be careful with the pairwise check being O(n^2) over all pairs, not a single linear scan, and with using strict `<`/`<=` exactly as specified — off-by-one comparisons here are what typically causes clean files to be wrongly rejected or evil files to slip through.
