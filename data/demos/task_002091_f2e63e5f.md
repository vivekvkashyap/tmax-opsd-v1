Separately, read the wav into a numpy array (scipy.io.wavfile or soundfile), run `np.fft.rfft` over the whole signal, take the magnitude spectrum, zero out or ignore the 0 Hz bin, and find the frequency bin with peak magnitude using `np.fft.rfftfreq(len(signal), 1/sample_rate)` to convert bin index to Hz — round to the nearest integer to get F_dom. For the simulation, parallelize across RPM candidates 1000..5000 step 100 with `multiprocessing.Pool`: for each RPM compute mu = 95 - RPM/100, draw 100,000 samples from N(mu, 5^2), compute the empirical 95th percentile and filter out RPMs where it exceeds T_max, then among survivors compute 1D Wasserstein distance (`scipy.stats.wasserstein_distance`) between the simulated sample and samples drawn from N(F_dom/2, 2^2), and pick the RPM minimizing it. Finally stand up a simple Flask/http.server app on 127.0.0.1:8000 with a GET /profile handler returning the three computed values as JSON, and run it persistently (background process, nohup, or long-lived terminal) so it stays queryable.
