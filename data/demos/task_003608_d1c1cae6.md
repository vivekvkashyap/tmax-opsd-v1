Structure this as a discrete pipeline with a clear filtering step first. Use `find /home/user/app_data -type f -newer /home/user/last_backup.stamp` to get exactly the incremental file set — this is the step naive attempts get wrong (comparing raw mtimes manually or including the whole directory instead of only newer files). For each matched file, branch on extension: parse CSV with the standard library `csv` module treating row 1 as header and converting each row to a dict of strings; parse XML with `xml.etree.ElementTree`, iterating `<record>` children of `<records>` and building a dict from each child tag/text (also as strings); for JSON files just copy/re-serialize as-is. Write every result to `/home/user/backup_staging/<basename>.json` as a JSON array of objects. Then build the manifest as a JSON list of the *original* absolute source paths (not the staged copies) for exactly the newer files. Finally tar the staged directory's contents (not the directory entry itself) — use `tar -czf /home/user/incremental_backup.tar.gz -C /home/user/backup_staging .` or list files explicitly so paths inside the archive are flat filenames, not `backup_staging/...`. Verify with `tar -tzf` that entries are bare filenames before finishing.
