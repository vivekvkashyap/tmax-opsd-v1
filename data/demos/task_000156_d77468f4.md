The core lesson is to never let `transaction_id` pass through any floating-point representation. Set up the Rust project with `cargo init` in `/home/user/cleaner`, add the `csv` and `serde` crates, and read `transactions.csv` using the `csv` crate with each record's `transaction_id` field pulled as a raw `&str`/`String` rather than deserializing it into an `f64`. Parse that string directly with `str::parse::<u64>()` (not via any float intermediate), treating an empty string as `0u64` (check `is_empty()` before parsing, or match on the parse error). Accumulate the sum in a `u64` (or `u128` if you're worried about overflow) using ordinary integer addition, which is exact and immune to the precision loss that floats suffer above 2^53. Write the final sum as a plain string to `/home/user/sum.txt` and run the compiled binary yourself (`cargo run` or `cargo build --release && ./target/release/...`) to confirm the file is actually produced, rather than just writing code that would produce it.
