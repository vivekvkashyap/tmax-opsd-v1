This bundles three unrelated deliverables — do them independently and verify each in isolation. (1) Video analysis: use `ffmpeg` to extract frames (e.g. one per video frame or at a known fps) into images, then write Rust code (using an image-decoding crate, or parse raw pixel data extracted via ffmpeg's rawvideo output) to sample the pixel at (100,100) per frame and count frames matching R>200,G<50,B<50 — validate your pixel-indexing (row-major, correct channel order) against a couple of frames you inspect manually before trusting the full count, then write the integer to the txt file. (2) Log classifier: build a small Rust binary (`cargo new classifier`) whose logic checks, in order, for the exact substring "UDP FLOOD" and for any single IP appearing with more than 50 connections (parse connection counts per IP from the log text) — print exactly "EVIL" or "CLEAN" with no extra output, and sanity-test it against a few log snippets you write yourself before running on the real corpus. (3) Mail spool: create the sparse file with `truncate -s 50M /home/user/mail.img` (or `fallocate`), hand-craft the fstab line with device, mountpoint, `ext4`, `loop` in the options field, and the two trailing `0 0` fields, `mkdir` the backup dir, then launch the podman container with `podman run -d --name mail_spooler -v /home/user/mail_backup:/var/mail alpine:latest sleep 3600` and confirm with `podman ps`/`podman inspect`.
