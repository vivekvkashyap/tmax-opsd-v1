Then group rows by `metric` only (not by sensor), sort each group's rows strictly by `timestamp` ascending, and keep `math.ceil(len(group)/2)` earliest rows — do not round or use integer division, since that silently drops the middle row when counts are odd. Write the kept rows to the output path with header `timestamp,sensor,metric,value`. Third, write an aggregator that streams every CSV in `/home/user/processed/`, accumulates a running sum and count per `metric`, and writes `{metric: round(avg, 2)}` to `/home/user/summary.json`. Test each stage on the sample files before wiring them together.
