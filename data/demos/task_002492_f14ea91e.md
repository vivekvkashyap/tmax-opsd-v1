Build the validator in clear stages rather than one monolithic parser. First, determine TOTAL_FRAMES: shell out from C with `popen("ffprobe -v error -count_frames -select_streams v:0 -show_entries stream=nb_read_frames -of csv=p=0 <video>", "r")` (or an equivalent ffprobe invocation) and parse the integer from its output — don't hardcode a frame count. Second, read the whole patch file into a buffer (fopen/fread, tracking size) so you can locate line boundaries with strchr/memchr rather than fragile fixed-width parsing. Parse the "FRAME <N>" line and reject if N is not a valid non-negative integer strictly less than TOTAL_FRAMES. Parse the "CRC32 <hex>" line, then compute CRC32 (implement the standard IEEE 802.3 polynomial/table yourself, or use zlib's crc32 if linking against it) over every byte from right after the newline terminating the CRC32 line through EOF, and compare against the parsed hex value case-insensitively. Finally check the payload begins with the exact bytes "@@ ". Reject on any rule violation (mismatched frame bounds, checksum, or missing diff-header) with a non-zero exit; only exit 0 when all three checks pass. Compile with `gcc -o /home/user/validate validate.c` and test against both a hand-crafted clean patch and deliberately corrupted ones before trusting the result.
