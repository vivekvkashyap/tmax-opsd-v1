Build run_pipeline.sh as a join-then-aggregate awk/join pipeline rather than a monolithic script that only works on the sample data — the hidden verifier will run it on new tx/fx files, so hardcoding is a trap. Approach: load fx.csv into an awk associative array keyed by `date,currency` (skip its header), then stream tx.csv (skip its header) and for each row look up the matching rate by that same composite key, multiply amount*rate, and accumulate into a per-date sum array; at the end iterate the sums in date-sorted order (`for date in sorted keys` via `asort`/`sort` piped through, or sort the final output lines with `sort`), printing `date,total_usd` with `printf "%.2f"`. Handle floating point carefully — awk's native arithmetic is fine here, but avoid locale/format surprises by always using printf with fixed precision, not string concatenation of raw floats. For test_accuracy.sh, invoke run_pipeline.sh with the three required positional args, then use `diff <(sed -e '$a\' actual) <(sed -e '$a\' expected)` or similar trailing-newline-tolerant compare, and write REPRODUCIBLE AND ACCURATE or FAIL to test_status.txt accordingly. Run test_accuracy.sh yourself to confirm it passes and generates the status file.
