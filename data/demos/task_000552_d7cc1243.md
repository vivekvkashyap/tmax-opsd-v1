Set up and validate the backend services first, independent of the C++ code. Load `users.csv`/`products.csv` into Postgres tables matching their given schemas (use `psql \copy` or a simple loader), run `load_redis.sh` to populate the blacklist, then confirm connectivity manually with `psql` and `redis-cli` before writing C++. In `sanitizer.cpp`, read stdin line by line as CSV (a simple split on commas is fine here since fields are simple, but watch for a trailing newline/CRLF), always echo the header through unchanged, and for each row apply the three checks in a cheap-to-expensive order: first the SQL-injection substring check on `notes` (case-sensitive exact matches on `DROP`, `--`, `;`, `UNION`) since it's pure string work; then a single Redis `EXISTS txn_id` call via `hiredis` to reject blacklisted transactions; then one `libpq` query per row (or batch if you prefer) joining `users` and `products` requiring `account_status = 'VERIFIED'` and `is_active = true` and both IDs present. Only print a row if it survives all three checks. Compile with `g++ -O2 sanitizer.cpp -o sanitizer -lpq -lhiredis`, then self-test by piping the given `reference/` sample data and a few hand-crafted malicious rows through `./sanitizer` and checking the header-only-vs-full-passthrough behavior before trusting it against the hidden corpora.
