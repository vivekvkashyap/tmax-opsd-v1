Sequence the pipeline in strict dependency order and validate each stage's intermediate output before moving to the next. First, extract frames with `ffmpeg -i traffic_feed.mp4 -vf fps=1 frame_%03d.jpg` and MD5-hash each extracted JPEG's file bytes (not the video timestamp) to build the set of unique frame hashes actually present in the video — duplicate hashes collapse to one entry. Second, stream-read `raw_telemetry.csv` and keep only rows whose `frame_md5` is in that unique-hash set; this naturally drops rows belonging to frames not in the video and duplicate-frame rows beyond the first. Third, for `location_name`, detect mis-decoded UTF-8: if the string is invalid UTF-8 (or exhibits mojibake patterns), re-interpret the raw bytes as Windows-1252 and re-encode to UTF-8 — in Go use `golang.org/x/text/encoding/charmap.Windows1252` with a decode attempt guarded by a UTF-8 validity check (`utf8.ValidString`), not a heuristic on individual characters. Finally bulk-insert into SQLite using a single transaction with a prepared `INSERT OR REPLACE` statement for speed, matching the exact schema `frame_md5 TEXT PRIMARY KEY, location_name TEXT, vehicle_count INTEGER`.
