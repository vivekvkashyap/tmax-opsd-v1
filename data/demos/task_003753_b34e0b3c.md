Structure this as three concurrent stream consumers feeding a join, built entirely with shell primitives. Open three separate connections (e.g. `exec 3<>/dev/tcp/127.0.0.1/9001`, similarly for 9002/9003, or `nc host port`) and pipe each through an `awk`/`sed` normalizer that: (a) converts the human-readable `YYYY-MM-DD HH:MM:SS` orders timestamp to epoch via `date -d ... +%s`, (b) leaves the payments epoch as-is, (c) converts the shipping ISO-8601-with-Z timestamp to epoch the same way — emit each stream as `order_id<TAB>field1<TAB>field2` sorted by `order_id` so `join` can be used. Since `join` only merges two files at a time, join orders+payments first on `order_id`, then join that result with shipping — using `sort -t$'\t' -k1,1` before each join is mandatory or `join` silently drops non-adjacent matches. An inner join naturally happens because `join` without `-a` only emits keys present in both inputs. After the three-way join, reformat fields into the required CSV column order, then pipe through `sort -t, -k2,2n -k1,1` for the final numeric-then-alphabetic ordering. Test incrementally against small hand-crafted socket data before trusting it against the fuzzed oracle comparison.
