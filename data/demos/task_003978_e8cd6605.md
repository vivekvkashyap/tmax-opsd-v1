Structure this as: parse -> validate -> hash-in-parallel -> reduce -> sort -> write. Read `/home/user/raw_configs.jsonl` line by line, dropping any record where `config_data` is missing `app_name` or it's null — do this filtering before hashing, not after. For the dedup key, you must produce a canonical serialization: `json.dumps(config_data, sort_keys=True, separators=(',', ':'), ensure_ascii=False)` concatenated with `server_id`, then SHA-256 of the UTF-8 bytes — using the default `json.dumps` (which escapes unicode and adds spaces) will produce different hashes for logically-identical records and break dedup. Use `concurrent.futures.ProcessPoolExecutor` to map records to (hash, record) pairs in parallel (hashing is the CPU-bound step worth parallelizing), then reduce sequentially in the main process: for each hash key keep only the record with the minimum `timestamp`. Finally sort the deduplicated records by `(server_id, timestamp)` and write them with `json.dumps(record, ensure_ascii=False)` per line — ensure_ascii=False is required both when building the hash key and when writing the final output, otherwise Unicode text gets escaped and the byte-level check fails.
