Process the CSV in strict pipeline order and get each stage exactly right before moving to the next, since the grader diffs the final file byte-for-byte. Load rows preserving original order, then: (1) dedupe by building a hash/tuple key of `SensorID,CoordX,CoordY,Temperature` and keep only the first Timestamp seen per key — do this before any sorting, using the raw file order (which is already chronological); (2) compute Euclidean distance from (0,0) per row, rounded to 2 decimals; (3) for the rolling average, group by SensorID in chronological order and, for each row, average only the up-to-3 *previous* readings for that sensor (excluding the current one) — track a per-sensor history list as you iterate; use the literal string `N/A` when there is no prior history; (4) flag ANOMALY only when a numeric rolling average exists AND current Temperature > 1.5 * that average, else NORMAL; (5) finally sort output rows by Timestamp ascending as integers (not strings) then SensorID lexicographically, and write exactly the five required columns with a header. Common mistakes: sorting before deduping/rolling-avg computation, including the current reading in its own rolling window, or comparing anomaly threshold using the post-rounding average instead of matching the spec's rounding order.
