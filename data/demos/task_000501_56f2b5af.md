Work through the pipeline in dependency order: extraction must finish and be verified before feature engineering, which must finish before you start the server. First run `ffmpeg -i /app/dashcam.mp4 -vf fps=5 -q:v 2 /home/user/frames/frame_%d.jpg` (create the directory first) and confirm the frame count matches what ffmpeg reports — don't assume a count. Compute grayscale brightness per frame with OpenCV (`cv2.cvtColor(img, cv2.COLOR_BGR2GRAY).mean()`) or PIL (`.convert('L')` then mean of pixel data), iterating frames in numeric order (sort by the integer in the filename, not lexicographically, or `frame_10` sorts before `frame_2`). Precompute the full features array (brightness, rolling_avg over a sliding window of the last up to 3 frames, is_flash comparing current brightness to the *previous* frame's rolling_avg) once at startup and cache it in memory, rather than recomputing per-request — this also makes the auth-gated GET handler simple: check the Bearer token first, then look up by frame_id in the cached list, 404 if out of range. Bind the HTTP server strictly to 127.0.0.1:8080, run it detached (nohup/background) so it survives the tool call, and only then write pipeline_status.log with the frame count. Round JSON floats to 2 decimals at serialization time.
