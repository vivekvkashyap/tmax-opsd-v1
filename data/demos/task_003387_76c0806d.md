Run `tesseract /app/glitch_criteria.png -` (installing tesseract-ocr if needed) to recover the exact REJECT_THRESHOLD multiplier printed in the scanned logbook image — use that recovered numeric value in your C program rather than assuming a round number. Write filter.c against the NetCDF C API: open the file with `nc_open`, get the `signal` variable's id and dimension length with `nc_inq_varid`/`nc_inq_dimlen`, then read the full 1D float array with `nc_get_var_float`. Compute the mean, then the sample standard deviation (divide the sum of squared deviations by N-1, not N, to match "sample" standard deviation), then find the maximum absolute deviation of any single point from the mean. Compare `max_abs_deviation / std_dev` against the OCR'd threshold: if it exceeds the threshold, `exit(1)` (anomalous/rejected); otherwise `exit(0)` (clean/accepted) — be careful with the strict-greater-than boundary since the hidden corpora are constructed close to that line. Install `libnetcdf-dev`, compile with `gcc filter.c -o /home/user/filter -lnetcdf`, and test against the sample files in /app/sample_data/ before trusting it against the hidden clean/evil corpora, since 100% pass on both is required.
