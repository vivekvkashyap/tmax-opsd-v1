Build this as an awk pipeline you test stage-by-stage on a small sample before combining. Stage 1 (validation): use `awk -F',' 'NR>1 && NF==5 && $3 ~ /^[0-9]+$/ && $3+0>0 && $4 ~ /^[0-9]+$/ && $4+0>0 && $5 ~ /^[0-9]+$/ && $5+0>0'` to drop the header and reject rows with wrong field count, empty fields, non-digit content, or non-positive values — regex-anchoring to digits-only is what correctly rejects negative numbers (a leading `-` isn't a digit) and non-numeric junk like "invalid" in one shot. Stage 2+3 (reshape + bucket): for each valid row emit three lines swapping in CPU/MEM/CONN as the metric and the corresponding column as value, with the timestamp truncated to the hour via `substr($1,1,13)` (works cleanly since the timestamp format is fixed-width ISO 8601). Stage 4+5 (aggregate + format): pipe through `sort` then `awk` grouping consecutive identical `(hour,server,metric)` keys (or use an associative array keyed by that triple accumulating sum/count), and in the END block `printf "%s,%s,%s,%.2f\n"` for each group — accumulate in one pass with arrays rather than group-by-sort-then-scan if that's simpler for you. Do a final `sort -t, -k1,1 -k2,2 -k3,3` to guarantee the required Hour/ServerID/Metric ordering, then diff against the expected output.
