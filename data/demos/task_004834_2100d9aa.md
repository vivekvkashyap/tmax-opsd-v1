Build this bottom-up and verify each layer before adding the next. First extract the secret: run `strings /app/operator-core | grep -i mock` (or similar) to locate the literal value following the `X-K8s-Mock-Auth` check in the binary's read-only strings — don't guess it. Start two instances of the binary on 8081/8082 and confirm each responds correctly to a curl request carrying the discovered header before writing any proxy code. Then write proxy.py as a minimal `http.server`-based reverse proxy that keeps a shared counter (or itertools.cycle) over the two backend addresses, forwards the client's method/path/body, and injects the auth header on the outbound request — test it directly on 9090 with curl before touching SSH. For the SSH tunnel, generate a keypair (`ssh-keygen`), append the public key to `~/.ssh/authorized_keys`, ensure sshd is running, then use `ssh -N -L 0.0.0.0:8000:127.0.0.1:9090 user@localhost` (as a background/supervised process, not interactively). Finally write supervisor.conf with `autorestart=true` for all four programs, and start `supervisord -c ...`, verifying each program's status with `supervisorctl status` before declaring done.
