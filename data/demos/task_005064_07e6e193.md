Build and test each script standalone before chaining them. `setup_env.py`: use `os.makedirs(..., exist_ok=True)` for all directories, generate the self-signed cert with `subprocess.run(["openssl","req","-x509","-newkey","rsa:2048","-nodes",...])` only if `server.crt`/`server.key` don't already exist (idempotency), write the status JSON, and create the symlink with `os.symlink` guarded by checking `os.path.islink`/removing a stale target first so reruns don't crash. `server.py`: use `http.server` wrapped with `ssl.SSLContext.wrap_socket` (or `ssl_context=...` on the socketserver), serving from the docroot, bound to 127.0.0.1:8443. `service_manager.py`: on `start`, `subprocess.Popen` the server detached and write `proc.pid` to the pid file; on `stop`, read the pid, `os.kill(pid, signal.SIGTERM)`, remove the file; on `status`, check `os.kill(pid, 0)` in a try/except. `alert_monitor.py`: hit the health endpoint with `requests.get(..., verify=False)` (or ssl context ignoring verification) in a try/except catching connection errors, write the alert text and call `service_manager.py start` via subprocess on failure. Finally execute the steps in the exact order given and assemble `final_report.json` by re-checking the server actually responds again.
