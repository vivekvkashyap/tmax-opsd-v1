Build/install with `maturin develop` (or `maturin build` + `pip install`) and confirm `python3 -c "import rust_ecc; print(rust_ecc.compute_checksum(...))"` works before touching the worker. Finally run `/app/benchmark.py` and read its reported `requests_per_second`; if it's under 800, profile whether the bottleneck moved elsewhere (e.g. Redis round-trips, worker polling loop) rather than assuming the Rust extension alone guarantees the target.
