First, `cat` the `Makefile`'s `build` target: protoc needs both `--python_out=.` for message classes AND `--grpc_python_out=.` for the service stubs — a common mistake is generating only one. Run `python3 -m grpc_tools.protoc -I. --python_out=. --grpc_python_out=. math_graph.proto` and confirm both `_pb2.py` and `_pb2_grpc.py` land, then `make build` should reproduce this. Second, read `server.py`'s Kahn's-algorithm loop carefully: when you dequeue a node and walk its `adj[curr]` neighbors, the in-degree you decrement must belong to the *neighbor being visited*, not the node you just popped — decrementing the wrong dictionary key silently breaks either the ordering or triggers a false "cycle" once counts go negative or never hit zero. Trace a small graph by hand (or add print statements) to confirm each neighbor's in-degree hits exactly zero once all its predecessors are processed. After fixing both, run `make build`, start the server in background, run the client, and diff `output.txt` against `expected.txt` to `diff_result.txt` — iterate until the diff is empty.
