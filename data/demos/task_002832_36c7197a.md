Work the pipeline in strict phase order, verifying each stage before moving on. Phase 1: use `cat dataset.tar.gz.part* > combined.tar.gz` (glob sorts lexically, matching the `split` suffixes) then `tar -xzf` into the processing directory. Phase 2: for each nested archive, test integrity without extracting first — `unzip -t` for zips, and for tar.gz use `gzip -t` or `tar -tzf` to detect corruption; on failure append just the basename to the log, on success extract into `extracted/<name-without-extension>/` (strip the archive's extension carefully, e.g. `.tar.gz` vs `.zip`). Phase 3: find broken symlinks with `find extracted -xtype l -delete` (or `-L` variants) — do this before dedup so broken links aren't mistaken for content. For deduplication, hash every regular file (e.g. `find ... -type f -exec sha256sum {} +`), group identical hashes, then replace all-but-one of each group with `ln` (hard link) to the first, not a symlink — the target is that all duplicates share one inode. Phase 4: archive with `tar -cjf` and confirm hardlink preservation (tar's default behavior already preserves them; verify via `tar -tvf` showing the `h` type for repeated files rather than re-storing full content).
