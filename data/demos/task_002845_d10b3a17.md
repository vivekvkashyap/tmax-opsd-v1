Build the detector first and validate it standalone against both corpora before wiring up services. For RFI: take `np.fft.rfft` along the time axis (axis=0) for each antenna column, look at the magnitude spectrum excluding the DC bin, and flag a file as evil if any bin's power is a large outlier relative to the rest of the spectrum (e.g. many standard deviations above the median/mean magnitude) — thermal noise has a flat-ish spectrum while a coherent sine wave creates one sharp spike. For NaN/Inf: check with `np.isnan`/`np.isinf` before doing FFT math (NaNs will silently propagate through FFT and corrupt everything, so check first). Tune your threshold by running against every file in `/home/user/sim_data/clean/` and `/home/user/sim_data/evil/` and iterating until it's 100% correct — don't guess a threshold and stop. For the pipeline, start redis-server, producer.py, and aggregator.py first and confirm each port is actually listening (e.g. with `curl`/`redis-cli`) before writing consumer.sh, since a naive consumer loop written against services that aren't up yet will silently do nothing. Use `redis-cli blpop raw_data_queue 0` (blocking pop) in a loop rather than a busy-poll, and make sure the loop keeps running after processing one item.
