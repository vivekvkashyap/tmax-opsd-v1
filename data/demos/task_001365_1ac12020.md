First measure the reference audio's real duration rather than assuming a value: run `ffprobe -v error -show_entries format=duration -of csv=p=0 /app/audio/ref.wav` (or `sox --info`) and use that number as the runtime argument to your filter. Implement the two checks as independent linear scans over the raw stdin buffer. For malformed unicode escapes: scan byte-by-byte for the two-character sequence `\u`, then check that exactly the next 4 characters are valid hex digits (0-9, a-f, A-F) and that there are actually 4 characters available before EOF/newline — both "wrong character" and "fewer than 4 characters before the string/line ends" must count as malformed, which is an easy edge case to miss. For timing anomalies, use lightweight parsing (locate `"start_ts":` and `"end_ts":` substrings per line, then `strtod`/`sscanf` the following number) rather than a full JSON parser, and reject if `start_ts >= end_ts` or `end_ts > duration + 0.1`. Since REJECT must fire on *any* anomaly anywhere in the stream, don't short-circuit reading — but you can exit as soon as the first violation of either kind is confirmed. Validate your binary manually against a couple of files in `/app/corpus/clean/` and `/app/corpus/evil/` with `./loc_filter <duration> < file` before assuming it generalizes to the full corpora.
