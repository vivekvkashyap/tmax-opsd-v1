Design the Rust program around three concerns kept separate: discovery, concurrent metadata extraction, and safe shared-state mutation. Use `walkdir` (or manual recursion) to enumerate `.gz` files under the raw logs root, then process the list in parallel with `rayon`'s `par_iter` (or a thread pool) — for each file, wrap it in `flate2::read::GzDecoder` and use a `BufReader::lines().next()` to read only the first line without decompressing/writing the whole file to disk, then `serde_json` parse it for `app_id`/`timestamp`. Create the target `app_id` subdirectory with `create_dir_all` (safe to call redundantly across threads), then use `std::fs::hard_link` to link the original `.gz` into it under `<timestamp>_<original_filename>`. For the manifest, don't just use a `Mutex<File>` in-process only — since multiple OS threads write concurrently, take an OS-level exclusive lock (e.g. `fs2::FileExt::lock_exclusive`) around each open-append-write-unlock cycle so lines never interleave. After all files are hard-linked, group results by `app_id` in memory, find the max-timestamp entry per group, and create a `latest.gz` symlink (`std::os::unix::fs::symlink`) pointing at that hard link's filename. Build with `cargo build --release` and run the resulting binary once to produce the final tree.
