This task has a calibration phase feeding a filtering phase — don't skip straight to writing the bash filter. First, extract frames from `/app/calibration.mp4` with `ffmpeg` (e.g. to a temp directory of PNG/PPM frames or piped raw frames), then programmatically test each frame for being "completely red" (every pixel equals RGB 255,0,0, not just mostly-red) — use a script (Python with PIL/numpy, or ImageMagick's `compare`/`identify`) that checks min/max per channel across the whole frame rather than eyeballing thumbnails, and count how many frames pass; this count is your parameter `C`, derived from the video, not guessed. Next, run `papermill /app/solve_ode.ipynb <output.ipynb> -p C <your_count>` to inject the parameter and execute the notebook, which produces `/tmp/reference_solution.csv`. Only then write `sanitize.sh`: for each input CSV, join on matching row order/timestamp against the reference (both have exactly 100 rows), compute `max(|value_i - ref_value_i|)` across all rows (e.g. with `awk` or `paste` + a numeric loop, or a short embedded Python one-liner), and copy the file to the output dir only if that max strictly is `< 0.5`; treat `>= 0.5` as reject with no copy.
