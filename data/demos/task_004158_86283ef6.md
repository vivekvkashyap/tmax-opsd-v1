Treat this as four separable subsystems, each independently testable. Run `tesseract /app/architecture.png stdout` (try `-psm` variants and simple preprocessing like grayscale/threshold if the raw OCR is noisy) and search the extracted text for the `KEY=value` pair naming the backup directory — don't guess or hardcode a path, read it from the OCR output. For backup.sh, remember it will be invoked with an emptied PATH (`env -i PATH="" bash backup.sh`), so the script itself must set `export PATH=/usr/bin:/bin:/usr/local/bin` (or similar) at the top before calling `mkdir -p`, `tar`, `gzip`; make directory creation idempotent (`mkdir -p`) rather than failing if it already exists. For rotate_logs.sh, use the classic copy-then-truncate or `mv` + reopen pattern (`mv server.log server.log.1 && gzip -f server.log.1 && : > server.log` or `truncate -s0`), being careful not to leave the log missing or a stale in-use descriptor. For sanitizer.c, read the whole file as bytes and scan for the literal substring `DROP_TABLE` and for any byte outside printable ASCII plus `\n\r\t`; compile with `gcc` and validate exit codes against a handful of files you construct yourself before relying on the hidden corpora at `/app/corpus/evil/` and `/app/corpus/clean/`.
