For the detector, build the pipeline as a chain of standard tools and test it manually on a crafted repeating-block string before trusting it: `base64 -d "$1" | xxd -p -c 256 | tr -d '\n'` gives one continuous hex string, then `fold -w 32` splits it into 16-byte (32 hex char) blocks one per line; pipe that through `sort | uniq -d` — if any duplicate block line comes out, exit 1 (malicious), otherwise exit 0 (clean). Be careful with `xxd -p`'s default line wrapping (use `-c 0` or a large `-c` value) so the hex isn't itself broken into short lines before you fold it into blocks. For the Nginx half, edit the `http`/`server` block to `listen 8080;`, add a `location /api/ { proxy_pass http://127.0.0.1:9000; }` block, and add an `if ($http_x_debug_bypass = "true") { return 403; }` check (Nginx has no native single-header equality primitive outside `if`, so this is the pragmatic approach) before the proxy_pass — verify header matching is case-sensitive to the exact header value expected. Start it with `nginx -c /home/user/nginx/nginx.conf` and test both a plain `/api/` request and one with the `X-Debug-Bypass: true` header using `curl -H` before considering it done.
