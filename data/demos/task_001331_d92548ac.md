Structure the C program as three clearly separated passes rather than one tangled loop. First write a normalization function that trims leading/trailing whitespace, collapses any run of internal spaces to a single space, and lowercases ASCII letters — apply it to each line right after reading with `fgets`/`getline`. Second, implement Levenshtein distance yourself as a classic dynamic-programming function over two null-terminated strings (a 2D DP table, or rolling two rows to save memory), since you'll call it repeatedly. Third, maintain a growable array (or fixed-size array sized generously) of accepted normalized strings; for each new normalized line, compute its edit distance against every currently accepted string, and only append it to the accepted list if that distance exceeds 1 for all of them — this is a sequential/incremental dedup, so comparisons are against strings accepted so far, not the full original list, and order of appearance matters for the "accepted" order in the output. Watch buffer sizes: allocate enough room for whitespace collapsing to not overflow, and null-terminate carefully. After writing the accepted list to the output file (one per line, in insertion order), compile with `gcc -O2` and run it, then diff the output against your expectation of what the normalization+dedup should produce.
