Build this in layers and test each independently before wiring them together. First write a standalone RLE encoder function that walks the byte stream once, counting runs of identical bytes (including `\n` and spaces) and emitting `<count><char>`; verify it against a couple of hand-checked strings. Second, implement the incremental-backup logic: for each `.txt` file, compute its RLE, then check whether `<backup_dir>/current` exists as a symlink — if so, read the corresponding `.rle` file from the target of that symlink and compare byte-for-byte with the freshly computed content. On exact match, use `std::fs::hard_link` (not copy) to link the new backup path to the existing file so inodes are shared; on any difference (or missing prior file), write the new bytes fresh. Third, after all files in the new backup dir are written, remove any existing `current` symlink and recreate it pointing at the new backup directory name (relative or absolute, but must resolve correctly with `ls -l`/`readlink`). Common pitfalls: comparing file mtimes instead of content, hard-linking when content actually differs, or updating `current` before the backup directory is fully populated. Finally write `verify.sh` to `stat -c %i` (or equivalent) each of the four `.rle` paths and print them in the exact required format.
