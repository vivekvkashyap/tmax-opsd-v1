Start by installing the watchdog library (pip install watchdog) before writing any code. Build the daemon around watchdog's Observer + FileSystemEventHandler, overriding on_any_event (or the specific on_created/on_modified/on_deleted handlers) so a single handler drives all three requirements per event: (1) print the `EVENT: <type> | PATH: <path>` line, (2) resync published/ to src/, (3) trigger a tar backup — in that order isn't critical but all three must fire on every event. For the hard-link mirror, walk src recursively, mkdir matching subdirectories in published, and use os.link (not shutil.copy) to create/update each file's hard link, removing the published-side hard link with os.remove when a src file disappears — don't try to hard-link directories themselves. Create the /home/user/docs/live symlink once at startup with os.symlink, checking os.path.lexists first so reruns don't crash. For backups, shell out to tar with --listed-incremental=.../snapshot.snar and pick the next sequential backup_N.tar name by counting existing files in backups/. Since stdout is redirected to a log file, run Python unbuffered (python3 -u) or flush() after each print so daemon.log actually fills in real time. Launch the daemon with nohup ... & and terminate it with SIGTERM after the simulation script finishes.
