Treat this as a sequential pipeline with a scheduling artifact at the end. First extract frames with `ffmpeg -i /app/traffic_monitor.mp4 -vf fps=1 frame_%04d.png` into a working directory, then walk the frames in order computing `md5sum` per file; collapse runs of consecutive identical hashes down to the first occurrence only (a simple "if hash == previous_hash: skip" pass over the sorted frame list, not a global unique-hash set). For each surviving frame, convert to grayscale (e.g. PIL `.convert('L')`) and compute mean pixel value as brightness. Apply a trailing rolling average with window 3 where early elements average over however many values exist so far (1st = itself, 2nd = mean of first two, 3rd+ = mean of last three) — don't pad with zeros. Compute min/max/avg over the rolling-average sequence, round to 2 decimals, and serve them as JSON from a lightweight HTTP server (`http.server`, Flask, etc.) bound to `0.0.0.0:8080` at `GET /stats`, left running in the background. Finally write `/home/user/pipeline.cron` with a standard 5-field cron line invoking `/home/user/process.sh` every 5 minutes (e.g. `*/5 * * * * /home/user/process.sh`) — no need to install it, just place the file correctly formatted.
