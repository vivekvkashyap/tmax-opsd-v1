Replicate this as a straightforward streaming state machine in Python, reading `sys.stdin` line by line rather than loading everything at once (inputs can be up to 5000 lines but streaming is still the correct mental model to match the oracle's semantics). Keep a single `prev_line` string variable and skip a line only if it is an exact string match to the immediately preceding non-skipped line (not to any earlier line — this is consecutive-only dedup, a common trap is deduplicating globally with a set). For each surviving line, split on commas into `timestamp, locale, event_type, word_count`, bucket the timestamp with integer floor division `int(timestamp) // 3600 * 3600`, and compute score as `word_count` for `translated`, `word_count * 2` for `reviewed`, and 0 (drop, don't aggregate) for anything else. Accumulate into a nested dict `{bucket: {locale: total}}`. At EOF, emit rows sorted by `(bucket, locale)` ascending, format `bucket,locale,total_score`, no header. Use `/app/oracle` locally with a handful of crafted inputs (including consecutive duplicates and mixed event types) to diff your script's output against the oracle's before considering it done — this catches off-by-one bucket or ordering bugs cheaply.
