First read the legacy Python 2 app to extract its exact checksum algorithm (hashing function, encoding, any error-correcting-code combination step) — your Python 3 `/verify` handler must reproduce that logic byte-for-byte, since a mismatch in a single edge case (empty payload, unicode, case) breaks equivalence. Build the Flask/FastAPI service on port 5001, add a Redis client (localhost:6379, db 0): on each request, check if `payload` is already a key in Redis; if present, compare the cached valid signature directly instead of recomputing, otherwise compute normally and only write payload->signature to Redis when the signature was valid (don't cache invalid attempts, since sig varies per request but only the correct one is deterministic). For Nginx, use two `location` blocks with `proxy_pass` values ending in a trailing `/` after the matched prefix (e.g. `location /v2/ { proxy_pass http://127.0.0.1:5001/; }`) so the prefix is stripped automatically — a missing trailing slash is the classic bug that leaves the prefix intact upstream. Start redis-server, both Python apps, and nginx in the background before finishing, and verify each is listening on its port with curl or `ss` before considering the task done.
