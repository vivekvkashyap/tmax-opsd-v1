Treat this as a pipeline with a hard ordering dependency: parse the log before touching audio. Parse `/app/metadata/ingest.log` by splitting on `---` into records, then within each record extracting `Record ID`, `Status`, `Start Time`, `End Time` with simple line-prefix matching (regex or `str.split(':')`), keeping only records where `Status == Valid`, in their original log order (order matters for the merge step). For each valid record, load `/app/audio/source.wav` with Python's `wave` module or `pydub`, slice the samples between `Start Time` and `End Time` (convert seconds to frame indices using the file's frame rate), and write each slice to `/home/user/chunks/chunk_<Record ID>.wav` preserving the original sample rate/format. Concatenate the same valid slices, in log order, into `/home/user/curated.wav`. Compute `manifest.json` by SHA256-hashing the raw bytes of each written chunk file (`hashlib.sha256(open(path,'rb').read()).hexdigest()`) keyed by filename — hash the file as written, not the in-memory samples, so the manifest matches what a verifier will independently hash. Finally install a local ASR tool (e.g. `openai-whisper`) and transcribe `curated.wav`, writing the result to `/home/user/transcript.txt` as one line (strip/join any newlines) since accuracy is scored by Word Error Rate against a hidden reference.
