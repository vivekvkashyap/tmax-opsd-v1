This is a multi-stage recovery-then-reverse-engineer task — do the stages strictly in order since each unlocks the next. First transcribe /app/intercepted_audio.wav using a speech-to-text tool you install (e.g. `pip install openai-whisper` and run it, or another ASR tool) — do not guess the passphrase, derive it from the actual transcription output, then normalize it (lowercase, strip spaces) per the instructions. Use that normalized string with `unzip -P <passphrase> /app/evidence.zip` to extract payload_decoder and core.payload. Since payload_decoder is stripped, reverse-engineer it with static tools (`objdump -d`, `strings`, `radare2`/`ghidra` if available) and by black-box probing: run it on a range of crafted inputs (single chars, known patterns, varying lengths) and look for arithmetic patterns in the hex output as a function of byte value and position — a per-character transform combining the input byte with its index is a common pattern to test for (e.g. XOR against a fixed byte, then an index-dependent offset). Use the core dump only as a secondary aid if static/dynamic analysis leaves ambiguity. Once you have a hypothesis, implement it in clean_decoder.cpp, compile, and validate by comparing outputs against the original binary on many random strings yourself before submitting, since the real grader fuzzes with hundreds of inputs and requires bit-exact match.
