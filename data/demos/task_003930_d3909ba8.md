Build this bottom-up and test each layer before wiring them together. First write the Rust server alone: a `/verify` endpoint that shells out to `/app/capacity_oracle <ram_mb> <cpu_cores>` (via `std::process::Command`, capturing stdout as-is) and an `/allocate` endpoint that only spawns `qemu-system-x86_64 -m <ram> -smp <cpu> -vnc 127.0.0.1:1 -daemonize` when the oracle prints "OK" — note VNC display :1 means the actual TCP listener will be on port 5901, not 5900. Test both endpoints manually with `curl` before touching git. For the git hook: `pre-receive` reads ref updates from stdin, so use `git diff --name-only <old> <new>` (or diff against the empty tree for the first push) to find changed `.json` files, extract their blob contents with `git show <new>:<path>`, POST each to `http://127.0.0.1:8080/verify`, and `exit 1` on any response containing "REJECT" to abort the whole push — remember to chmod +x the hook. Only after both pieces work standalone, wrap the server start in a `systemd --user` unit (`ExecStart` running `cargo run --release` or the built binary), enable lingering so it survives without an active login session, and confirm with `systemctl --user status` and a `curl` to 127.0.0.1:8080 before considering it done.
