Work through the pipeline as discrete, verifiable stages rather than one big script. First unzip `docs_archive.zip`, then extract every `.tar.gz` found inside it into `/home/user/extracted_docs` (loop over the extracted `.tar.gz` files and `tar -xzf` each into the target directory, preserving or flattening subdirectories — either is fine as long as the resulting `.txt`/`.csv` files are discoverable). Second, find all `.txt` files recursively (`find ... -name '*.txt'`) and for each, convert with `iconv -f ISO-8859-1 -t UTF-8 file.txt > file.txt.tmp && mv file.txt.tmp file.txt` — converting in place without a temp file risks truncating the source before iconv finishes reading it. Third, find all `.csv` files, parse each line as `filename,author,date` (no header, so read from line 1), and merge every row from every CSV into one JSON array of `{"file":..., "author":..., "date":...}` objects written to `/home/user/metadata.json` — a Python or Node script with `csv` module is more robust here than raw shell parsing since values could theoretically contain commas. Finally, build `final_docs.tar.gz` containing only the converted `.txt` files and `metadata.json`, explicitly excluding the `.csv` and nested `.tar.gz` artifacts (e.g. stage the wanted files in a clean directory before tarring, rather than trying to `--exclude` from the messy extracted tree).
