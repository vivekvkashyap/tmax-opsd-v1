This is a transitive-closure-then-rank problem in SQLite. Use a recursive CTE that starts from `nodes` where `node_type='User'`, and recursively follows `edges` where `relation_type='OWNS'` from each user's id (base case: user id as the starting node), joining `edges.source_id = previous.target_id` to walk arbitrarily deep chains — this correctly captures cases like User -> SubEntity -> Asset. Union in the anchor row so you retain the original user_id at every recursion depth (i.e., carry a `root_user_id` column through the recursion rather than losing track of which top-level user started the walk). After the recursive CTE, filter for reached nodes where `node_type='Asset'`, group by `root_user_id`, and `SUM(asset_value)`, joining back to `nodes` to get `group_name`. Then apply `RANK() OVER (PARTITION BY group_name ORDER BY total_asset_value DESC)` for `group_rank`. Watch for two traps: only users with at least one reachable asset should appear (an inner join naturally excludes zero-asset users), and shared ownership (two different users both owning the same downstream sub-entity) must let both users count that entity's assets independently — don't dedupe across users. Export via `sqlite3 -header -csv` sorted by `group_name, group_rank`.
