First derive the secret key precisely, since any off-by-one in frame counting breaks the whole downhoneypot. Extract every frame from /app/evidence.mp4 losslessly (e.g. `ffmpeg -i evidence.mp4 frame_%04d.png`), then for each frame check programmatically that every single pixel equals exactly RGB(255,0,0) — a quick way is to load each frame and verify both its min and max pixel value equal that exact color (e.g. with Python/PIL/numpy, or ImageMagick's `convert frame.png -format "%[fx:...]"`); don't eyeball colors, since near-red or compressed artifacts can be misleading — use ffmpeg with a lossless intermediate format (png) to avoid codec color-shift. Count exactly how many frames pass that pure-red test, then compute `sha256sum` of the exact string `RED_FRAMES_<count>` with no trailing newline (use `printf` or `echo -n`, not `echo`) — this hex digest is your SECRET_KEY. Then write a small Go net/http server: `/health` returns 200/"OK"; `/command` reads header `X-Secret-Key`, compares it to your derived key (constant-time or plain string compare is fine), returning 200/"COMMAND_ACCEPTED" on match and 401 otherwise. Run it backgrounded on 127.0.0.1:8080 and curl-test both endpoints before finishing.
