Work this in stages, not as one monolithic C program written blind. First run `tesseract /app/org_chart.png stdout` (or via `system()` from C) to extract the `Manager,Employee` lines, and parse them into an adjacency list mapping each manager to its direct reports. Build a helper (recursive DFS or BFS) that, given a starting username, collects that user plus every transitive subordinate. For the SQLite side, open `/app/audit.db` and explicitly avoid the corrupted index — either `DROP INDEX idx_access_time;` before querying, or add `NOT INDEXED` / force a full scan on `access_logs`, then `SELECT COUNT(*) FROM access_logs WHERE username = ?` per name in the collected set (or one query with an IN clause) and sum. For the server, use standard POSIX sockets: `socket()`, `bind()` to 127.0.0.1:8888, `listen()`, then loop on `accept()` so each connection is handled and closed but the listening socket stays open for subsequent clients — a single-shot server that exits after one request will fail verification. Read until `\n`, strip it, compute the sum, write `"%d\n"`, close the client fd, loop back to `accept()`. Compile with `gcc ... -lsqlite3`, then launch it backgrounded (`./server & disown`) so it survives after your shell command returns.
