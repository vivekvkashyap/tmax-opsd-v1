In `pipeline.py`, after `pd.merge(...)`, the `item_id` column becomes `float64` because pandas can't hold `NaN` in a plain `int64` column. Fix it by casting the merged column to pandas' nullable integer dtype: `merged['item_id'] = merged['item_id'].astype('Int64')` (capital I — this is the extension dtype that supports `pd.NA`). When written with `to_csv`, missing values render as an empty string and present values render as plain integers (no `.0`), and no rows are dropped.
