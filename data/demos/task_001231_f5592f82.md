Diagnose each sub-system independently before writing code. For SSH, don't touch the key material — the failure is almost certainly permissions: SSH clients silently refuse private keys that are group/world readable, so check `stat -c "%a" /home/user/.ssh/id_rsa` and `chmod 600` it (and confirm `~/.ssh` itself is `700` and `authorized_keys`/`config` are appropriately restrictive); verify with `ssh router-01 cat /home/user/router_config.txt` actually succeeding non-interactively before moving on. For the mount point, open `/home/user/network_fstab` and find the line whose device field is exactly `//storage-node/router-backups` — the second whitespace-separated field is the mount point path; `mkdir -p` that exact path (don't invent your own directory name). For `net_manager.py`, read stdin line-by-line in a loop (strip trailing newlines): on `BACKUP`, invoke `ssh router-01 cat /home/user/router_config.txt` via `subprocess.run(capture_output=True)`, write its stdout bytes verbatim (no extra newline added/stripped) to `<mountpoint>/router-01-latest.bak`, then print `BACKUP_SUCCESS`; on `EXIT`, exit(0); otherwise print `UNKNOWN_COMMAND` and keep looping. Test by piping a sequence of commands through the script and diffing the resulting backup file against the source config.
