Extract `/home/user/cJSON.tar.gz`, compile `cJSON.c` with `gcc -c -fPIC cJSON.c -o cjson_build/cJSON.o` (or build a shared lib) so it can link with your C client. Start `mock_server.py` in the background (`python3 mock_server.py &`) and verify it's listening on 127.0.0.1:9090 with a quick `curl` POST before writing client code against it. In `telemetry_client.c`, construct the exact JSON payload with `cJSON_CreateObject`/`cJSON_AddNumberToObject`/`cJSON_AddArrayItem`, send it via `libcurl` (`CURLOPT_POSTFIELDS`, `CURLOPT_HTTPHEADER` with `Content-Type: application/json`), capture the response body into a buffer via a write callback, parse it with `cJSON_Parse`, extract `processing_time_ms` as an integer, print only that number, and free all cJSON/curl resources. Compile with `gcc telemetry_client.c cjson_build/cJSON.o -lcurl -o /home/user/client`. Write `benchmark.sh` as a simple loop invoking `/home/user/client` 50 times, accumulating the printed integers in a shell variable, and writing `Total Time: <sum>` to the result file — test the client standalone first so loop errors don't mask client bugs.
