For the sampler script, seed with `RANDOM=$seed` so runs are reproducible, and on each step pick an index 1-10 via `$((RANDOM % 10 + 1))` and flip that character of the 10-bit string (careful with 1-indexing into the string). Query the API for both current and proposed state energies via `curl` and extract the float with `grep -o` or a small `python3 -c`/`jq` one-liner — don't try to parse JSON with pure string slicing, it's fragile. Implement Metropolis-Hastings with `awk`: compute `exp(E_current - E_proposed)` and compare against a uniform random float (e.g. `awk -v seed=... 'BEGIN{srand(seed); print rand()}'`) inside the same awk invocation to avoid subshell precision mismatches. Log the energy of whichever state you end up in after each step. For convergence, launch the three chains with `&`, `wait` for all, then `tail -n 100` each log, concatenate, and average with `awk '{s+=$1}END{print s/NR}'`.
