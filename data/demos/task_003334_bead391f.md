Build the filter as a brightness threshold check, not a fixed pixel-color match, since holdout images will differ from the calibration corpus. With ImageMagick, `identify -format "%[fx:mean]" <image>` (or `convert image -colorspace Gray -format "%[mean]" info:`) gives a 0–1 mean luminance you can scale to a percentage; compare against the ~5% threshold stated in the spec and exit 1 below it, exit 0 above. Calibrate against `/app/corpora/clean/` and `/app/corpora/evil/` by running your script over every file in both directories and checking the exit codes land correctly, adjusting the threshold if real samples cluster differently than expected — do not hardcode filenames or hashes, only brightness logic. For the pipeline: extract frames with `ffmpeg -i /app/rollout.mp4 -vf fps=1 /home/user/frames/frame_%04d.png`, loop over the resulting files running `filter.sh` on each and `rm` any that get rejected, then for survivors compute `sha256sum` and write `filename,sha256` rows to `/home/user/valid_rollout.csv` sorted alphabetically by filename (a straightforward `sort` on the frame filenames before writing works, since they're zero-padded).
