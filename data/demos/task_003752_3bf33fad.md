Build the payload in three independent stages, then combine. First, use `zipfile.ZipInfo` directly rather than `ZipFile.writestr(name, ...)`: construct a `ZipInfo` object whose `.filename` attribute is set to the raw traversal string `../../../home/user/success.txt` and write it into the archive via `zf.writestr(info, data)` — this bypasses the sanitization that occurs when you pass a plain string filename to some higher-level APIs, letting the raw path land unmodified in both local and central-directory headers. Second, generate the self-signed certificate with the `cryptography` library (`generate_private_key` + `x509.CertificateBuilder`), setting the subject/issuer Name's CN attribute to the exact required value, then serialize to PEM and add it as a normal zip entry named `cert.pem`. Third, only after the ZIP file is fully written and closed, read it back as raw bytes, compute `hashlib.sha256(data).hexdigest()`, and append that 64-character lowercase hex string directly onto the end of the file bytes (not as a zip entry — the checksum must be appended after the entire zip structure, so any tool reading the file as a strict zip should still find the traversal entry when the trailing 64 bytes are stripped).
