Build this as a small HTTP service (Python's http.server or Flask is simplest) bound to `127.0.0.1:9000` with a POST `/upload` handler that reads the raw request body as zip bytes. Use a real zip library (e.g. Python's `zipfile`) rather than shelling out to `unzip`, since you need per-entry control: iterate `ZipFile.namelist()`/`infolist()` and for each entry compute the resolved absolute destination path under your staging dir (e.g. `os.path.realpath(os.path.join(stage_dir, name))`), then check it still starts with `stage_dir` — reject/skip any entry whose resolved path escapes the staging directory (this is the standard zip-slip defense; don't just check for `..` substrings in the name, since normalization can hide traversal). Only extract entries that pass this check and end in `.wal`. For each safely-extracted `.wal` file, invoke `/app/sensor_decoder <path>` as a subprocess, capture stdout, and figure out by inspection which line/prefix carries the sensor identifier — write a small parsing rule (e.g. match a line containing a known label and split it) rather than hardcoding any specific ID you observe during testing. Rename the file to `<SensorID>_<originalbasename>`, collect the renamed basenames of only the successfully processed files in extraction order, and respond 200 with them as a JSON array. Test manually with a hand-built zip containing both a normal entry and a `../`-style malicious entry to confirm isolation before considering it done.
