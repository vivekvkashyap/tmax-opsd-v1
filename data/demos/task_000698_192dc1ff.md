Design the traversal as an explicit recursive walk that tracks a "current ancestor stack" of canonicalized directory paths (a `HashSet<PathBuf>` or `Vec<PathBuf>` passed down the recursion), not just a global "visited" set — a global visited set would wrongly block legitimate re-entry into a directory reached via two different non-looping paths, while an ancestor-stack check correctly identifies only true cycles (a symlink resolving to a path already open above it in the current recursion). For each directory entry: if it's a symlink, canonicalize its target (`fs::canonicalize`); if the canonicalized target is already in the ancestor stack, log `LOOP DETECTED: <symlink path>` to the log file and skip that branch (do not recurse); otherwise push the canonicalized target onto the stack, recurse, then pop it on return. For regular files, compare `metadata().modified()` (as unix seconds) against `last_backup_time` before copying, and mirror the relative path (based on the symlink's own name, not its target's name) under the destination root using `fs::create_dir_all` for parent directories.
