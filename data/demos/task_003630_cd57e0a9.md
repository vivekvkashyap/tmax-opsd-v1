Then write `/home/user/extractor.cpp` that opens the WAV file in binary mode, seeks to that offset, and copies the remaining bytes to a new file (an embedded ELF). Verify the extracted bytes actually start with the ELF magic `\x7fELF` before trusting the offset. Use `readelf -S` or `objdump -h` on the extracted ELF to locate the `.gcode` section, then `objcopy -O binary --only-section=.gcode <elf> <out>` (or parse section headers yourself) to dump its raw contents as text — don't just `strings` the whole binary, since other sections may contain confusable text. Once you have the GCode text, parse only lines that are `G1` commands with both an `X` and a `Y` token, track the running `(X, Y)` position starting at `(0,0)`, and accumulate the Euclidean distance between consecutive points across only those matching lines. Round the final sum to 2 decimals and write it to `/home/user/distance_metric.txt`.
