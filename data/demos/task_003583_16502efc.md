This is a two-stage data-repair-then-analysis pipeline — get the cleaning exactly right before running PCA, since PCA amplifies any residual corruption. Load the CSV, drop rows where `concept_id` is NaN entirely (these are unrecoverable), then convert the remaining `concept_id` floats to ints with rounding/`astype(int)` (they're exact integers like `1042.0`, so plain truncation is safe). Separately, find rows where the `dim_0..dim_127` columns are NaN despite having a valid concept_id — for each such row, shell out to `/app/embedder_oracle <concept_id>` and parse its comma-separated stdout back into the 128 float columns for that row, being careful to preserve column order and row alignment. After the matrix is fully reconstructed with no remaining NaNs, run `sklearn.decomposition.PCA(n_components=2, svd_solver='full', random_state=42)` on the full standardized-or-raw feature matrix (use raw values unless the task implies scaling — match the reference which does not mention standardization) and transform it. The centroid of any PCA-transformed dataset is mathematically the zero vector regardless of the data, so compute `transformed.mean(axis=0)` yourself and write it to confirm rather than hardcoding zeros — a bug earlier in the reconstruction will show up as a nonzero centroid, which is exactly what the tolerance-based grader would catch.
