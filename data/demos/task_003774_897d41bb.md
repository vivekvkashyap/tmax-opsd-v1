Treat this as two independent subsystems. For the flash detector: fix the `requirements.txt` version conflict first (pin opencv-python and numpy to a mutually compatible pair) so the install actually completes, then read `flash_detector.py` carefully — OpenCV's `cv2.split`/array order is BGR, so any luminance formula must weight the channels in the correct BGR order with the standard perceptual weights (roughly 0.299 R / 0.587 G / 0.114 B, applied to the correct channel each came from), not an equal-weighted or swapped formula; after fixing, run it on the given video and confirm the flash count written to the output file looks like a small integer consistent with visually/programmatically detected bright frame runs (a rising edge over a luminance threshold with debouncing/hysteresis so a single flash isn't double-counted across consecutive bright frames). For the sanitizer: read the raw bytes of the file, attempt strict UTF-8 decoding (reject on `UnicodeDecodeError`), then attempt `json.loads` (reject on parse failure), then validate the schema requires a string "timestamp" key and a dict "sensor_data" key (reject if either is missing/wrong type) — exit 0 only if all three checks pass, exit 1 otherwise. Test your sanitizer against both a valid log and deliberately malformed ones (bad bytes, missing fields, broken JSON) before considering it done.
