Git post-receive hooks read ref updates from stdin as `oldrev newrev refname` lines (one per updated ref) — parse stdin, not argv, and handle the case of multiple lines if present. To list `.json` files at `newrev`, run `git ls-tree -r --name-only <newrev>` from within the hook's working directory (a bare repo hook already runs inside the repo) and filter for `.json` suffix; get each file's content at that commit with `git show <newrev>:<path>` rather than assuming a working tree exists (bare repos have none). Parse each JSON's `endpoints` list, and for each `{host, port}` attempt `socket.create_connection((host, port), timeout=1)` wrapped in try/except — only on failure (ConnectionRefusedError, timeout, OSError) append the `ALERT: Cannot connect to {host}:{port}` line to `/home/user/alerts.log`; successful connections must produce no log line at all, so don't log the success case at all. Make the hook file executable (`chmod +x`) and start with a `#!/usr/bin/env python3` shebang. For the test trigger, start `python3 -m http.server 10000 &` in the working clone's environment first, then add/commit/push `services.json` to `master` — the push is what fires the hook synchronously on the receiving end.
