Work in three independent passes and test each in isolation before wiring them together. For `auth.py`: split the token on `.`, validate you get exactly 3 non-empty parts (else `INVALID_FORMAT`), base64-decode header/payload with correct padding handling (catch decode errors as `INVALID_FORMAT`), check the header's `alg` field is exactly `HS256` (anything else, including `none`/missing, is `UNSUPPORTED_ALG`), then recompute HMAC-SHA256 over `header.payload` using the shared secret and compare against the decoded signature bytes with a constant-time comparator — mismatch is `INVALID_SIGNATURE`, match is `VALID`. Since a fuzzer checks bit-exact equivalence to a reference, be careful about ordering of checks (format vs alg vs signature) and about malformed base64/JSON edge cases — write and run your own quick fuzz harness with garbage/truncated/randomly-cased tokens to sanity check behavior before trusting it. For the SSN redaction in `worker.py`, use a regex substitution that keeps only the last four digits.
