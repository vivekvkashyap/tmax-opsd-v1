Start by reading sanitize.c: a fixed-size stack buffer is filled with `strcpy(buffer, argv[1])` with no length check, so any input over the buffer's capacity overflows and segfaults. Fix it by sizing the buffer to safely hold the required input length (up to 100 chars) plus a null terminator, and use a bounded copy (`strncpy`/`snprintf` with explicit truncation/null-termination, or a same-size heap allocation) instead of raw `strcpy` — do not change what gets printed (`CLEAN: <input>`) since the test suite checks exact stdout. Then write `build_and_test.sh` as a single script: `mkdir -p` the bin dir, compile the C++ file with `g++ -c` and the C file with `gcc -c` (or compile+link directly), link both object files into one executable with `g++` (needed since one TU is C++) at the target bin path, then run `python3 -m pytest ... -v` redirecting stdout to the log file — remember pytest's pass/fail summary goes to stdout by default so a plain `>` redirect captures it. `chmod +x` the script, then actually execute it end-to-end rather than just writing it, and check the resulting log and binary output match expectations before finishing.
