Build the ETL script incrementally and verify each stage before chaining them. Use `concurrent.futures.ProcessPoolExecutor` (or ThreadPoolExecutor) to launch one worker per sensor file so the three reads happen in parallel; each worker must open its file with the correct encoding (`utf-8`, `utf-16`, `iso-8859-1` respectively — detect or hardcode per known filename, don't assume one encoding works for all). After reading, coerce `temperature`/`humidity` to numeric, drop rows with NaN or out-of-range values (temperature outside [-50,150], humidity outside [0,100]) before any resampling — validating after resampling would let bad readings pollute the mean. Set `timestamp` as a DatetimeIndex, `resample('5min').mean()`, then `ffill()` only (never bfill) to fill gaps — leave a leading empty bin as NaN. Concatenate all three resampled frames, `reset_index()`, then write to Parquet with `to_parquet`. Compute report stats (record counts, hottest sensor by mean temperature) directly from the final combined DataFrame so the numbers in report.md are guaranteed consistent with the Parquet output, and format the average temperature to exactly 2 decimal places.
