Break this into independent, testable pieces before wiring them together. First, extract the secret: use `ffmpeg` (e.g. `ffmpeg -i input.mp4 -vf "blackdetect=d=0:pix_th=0.00"` or per-frame extraction with a Python/PIL/OpenCV script that checks every pixel of each decoded frame is exactly (0,0,0)) to count truly-black frames precisely — don't rely on approximate blackdetect thresholds alone since the task requires an exact count; cross-check by extracting frames to PNG and verifying programmatically. Build the token as `op-secret-<count>` only after you're confident in the count. Second, set up the manifest directory, YAML, ACLs (`setfacl -m u:user:rwx`, then `setfacl -m g::--- ,o::---` and confirm with `getfacl`), the fstab line, and TLS certs (`openssl req -x509 -newkey rsa -nodes -days 365 ...`) as isolated, verifiable steps. Third, write the Go server: use `crypto/tls` for the HTTPS listener, compare the `Authorization` header exactly, and for the TCP trap use `net.Listen` on 8022, read any bytes, then immediately `conn.Close()` without writing anything. Test each protocol independently with `curl -k` and `nc`/a raw socket before finalizing. Run the binary in the background (e.g. `nohup ... &` or `setsid`) so it survives your terminal session, and verify it's still listening afterward.
