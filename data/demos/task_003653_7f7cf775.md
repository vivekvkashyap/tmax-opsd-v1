For the setup script, favor idempotent primitives: `mkdir -p` for every directory, `chmod` unconditionally (it's a no-op if already set), and critically use `ln -sfn <target> <linkname>` (not plain `ln -s`) for the symlink — the `-f` overwrites an existing link and `-n` treats an existing symlink as a file to replace rather than a directory to link inside of, which is what prevents nested symlinks on re-runs. Set `releases` to 0755 and each versioned dir to 0700 after creating them. For the Go validator, first run OCR (e.g. `tesseract`) against `/app/restore_secret.png` to recover the literal text after `SECRET_KEY:` — treat that recovered string as your secret constant. In the program, normalize input with `path.Clean`, then check rules in this exact precedence: secret-substring match (0777) first, then `/restricted` prefix (0600), else MD5 the cleaned path string and print the lowercase hex hash's first 4 characters. Build with `go build -o path_validator path_validator.go` and test manually against varied inputs (empty strings, `..`, trailing slashes) before relying on the fuzz harness.
