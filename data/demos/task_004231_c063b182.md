First diagnose the vendored package by hand: `cat -A` or `grep` the `Makefile`'s `run:` target to spot indentation that uses spaces instead of a literal tab (Make requires a hard tab before recipe lines) and fix it with a real tab character; then find the DB port constant in the Python source (likely `config.py`) and change it from the wrong port to 8080 to match `db_service.py`. Don't just eyeball the file in an editor that might convert tabs — verify with `cat -A Makefile | grep -A1 run:` showing `^I` before the command. For the orchestration script, use `os.makedirs(path, exist_ok=True)` for idempotent directory creation, `subprocess.Popen` to launch `db_service.py` in the background, then poll with a short-timeout `socket.connect_ex(('127.0.0.1', 8080))` in a retry loop until it returns 0 before launching `make run` (also via Popen, cwd set to the aggregator directory) — starting the aggregator too early is exactly the failure mode described. For backups, spawn a background thread or subprocess loop that sleeps 2 seconds and copies `/home/user/data.db` to `/home/user/backups/data_backup.db` repeatedly, and keep the main process alive so both children keep running after the script logically finishes.
