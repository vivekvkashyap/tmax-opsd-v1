Write audit_and_fix.sh as a single script that takes `$1` as the target dir and performs each remediation independently, then run it once at the end. For integrity, `cd` into `$1/bin` (or use `--check` with relative paths) and run `sha256sum -c known_hashes.txt --quiet` (or parse its output) to identify which files fail, appending their absolute paths (`$1/bin/<name>`, resolved with e.g. `realpath`) to the compromised-files list. For permission auditing, use `find "$1" \( -perm -4000 -o -perm -2000 -o -perm -o+w \)` to locate SUID/SGID/world-writable files in one pass, append each absolute path to the violations list, then remediate each with `chmod u-s,g-s,o-w` (clears only those bits, leaves the rest of the mode untouched). For redaction, use a regex like `API_KEY=[A-Za-z0-9]{16}` restricted to files under `$1/logs/`, and rewrite matches in-place (e.g. `sed -i -E 's/API_KEY=[A-Za-z0-9]{16}/API_KEY=REDACTED/g'`), being careful the pattern requires exactly 16 chars so shorter keys like the "short" example are left alone. For the CSP check, grep the config file for the exact required line and append it only if absent. Sort both output lists alphabetically before writing (`sort` or `LC_ALL=C sort`), and chmod +x the script.
