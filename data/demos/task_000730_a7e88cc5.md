Design extractor.py around streaming tail-follow semantics: open live.log, seek to current end (or start), read in a loop, and buffer any trailing partial record (fewer bytes available than the header+payload requires) rather than discarding or crashing on it — retry after a short sleep once more bytes arrive. To handle rotation without dropping records, detect rotation by tracking the inode (`os.fstat(fd).st_ino`) of the currently open live.log; if a `stat()` on the live.log path shows a different inode, first drain any remaining bytes from the still-open old file descriptor (it still holds the rotated file's final ~512KB content) before switching to the new live.log — closing/reopening the path too early loses the tail of the rotated file.
