Build this as a linear pipeline and cache intermediate results so the API can serve them cheaply. Use OpenCV to read frames from vibration.mp4, convert each to grayscale, threshold at >200, and compute the centroid x via `cv2.moments` (or mean of thresholded pixel coordinates) to get x(t) as an array indexed by frame. For velocity, implement central differencing yourself: `v[i] = (x[i+1]-x[i-1]) / (2*dt)` with dt=1/30, using one-sided differences only at the two endpoints. For SSA: build the trajectory (Hankel) matrix from x(t) with window length 30, run `numpy.linalg.svd`, reconstruct using only the top-2 singular components (zero out the rest before inverse-transform/diagonal-averaging back to a 1D signal) — this is the step naive attempts get wrong by skipping the anti-diagonal averaging step of Hankelization. For dominant frequency, `np.fft.rfft` on the filtered signal, take `np.abs`, and pick the peak bin excluding index 0 (DC), then convert bin index to Hz using the frame rate. For the bootstrap CI, resample in contiguous blocks of 15 frames (not iid single points) 500 times, recomputing the dominant-frequency FFT peak each time, then take the 2.5/97.5 percentiles. Compute MSE against reference_signal.csv directly. Finally serve everything from a Flask/FastAPI app on 127.0.0.1:8000, checking the `Authorization: Bearer data-sci-token-882` header manually for /api/reference_diff and returning 401 on mismatch.
