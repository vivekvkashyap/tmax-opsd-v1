For the git hook: `git init --bare /home/user/metrics-config.git`, then place your compiled binary at `hooks/post-receive` inside that bare repo (must be the exact filename, executable) — read all of stdin (each line is "oldrev newrev refname"), and for each line emit one JSON line with the literal field order `{"type": "push", "ref": "<refname>"}` to `/home/user/dashboard/events.log`, creating the dashboard directory first. For the SMTP daemon: use `std::net::TcpListener` bound to 127.0.0.1:8025, accept connections in a loop, read line-by-line with a buffered reader, and when a line starts with `Subject: ` strip that prefix and any trailing `\r\n`/whitespace before appending it plus a newline to alerts.log; keep the listener loop running as a detached background process (e.g. `nohup ./smtp-alerter &`). For monitoring, write monitor.sh using `pgrep -f smtp-alerter` to check liveness and restart if absent, then install it via `crontab -l | { cat; echo "* * * * * /home/user/monitor.sh"; } | crontab -`.
