Build this as a straightforward Flask/http.server app that does real signal processing per request rather than precomputing anything. On each POST to /analyze_mesh: (1) extract frames from `/app/aero_mesh_sim.mp4` with OpenCV's `VideoCapture` (cache the decoded frames in memory at startup so you don't re-decode the whole video on every request — the FPS is given as exactly 30), (2) for each frame slice the region `frame[y_min:y_max, x_min:x_max]`, convert to grayscale, and take `.mean()` to build a 1-D time series across frames, (3) run `numpy.fft.rfft` on that series, compute frequency bins with `numpy.fft.rfftfreq(n, d=1/30)`, and find the index of the peak magnitude while excluding bin 0 (the DC component) — this exclusion is the detail naive attempts miss and it skews the result badly if left in, (4) load `/app/baseline_freqs.json`, look up the key formatted exactly as `"x_min,y_min,x_max,y_max"` matching the request, and compare `abs(computed - baseline) > 0.05` to set `is_divergent`. Start the server as a background process, then write `/home/user/service_ready.txt` only once the socket is actually accepting connections.
