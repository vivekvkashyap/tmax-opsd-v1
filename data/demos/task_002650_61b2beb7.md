Build the pipeline as separate, testable stages rather than one big script. First load each source with its declared encoding explicitly (pandas `read_parquet` for the parquet file, `open(..., encoding='shift_jis')` or `pd.read_csv(..., encoding='shift_jis')` for the Asia CSV, and `open(..., encoding='utf-16')` + `json.load` for the EU file) — a naive attempt that assumes UTF-8 everywhere will produce garbled or unparseable text. Standardize both review sources to the same two columns (id, text) before concatenating them, then do an inner merge against the users frame on the id column so only matched users survive. For normalization, apply the transformations in order: lowercase first, then strip everything outside `[a-z0-9\s]` with a regex, then split on whitespace and rejoin with a single space (`' '.join(text.split())`) so tabs/multiple spaces collapse — order matters, since removing punctuation before lowercasing or splitting before stripping symbols can leave stray characters. Finally sort records by username and write one compact JSON object per line (keys: username, signup_date, normalized_review) to the exact output path, verifying the file with a quick read-back before finishing.
