For the missing-config issue, don't guess the path — build the existing `processor.cpp` as-is, then run it under `strace -f -e trace=open,openat ./processor <somefile>` (with `SENSOR_CONF` unset) and read the strace output to find the exact absolute file path it attempts to open; create that path with `mkdir -p` on its directory and `touch` on the file itself. For the negative-VarSum bug, look at how the per-sample squared difference from the mean is computed and accumulated: if the intermediate `diff` and `diff*diff` are computed in a 32-bit `int` type, a diff of several thousand squared can overflow a 32-bit signed integer and wrap negative even though the true value is far below INT_MAX in the final sum — fix this by widening the diff (or at least one operand of the multiplication) to `long long` before squaring, so the multiplication itself happens at 64-bit width, not just the accumulator.
