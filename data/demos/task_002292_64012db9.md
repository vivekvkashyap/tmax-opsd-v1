Build this bottom-up: directories/links first, then the monitor script, then the pipeline wrapper, verifying each piece independently before chaining them. Create `/home/user/services-enabled/` and symlink only `web.json` and `api.json` from `services-available` (leave any other services, e.g. `db.json`, un-linked — check the directory listing first rather than assuming which files exist). In `monitor.py`, glob `*.json` in the enabled directory, parse each for `name`/`port`, and probe liveness with a short-timeout `socket.connect_ex(('127.0.0.1', port))` (a non-zero result means DOWN). Compute disk usage with `shutil.disk_usage('/home')` and cast `int((used/total)*100)`. For the timestamp, use `zoneinfo.ZoneInfo('Asia/Tokyo')` (not a hardcoded UTC+9 offset, which breaks on DST edge cases) and manually append the literal string `JST` since Python's default tz abbreviation won't necessarily print that label — match the log line format character-for-character. In `pipeline.sh`, run `python3 -m py_compile monitor.py`, `exit 1` on failure, else `mkdir -p bin`, symlink `bin/monitor -> monitor.py`, and `chmod +x monitor.py`. Finally run the pipeline then execute `/home/user/bin/monitor` once to populate `report.txt`.
