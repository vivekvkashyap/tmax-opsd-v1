Build the pipeline bottom-up and test each stage independently before chaining them. For `etl.sh`, use `awk -F,` (or a careful `csv`-aware parser) to skip the header, drop rows whose `split` field is `ignore`, drop rows whose `text` field is empty or all-whitespace (trim and check length), then route remaining rows to `clean_train.tsv`/`clean_test.tsv` by the `split` value, emitting only `id<TAB>text` with no header — watch out for trailing whitespace-only text fields that look non-empty until trimmed. For `reduce.sh`, remember the input CSV's first column is `id` and columns 2..K+1 are the first K embedding dims, so use `cut -d, -f1-$((K+1))` against the header-less embedding CSV. For `tune.sh`, nest nested loops over the K and T grids, generate temp reduced train/test files per combination (name them uniquely or reuse a temp dir cleaned each iteration), call `evaluate.py` and parse its single float from stdout, and track the running best using shell arithmetic/`bc` comparisons — then write `best_params.txt` in the exact `K=<k>,T=<t>,Score=<score>` format once the full grid has been searched. Don't hardcode a winner; let the grid search determine it.
