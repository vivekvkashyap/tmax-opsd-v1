Treat this as a bash-orchestrated pipeline where each stage writes files the next stage consumes, and the Bash scripts are the graded orchestration layer even though computation (SVD, regression) happens in short Python helpers they invoke. In `extract.sh`, use ffmpeg with a scale + grayscale filter to dump every frame of `calibration_video.mp4` as 128x128 JPEGs into `data/frames/`. In `compute_features.sh`, loop over the frame files (via `find ... | sort | while read` or `xargs`, in Bash) and for each call a small Python helper that loads the image as a 128x128 array and computes `numpy.linalg.svd`, keeping only the top 5 singular values; append each result as a CSV row `frame_idx,sv1..sv5` to `data/dataset.csv`, deriving `frame_idx` from the filename ordering (1-indexed) rather than hardcoding it. In `train.sh`, fit `sklearn.linear_model.Ridge` with 5-fold CV over alpha in `[0.1, 1.0, 10.0]` (`GridSearchCV` or manual `KFold` loop) to pick the best alpha by CV MAE, bootstrap-resample the CV fold errors ~100 times to get a 95% CI on MAE and log it, then persist the fitted pipeline with `pickle`/`joblib` to `model.pkl`. `inference.sh` should load the model, recompute the same top-5-SVD feature vector for the given image path, and print only the numeric predicted frame index. Chain all steps in `pipeline.sh` and confirm end-to-end MAE on held-out frames is comfortably under the 20.0 threshold before finishing.
