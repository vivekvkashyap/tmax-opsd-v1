Work through this as a pipeline of discrete, independently-testable stages rather than one big script. First extract the tarball with `tar -xzf`, then parse `build.conf` line by line: for `REPLACE:Old:New` lines, loop over files under `extracted_docs/` and apply substitutions with `sed -i "s/Old/New/g"` (watch out for regex metacharacters in Old/New — treat them literally if they could contain slashes or special chars, or escape them). For `COMPRESS_EXT:.ext`, remember only files matching that exact extension get the compression step — other files just get the text replacements. For the C++ compressor, write a simple state-machine scanner over the file's characters: count consecutive runs of the space character, and only when a run length is 2 or more do you emit `&N&`; a lone space must pass through unchanged, and don't let runs of other whitespace (tabs, newlines) trigger it. Read/write via stdin/stdout or explicit file paths, and process each matching file with the compiled binary in place. Finally repackage with `tar -czf` so the extracted files are recoverable at the tarball root or inside a top-level directory. Test each stage against a small file before running the whole pipeline.
