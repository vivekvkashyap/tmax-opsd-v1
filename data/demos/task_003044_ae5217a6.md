Extract `/home/user/incoming_logs.tar` into `/home/user/processing`, then use `find /home/user/processing -name '*.log'` to locate all log files regardless of nesting depth. For each one, open a file descriptor and take a shared lock before reading — in bash this looks like `exec 9<"$file"; flock -s 9; split -l 50 "$file" "${dir}/${base}_chunk_"; exec 9<&-` (the grader specifically checks for `flock` usage in the script, so don't skip it even though nothing else is writing concurrently). `split` produces suffixes like `aa`, `ab`, ...; rename each resulting piece from `<prefix>aa` to `<name_without_ext>_chunk_aa.txt` (strip the `.log` extension from the original name, keep the split suffix, append `.txt`) using a loop over the split output files, not a hardcoded suffix list, since file sizes determine how many chunks exist. After all files are split and renamed, gather every `*chunk*.txt` across all subdirectories and tar them into `/home/user/final_chunks.tar.gz` flat — use `tar -czf final_chunks.tar.gz -C <dir> file1 file2 ...` per source directory, or copy/symlink all chunks into one staging directory first, then tar just the basenames (avoid `tar -C` per-file with full paths, which would preserve directory structure). Finally list the archive's members with `tar -tzf`, sort them, and write them one-per-line to `/home/user/report.txt`.
