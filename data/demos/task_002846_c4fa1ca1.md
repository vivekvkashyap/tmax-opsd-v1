Build the C daemon as a straightforward loop: create a Unix domain socket with `socket(AF_UNIX, SOCK_STREAM, 0)`, `bind` to `/home/user/account.sock` (unlink any stale socket file first), `listen`, then loop `accept()`-ing one connection at a time (sequential is fine, no threads needed). Per connection: write "PASSWD:\n", `read()` the response, compare exactly to "adminpass\n" — if it doesn't match, write "DENIED\n" and `close()` the fd immediately (don't keep reading). On success write "AUTH_OK\n" and enter an inner loop reading further lines, replying "PONG\n" to "PING\n" and closing on "QUIT\n". Test the daemon manually with `socat - UNIX-CONNECT:/home/user/account.sock` before writing service.sh or the Expect script, so you can see the exact byte sequence going back and forth. For service.sh, use `$!` after backgrounding the daemon to capture the PID and write it to the pid file; for stop, `kill $(cat pidfile)` then clean up both the pid file and the socket file. For health_check.exp, use `spawn nc -U ...`, then `expect "PASSWD:"`, `send "adminpass\r"`, `expect "AUTH_OK"`, `send "PING\r"`, `expect "PONG"` with a reasonable `timeout` set, exiting 0 only after the final match. Finish by leaving the daemon started via service.sh.
