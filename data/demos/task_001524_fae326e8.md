Break this into independently-testable stages before wiring them into the server. (1) Extract the transcript: `strings /app/audio/transmission.wav | grep TRANSCRIPT_DATA:` to pull the raw line, then strip the `TRANSCRIPT_DATA:` prefix and pipe the remainder through `iconv -f UTF-8 -t ASCII//TRANSLIT` (or similar transliteration) to fold accented/non-ASCII characters into plain ASCII, dropping anything undroppable with `//IGNORE` as needed — verify the result contains no non-ASCII bytes with `grep -P '[^\x00-\x7F]'` before trusting it. (2) Telemetry: for each CSV in `/app/data/telemetry/`, process files in parallel via `find ... | xargs -P<n> -I{} ...`; within each file's processing, read min/max timestamp, then for every second in that range emit the most recent value seen so far (forward-fill) using `awk`, and track a running max per file, writing results somewhere collectible; after all parallel jobs finish, take the max across all per-file maxes. (3) Server: a bash+`nc`/`socat` loop that accepts a connection, reads the request (can ignore the body), and writes back a correct `HTTP/1.1 200 OK` status line, a `Content-Type: application/json` header, blank line, and the JSON body — test manually with `curl -v http://127.0.0.1:8080/` and validate with `jq` before considering the server correct, and make sure it loops to serve repeated requests rather than exiting after one.
