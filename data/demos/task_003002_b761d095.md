Decompose into OCR recovery, DB repair, graph query, and a tiny HTTP server. Run OCR on `/app/pattern.png` (e.g. `tesseract /app/pattern.png stdout`) to read off the exact sequence of node *types* it specifies — do not guess the pattern, extract it from the image text. Open `/app/backup.sqlite` with `sqlite3` and run `REINDEX;` (or `REINDEX <table>;`) on the `edges`/`nodes` tables to clear the corrupted-index condition before querying. Then write a query (or a short script) that walks `edges` joined against `nodes.type`, matching a directed path whose node types follow the sequence recovered from OCR — a multi-way self-join or a small graph-search script over adjacency built from `edges` both work; return the corresponding node `name`s in path order. Finally implement a minimal HTTP server (Python's `http.server`/Flask, or another language) bound to `127.0.0.1:8080` exposing `GET /pattern` that returns `{"path": ["name1", "name2", "name3"]}` built from your query result — serialize with a JSON library rather than hand-built strings to avoid quoting bugs, and launch it backgrounded (`nohup ... &`) so it stays up for the verifier.
