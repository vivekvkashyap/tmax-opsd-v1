Structure `process_loc.sh` as: dedupe -> mask -> sort/write cleaned -> aggregate cumulative -> done. For dedup-by-highest-timestamp per (translation_id, language_code), sort the CSV by id, language, then timestamp descending, and use `awk -F, '!seen[$1","$2]++'` (or equivalent) to keep only the first (highest-timestamp) row per key — don't just dedupe on full-row equality. Mask emails with `sed -E 's/@[^,]+/@anonymized.local/'` applied only to the email field. Sort the cleaned rows numerically by timestamp (`sort -t, -k3,3n`) before writing `cleaned.csv`. For `cumulative_stats.csv`, first sum word_count per (timestamp, language) pair (since ties at the same timestamp must be merged into one output row and added to the running total before that row is emitted), then walk the timestamp-sorted, language-grouped totals accumulating a running sum per language — an awk associative array keyed by language, updated in a single pass over rows sorted by timestamp, works well. Finally install the cron entry with `(crontab -l 2>/dev/null; echo "15 3 * * * /home/user/process_loc.sh") | crontab -`, and confirm with `crontab -l` that the exact schedule and path match.
