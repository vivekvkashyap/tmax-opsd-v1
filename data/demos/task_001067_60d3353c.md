Build this as one bash pipeline script rather than solving pieces ad hoc. Tokenize with `tr '[:upper:]' '[:lower:]' < raw_data.txt | tr -c 'a-z' ' ' | tr -s ' ' '\n' | grep -v '^$' > tokenized.txt` (or equivalent sed/tr chain) so every non-letter becomes a separator and runs collapse to one word per line. For the round-robin fold split, read tokenized.txt line by line with a counter and use modulo-3 arithmetic (e.g. `awk 'NR%3==1{print > "fold_1.txt"} NR%3==2{...} NR%3==0{...}'`) rather than manually chunking — line 1 to fold_1, line 2 to fold_2, line 3 to fold_3, line 4 back to fold_1, etc. For the hyperparameter search, concatenate fold_1+fold_2, compute word frequency counts (`sort | uniq -c`), then for each candidate T from 1 to 5 count how many distinct words have frequency >= T, and find the T where that count equals exactly 4 — write only that integer to best_param.txt. Put `export OMP_NUM_THREADS=1` as the very first line of process.sh, chmod +x it, and actually execute it so all output files are generated fresh.
