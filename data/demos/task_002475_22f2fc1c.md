Structure the Go daemon as: (1) a poll loop over `/home/user/incoming` checking for new `.tar.gz` files every second (simplest robust approach — avoid overengineering with fsnotify unless comfortable with it); (2) on detection, extract with `archive/tar` + `compress/gzip` into a temp dir; (3) walk the extracted files, and for any file with size strictly greater than 1,024,000 bytes, split it into sequential 1,024,000-byte chunks named `<name>.part0`, `.part1`, etc. (last chunk shorter), then delete the original — files exactly at or under 1,024,000 bytes are left untouched, so use strict `>` not `>=`, a common off-by-one mistake given the task's edge_case.bin test. (4) Zip everything (chunks + untouched files) with `archive/zip` into `/home/user/outgoing/<basename>.zip`. (5) Append one JSON line to `/home/user/artifact_log.jsonl` with `original_archive`, `repackaged_archive`, and a `chunked_files` map containing ONLY files that were actually split (name -> chunk count). After building, `go run`/`go build` and start the daemon in the background, then run `generate_test_data.sh`, move the payload into `incoming/`, and manually inspect the resulting zip contents and log line before finishing.
