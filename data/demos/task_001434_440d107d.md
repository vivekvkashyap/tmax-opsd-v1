Approach this as static analysis first, then build tooling. Use `strings /home/user/workspace/auth_service | grep SEC_KEY=` (or similar) to recover the plaintext secret embedded in the binary — do not try to disassemble or run the untrusted binary directly on the host for this step. Once you have the secret string, write `token_gen.cpp` that takes the secret and a date string as argv, XORs each secret byte with the date string cycled to match length, and prints the result as lowercase hex (build the hex manually with `sprintf("%02x", ...)` or similar, byte by byte, in the same order as the secret). Compile it, run it with the extracted secret and the date `"20241231"`, and redirect the output into `new_token.txt`. For verification, write `verify.sh` that wraps execution of `./auth_service <token>` inside `unshare --net --ipc -- ...` so the binary runs in isolated network and IPC namespaces (you likely need `sudo` or appropriate privileges for `unshare` to create new namespaces), read the token from `new_token.txt` inside the script, `chmod +x verify.sh`, and confirm it exits 0 before finishing.
