Design this as a single-pass, chronological state machine in C, since every rule (dedup, rate anomaly, version anomaly) depends on comparing each new valid record against state accumulated from prior records — do not try to do it with a batch/sort-then-filter approach. Compile a strict `regex.h` pattern for the log line format and skip (regexec != 0) anything that doesn't match. Parse the bracketed timestamp with `strptime` and `timegm` (not `mktime`, to keep it UTC) to get a Unix epoch. For dedup, compare each newly parsed valid record only to the immediately preceding *valid* record (same user/action/config_v, and epoch delta <= 10s) — a duplicate must not become the new "preceding" record for the next comparison, keep the first occurrence as the anchor. Maintain, per user, a small list/queue of that user's valid-action epoch timestamps; before appending the current one, count how many prior timestamps for that user fall within 60s of it — if that count is >= 3, flag ANOMALY_RATE (this is the 4th-or-later action in the window). Separately track a single global running maximum CONFIG_V seen so far (updated only after the comparison, and never flag the very first valid record) — if the new record's CONFIG_V exceeds that running max by more than 50, flag ANOMALY_VERSION. Join both flags with `|` when both apply, else NONE. Write CSV rows only for surviving (non-duplicate) records, compile with `-Wall`, and run it to produce processed_audit.csv.
