Split this into independently-checkable stages. First get a transcript: install ffmpeg + a speech-to-text tool (e.g. `pip install openai-whisper`, or another ASR you can get working offline), run it on /app/metrics_dictation.wav, and print the raw transcript so you can visually confirm it captured a sequence of "Measurement N: <value>" style utterances before trusting it. Second, parse the transcript into a plain numeric list: strip narration words, convert any spelled-out numbers to digits (a small word-to-number map, or a library like `word2number`, handles cases where the ASR outputs words instead of digits) — don't assume the ASR always emits digits. Third, compute the rolling average with window=3 using the exact edge-case rule given (avg of available elements when fewer than 3 exist so far, not zero-padding), rounding each to 2 decimals, and write Index (1-based),Value,RollingAvg to the CSV via csv.writer. Fourth, write process_metrics.sh wrapping this logic (or invoking your script) and `chmod +x` it, then create the cron file with a line matching `*/15 * * * * /home/user/process_metrics.sh`. Since verification tolerates MSE<=2.0, minor per-value ASR slips are fine — but a wrong window definition or unit/order errors will blow the threshold, so hand-check your rolling average formula against a couple of manually computed points from the printed transcript.
