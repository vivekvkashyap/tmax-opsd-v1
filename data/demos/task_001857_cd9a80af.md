Treat this as three separable stages: parse the log to find which files to back up, convert+rename those files, then archive+encode. First, parse `/home/user/changes.log` structurally (split on blank lines or `COMMIT:` markers) and take the LAST commit block only, extracting filenames from its `FILES:` list (lines starting with `- `). Don't just grep the whole file for filenames — you need the ones tied to the final commit block specifically. For each of those files, read the `key=value` lines from `/home/user/configs/<file>.conf` and build a JSON object (all values as strings is fine), writing it to `/home/user/backup/backup_<name>.json` (strip `.conf`, add `.json`, prepend `backup_`). Then build the tar so the JSON files are at the archive root, not nested under `backup/` — from inside `/home/user/backup/`, run `tar -czf` on just the filenames (or use `tar -C /home/user/backup -czf ...`) rather than tarring the parent path. Finally base64-encode the resulting tarball (`base64 archive.tar.gz > /home/user/archive.b64`) and confirm by round-tripping: decode, extract, and check the JSON contents match.
