Start the MLflow server as a background process first (`mlflow server --backend-store-uri sqlite:///... --default-artifact-root ... --host 127.0.0.1 --port 5000 &`, rooted under `/home/user/mlruns`) and confirm it's reachable before running any training. When fixing `train.py`, the core discipline is: call `train_test_split` on the raw, unimputed/unscaled features first, then `fit_transform` the imputer and scaler on the training split only, and `transform` (never fit) the test split with those same fitted objects — this is the standard way to eliminate train/test leakage. Apply the identical ordering fix to the correlation matrix: compute it only from the scaled training features, not the full dataset. For the confidence interval, compute per-sample squared errors on the test set, then use `scipy.stats.t.interval` with the sample mean and standard error of the mean (`scipy.stats.sem`) over those squared errors at 95% confidence — do not just use a normal approximation. Log `mse_ci_lower`/`mse_ci_upper` via `mlflow.log_metric`, then after the run finishes, pull the run_id from `mlflow.active_run()` or the client, and write `fix_summary.json` with values rounded to 4 decimals, keeping `RANDOM_STATE=42` untouched throughout.
