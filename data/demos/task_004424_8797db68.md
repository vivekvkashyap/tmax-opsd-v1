Set up a Rust binary (cargo new) rather than a single-file script so you can pull in `regex`/`chrono`/`csv` if useful, but plain `std` parsing (split on whitespace, strip `key=` prefixes) is simpler and sufficient here. Parse each log line with a regex or manual split to pull out the date (first 10 chars of the bracketed timestamp), `target_server`, and `snapshot_hash`; keep the full timestamp too so you can compare within a day. Build a map keyed by (date, server) and, when a duplicate (date, server) appears, overwrite only if the new timestamp is later than the stored one — do not just keep the last line seen in file order, since lines may not be chronological within a day. Then, per server, find min/max date, walk every calendar date in that inclusive range (use a proper date library or manual day arithmetic, watching month/year rollovers), and for any date missing from the map, carry forward the hash from the nearest prior date that has one (not necessarily the previous calendar day — skip through gaps). Finally collect all (date, server, hash) rows, sort by date ascending then server name ascending, and write the CSV with header `date,server,hash` to the exact output path. Test against the raw log by hand-tracing which timestamp wins for servers with same-day duplicates before trusting the program's output.
