Handle the recursive-archive extraction with a loop, not a single pass: extract `project_backup.tar.gz` into `/home/user/extracted_project`, then repeatedly scan for any remaining `.zip` or `.tar.gz` files anywhere under that tree (`find ... -name '*.zip' -o -name '*.tar.gz'`), extract each into its containing directory (`unzip` for zip, `tar -xzf` for tar.gz) and delete the archive file immediately after extracting it, and keep repeating the scan-extract-delete cycle until no archive files remain (nesting could be more than one level deep, so don't assume a fixed depth). Once extraction is fully flat, use `find ... -name '*.py'` to locate every Python file recursively and run an in-place text substitution replacing the literal string `FIXME:` with `TODO:` in each (e.g. `sed -i 's/FIXME:/TODO:/g' file`) — do this after all extraction/deletion is complete so you don't miss files still packed in nested archives. Then `tar -cjf /home/user/clean_project.tar.bz2 -C /home/user extracted_project` (or equivalent) to produce a single bzip2 tarball. Finally, for the JSON report, for each `.py` file count occurrences of `TODO:` (post-replacement, so include pre-existing TODOs too) and write a `{basename: count}` dict to `/home/user/todo_report.json` — a quick Python script with `collections` or plain grep -c per file works well here.
