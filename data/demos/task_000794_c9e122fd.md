Break this into independently verifiable stages. (1) Barcode: extract one frame per second with ffmpeg, read each frame's average RGB (e.g. Pillow), and classify against the four given color rules to map each frame to a nucleotide — derive the 12-char string this way, don't assume an order. (2) Assembly: from `reads.txt`, extract all k=4 substrings as graph edges (k-1-mers as nodes), build adjacency with multiplicity, then find an Eulerian path (Hierholzer's) starting from the node with out-in degree +1 (or any node if balanced); reconstruct by concatenating path node prefixes plus the final node. Sanity-check that the result starts with your step-1 barcode. (3) MCMC: align each read to its exact matching k-mer position in the consensus, tally matches/mismatches, run Metropolis-Hastings with Uniform(0,0.5) prior and binomial likelihood for >=5000 iterations, report the posterior mean. (4) Server: implement a minimal HTTP server (raw sockets or a header-only lib) exposing the three GET routes as plain-text bodies, compile, and run bound to 0.0.0.0:8080 in the background so it stays reachable.
