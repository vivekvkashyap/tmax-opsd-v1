Design the supervisor around a fixed pool of worker slots, each backed by its own pair of pipes (one for stdin to the child, one for stdout from the child) and its own PID. Use `fork`+`exec` per worker, `dup2` the pipe ends before `exec`, and reap deaths with a `SIGCHLD` handler (setting a flag) or non-blocking `waitpid(..., WNOHANG)` polled in the main loop — on detecting a dead child, immediately re-fork a replacement into that same slot and rewire its pipes so the slot count never drops below `WORKER_COUNT`. Route incoming stdin lines round-robin across slots, but check each slot's liveness/pipe-writability before writing (a `SIGPIPE`/`EPIPE` on a crashed worker must not kill the supervisor — ignore `SIGPIPE` and skip/reroute to the next live worker instead of losing lines). Use `poll`/`select` over all worker stdout fds to aggregate output non-blockingly so throughput isn't serialized. For log rotation, track the current file's byte size as you write and only rotate (rename to `.1`, open a new file) right after a complete line pushes size over 1MB. Compile with `gcc -O3` in `build.sh` and test end-to-end timing before assuming correctness.
