Compile `generator.c` with gcc, run it, and redirect stdout to samples.txt. Then in fit.py: load the samples with numpy, and derive the negative log-likelihood analytically from the given density — since p(x|a) = exp(-x^4 - a*x^2)/Z(a), the per-sample log-likelihood is -(x^4 + a*x^2) - log(Z(a)), so the total NLL over n samples is sum(x^4) + a*sum(x^2) + n*log(Z(a)) (the x^4 term is constant in a and can be dropped from optimization). Define Z(a) using `scipy.integrate.quad(lambda x: np.exp(-x**4 - a*x**2), -np.inf, np.inf)`, wrap it in a function of a. Minimize the NLL over a in [-5,5] using `scipy.optimize.minimize_scalar(..., bounds=(-5,5), method='bounded')`. For numerical stability, avoid recomputing Z redundantly per sample (compute it once per candidate a), and guard against Z becoming 0 or negative before taking log. Round the optimizer's result to 3 decimal places and write to result.txt in the exact `a=X.XXX` format — do not hardcode a value, let the optimization determine it from the actual samples.
