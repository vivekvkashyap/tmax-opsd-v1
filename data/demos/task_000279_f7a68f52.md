For each row/column cell: treat an empty string as "no packet" and drop it; also drop values that parse to `0`, per the "drop empty or zero" rule — don't accidentally drop other falsy-but-valid values. Aggregate per stream_type into a list of ints, then compute `max()` (as int) and `mean` formatted with exactly two decimals via `f"{avg:.2f}"` (a string, not a float). Decide explicitly what happens when a stream_type has zero valid packets after filtering (e.g. omit it from output) — this is a natural place your implementation could silently diverge from the oracle's, so test it deliberately. Sort the output list alphabetically by stream_type and emit via `json.dumps` to stdout. Build the ffprobe wrapper separately with `-show_packets -print_format csv` to produce real CSV input for a manual end-to-end run, but keep the core transform logic self-contained and stdin/stdout-only so the fuzzer can drive it directly.
