Start by feeding `/app/kg_matcher` small, hand-crafted inputs via stdin (a handful of triples with obvious 2-hop patterns) and diffing the exact stdout byte-for-byte across variations: try empty input, no matches, one match, multiple matches with ties, whitespace around the `|` separators, and duplicate triples — this tells you the precise JSON key names, field order, separators/whitespace (`json.dumps` with default vs compact separators), sort order, and how duplicates/empty results are handled, all things you must not guess. Only encode a rule into your Python script once you've confirmed it against actual binary output. Implement the core join with simple dict/list building: group `(subject, object)` pairs by predicate, then for each `X --pred1--> Y` pair look up `Y --pred2--> Z` pairs, emit the composite dict, sort the full result list by the tuple `(start_node, bridge_node, target_node)`, and serialize with the exact separators you observed from the oracle binary. Test against many more random inputs than you probed manually before considering the task done, since the grader will fuzz-test with fresh random inputs.
