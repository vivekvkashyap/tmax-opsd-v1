Build the pipeline bottom-up and test each stage before wiring it together. Init the bare repo with `git init --bare /home/user/dashboards.git`. Generate the cert/key with `openssl req -x509 -newkey rsa:2048 -keyout key.pem -out cert.pem -days 365 -nodes -subj "/CN=localhost"` (the `-nodes` flag avoids a password prompt). Write `deploy_env.sh` as a simple `export DEPLOY_ENV=prod` file. For the `post-receive` hook: read stdin with `read oldrev newrev refname`, `source /home/user/deploy_env.sh`, then `if [ "$DEPLOY_ENV" = "prod" ]; then python3 /home/user/deploy.py "$newrev"; fi` — remember hooks run with the repo as CWD and need `chmod +x`. In `deploy.py`, use `subprocess.run(["git", "archive", newrev], cwd="/home/user/dashboards.git", stdout=...)` piped into `tar -x -C /home/user/www/dashboards/` (create the dir first) rather than trying to parse git objects manually. For `server.py`, wrap a standard `http.server` with `ssl.wrap_socket`/`SSLContext.load_cert_chain` bound to port 8443, serving from `/home/user/www/`. Launch it with `nohup python3 server.py & echo $! > server.pid`, then verify end-to-end by cloning the bare repo, pushing a test commit, and curling `https://localhost:8443/...` with `curl -k`.
