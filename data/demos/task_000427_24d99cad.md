Build this as a pipeline orchestrator, not a reimplementation of the extractor. In your C program: (1) read events.jsonl line by line without modifying the source file; (2) sanitize each line with a simple scan-and-replace turning every `\u` occurrence into `__` before it ever reaches the legacy binary; (3) spawn `/app/legacy_extractor` as a child process using `fork`/`exec` with `pipe()` (or `popen`), write the sanitized line(s) to its stdin, and read its CSV stdout back — do this per-line or via a bidirectional pipe so you can correlate each output row to its source; (4) parse the ISO 8601 timestamp the extractor echoes back (format `YYYY-MM-DDTHH:MM:SSZ`) using `strptime` with `TZ=UTC`/`timegm` (not `mktime`, which uses local time and will shift results) to get a correct Unix epoch integer; (5) write `unix_timestamp,feature_score` rows with no header to output.csv; (6) count successfully written rows and write the exact summary line to summary.txt. The two easy failure points are: forgetting the sanitization step (causing segfaults that kill the whole pipeline) and using local-time conversion instead of UTC (causing timestamp misalignment against the reference).
