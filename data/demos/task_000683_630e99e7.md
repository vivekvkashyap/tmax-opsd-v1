Work in two stages: recovery then service. First, run `tesseract /app/data_plot.png stdout` (or use pytesseract) to OCR the numbers printed on the image; don't assume the values, actually extract and parse them into a numeric array — clean up OCR noise (stray characters, spacing) before parsing. Compute the z-score embedding yourself: mean = sum/n, std = population or sample stddev consistent with how you'll describe it, then (x-mean)/std per element. Second, write a minimal C++ HTTP server (raw sockets or a header-only lib) that binds to 127.0.0.1:8080, parses just enough of the request line to detect `GET /embedding`, and writes a valid HTTP response with `Content-Type: application/json` and a body matching `{"mean":..., "std":..., "embedding":[...]}`. Common pitfalls: forgetting the blank line separating HTTP headers from body, not flushing/closing the socket properly, and hardcoding numbers instead of deriving them from the OCR'd image at runtime (the grader may swap the image). Put compilation (`g++ ...`) and the server launch (backgrounded, e.g. `./server &`) into `/home/user/run_server.sh`, and verify with `curl` locally before finishing.
