Work this in three separate phases and verify each before moving on. (1) Recovery: transcribe `/app/admin_note.wav` yourself using an available speech-to-text tool (e.g. `whisper`, or any offline STT you can install/run) to get the base passphrase — don't guess it. Analyze `/app/legacy_auditor` with `strings`/`objdump`/`nm` to recover the literal header name and salt strings embedded in the binary. Then construct candidate tokens as `<passphrase><salt><NN>` for NN from 00 to 99, sha256-hash each, and compare against the hash in `/app/token_hash.txt` to find the matching two-digit suffix — a simple bash or python loop over `sha256sum` is enough. (2) Implementation: write `/app/cgi-bin/upload.sh` as a standard CGI script — check the auth header via its `HTTP_<HEADERNAME>`-style env var first and emit `401` if wrong; sanitize `HTTP_X_FILENAME` by stripping directory components (e.g. `basename`) and rejecting any input still containing `..` or a leading `/` with `400`; save the POST body (read from stdin) to `/app/audit_logs/<sanitized-name>`; recompute sha256 of the saved file and compare to `HTTP_X_FILE_HASH`, deleting and returning `422` on mismatch, else `200 SUCCESS`. Always print `Content-Type: text/plain` then a blank line before any body. (3) Deployment: `chmod +x` the script, create `/app/audit_logs`, and launch `python3 -m http.server 8080 --cgi` from `/app` in the background so it stays up.
