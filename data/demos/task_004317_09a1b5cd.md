Build and test this in stages rather than all at once. Set up the module with `go mod init auto_archiver` in /home/user/archiver and `go get github.com/fsnotify/fsnotify`. In main.go, take the watch-root as argv[1], walk it recursively with `filepath.Walk`/`filepath.WalkDir` adding every directory (not files) to a single `fsnotify.NewWatcher()`, then select on `watcher.Events`/`watcher.Errors` in a loop, checking `event.Op & (fsnotify.Create|fsnotify.Write) != 0`. On the first qualifying event, immediately capture the triggering file's absolute path (use `filepath.Abs`) for trigger.log, then build the tar.gz: walk the watched root again, and for each file compute its path *relative to the watch root* for the tar header name (`filepath.Rel(root, path)`) so archive entries look like `subdir/trigger.txt`, not absolute paths — this relative-path handling is the easiest part to get wrong. Write the tar entries through `gzip.NewWriter` -> `tar.NewWriter`, close both writers in the correct order before exiting, then write trigger.log, and `os.Exit(0)`. Separately create the /home/user/workdir/subdir test fixture with initial.txt containing "INIT", `go build` and run the binary in the background against workdir, then create subdir/trigger.txt with "BACKUP_ME" to fire the event, and wait for the process to exit and both output files to appear.
