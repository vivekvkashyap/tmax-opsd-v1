Spawn the mock script with `std::process::Command::new("/home/user/qemu_serial_mock.sh").stdin(Stdio::piped()).stdout(Stdio::piped()).spawn()`. The critical pitfall: you cannot just blast all 4 answers at once and hope it works, and you must flush stdin after every write (`writeln!` + `.flush()`) or the child process — buffered on its own read — can stall waiting for input that's stuck in your process's output buffer, causing a deadlock. Best approach: wrap the child's stdout in a `BufReader`, read it line-by-line (or watch for each expected prompt substring) interleaved with writing the next answer to stdin in the documented order (Timezone, Locale, IP, Gateway), flushing each time. After `child.wait()` succeeds, parse the gateway IP (4th line of the spec file) and write it into `/home/user/route-eth0.conf` in the exact `default via <GATEWAY_IP> dev eth0` format — don't hardcode the IP, read it from the spec file so it's correct for any input.
