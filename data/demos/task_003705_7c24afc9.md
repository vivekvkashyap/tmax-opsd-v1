Split the raw CSV before computing any statistics, never after. In `prepare_data.sh`: use `head -n 80 raw.csv > train_raw.csv` and `tail -n 20 raw.csv > test_raw.csv` first, then run a single `awk` pass over ONLY `train_raw.csv` to accumulate sums and sums-of-squares for columns 1 and 2, computing population mean and std (std = sqrt(E[x^2]-E[x]^2), no Bessel correction). Pass those four scalars (via `-v`) into a second `awk` invocation applied separately to `train_raw.csv` and to `test_raw.csv`, standardizing columns 1-2 and leaving column 3 (`y`) untouched, writing to `train_scaled.csv` and `test_scaled.csv` respectively. The classic bug is computing mean/std over the whole file before splitting, or reusing the same combined stats for both outputs — avoid that. For `bootstrap.sh`, read `train_scaled.csv` into an array (e.g. via `mapfile`/`readarray`), then for each of 5 output files loop 80 times picking a random line via `$(( RANDOM % 80 ))` (note `$RANDOM` alone can exceed array bounds, so mod it) and append the sampled line, producing `boot_1.csv`...`boot_5.csv` each with exactly 80 rows sampled with replacement from the 80 training rows.
