Get each moving part working in isolation before wiring them together. Have `provision.py` `subprocess.Popen` four `python3 /app/backend.py --port 800{1..4}` background processes, then actually poll each port (e.g. loop attempting a socket connect or `curl`) until it accepts connections rather than assuming they're up immediately — this is the flakiest naive-attempt failure point. Generate `nginx.conf` with an `upstream` block listing all four `127.0.0.1:800N` backends (default round-robin load balancing is exactly what "load balance evenly" needs — no special directive required) and a `server { listen 8080; }` proxying to that upstream, with `access_log /home/user/proxy_logs/access.log;`. Before starting nginx, create `/home/user/proxy_logs` and `chmod 700` it, and after nginx writes its first log line, `chmod 600` the log file (or set the file's permissions immediately once it exists). Launch nginx with your config and confirm with `curl localhost:8080/` that it round-trips. For `rotate.py`, `os.rename` the log to `.1`, then send `SIGUSR1` (or equivalent nginx reopen signal) to the nginx master PID, guarding the rename in a check for file existence first. Finally benchmark with concurrent requests yourself to sanity-check you're well under 3.5s before considering it done — four parallel 50ms backends should comfortably clear 200 requests.
