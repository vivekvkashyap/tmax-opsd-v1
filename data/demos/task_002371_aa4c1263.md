Treat this as a self-join problem over JSON Lines, most robustly done via `jq` or a short inline Python invoked from the bash script (bash receiving the date as `$1`). Load all transactions, filter to those whose timestamp date-prefix matches the given `YYYY-MM-DD` argument, then find pairs (A, B) with A.sender == B.receiver AND A.receiver == B.sender AND A.timestamp == B.timestamp AND A.tx_id != B.tx_id. Key traps: (1) each unordered pair will be found twice via the self-join (A,B) and (B,A) — dedupe by only keeping pairs once, e.g. by sorting the two tx_ids and using a set/dict keyed on the sorted pair, or requiring tx_id_A < tx_id_B; (2) within each output pair, sort the two tx_ids alphabetically before joining with a comma (not by original order); (3) sort the final list of "id1,id2" lines alphabetically before writing to `/home/user/deadlocks.log`. Also don't accidentally match a transaction against itself. Make the script executable, parameterize strictly on `$1` for the date, and run it with `2023-10-01` at the end to actually produce the log file the grader reads.
