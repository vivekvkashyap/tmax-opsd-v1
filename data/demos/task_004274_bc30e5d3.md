Build and verify each component independently before wiring them together. Create the raw data dirs and size the dummy files precisely with `fallocate -l 1M`/`2M` (or `dd`), then create symlinks with `ln -s <absolute-path-of-target> /home/user/symlinks/<name>` — use absolute, not relative, targets since the check does `readlink` and compares against the full path. For `gather_stats.sh`, don't hardcode `/home/user`: reference `$STORAGE_ROOT/symlinks/` so the script only works when `runner.sh` exports that variable first; compute each symlink's size with something that follows the link to the real target (e.g. `du -sb "$(readlink -f ...)"`) and emit strict `{"alpha": N, "beta": N}` JSON. Run `runner.sh` once and inspect `usage.json` before touching Rust. For the server, a small `std::net::TcpListener` loop is sufficient — no crates needed — just parse the request line for path, and read/return the JSON file's raw bytes for `/metrics`. Start it detached (e.g. `nohup ... &`) and confirm both endpoints with `curl` yourself before declaring done.
