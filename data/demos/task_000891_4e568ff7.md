Work in order: first apply the patch with `patch parser.cpp < update_signatures.patch` (or `patch -p1` depending on the diff headers — inspect the `---`/`+++` lines to pick the right strip level) so the new signature checks land in `parser.cpp`. Compile the shared library position-independent: `g++ -fPIC -shared -o libwafparser.so parser.cpp`. Compile the executable separately and link it against that library, then set the runpath explicitly (not just `-L.`): use `g++ -o waf_analyzer main.cpp -L. -lwafparser -Wl,-rpath,'$ORIGIN/../lib'` — remember `$ORIGIN` must reach the linker literally (escape or single-quote it so the shell doesn't expand it) and it is relative to the executable's own location once moved into `bin/`, so it must point at `../lib`, not `.`. After building, create `deploy/bin` and `deploy/lib`, move the binary and library respectively, then verify the rpath survived with `readelf -d waf_analyzer | grep -E 'RUNPATH|RPATH'` before running. Finally `cd` into `deploy/bin` and execute `./waf_analyzer /absolute/path/to/requests.log > ../analysis_results.log` — running from that directory is what proves `$ORIGIN` resolution actually works, so don't just run it in place from the build directory.
