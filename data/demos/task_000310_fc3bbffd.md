Then design the C program: load base state into a small server/param -> float map, parse `deltas.csv` (skip header), and for each of the 24 hour buckets (bucket h covers `[base_time + h*3600, base_time + (h+1)*3600 - 1]`) apply all deltas whose timestamp falls within or before that bucket's end in timestamp order, updating the in-memory state, then snapshot the current values as that hour's row — carrying forward unchanged params rather than resetting them each bucket. Since deltas.csv is presorted by timestamp in this task, a single sequential pointer walk (advance while `delta.timestamp <= bucket_end`) is simpler and correct than repeatedly re-scanning. Write the wide CSV with the exact header and one row per hour, formatting floats with `%.1f`.
