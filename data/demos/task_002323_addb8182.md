Build the ETL as discrete, testable stages: (1) for each user, compute the floor-average of their non-empty transaction amounts using `datamash -t, -g <user_col> mean <amount_col>` (or sum/count then floor-divide) restricted to rows with a non-empty amount field; (2) join that per-user average back onto the transactions so any row with an empty amount is imputed with the floored average (0 if the user has no valid transactions) — do this substitution with `awk`, checking `$field == ""`; (3) join the imputed transactions to `products.csv` on `product_id` (sort both files on the join key first, since `join` requires sorted input) to attach `category`; (4) aggregate total spend per (user_id, category) with `datamash -g user,category sum amount`; (5) for each user pick the category with max spend, breaking ties alphabetically by sorting on `(-spend, category)` before taking the first row per user group. Write the final `user_id,top_category` pairs sorted numerically by user_id to `/home/user/recommendations.csv`.
