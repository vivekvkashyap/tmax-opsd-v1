Write the Python script in three clearly separated phases and test as you go. Classification: open each file in `/home/user/project_assets/` in binary mode, read the first 1024 bytes, and check `b'\x00' in chunk` to decide binary vs text — don't rely on file extensions or MIME guessing. Linking: create `/home/user/organized/bin/` and `/home/user/organized/txt/`, then for binaries use `os.link(src, dest)` (hard link, so `os.stat` inode matches the original) and for text files use `os.symlink(os.path.abspath(src), dest)` (absolute path, verifiable with `os.path.islink`) — using the wrong link type for either category is the most common mistake here. Manifest: build the dict mapping each basename to its type and destination path, then write it atomically: serialize to a temp file in the same directory as the target (e.g. `tempfile.NamedTemporaryFile(dir='/home/user/organized', delete=False)`, or a fixed `.tmp` name) and call `os.rename(tmp_path, '/home/user/organized/manifest.json')` — rename within the same filesystem is what makes the write atomic, writing directly to the final path is not. Run the script once, then double-check with `stat -c %i` that hardlinked binaries share inodes with the originals and `ls -l`/`readlink` that text links are proper symlinks, and confirm the original files in `project_assets/` were never modified or deleted.
