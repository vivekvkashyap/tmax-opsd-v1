Before touching the backend, `tar --same-owner -czpf /app/backup/backend_backup.tar.gz -C /app backend` to preserve permissions/ownership. For the interactive init, write an expect script with `spawn /app/backend/init.sh`, `expect "PIN"` (match whatever prompt text init.sh actually shows), `send "<recovered-pin>\r"`, `expect eof`. Start the backend server in the background with `nohup python3 server.py --socket <recovered-socket-path> &`, then poll until the socket file appears before `chmod 666` on it — starting nginx or chmod'ing too early will race the socket's creation. Edit nginx.conf's upstream/proxy_pass line to reference the recovered socket path exactly, then launch with `nginx -p /app/nginx -c nginx.conf` (nginx must run as your current non-root user, so its conf's pid/log paths must be writable). Verify with `curl http://127.0.0.1:8080/status`.
