Build this pipeline in the order it will execute, testing each stage before moving on. First generate the self-signed cert/key with `openssl req -x509 -newkey rsa:2048 -keyout key.pem -out cert.pem -days 365 -nodes -subj "/CN=localhost"`, create an empty index.html, then launch `python3 -m http.server` wrapped with an `ssl.SSLContext` (load the cert/key, wrap the socket) in the background with `&` or `nohup`, and confirm it answers with `curl -sk https://127.0.0.1:8443/index.html`. Next set up the bare repo (`git init --bare`) and write the post-receive hook as a script that reads stdin for the ref, uses `git --work-tree=... --git-dir=... checkout -f` to populate the deploy directory, then `gcc checker.c -o checker -lcurl` and runs `./checker > /home/user/hook_output.log`; remember to `chmod +x` the hook — a non-executable hook silently does nothing. Write checker.c using easy libcurl handle setup (curl_easy_init, CURLOPT_URL, CURLOPT_SSL_VERIFYPEER=0, CURLOPT_SSL_VERIFYHOST=0), fetch the status code via curl_easy_getinfo(CURLINFO_RESPONSE_CODE), and print exactly `STATUS: <code>\n`. Clone the bare repo, add/commit/push to master, and check the log afterward to confirm the hook actually ran (a common failure is pushing before the web server is up, or forgetting the working directory for the checkout).
