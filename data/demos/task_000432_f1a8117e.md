Implement rational arithmetic using only Bash integer arithmetic `(( ))`/`$(( ))` — no `bc`/`awk`/`python`. Write a `gcd` helper using the Euclidean algorithm with absolute values (`(( a=(a<0)?-a:a ))` etc.) since Bash's `%` and integer division truncate toward zero and can misbehave with negative operands if not handled carefully. For `add_rat N1 D1 N2 D2`, compute numerator `N1*D2 + N2*D1` and denominator `D1*D2`; for `mul_rat`, compute `N1*N2` and `D1*D2`. After computing the raw N/D, normalize sign so D is always positive (if D is negative, negate both N and D), reduce by `g = gcd(|N|, |D|)` (treating N=0 as gcd=D itself, or just special-case zero to output `0/1` directly), and echo `"$N/$D"`. The two classic bugs here are: not flipping sign correctly when only one of N or D is negative (getting `3/-4` instead of `-3/4`), and computing GCD with signed inputs so it returns a negative divisor and corrupts the reduction. For the test script, use `$RANDOM` to generate 100 sets of N1,D1,N2,D2 (D nonzero, range -50..50), call `add_rat` both ways, compare strings exactly, exit 1 immediately on first mismatch, and only write `ALL_PASS` to test_result.log after all 100 pass.
