Structure pipeline.py in clear stages. First melt the wide CSV: for each of the three language suffixes (en/fr/es), extract vendor_id plus that language's name/desc pair into long-format rows, skipping any row where product_name is empty/NaN, and concatenate into one long DataFrame. Second, for near-duplicate detection, group rows by language, and for each language's list of (vendor_id, name) pairs generate all unique combinations with `itertools.combinations` filtered so vendor_id_1 < vendor_id_2 (string compare) — then dispatch the pairwise `difflib.SequenceMatcher(None, n1, n2).ratio()` computation across a `concurrent.futures.ProcessPoolExecutor` (chunk the pair list across workers) and keep only pairs scoring >= 0.85. Third, open a sqlite3 connection, create the two tables with the specified schemas, and bulk-insert both DataFrames using `executemany`. Fourth, tally similar_pairs counts per language directly from what you inserted (or re-query the DB) and dump that dict to summary.json with `json.dump`, so the JSON counts are guaranteed consistent with the DB contents. Run the script and spot check the DB tables and JSON against each other before finishing.
