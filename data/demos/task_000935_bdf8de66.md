Solve the tricky parts (encoding + embedded newlines) in a language with a real CSV parser rather than fighting `awk`/`grep` on this — a short Python script invoked from the bash entry point is the pragmatic choice here. First convert encoding with `iconv -f ISO-8859-1 -t UTF-8 raw_logs.csv > /tmp/utf8.csv` (or do the decode in Python with `open(..., encoding='latin-1')` then re-encode) — do this before parsing, since decoding must happen on the raw bytes, not after CSV splitting mangles them. Then use Python's `csv` module to read the UTF-8 file: `csv.DictReader` correctly handles quoted fields containing embedded newlines and commas as long as the file is opened with `newline=''`, so let the library do that work instead of manual line-splitting. Deduplicate by keeping only the first row seen per `log_id` (track a seen-set, skip subsequent duplicates). Clean the `message` field by replacing `\n`/`\r\n` inside the value with a single space. Load `users.json` with `json.load` into a dict keyed by `user_id` for an O(1) join.
