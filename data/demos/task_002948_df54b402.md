Build and test the pipeline script in isolation before wiring up the HTTP layer, so you can debug the math separately from networking. In `pipeline.sh`, extract frames with `ffmpeg -i /app/dashcam.mp4 -vf fps=1 frame_%04d.png` into a temp dir, then loop over frames in order computing grayscale mean brightness per frame (ImageMagick's `convert frame.png -colorspace gray -format "%[fx:mean*255]" info:` works well). For each frame, classify Evidence=1 if brightness >= threshold else 0, then apply Bayes' rule sequentially: unnormalized posterior for S=1 is `prior * P(evidence|S=1)`, for S=0 is `(1-prior) * P(evidence|S=0)`, normalize by their sum, and use the resulting P(S=1) as the new prior for the next frame. Log each step's index/brightness/evidence/posterior into `trace.csv` inside the timestamped experiment directory, and print only the final probability to stdout. For the server, use Python (Flask/FastAPI) purely as a thin wrapper that checks the `Authorization: Bearer mlops-track-99` header, shells out to `pipeline.sh <prior> <threshold>` via `subprocess`, captures stdout, and returns it as JSON — test both the 401 and success paths with `curl` before leaving it running.
