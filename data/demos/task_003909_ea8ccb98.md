The core challenge is a reader racing an active-log/rotated-log rename under `flock`, so mirror the writer's locking discipline instead of polling blindly. In `archiver.cpp`, each pass: take a shared `flock(fd, LOCK_SH)` on `active.log`, read new bytes since your last per-file offset, release quickly; also check `rotated.log` under its own shared lock for unarchived content (track processed offsets/sequence numbers per file to avoid double-counting or missing a record caught mid-rename). Since the writer holds an exclusive lock during rotation, shared-lock reads serialize naturally with it — poll in a tight loop, since rotation happens roughly every 100 writes. For decoding: parse each JSON line's `data` hex string, convert byte pairs into UTF-16LE code units, then reencode as UTF-8, appending `seq,utf8_text` to the CSV. Build with `g++ -O3 -std=c++17`, then write `run_test.sh` to clear the log dir, launch log_writer and archiver in the background, wait for log_writer to exit, then kill the archiver — test end-to-end and check the captured fraction of the 5000 records before considering it done.
