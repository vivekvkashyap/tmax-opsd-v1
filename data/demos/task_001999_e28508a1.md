This is a video-analysis + curve-fitting task feeding a Rust build. In `extract_math.py`, open the video with OpenCV, iterate frames, and on each frame isolate the dark dot: threshold the grayscale frame (the dot is black on white, so something like `cv2.threshold(..., THRESH_BINARY_INV)`), find contours or use `cv2.findNonZero`/moments on the thresholded mask to get the centroid (x, y) in pixel coordinates for that frame. Collect all (x, y) centroid pairs across frames where a dot was actually detected (skip frames with no detection rather than recording bogus (0,0) points, which would badly corrupt the fit). Fit a degree-2 polynomial with `numpy.polyfit(xs, ys, 2)` to get (A, B, C) directly — don't hand-derive the least-squares normal equations yourself. Double check axis orientation matches the task's stated convention (y measured top-left origin, increasing downward) before fitting. Write the three coefficients into `trajectory_constants.rs` in the exact `pub const COEFF_X: f64 = ...;` format shown, then have `ci_pipeline.sh` install Python deps, run the extraction script, and run `cargo build && cargo test` in sequence, failing loudly if any step errors.
