For each CSV: read the 1000 amplitude samples, build a complex FFT input buffer, run a real-to-complex or complex FFT via `rustfft`, compute magnitude = sqrt(re^2+im^2) per bin, and map bin index to frequency using `bin_index * sample_rate / n_samples`. Only scan bins whose frequency falls in [0, 500] Hz inclusive (remember bin 0 is 0 Hz, and with 1000 samples at 1000 Hz each bin is 1 Hz wide, so bins 0..=500 cover that range) — pick the bin with strictly greatest magnitude, breaking ties consistently (e.g. keep first-seen maximum) since float equality is fragile. Collect the 100 peak frequencies for a directory, compute the mean, then bootstrap: create one `ChaCha8Rng::seed_from_u64(42)`, and reuse that exact same RNG instance across 10,000 resamples of the reference set first, then continue drawing from the same RNG for 10,000 resamples of the simulation set — do not reseed between the two. Each resample draws N=100 indices with replacement, averages, and you sort the 10,000 bootstrap means to take the 2.5th/97.5th percentiles. Serialize both results into the exact nested JSON structure at `/home/user/report.json`, then `cargo build --release && ./target/release/<bin>` to actually produce the file.
