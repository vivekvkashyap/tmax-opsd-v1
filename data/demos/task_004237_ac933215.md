You do not need to reverse-engineer `/app/meta_bridge` at all — the service must invoke the real binary live for every request, not reimplement its algorithm. Build a TCP server (e.g. Python `socketserver.ThreadingTCPServer` or a plain accept-loop) bound to `127.0.0.1:8081` that stays running persistently and handles requests sequentially. On each connection, read the line, parse the two table names with a regex like `SELECT \* FROM (\S+) JOIN (\S+) ON` (match up to whitespace before `JOIN`/`ON`, don't assume fixed-width names), then for each table run `subprocess.run(['/app/meta_bridge', table], capture_output=True, text=True)` and take its stdout, stripped of trailing newline, as the graph node id. Format the reply as `PIPELINE: <node1> -> <node2>\n` and send it back before closing the connection. Log each incoming request line (and ideally the parsed tables/response) to `/home/user/wrapper.log`. Launch the service detached (e.g. `nohup python3 service_wrapper.py &` or a systemd/background process) so it keeps running after your shell session ends, and test it yourself with a raw `nc`/socket client before considering it done.
