First, telemetry recovery: use `ffmpeg -i /app/cloud_telemetry.mp4 frame_%04d.png` (or OpenCV's `VideoCapture`) to dump frames, then decode each with `pyzbar.decode` (install via pip if missing) on a PIL/OpenCV image. Parse each decoded string as JSON, key results by the `service` field in a dict to dedupe repeated frames of the same QR code, and only then sum `base_cost - idle_waste` per unique service to get `total_optimized_cost` — a naive per-frame sum without deduping will overcount since each service's code repeats across many consecutive frames. Second, the dashboard: build a minimal Flask/stdlib HTTP server bound to 127.0.0.1:8080 with `/health`, `/costs` (serving the float computed above), and `/crash` calling `os._exit(1)` so the process dies hard with no graceful shutdown.
