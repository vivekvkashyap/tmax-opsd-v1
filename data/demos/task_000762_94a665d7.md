For each .tar.gz in /home/user/archives/, first test integrity without fully committing to extraction: `gzip -t file.tar.gz` and/or `tar -tzf file.tar.gz > /dev/null 2>&1`, checking the exit code — a nonzero exit means corrupted/truncated, so skip that file entirely (do not attempt to extract or list it further). For files that pass, extract into a per-archive subdirectory under /home/user/extracted/ (e.g. `tar -xzf file.tar.gz -C /home/user/extracted/<basename>/`) so readme.md and app.bin from different archives don't collide. For each surviving archive, grab the first line of its readme.md with `head -n 1 readme.md`, and get the entry point with `readelf -h app.bin | grep "Entry point address:" | awk '{print $4}'` (the address text as readelf prints it, e.g. `0x401050`). Build each summary line with the exact format `<archive_filename> | <first_line> | <entry_point>` (single space around each pipe), collect all lines, and pipe them through `sort` (plain alphabetical on the whole line, which sorts by filename since it's first) before writing to /home/user/docs_summary.txt.
