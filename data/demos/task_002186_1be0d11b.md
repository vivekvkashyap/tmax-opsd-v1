Install `libhdf5-dev` and `libgsl-dev` first. In C, use the HDF5 C API (`H5Fopen`, `H5Dopen2`, `H5Dread`) to load the `/x` and `/y` datasets (50 doubles each) from `/home/user/data.h5`. Build the Vandermonde-style design matrix as a `gsl_matrix` of shape 50x4 with columns `[1, x, x^2, x^3]`, and the target as a `gsl_vector` of length 50. Instead of forming the normal equations (`X^T X`), call `gsl_multifit_linear_alloc`/`gsl_multifit_linear` (or the SVD-based workspace functions), which internally uses SVD and avoids the catastrophic conditioning problem of `X^T X` for this near-collinear cubic basis. Extract the 4 resulting coefficients from the solution vector and print them to `/home/user/coeffs.txt`, one per line with `%.6f`. Compile linking `-lgsl -lgslcblas -lm -lhdf5`, run the binary, then write a short Python (matplotlib) or gnuplot script that reads the same `/x`,`/y` data plus the fitted coefficients to plot the scatter and the fitted cubic curve, saving to `/home/user/fit.png`. Sanity-check the fit visually before finishing — a badly wired design matrix (wrong column order/powers) is the most common failure.
