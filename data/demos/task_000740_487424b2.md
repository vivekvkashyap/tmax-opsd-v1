Structure the script around three stages: parse config, compare versions, build the mock if needed. Use `jq -r '.lib_name'`, `.min_version`, `.symbol` on `/home/user/deps.json` to extract values dynamically — never hardcode the symbol/lib names, since the checker likely varies these.

For semantic version comparison, don't do naive string comparison (`"2.10.0" < "2.9.0"` as strings is wrong). Instead split on `.` into major/minor/patch integers and compare numerically, or use `sort -V` (version sort) to compare the installed and minimum version strings robustly — e.g. `[ "$(printf '%s\n%s' "$MIN" "$INSTALLED" | sort -V | head -1)" = "$MIN" ]` tells you if installed >= min.

If installed >= min, log "SYSTEM_UP_TO_DATE" and exit early — don't build anything in that branch. Otherwise, generate a minimal GNU AS `.s` file that declares `.global <symbol>` and a matching label that zeroes the return register (`xorq %rax,%rax`) and returns, assemble/link it into a `.so` (e.g. via `gcc -shared -fPIC`) named after `lib_name` into `/home/user/build/`, then log "MOCK_BUILT". Verify with `nm -D` that the symbol is exported as a defined text symbol before considering it done. Remember to `chmod +x` the script.
