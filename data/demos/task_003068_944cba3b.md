Build this as three composable scripts, each independently testable, rather than one monolithic script. For `init_db.sh`: create the schema with `sqlite3 network.db "CREATE TABLE edges (source TEXT, target TEXT, latency INTEGER);"` then import the CSV with `sqlite3 network.db ".mode csv" ".import '| tail -n +2 /home/user/network_edges.csv' edges"` (or equivalent skip-header approach) — a naive `.import` without skipping the header row will insert the literal header as a bogus data row and break every downstream query. For `find_shortest_path.sh`: write a recursive CTE that starts from `start_node` with cumulative latency 0 and an accumulating path string, recursively joins `edges` to extend the path while avoiding revisiting nodes (to prevent infinite loops on cycles), then selects the minimum-total-latency row where the path reaches `end_node`; format the final row as `TOTAL_LATENCY|PATH` using `sqlite3 -separator '|'` or string concatenation in SQL. For `pipeline.sh`: source/call `init_db.sh`, then loop over `routes_to_calc.csv` with `while IFS=, read -r start end; do ...; done < file`, call `find_shortest_path.sh "$start" "$end"`, and append `START,END,LATENCY,PATH` to the log — remember `chmod +x` all three scripts and actually execute `pipeline.sh` at the end so the log file is produced.
