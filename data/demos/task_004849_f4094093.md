In `extractor.c`, read the TSV line-by-line (`fgets`/`getline`), split on tabs to get timestamp, sensor ID, and hex payload. Decode hex to ASCII by consuming two hex chars at a time (`sscanf("%2hhx", ...)` or manual nibble parsing) into a buffer sized at half the hex string length plus one for the null terminator — get the buffer sizing right or you'll overflow/truncate. For word counting, scan the decoded string with a simple state machine: track whether you're "inside" a run of `isalpha()` characters, incrementing the word count only on the transition from non-alpha to alpha (this correctly handles adjacent digits/punctuation as delimiters without needing `strtok`, which would mis-split on this custom alphabetic-only rule). Emit `Timestamp,SensorID,WordCount\n` with no extra spaces. In the Makefile, make `all` compile `extractor.c` (e.g. with `gcc -o extractor extractor.c`), and make `run` depend on `all`, `mkdir -p /home/user/output`, and pipe/redirect `/home/user/data/raw_logs.tsv` through the compiled binary into `/home/user/output/features.csv`. Test by running `make run` and diffing the output.
