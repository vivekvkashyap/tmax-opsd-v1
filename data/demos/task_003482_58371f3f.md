Build this bottom-up and test each layer in isolation before wiring them together. First generate the CA with `openssl genrsa`/`req -x509`, then a server cert for CN=localhost and a client cert, both signed by that CA; put keys under `/home/user/certs/` and `chmod 400` the server key immediately (a common mistake is leaving it group/world readable). Next write `auth_filter.cgi` as a standalone script you can test by exporting `HTTP_AUTHORIZATION` manually and running it directly — it must emit a proper CGI response (`Status:` header line, blank line, body) since Nginx's `auth_request` only cares about the status code. To detect the vulnerable token, split the JWT on `.`, base64url-decode the first segment (handle missing padding), and check for `"alg"` set to `none` case-insensitively — do this decode yourself rather than assuming a fixed byte pattern. Every request with a JWT must append one audit line before returning, using the exact log format given. For Nginx, configure `listen 8443 ssl`, `ssl_client_certificate`/`ssl_verify_client on`, an internal `/auth` location with `fastcgi_pass unix:/home/user/fcgiwrap.sock`, and `auth_request /auth` on the proxied location pointing to `127.0.0.1:8080`. Finally have `start_gateway.sh` launch `fcgiwrap` on that socket, fix permissions, then start nginx — verify each service is actually listening before declaring success.
