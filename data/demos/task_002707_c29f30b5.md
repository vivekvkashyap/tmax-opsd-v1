Use `find` with `-name "*.log"` (not `.txt`/`.dat`) to recurse the source tree, and `grep -h` across all matches to extract lines containing the literal string `[CRITICAL]` while stripping filenames from the output. Concatenate all matches, then pipe through `sort` for a fully deterministic alphabetical order — do this before splitting so chunk boundaries are stable regardless of scan order. Use the `split` utility with `-l 10` and a `chunk_` prefix to produce fixed-size files in the destination directory (create it first with `mkdir -p`); `split`'s default suffix naming (aa, ab, ac...) already matches the required pattern. For the manifest, loop over the produced chunk files, compute `sha256sum` for each, and format each line as `filename,hash` — watch the comma placement and that you're emitting the bare hex hash, not the `sha256sum` tool's default `hash  filename` order. Finally sort the manifest lines by filename before writing, since chunk creation order should already match but don't assume it — sort explicitly to be safe.
