Set up your Rust project (`cargo init`), then structure the solution as four clear passes over the parsed records rather than trying to do everything in one pass. First write a line parser (regex or manual splitting on `] Server: `, `| Key: `, `| Value: `) that extracts the bracketed timestamp field, IP, key, and value from each log line. For timestamps, detect the two formats: if the bracketed content parses as a plain integer, treat it directly as Unix epoch seconds; otherwise parse it as ISO8601 (a `chrono` crate DateTime parse with `%Y-%m-%dT%H:%M:%SZ`) and convert to epoch via `.timestamp()` — do not assume one format applies to all lines within a server's records. Group parsed records by server IP into a map, and for masking, lowercase the key and check `contains("pass") || contains("secret") || contains("token")`. After grouping, sort each server's records by epoch ascending, then compute consecutive-interval population variance (sum of squared deviations from the mean, divided by N, not N-1) — remember servers with fewer than 3 total changes (i.e. <2 intervals) must report variance 0.0. Round the variance to exactly 2 decimals and serialize the map (e.g. with `serde_json`) to the required output path, then build and run to produce the file.
