This is a classic "recover an unlinked-but-open file via /proc" exercise: find the running `transaction_generator.py` process with `ps aux | grep transaction_generator`, get its PID, then look in `/proc/<PID>/fd/` for the file descriptor pointing at the deleted `transactions.log` (it will show as a broken symlink like `/home/user/data/transactions.log (deleted)`). Copy its live contents out with `cat /proc/<PID>/fd/<N> > /home/user/recovered_logs.txt` — since the process may still be writing/holding it open, a straight `cat`/`cp` of the fd captures what's been flushed to it. For the aggregator, wrap the `json.loads` call per line in a try/except that catches `json.JSONDecodeError` (and consider stripping/ignoring null bytes) and just `continue`s on failure rather than crashing. For precision, replace the float accumulator with `decimal.Decimal`, parsing each amount as a string into `Decimal` (not via float first) so no binary floating-point rounding creeps in, then sum exactly. Run the fixed script against the recovered log and write the final Decimal sum, formatted as a plain decimal number, to the output file.
