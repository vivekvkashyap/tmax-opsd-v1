Treat the log lines as semi-structured key:value pairs separated by `|` and parse them robustly rather than with fragile fixed-offset substring math. For each line, split on `|`, trim whitespace, then split each field on the first `:` to get a key/value pair; only act on fields whose key (after trimming) is exactly `LR` or `ValLoss`, parsing the value with `std::stod`, and ignore `Exp_ID`, `BS`, `Acc` and any `[INFO]` prefix noise. Accumulate parallel `std::vector<double>` for LR (x) and ValLoss (y) in file order, pairing them per line. Compute Pearson's r manually with the standard formula: means of x and y, then `sum((x_i-mean_x)*(y_i-mean_y))` over `sqrt(sum((x_i-mean_x)^2) * sum((y_i-mean_y)^2))` — use `double` throughout and `<cmath>`'s `sqrt`, no external stats library. When writing the result, use `std::ofstream` with `std::fixed << std::setprecision(4)` to get exactly 4 decimal places (watch that negative correlations format correctly, e.g. `-0.9594` not `-0.9594000000001` or losing the sign), followed by a newline, then compile (`g++ -std=c++17 -O2 analyze.cpp -o analyze`) and run before checking the output file.
