Scaffold with `cargo new --bin safe_archiver` and add tar, flate2, reqwest (blocking or async+tokio), redis, and walkdir to Cargo.toml. The core correctness challenge is symlink validation before any archiving/network activity: recursively walk the input directory without following symlinks by default (walkdir with follow_links(false)), and for every symlink entry read its target, canonicalize it relative to the symlink's containing dir, and check two things — (a) does resolving it (including transitively through other symlinks) ever revisit a path already seen in the current resolution chain, which indicates a cycle (use a visited-set / bounded resolution depth rather than blindly following, since a naive resolver can hang or stack-overflow on A->B->A loops); (b) does the fully resolved absolute path fall outside the canonicalized input_directory_path (use `Path::starts_with` on canonical paths, not string prefix on raw paths, since ../ and symlink chains can bypass a naive string check). If either check trips, print to stderr and exit(1) with zero network/Redis activity. Otherwise build the tar.gz preserving internal symlinks as symlinks (tar crate's `append_link`/appropriate builder method, not append_file which would dereference), POST it via multipart to /upload, then INCR backup:success in Redis, and exit(0). Build with `cargo build --release` and test against both a clean and a hand-crafted malicious directory before finishing.
