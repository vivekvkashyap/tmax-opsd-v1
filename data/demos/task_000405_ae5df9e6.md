Build and test the two pieces separately before combining. In the C++ checker, parse the config file line by line, and for each `hostname:port` open a standard `socket()`/`connect()` (with `inet_pton` or `getaddrinfo` for the hostname, since it may not always be a literal IP) — set a short connect timeout so a non-listening port fails fast rather than hanging; on the very first failure, print `DOWN_REASON: <hostname>:<port>` and exit 1 immediately (don't keep checking the rest), otherwise after all succeed print `ALL_UP` and exit 0. Close every socket you open to avoid fd leaks across the monitoring loop's repeated invocations. In the deploy script, compile first, launch `socat TCP-LISTEN:8080,fork,reuseaddr TCP:127.0.0.1:8081` in the background with `&`, then start a `while true; do ...; sleep 2; done &` loop that runs the compiled checker, checks `$?`, and appends the correct `CLOUD_HEALTH: PASS`/`FAIL` line — disown or nohup both background jobs so the script itself returns rather than blocking. Test the checker binary alone against a known open/closed port before wiring the full loop.
