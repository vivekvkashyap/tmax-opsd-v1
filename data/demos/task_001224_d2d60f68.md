Split this into three testable pieces: feature extraction, a reproducibility wrapper, and a TCP server. For `extract_features.c`, read `/app/fixture.wav` — first parse the standard RIFF/WAVE header to locate the `data` chunk (don't assume a fixed offset; different files may have extra chunks like `LIST` before `data`), then read the PCM samples as signed 16-bit little-endian integers and compute: mean absolute amplitude, max absolute amplitude, zero-crossing rate (count sign changes between consecutive samples, divided by sample count as stated in the spec), and RMS (sqrt of mean of squared samples) — normalize/scale amplitudes consistently with however the verifier's Python reference likely computes them (e.g. raw int16 values vs normalized to [-1,1]; test both if unsure by reasoning about typical conventions) since a scale mismatch will silently fail. For `run_pipeline.sh`, compile both C programs, run the extractor against the fixture, save the 4 numbers to `experiment_results.txt`, and log the SHA-256 of the compiled binary. For `embedding_server.c`, parse the 4 floats from that results file at startup, then run a standard accept-loop TCP server on `127.0.0.1:8080` that reads a comma-separated 4-float query per connection, computes Euclidean distance to the stored embedding, replies `Distance: X.XXXX\n`, closes that connection, and loops to accept the next one. Test manually with `printf`/`nc` before backgrounding the server and leaving it running.
