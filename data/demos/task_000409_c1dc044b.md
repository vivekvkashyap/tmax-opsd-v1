Build this as a stateless L7 proxy plus a firewall rule, and keep the two concerns separate. First lock down the network layer: use `iptables -A INPUT -p tcp --dport 8080 ! -i lo -j DROP` (or an equivalent rule scoped to non-loopback interfaces) so port 8080 is unreachable except via 127.0.0.1, and verify with a quick external-looking connection attempt that it times out. Then write the C++ proxy as a simple blocking (or thread-per-connection) TCP server on 0.0.0.0:8000: accept a connection, read the raw HTTP request until you have the full header block (up to `\r\n\r\n`, then read `Content-Length` bytes of body if present), and parse out the `Cookie` and `Authorization` header values with straightforward string search — no need for a full HTTP parser. Apply the checks in order: missing `session_token=` in Cookie -> 403 + log `BLOCKED_NO_COOKIE`; dangerous character in Authorization -> 403 + log `BLOCKED_MALICIOUS`; otherwise open a fresh socket to 127.0.0.1:8080, forward the exact original request bytes, read the full backend response, relay it to the client, and log `ALLOWED_PROXIED`. Append one log line per request with `date +%s`-style epoch timestamps, and make sure the log write and forwarding happen for every request even under concurrent load — buffer partial reads/writes since sockets can deliver data in chunks.
