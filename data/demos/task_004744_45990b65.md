Design the C program around line-level integrity checking rather than naive CSV field splitting. Read the file with `fgets`/`getline` line by line (or read the whole file and split on `\n`), and for each line count commas with a simple loop; only proceed if the count is exactly 3, otherwise skip — this is what correctly discards rows broken by embedded newlines (a row split by a stray newline becomes two lines, each with a comma count other than 3). For a valid line, split on commas into timestamp/email/event_type/amount, derive the bucket by slicing the first 13 characters of the ISO8601 timestamp (`YYYY-MM-DDTHH` → replace the `T` with `-`). For anonymization, find the `@` in the email, keep the character before position 0 of the username plus `***`, then append everything from `@` onward — do not just truncate the domain. Aggregate sums in an in-memory structure keyed by `bucket,email` (a simple linked list or sorted array is fine given small input size), then sort the aggregated rows lexicographically by bucket then email before writing CSV. Compile with `gcc -o /home/user/bin/process_events`, write the exact cron line `15 * * * * /home/user/bin/process_events` to `/home/user/pipeline.cron`, and run the binary once to produce the output file.
