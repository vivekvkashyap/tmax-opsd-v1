Build the sanitizer as a pure recursive function first, independent of any networking: walk the parsed JSON, and for dicts, drop keys starting with `_` then rebuild the dict with `sorted(keys)` (use `json.dumps(obj, sort_keys=True, separators=(",", ":"))` for the compact/sorted serialization instead of hand-rolling ordering); for lists, recurse into each element but never reorder them; leave scalars untouched. Write pytest cases that isolate each rule (top-level underscore removal, nested underscore removal, list order preservation, key sorting) rather than one giant test. For the proxy, don't just pipe bytes through — read the request body via `Content-Length`, run it through `sanitize_payload`, then when forwarding recompute `Content-Length` from the *new* sanitized body length (a common bug is forwarding the original header, which desyncs the upstream read) and set `Content-Type: application/json` explicitly. Relay the upstream's status code and body verbatim back to the client. Test the whole chain locally with `curl -X POST` against the proxy before declaring done, and only write SUCCESS once `pytest` passes and a manual curl round-trip through to the upstream on port 9000 returns the sanitized body.
