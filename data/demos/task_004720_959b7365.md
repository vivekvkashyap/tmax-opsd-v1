Break this into extraction, ETL, and serving stages. First, run `tesseract /app/pipeline_config.png` (or convert/preprocess the image first if OCR is noisy — upscale, threshold) to recover the `PIPELINE_ID` and `INTERVAL` values from the image; store them in variables/config rather than assuming any value, since these are exactly what you must discover from the image, not invent. For the ETL: load all CSVs from `/app/data/` with pandas, concatenate, sort by `timestamp`; for imputation, group by `server_id`, set the timestamp as a sorted index, and use `.interpolate(method='time')` (or linear on the sorted sequence) for missing `cpu`/`mem`, then `.ffill().bfill()` to cover boundary NaNs that interpolation can't fill. For aggregation, group by `server_id` and `pd.Grouper(freq=INTERVAL, key='timestamp')` (using the interval string you OCR'd) and take the mean of `cpu`/`mem` per bucket, rounding to 2 decimals. For serving, stand up a Flask/FastAPI app on 127.0.0.1:8080 with `GET /api/metrics?server_id=...`, checking the `Authorization: Bearer <PIPELINE_ID>` header against the OCR'd token and returning 401 on mismatch, else a chronologically sorted JSON array of `{timestamp, cpu, mem}` objects. Run the server as a persistent background/foreground process so it stays up for the verifier to query.
