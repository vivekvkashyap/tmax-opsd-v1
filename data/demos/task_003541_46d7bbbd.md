For chunking, open the .dat file in binary mode, read/write in a loop using a fixed 100000-byte buffer, tracking a part index starting at 0 — the last read will naturally be short, so just write however many bytes `read()` actually returned rather than assuming a fixed size, and write each chunk to `/home/user/dataset/chunks/<name>_partN.bin`. For checksums, shell out per chunk with `popen("sha256sum <path>", "r")` (or fork/exec) and parse the first whitespace-delimited token from the output, then append `<hash>  <chunk>\n` (two spaces, matching sha256sum's own format) to manifest.txt — do this immediately after each chunk write so ordering matches. For the hard link, use `::link(dat_path, archive_path)` (not copy) so inodes match; verify test-side expectations by checking `stat -c %i` on both files after linking. Watch out for race conditions: only start processing a .dat once its matching .ready file exists, since the .dat may still be growing before that.
