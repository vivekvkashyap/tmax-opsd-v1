Use `git bisect` properly rather than manually diffing 200 commits. First write a tiny wrapper script (e.g. `test_commit.sh`) that: builds the current checkout (cmake/make into `build/engine`), runs it on `test_queries.csv` producing an output file, runs the oracle `/app/oracle_engine` on the same input, and compares the two outputs numerically (e.g. with a small Python/awk diff allowing tiny floating tolerance) — exit 0 if they match, 1 if not. Run `git bisect start`, mark `HEAD` bad and `HEAD~200` good, then `git bisect run ./test_commit.sh` to let git automate the search instead of testing manually. Once the culprit commit is found, inspect its diff (`git show <bad-commit>`) to see exactly what changed in `src/parser.cpp` or `src/integrator.cpp` — look specifically for off-by-one errors in loop bounds, string-slicing/substring indices, or integration step counts. Fix the bug on `main` at `HEAD` (not by reverting, since later commits may depend on other changes), rebuild into the exact required path, and regenerate the final output file at the exact required path.
