Treat this as a graph reachability problem over three merged data sources, not three separate lookups. First build an in-memory adjacency structure: from `employees.csv` map employee -> role_id, from `roles.json` map role_id -> set of direct-access systems (an employee's starting nodes), and from `network_graph.txt` build a directed adjacency list of system-to-system edges. Then for each employee, run a BFS (not DFS) from each of their directly-accessed systems simultaneously, treating each directly-accessed system as distance 1, and its outgoing edges as distance 2, 3, etc. — the key subtlety is that "direct access" itself already counts as path length 1, so BFS depth must be offset by one, and if an employee has multiple directly-accessed systems you take the minimum distance to SYS_OMEGA across all of them. Skip employees whose BFS never reaches SYS_OMEGA. Write a small Python script using `csv`, `json`, and `collections.deque` for the BFS rather than trying to do graph traversal in pure shell/awk — it's much easier to get the distance semantics right. Finally sort results by shortest_path ascending, then employee_name alphabetically, and dump to the exact two-key JSON schema at `/home/user/audit_report.json`.
