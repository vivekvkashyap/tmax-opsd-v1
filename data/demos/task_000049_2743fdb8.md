Write the C parser as a simple state machine over lines read with `fgets`/`fscanf`: on seeing `BEGIN_JOB`, reset per-record buffers for `status` and `path`; while inside a block, check each line's prefix with `strncmp` for `Status:` or `Path:` (ignore `JobID:`/`Priority:` — they can appear in any order, so don't assume a fixed line order) and store the matched value, trimming the trailing newline; on `END_JOB`, if the stored status string exactly equals `REQUIRED`, write the stored path to the output file. Use fixed-size buffers large enough for a path line, or dynamically allocate, and be careful trimming both `\n` and any `\r`. Compile with `gcc -o queue_parser queue_parser.c` and run it against the log to produce `/home/user/targets.txt`. Finally, build the tar archive directly from that file list rather than typing directory names by hand, e.g. `tar -czf /home/user/required_backups.tar.gz -T /home/user/targets.txt` (or a loop appending each line to a `tar -czf` invocation), so only REQUIRED directories end up archived — verify with `tar -tzf` that no IGNORED/COMPLETED directory appears.
