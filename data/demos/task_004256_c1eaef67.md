Treat this as union-find over a similarity graph, not pairwise nested loops with manual bookkeeping. First normalize every row: email = `strip().lower()`; for name/address, apply Unicode NFC normalization via `unicodedata.normalize('NFC', s)` (this is what collapses composed vs. decomposed accented characters like é written as one codepoint vs. e+combining-accent); address additionally needs `re.sub(r'[^A-Za-z0-9]', '', s).lower()` after NFC to strip punctuation/spacing before comparison. Build a Levenshtein distance function (or use `python-Levenshtein`/`rapidfuzz` via pip) for the name check. For every pair of rows (i, j), mark them connected if normalized emails are identical OR (name Levenshtein distance <= 2 AND normalized addresses are identical) — an O(n^2) comparison is fine for reasonable row counts. Feed each connected pair into a union-find (disjoint set) structure so transitive chains merge correctly even when no single condition links all members directly. After resolving all pairs, group row indices by their root, keep only groups with size > 1, and key the output dict by the string of the minimum index in each group with a sorted integer list as the value. Verify the JSON keys are strings and values are sorted lists before writing.
