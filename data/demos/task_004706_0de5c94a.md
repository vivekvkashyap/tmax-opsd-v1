Approach this as config-driven, diff-based backup logic rather than hardcoding directories. Parse `/home/user/repo_config.ini`'s `[Backup]` section (Python `configparser`) to get `include_dirs` and `exclude_patterns` dynamically, then walk only those directories, using `fnmatch`/`glob` pattern matching against each filename (not full path) to apply `exclude_patterns` before hashing. For every surviving file, compute SHA-256 over its raw bytes and record it keyed by the path relative to `/home/user/repo/` (use `os.path.relpath`), writing this as `manifest_v2.json`. Load `manifest_v1.json` and diff: a relative path present only in v2 is NEW, present in both but with a different hash is MODIFIED, and anything in v1 but excluded/gone should be ignored (not deleted-tracking, since spec doesn't ask for that). Sort the NEW+MODIFIED paths alphabetically for `backup_summary.txt`, writing one `NEW: <path>` or `MODIFIED: <path>` line each. Build `incremental_v2.tar.gz` containing exactly those changed files with `tarfile`, adding each with `arcname=<relative_path>` so the archive doesn't embed the absolute `/home/user/repo/` prefix — this is the step naive attempts usually get wrong.
