Build the Rust extractor first, test it in isolation before touching the real archive. Use the `tar` crate's `Archive::entries()` (not the high-level `unpack()`, which is itself vulnerable) so you inspect each entry's path yourself: for every entry, read `entry.path()`, and reject it if it is absolute OR if, after joining it onto the target directory and lexically normalizing (or canonicalizing the *parent* dirs that already exist), the resulting path is not still prefixed by the canonical target directory. A `..` component check alone isn't sufficient — verify by path-joining and prefix-comparison, not string matching. Skipped entries get their exact archive path (as stored in the tar header) appended to the log, one per line; safe ones get extracted and their name printed to stdout. Create parent directories for legitimate nested paths before writing files. After the tool works, run it against `/app/backup.tar.gz` extracting into `/home/user/extracted/`, then locate `voicemail.wav` there. For transcription, install a lightweight ASR tool (e.g. `pip install openai-whisper` plus ffmpeg, or `whisper-cpp`) and run it on the wav, writing the raw output text (trim extraneous timestamps/formatting) to `/home/user/transcript.txt`. Verify the log contains the traversal path before considering the extraction step done.
