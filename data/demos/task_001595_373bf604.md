Build and test each phase script in isolation with `echo`/`printf` before wiring them into `run_experiment.sh`, since a bug in any one script silently corrupts everything downstream. `tokenize.sh`: pipe stdin through `tr '[:upper:]' '[:lower:]'`, then strip punctuation with `sed 's/[^a-z0-9 \n]//g'` (careful to keep spaces/newlines), then collapse repeated spaces and trim with `sed -E 's/ +/ /g; s/^ //; s/ $//'`. `bootstrap.sh`: count lines of the input file with `wc -l`, then `shuf -r -n <count> <file>` to sample with replacement to the same line count. `inference.sh`: for each input line, tokenize into words (e.g. via awk), look up each word against a fixed weight table (critical=-5.0, error=-3.0, warning=-1.0, info=0.5, success=2.0, resolved=3.0, else 0.0), sum per-line, then average across all lines and format to exactly 4 decimals using `bc -l` with `printf "%.4f"` (bc alone won't pad zeros, so post-format). `run_experiment.sh`: create `/home/user/raw_data.txt` with varied mock logs first, then chain tokenize -> inference (baseline) -> 5x bootstrap+inference, appending each bootstrap result to `bootstrap_scores.txt`. Make every script executable with `chmod +x`.
