Use `pandas` for a clean, reliable pipeline rather than hand-rolling bucket math. Steps: (1) read the log with `re.findall`/`re.match` using a pattern like `r'\[(\d{4}-\d{2}-\d{2} \d{2}:\d{2}:\d{2})\]'` to pull every timestamp, parsing each with `datetime.strptime`. (2) Floor each timestamp to the hour (`dt.replace(minute=0, second=0)`), then build a `pandas.Series`/`groupby` count per hour. (3) Critically, reindex the counts against a complete hourly range built with `pd.date_range(start=floor(min_ts), end=floor(max_ts), freq='H')` and fill missing hours with 0 — this is the step naive scripts skip, silently dropping empty hours instead of emitting zero-count rows. (4) Compute the rolling 3-hour sum with `.rolling(window=3, min_periods=1).sum()` on the *filled* series (min_periods=1 matters for the first two hours, which only have 1-2 prior buckets). (5) Format `hour_bucket` as `YYYY-MM-DD HH:00:00` and write out exactly the headers `hour_bucket,change_count,rolling_3h_sum` to `/home/user/hourly_rolling_stats.csv` with `index=False`.
