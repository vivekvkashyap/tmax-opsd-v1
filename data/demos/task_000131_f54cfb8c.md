Build the audit script as four independent checks, each writing lines directly to the report file (append mode) so partial failures in one check don't block others. For JWTs: for each `.jwt` file, split on `.`, take the first field, translate `-_` back to `+/`, pad with `=` to a multiple of 4 (base64 -d is strict about padding), decode, then grep/jq for `"alg"` followed by `"none"` — handle both compact and spaced JSON (`"alg": "none"` vs `"alg":"none"`), e.g. with a regex like `alg"[[:space:]]*:[[:space:]]*"none"`. For log injection: use `grep -n` for the literal strings `<script>` and `UNION SELECT` on the log file, and use grep's own line-number output rather than re-counting lines yourself. For the cert check: run `openssl verify -CAfile /home/user/certs/ca.pem /home/user/certs/server.pem` and check its exit status/output for "OK" vs failure. For permissions: use `stat -c %a` (or equivalent) on each file in the keys dir and flag anything whose mode isn't exactly 600. Always strip directory prefixes with `basename` before writing filenames to the report. Make the script executable and actually run it before finishing.
