Treat this as a time-series alignment problem, not a line-by-line diff of the two logs. Parse each log line (regex on `\[(\d+)\] host=\S+ reqs=(\d+)`) into a `{timestamp: count}` map per server. Find the global min and max timestamp across both maps combined — don't use each server's own min/max separately, since the window must be identical for both. Then iterate every integer second from global_min to global_max inclusive, looking up each server's count via the map with a default of 0 when the second is absent (this is the gap-fill step — a naive script that only sums over timestamps actually present in a log will silently ignore seconds where one server logged and the other didn't, producing a wrong total). Sum `abs(alpha[t] - beta[t])` over the whole window and write it as `Total Distance: <integer>` to `/home/user/distance_result.txt`.
