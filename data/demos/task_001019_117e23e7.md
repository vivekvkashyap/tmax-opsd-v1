Once you have concrete `N` and `M` values, build `sanitizer.py` with two independent checks per log file: (1) unicode validity — scan the *raw* file text for `\u` escapes and verify each is followed by exactly 4 hex digits using a regex like `\\u(?![0-9a-fA-F]{4})`, checked before/instead of relying on `json.loads` (which may raise or may silently accept things differently depending on Python version) — flag as evil on any match; (2) rate limiting — group lines by `user_id`, sort each user's timestamps, and using a sliding window check whether any window of `M` seconds contains more than `N` events (a two-pointer scan over sorted timestamps is simplest and avoids O(n^2) blowup). Exit 1 if either check fails on any line, else exit 0.
