Build the pipeline as discrete, testable stages inside one script. First, extract candidates with grep/awk: match only lines that both start with/contain `INFO` and have a `Pos: <lat>, <lon>` field — explicitly exclude ERROR and WARNING lines even if they carry position data, since the spec restricts validity to INFO lines only (this is the main trap: a naive `grep Pos:` would wrongly include WARNING/ERROR lines). Pull out Drone ID and the two coordinates with awk/sed/grep -oP per matching line. Second, compute Euclidean distance to the fixed depot (45.5000, -122.6000) per drone using `awk` or `bc -l` for floating point (`sqrt((lat-45.5)^2+(lon-(-122.6))^2)`), rounding to 4 decimal places at the end, not at intermediate steps. Third, sort/select the minimum-distance drone (`sort -t, -k2 -n | head -1` style, or track min in awk). Finally emit JSON with `jq -n` or a printf template matching the exact two-key schema, write to `/home/user/closest_drone.json`, chmod +x the script, and run it once to confirm the file is produced correctly before finishing.
