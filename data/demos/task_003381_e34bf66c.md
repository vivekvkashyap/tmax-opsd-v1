Recover the config from the image before building anything else, since every later step depends on those three values: run `tesseract /app/monitor_spec.png stdout` (or similar OCR) and parse out the `PORT=`, `BACKUP_DIR=`, and `TOKEN=` lines exactly as printed — treat OCR output as noisy and sanity-check it (e.g. port should be numeric, token should look like the expected token format) rather than blindly trusting the first pass. Then build the pipeline bottom-up: create the bare repo with `git init --bare /home/user/config.git`; write `hooks/post-receive` as an executable script that does `git --work-tree=/home/user/app_state --git-dir=/home/user/config.git checkout -f` followed by invoking `/home/user/deploy.sh`. In `deploy.sh`, `mkdir -p` the OCR'd BACKUP_DIR, `tar` the app_state directory into a timestamped archive there, kill any existing service process (e.g. `pkill -f` a distinguishing string, or track a pidfile), then launch the monitoring service backgrounded/detached (`nohup ... & disown`) so the hook itself returns promptly. The service (Python http.server subclass, socat, or similar) must serve GET /status on the OCR'd PORT with 200 and body exactly `TOKEN=<extracted_token>`. Finally clone the bare repo, commit a dummy file, push it to trigger the hook, and curl your own service locally to confirm before finishing.
