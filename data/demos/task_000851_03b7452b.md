Build the sanitizer to fail closed: read allowed_encodings and target_encoding from /app/config/sanitizer_config.json, then for each input file try decoding with each allowed encoding in listed order using strict error handling (e.g. `bytes.decode(enc, errors='strict')` in a try/except, not `errors='ignore'`) — the first encoding that succeeds without raising wins; if all fail, the file is evil. After a successful decode, still check the decoded text for embedded `\x00` (rare post-decode but check raw bytes too) and case-insensitively for `<script>` before accepting — checking the substring only pre-decode or only post-decode is a common gap since some encodings can produce different byte/char views. Only for files passing all checks: write re-encoded bytes to the output dir in target_encoding, hash the *written output content* (not the original bytes) with hashlib.sha256, and record filename+hash in manifest.json exactly as specified. For Phase 2, treat it as standard reverse-proxy configuration: nginx `listen 8080; daemon off;` isn't required if backgrounding, a `location = /manifest.json { alias ...; }` for the static file, and `location /docs/ { proxy_pass http://127.0.0.1:5000/; }` for the Flask backend; start redis, then Flask with DOC_DIR set, then nginx, and curl both endpoints to confirm end-to-end wiring before finishing.
