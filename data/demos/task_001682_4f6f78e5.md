This is a three-stage decode-clean-detect pipeline; do each stage in isolation and sanity-check before moving on. First, decode `/app/metadata.dat`: read the raw bytes and decode with `.decode('cp500')` (Python's codec name for IBM EBCDIC cp500), then `json.loads` the result to get `min_valid`/`max_valid` — don't assume ASCII/UTF-8, the raw bytes will look like garbage otherwise. Second, read `/app/telemetry.wav` with Python's `wave` module (16-bit mono PCM at 10 Hz means each frame is one 2-byte signed sample); unpack all frames with `struct.unpack(f"<{n}h", frames)` into a list of ints. Clean the samples in a single forward pass: track "last valid value" starting from sample 0, and whenever a sample falls outside `[min_valid, max_valid]` replace it with that last-valid value (don't use interpolation or a different fill strategy). Third, run changepoint detection on the cleaned series to find where the baseline permanently jumps upward — a simple robust approach is scanning for the index that best splits the series into two segments with maximally different means (e.g. via a sliding-window mean comparison or `ruptures`/manual sum-of-squares minimization), not just the first sample that exceeds some fixed cutoff. Write the integer index to `/home/user/result.json`; verification tolerates being within a few samples of the true changepoint.
