Build this as a small HTTP service (Flask/FastAPI/http.server all work) bound to `0.0.0.0:8000` exposing `POST /process`. In the handler: open the SQLite db at the given `db_path` with `sqlite3`, introspect `wide_readings`' columns (via `PRAGMA table_info` or the cursor description) to find all sensor columns dynamically (don't hardcode `s1..sN`), then melt wide-to-long (timestamp, sensor_id, value), dropping NULL values. Per sensor, sort by timestamp and compute first-order differences with the first diff set to 0.0. For anomaly detection, treat `/app/detect_outliers` as a black-box CLI oracle: for each sensor spawn it as a subprocess (`subprocess.Popen`/`run`), write its diff values one-per-line to stdin, and read back one 0/1 per line in the same order — run these subprocess calls concurrently across sensors using `concurrent.futures.ProcessPoolExecutor` or `ThreadPoolExecutor` since the binary itself is the parallel unit, not your own detection logic. Filter to rows flagged `1`, bulk-insert into a new `anomalies(timestamp INTEGER, sensor_id TEXT, diff_value REAL)` table (CREATE TABLE IF NOT EXISTS first), commit, and return `{"status": "success", "anomalies_inserted": <count>}`. Keep the server process alive (don't let the script exit) and test with a `curl` POST against a sample db before finishing.
