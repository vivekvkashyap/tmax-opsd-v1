Do not attempt to shell out to /app/citation_oracle per query in your script — it exists only as a slow correctness reference to compare against, and repeatedly invoking it is exactly the anti-pattern you're replacing. Instead, build fast_query.sh to: create a temp sqlite3 database (file-based or `:memory:` via a here-doc piped into `sqlite3`), bulk-load the edgelist with `.mode csv`/`.import` (converting space-separated to a loadable format first, e.g. with `tr`/`awk`), then `CREATE INDEX` on the source column (and target if useful) before querying — indexing after bulk load, not before, is faster and correctness-critical for the CTE performance. For each query pair, or better, in a single batched pass, run a recursive CTE that does BFS-style shortest-path expansion: seed with the source node at depth 0, recursively join to edges where source=frontier and target not yet visited, track depth, and take MIN(depth) where node=target, returning -1 if no row is found. Loop over the queries file in bash, feeding each source/target into the parameterized SQL (or build one big multi-query SQL script to minimize sqlite3 process startup overhead, which matters for hitting the 3-second threshold), and print results to stdout in input order. Test correctness against a few known cases before optimizing further.
