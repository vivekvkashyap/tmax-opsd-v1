Build and test each phase independently before wiring them together. For the mount script: idempotency means checking state before acting — use `mkdir -p` for dirs, and before creating `<dest>/mnt` check with `[ -L "$dest/mnt" ]` and compare its current target (`readlink`) to the desired source; only (re)create the symlink if it's missing or wrong, using `ln -sfn` (the `-f` avoids "file exists" errors, `-n` avoids following an existing symlink into the wrong place) so reruns are safe. For the C supervisor: fork/exec per attempt, and crucially `fflush(stdout)` in the parent before each `fork()` (or set stdout unbuffered) so `[Supervisor] Restarting...` doesn't get buffered out of order relative to the child's own output; use `waitpid` + `WIFEXITED`/`WEXITSTATUS` to get the real exit code, loop up to max_retries+1 total executions, exit 0 immediately on a zero exit, exit 1 only after exhausting retries. Test the supervisor standalone against the failing service first, checking the exact log content, before running the full mount+supervisor pipeline.
