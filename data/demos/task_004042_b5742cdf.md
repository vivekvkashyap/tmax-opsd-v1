Design the parser to validate incrementally and never trust size fields before checking them against reality. Open each `.frm1` file and read only the fixed 20-byte header first (4-byte magic + two 8-byte little-endian uints via `struct.unpack("<8sQQ", ...)` or similar) — reject immediately if magic != `FRM1` or Original Size != 6220800, without reading further. Then check the actual remaining file size (via `os.fstat`/`os.path.getsize` minus header, or by seeking) against the claimed Compressed Size field and reject on mismatch before decoding any RLE. Decode the RLE stream incrementally (pair by pair), tracking a running decompressed-byte counter as you go; reject as soon as you see `count == 0`, and reject if the running total ever exceeds 6220800 (this stops RLE-bomb memory exhaustion — don't decode into an unbounded buffer first and check after). After the full stream is consumed, reject if the final decompressed total != 6220800. Only files that survive every check get processed: SHA-256 the raw bytes, write to `<output_dir>/objects/<hash>.raw` with `os.link` when the hash file already exists (else write fresh), and `os.symlink` from `<output_dir>/sequence/<name>.raw` to the objects path. Test against your own encoder plus deliberately corrupted files before trusting the hidden corpora.
