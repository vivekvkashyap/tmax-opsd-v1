Treat `/app/wave_sim` as a black-box data generator only — run it to produce the binary, then do all analysis yourself in your own C++ (or verify numerically first with a quick Python/NumPy prototype before committing to the C++ pipeline). Key steps: (1) run the simulator to get `wave_data.bin`; (2) read it as float32 and reshape with time (`nt`) as the fastest-varying axis — i.e. index as `data[x][y][t]` with `t` innermost, matching the stated row-major layout; (3) for each of the 64x64 spatial points, take a 1D DFT over the `nt=512` time samples; (4) sum squared magnitudes of the complex coefficients across all spatial points per frequency bin to get a per-frequency energy spectrum, zero out (or skip) index 0, and search only indices 1..nt/2-1 for the max — this gives k_peak; (5) build the 64x64 complex matrix of Fourier coefficients at k_peak and run a complex SVD (Eigen's JacobiSVD or BDCSVD supports complex matrices) to get singular values; (6) take the top 3 descending and write them as float64 into an HDF5 dataset literally named `/top_svals` using the HDF5 C/C++ API. Cross-check your k_peak and singular values against a NumPy `np.fft.fft` + `np.linalg.svd` prototype on the same binary before finalizing, since axis-order or FFT-normalization mistakes are the most common failure mode.
