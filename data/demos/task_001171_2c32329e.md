Build this as one idempotent bash script that classifies files by content, not by extension or filename alone, since claims in filenames/extensions can be wrong. Extract the archive first (`tar -xzf /home/user/raw_artifacts.tar.gz -C /home/user/artifacts`). For traversal, use `find /home/user/artifacts -type f` (not `-follow`/recursive symlink traversal) so you visit each real file exactly once and never chase directory symlinks into a loop. For each file: use `file <path>` or check the raw ELF magic bytes plus `readelf -h <path>` (look at the `e_machine` field, or grep `file`'s output for "ARM") to confirm ELF+ARM before copying to `elf/`; for `.gcode` files, grep with a word-boundary pattern (e.g. `grep -qw 'G28'`) so you don't match substrings inside other tokens; for WAL detection, read the first 4 bytes of every file with `xxd -l4 -p <path>` or `od -An -tx1 -N4` and compare against the two valid magic byte sequences, regardless of the file's extension. Create the three output subdirectories before copying anything. Once populated, `cd /home/user/curated && zip -r /home/user/curated_artifacts.zip elf gcode wal` so the zip's internal paths are relative to `curated/`.
