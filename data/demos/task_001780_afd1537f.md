Two independent components: an extraction pipeline and a long-running Rust HTTP service. For extraction, use `ffmpeg` to dump frames from `/app/dataset_log.mp4`, then run `zbarimg` over the frames to decode QR codes, filter/parse lines matching `GRAPH_EDGE;<source>;<target>;<weight>`, dedupe if the same code appears across multiple consecutive frames, and write a clean CSV of edges that your Rust program will read at startup (never hardcode the edges into Rust — read the generated file). For the service, use a Rust web framework (e.g. axum) to build an adjacency list from the CSV, implement Dijkstra's shortest-path algorithm for `POST /shortest_path`, check the `Authorization: Token research-2024` header before doing any work (return 401 immediately if missing/wrong), return 404 with the specified error JSON when no path exists, and otherwise return the path list and total integer weight in the exact schema. Bind to `127.0.0.1:8080`, run it in the background (e.g. with `&` or `nohup`), and only write `/home/user/service_ready.log` containing "SERVICE_UP" once the listener is actually bound — writing it too early causes the verifier to hit a connection refused.
