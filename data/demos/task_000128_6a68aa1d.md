Implement this as four sequential passes over the data using only stdlib (json, csv, hashlib, itertools), writing intermediate results to disk between phases so each phase is easy to check independently. Phase 1: for each raw line, json.loads it, then validate the exact field set (`set(obj.keys()) == {"timestamp","ip_address","user_email","endpoint","status"}`) and exact types (str for the first four, int for status — note bool is a subclass of int in Python, so explicitly exclude bool if being strict); anything failing either check gets its original raw line written verbatim to invalid.jsonl. Phase 2: for valid entries, hash user_email with `hashlib.sha256(email.encode()).hexdigest()` and zero the last IPv4 octet by splitting on '.' and replacing the 4th part with '0'; write these masked records to masked_logs.jsonl. Phase 3: tally masked_logs by ip_address with a dict/Counter, then sort entries by count descending, ip_address ascending as tiebreak, and write to summary.csv. Phase 4: build a dict mapping each masked user_email hash to a set of endpoints they hit, use `itertools.combinations` over all user pairs to compute Jaccard = len(intersection)/len(union), track the max, and on ties choose the pair whose (min_hash, max_hash) tuple sorts first; write the winning pair to anomalies.json with user1 < user2 alphabetically and similarity rounded to 4 decimals via `round(x, 4)`.
