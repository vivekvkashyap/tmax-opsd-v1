Use `multiprocessing.Pool.map` with one worker function per input file (glob all CSVs in `telemetry_input/`), having each worker return a tuple of (valid_records, rejected_rows) for its file rather than trying to share mutable state across processes — then merge the per-file results in the parent after `pool.map` completes. Inside each worker, for every row: try `float(value)` in a try/except (catch failures as immediate rejects), then if metric_type is temp and unit is F, convert with `(F-32)*5/9` and treat the value as the Celsius result and unit as "C" for subsequent validation and output; then apply the range check appropriate to metric_type (temp: -50..60, humidity: 0..100) — rows with other metric_types (e.g. pressure) aren't range-constrained by the spec, so let them through as valid unless unparseable. Rejected rows must be written back in their exact original CSV form (original value/unit strings, not the converted ones) — keep the raw row alongside the parsed version so you can emit it unchanged if rejected. After merging, group valid records by (sensor_id, metric_type), average the (possibly-converted) values, round to 2 decimals, and sort by sensor_id then metric_type before writing `aggregated.json`; write rejected rows to CSV preserving the original header.
