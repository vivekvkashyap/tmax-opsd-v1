Use `pathlib` for path resolution, not string matching against `../`: for each file path in a manifest, build the candidate as `(Path("/home/user/data") / path).resolve()` if relative, or `Path(path).resolve()` if absolute — resolve() collapses both `../` and `./` regardless of whether the raw string looks suspicious. Then validate by checking the resolved path is equal to or a descendant of `Path("/home/user/data").resolve()` (e.g. compare via `resolved.is_relative_to(root)` in Python 3.9+, or check `str(resolved).startswith(str(root) + os.sep)`) — this correctly lets `project/../images/logo.png` through (still inside data/) while rejecting `../../etc/passwd` and other absolute paths outside the root. Collect (backup_id, resolved_path) tuples across all manifests found via `Path(...).rglob("*.json")`, sort by backup_id then path, and write the CSV. For atomicity, write to a temp file in the same directory (e.g. `tempfile.NamedTemporaryFile(dir=..., delete=False)`) then `os.replace(tmp, final)` — never write directly to the target path. Finally use `awk -F, 'NR>1{print $1}' processed_manifest.csv | sort -u > valid_ids.txt` for the shell step.
