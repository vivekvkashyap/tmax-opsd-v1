Build the `/audit` handler in two independent data-access steps, then merge. First, resolve role inheritance in PostgreSQL with a single recursive CTE: start from the roles directly in `user_roles` for the given username, then recursively join `roles.parent_role_id` upward (bounded, since hierarchy is at most 5 deep) to collect the full closure of role ids. Join that closure against `role_permissions`/`permissions` filtered by the requested resource_name to get the list of granting role_names — sort them alphabetically, and set `authorized` based on whether the list is non-empty. Second, independently query Redis with `ZREVRANGEBYSCORE access:<resource> +inf -inf WITHSCORES`, filter members equal to the requested username, and take the top 3 scores (already descending) as floats. A naive approach that queries roles with N+1 lookups or walks parent chains in application code (round-tripping to Postgres per level) will be slow and risk breaking under fuzz load — do it in one CTE. Also watch for the empty-list vs missing-key edge case in Redis, and make sure JSON booleans are real booleans, not strings. Wire the Flask app to listen on 0.0.0.0:8080.
