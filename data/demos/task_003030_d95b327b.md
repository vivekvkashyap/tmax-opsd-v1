Implement this purely with bash/coreutils/awk — no Python/Perl/R. First filter `/home/user/mutation_rates.txt` to drop lines that are exactly `NaN` (e.g. `grep -v '^NaN$'`) and count the remaining valid lines as N. For each of the `<num_iterations>` bootstrap rounds, sample N values with replacement from the valid data — the simplest robust approach in bash is an awk script that picks N random indices (`int(rand()*N)`) into the valid-values array and computes their mean in one pass, since spawning `shuf`/subprocesses per-value per-iteration for 1000 iterations will be slow but is acceptable if correctness matters more than speed; accumulate each iteration's mean into an array/file. Every 100th iteration, compute the running average of all means seen so far and append `Iteration <i>: <cumulative_average>` to `/home/user/convergence.txt` — do this incrementally rather than recomputing from scratch each time if performance matters. After all iterations, sort the collected bootstrap means numerically (`sort -n`) and pick the values at the 2.5th and 97.5th percentile positions (for 1000 sorted values, roughly the 25th and 975th, 1-indexed) to form the CI, writing `Lower: X.XXX, Upper: Y.YYY` to `/home/user/ci_output.txt` rounded to 3 decimals via `awk`/`printf`. Run it with `1000` iterations on the real data file and sanity-check the CI brackets the sample mean.
