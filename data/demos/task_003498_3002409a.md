Build this in clearly separated, independently-testable stages inside the Cargo project. First, read the raw f32 binary with a fast bulk read (bytemuck or manual byte-slice reinterpretation) rather than reading value-by-value. Second, compute the moving average with a running-sum sliding window (add the incoming value, subtract the one leaving the window) rather than recomputing the 11-element sum from scratch at every index — this matters for performance but also correctness: initialize S[k]=0.0 for indices outside [5, 999994] exactly as specified, don't extrapolate. Third, find k_max via a single linear scan keeping the first index seen on ties. Fourth, for the grid search, precompute the 1,000,000 alpha values with the exact formula given, then use rayon's `par_iter()` (e.g. over a 0..1_000_000 index range) mapping each alpha to its SSE over the fixed 101-point window [-50,50] around k_max, and reduce with `min_by`/`fold` to find the minimizing alpha and its error — verify you actually see multi-core CPU usage during this step to confirm real parallelism, not just crate presence. Finally round the four output floats to 4 decimals per the spec and write result.json, then build with `cargo build --release` and run the release binary (not debug) since the grid search size makes debug builds impractically slow.
