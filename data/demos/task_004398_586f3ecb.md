Recover the two secret inputs before writing any server code, and keep them as variables you derive rather than assume. For the master key, run `tesseract /app/new_master_key.png stdout` (preprocess the image first if OCR output looks noisy) and strip trailing newline/whitespace from the result. For the legacy salt, use ELF tooling — `objdump -s -j .secret_salt /app/legacy_crypto.so` or `readelf -x .secret_salt` — to dump the raw bytes of that custom section and convert the hex dump back to an ASCII string, being careful to exclude any trailing NUL byte so it isn't included in the key material. Build the HMAC key as `<master_key>:<elf_salt>` exactly, with a single colon separator. Then write the C++ HTTP server (e.g. via a vendored single-header `cpp-httplib`) listening on `127.0.0.1:8080` handling `POST /verify`: parse the JSON body for `token`, split on the last `.` into hex payload and hex HMAC, hex-decode the payload, compute HMAC-SHA256 over the decoded payload using OpenSSL's `HMAC()` (link `-lcrypto`) with your derived key, compare against the decoded HMAC hex (constant-time compare is good practice), and return 200/`{"status":"success"}` or 401/`{"status":"failed"}` accordingly. Run it as a background process and confirm it's still listening before finishing.
