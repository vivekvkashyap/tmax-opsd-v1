Work through this in dependency order. (1) Compile `/app/src/libaudiofeat.c` to a shared object with `gcc -shared -fPIC -o /app/bin/libaudiofeat.so /app/src/libaudiofeat.c`, then in `/app/service/ffi_wrapper.py` load it with `ctypes.CDLL`, and set `.argtypes = [ctypes.c_char_str]` and `.restype = ctypes.c_float` on `get_audio_feature` — ctypes defaults to `int` return type, which corrupts float results if you skip setting `restype`. (2) Initialize the SQLite DB by executing `001_init.sql` then `002_add_columns.sql` in order against `/app/data/ci_service.db` (via `sqlite3` CLI or Python's `sqlite3` module) so the `jobs` table ends up with `id, filepath, feature_value, status` columns. (3) Build `/app/service/server.py` (Flask/FastAPI/stdlib) binding to `127.0.0.1:8080` with `POST /process` (calls the FFI wrapper, inserts a row, returns `{"job_id": <id>}`) and `GET /job/<id>` (returns the row as JSON or 404 if missing — use `cursor.lastrowid` for the insert id). (4) Launch the server in the background, redirecting stdout/stderr to `/app/logs/server.log`, and confirm with `curl` against `/app/test_audio.wav` before finishing.
