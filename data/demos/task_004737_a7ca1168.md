Build the pipeline as discrete, verifiable stages inside one Bash script rather than one big pipeline. First derive `y`: parse the FASTA with `awk`/`grep -v '^>'` to strip headers, then measure each sequence's character length in file order. Second, extract the matrix `X` from the HDF5 file using `h5dump -d /X` (or `h5dump` with an XML/CSV-like output flag) and post-process the text output with `awk`/`grep` into clean rows of numbers — this text-scraping step is the easy place to introduce off-by-one or extra-token bugs, so sanity-check the parsed matrix shape before proceeding. Third, hand `X` and `y` to a short embedded Python snippet (`python3 -c "..."` or a heredoc) that loads them via numpy and calls `numpy.linalg.solve` (or `lstsq` if not square) to get `w`, then write one value per line to the weights file. Finally, compute the L2 norm independently using only `awk`/`bc` (sum of squares, then `bc -l` sqrt), formatting to exactly 2 decimals with `printf`/`bc` rounding — do not just reuse Python's norm, since the task requires bash/awk/bc tooling for that step. Make the script executable at the end.
