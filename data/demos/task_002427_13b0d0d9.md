Treat the evaluator as a black-box scoring function you must probe and optimize against, not reverse-engineer by reading its binary. First load raw.csv and inspect each column for NaNs and extreme values (e.g. values many standard deviations from the mean) — impute or drop rows/columns with missing data, and clip or remove rows containing implausible extreme outliers so no column has values far beyond its typical range; the evaluator hard-fails to a zero score if any cell is missing/non-numeric or any value exceeds roughly mean+5*std. Once the data is clean, iterate on feature selection: keep the target column plus a small set of features that are each strongly correlated (positively or negatively) with target, since the score rewards high average |correlation with target|. Then check pairwise correlations among your chosen features and drop or merge any that are highly correlated with each other (redundant/derived duplicates), since the score is penalized by average pairwise |correlation| among features. Test different subsets by rerunning the evaluator against your candidate clean.csv, comparing scores, and converging on a small, low-collinearity, high-target-correlation feature set — treat this as an empirical search loop rather than a one-shot guess.
