This is a graph shortest-path-from-root problem layered with filtering, not a plain SQL join. Load the `citations` edges into memory (or use a recursive CTE) and do a BFS from `ROOT-01` following edges in the `citing_id -> cited_id` direction reversed (you want all papers that cite ROOT-01 transitively, so build reverse adjacency: for each citation row, `citing_id` is one step closer to citing `cited_id`, so from `ROOT-01` walk to all `citing_id`s whose `cited_id` matches, then repeat from there) — track the depth at which each paper is FIRST reached via BFS, since BFS naturally gives shortest depth per node and later encounters at deeper levels must be ignored (don't overwrite an existing depth). Watch for the trap in this data: some papers are reachable via multiple paths at different depths, so a naive DFS or recomputation could pick the wrong (longer) depth — always keep the minimum. After computing depth per paper, join against `papers`, filter to `field_of_study = 'Artificial Intelligence' AND year >= 2020`, sort by year desc, depth asc, paper_id asc, take top 5, and write a JSON array with keys `paper_id, title, year, depth` in that field order to `/home/user/lineage_results.json`.
