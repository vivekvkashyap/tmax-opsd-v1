Start by reading `/home/user/server.log` and `/home/user/config.json` to see the actual crash reason rather than assuming — the log will show a `load_cert_chain` failure (missing cert/key files) and/or a permission error from binding a low-numbered port as non-root. Generate a self-signed cert/key pair with `openssl req -x509 -newkey rsa:2048 -keyout /home/user/key.pem -out /home/user/cert.pem -days 365 -nodes -subj "/CN=localhost"` (the `-nodes` flag is what keeps the private key unencrypted, which this server's code requires since it never prompts for a passphrase). Edit `/home/user/config.json`'s `port` field from its privileged value to `8443` using `jq '.port = 8443' config.json > tmp && mv tmp config.json` (or `sed`/`awk`, but confirm the result is still valid JSON afterward — `jq . config.json`). Launch with `python3 /home/user/server.py &`, give it a moment to bind, then confirm it's actually listening on 8443 (e.g. `ss -tlnp` or just try the curl) before running `curl -k -s https://localhost:8443/health > /home/user/health_check.txt` — the `-k` flag is required because the cert is self-signed and untrusted.
