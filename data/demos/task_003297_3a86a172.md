Work in clear stages and verify each before moving on. First convert encoding: read the raw bytes and decode explicitly as UTF-16LE (e.g. `open(path, encoding='utf-16le')` or `iconv -f UTF-16LE -t UTF-8`), then re-emit as UTF-8 — do not rely on auto-detection, which often misreads BOM-less UTF-16LE. Second, deduplicate by comparing full rows (all columns, header excluded), keeping the header exactly once. Third, sort chronologically per device before computing the rolling window — a common mistake is computing the moving average on the original (possibly duplicated/unsorted) order, which corrupts the window contents. Compute the simple moving average per `device_id` independently using a window of the current row plus up to 2 prior rows for that same device (fewer than 3 available just averages what exists), rounding to exactly 2 decimals with standard rounding (watch for banker's rounding differences between languages). Finally, re-sort the output by `device_id` ascending then `timestamp` ascending for the final CSV — this final sort order may differ from the order used to compute the rolling window, so do it as a separate last step, not combined with the windowing sort.
