Separate the offline analytics computation from the web-serving layer. First extract frames at 1 fps deterministically, e.g. `ffmpeg -i /app/traffic_camera.mp4 -vf fps=1 frame_%03d.png`, then for each frame compute mean R, G, B channel intensity (e.g. with PIL/numpy: load image as array, `.mean(axis=(0,1))` per channel) to build an (N,3) embeddings matrix. Compute the 3x3 covariance with `numpy.cov(embeddings.T)` — be aware the verifier tolerates either `ddof=0` or `ddof=1`, so numpy's default (ddof=1) should be fine, and round to 4 decimals only at serialization time, not before. For the bootstrap CI, `numpy.random.seed(42)` immediately before resampling, draw exactly 10,000 bootstrap samples with replacement from the Red-channel values, compute the mean of each resample, and take the 2.5th/97.5th percentiles of that distribution of means (percentile method) — do the seeding and resampling once at startup and cache the result so repeated HTTP calls return identical, deterministic values rather than reseeding/resampling per request. Then stand up a minimal HTTP server (Flask/http.server/FastAPI) on 127.0.0.1:8080 with `/covariance` and `/bootstrap_red` returning plain CSV strings (no header) as specified, and leave it running.
