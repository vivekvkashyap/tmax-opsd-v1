Handle each of the four requirements with the right defensive tool rather than a naive recursive walk. To avoid the symlink loop, use `find /home/user/research_data -type f` (the default `find` does NOT follow symlinks unless you pass `-L`, so plain `find ... -type f` naturally skips descending into the loop symlink while still listing regular files everywhere else) — do not use `-follow` or `-L`. For PNG detection, don't trust the `.dat` extension: read the first 8 bytes of each `.dat` file (e.g. `head -c8 file | xxd -p` or Python's `open(path,'rb').read(8)`) and compare against the literal PNG magic byte sequence before copying+renaming to `.png` in `/home/user/recovered_pngs/`. For CSVs, filter by modification time with `find ... -name '*.csv' -mtime -2` (last 2 days) before converting, and use `iconv -f WINDOWS-1252 -t UTF-8` per file into `/home/user/clean_csvs/`, preserving filenames. For archive integrity, iterate all `*.tar.gz` files and run `tar -tzf <file> >/dev/null 2>&1`, checking the exit code — non-zero means corrupt/invalid — and append the absolute path to `/home/user/bad_archives.log` only for failures. Create both output directories up front with `mkdir -p`.
