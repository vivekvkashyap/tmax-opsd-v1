Start by running `tesseract` (install via apt if missing) on `/app/error_snapshot.png` to OCR out the EPSILON and MAX_ITERATIONS values printed there — inspect the raw OCR text carefully since OCR can mis-render characters like `1` vs `l` or scientific notation, and cross-check the extracted numbers make physical sense before hardcoding them into the C++ source. Then set up the project: create the directory, write the server code with your extracted constants substituted in, switch every `float` in the pricing computation (the variable types and the `S`/`x`/`prev_x` chain, plus the function's parameter and return type) to `double`, since a `float`'s ~7 significant digits can't resolve differences down to a very small epsilon and the convergence loop will either never terminate correctly or return an imprecise root. Download the header-only `httplib.h` into the project directory as instructed, compile with `g++ -O2 -std=c++11 server.cpp -o server -lpthread`, and launch it in the background (`./server &` or nohup) bound to 127.0.0.1:8080. Manually curl `/price?s=123456789.0` afterward and sanity-check the returned value looks like a square root before considering the task done.
