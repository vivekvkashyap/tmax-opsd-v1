Build this as a single awk-centric pipeline rather than chaining many fragile sed/date calls per line. Read each line, split on `|`, trim whitespace on each field (awk's gsub with a leading/trailing space pattern, or use `sub`). For the timestamp, detect its format by pattern (does it match `YYYY-MM-DD HH:MM:SS`, `MM/DD/YYYY HH:MM:SS`, or is it purely digits already an epoch?) and convert non-epoch forms to epoch using `date -u -d "..." +%s` (GNU date) — since you must treat all human timestamps as UTC, be careful to pass `-u`/`--utc` or an equivalent so the local timezone of the shell never leaks in, which is the most common way naive attempts silently produce wrong epoch values. Emit `epoch,level,message` for every row, then deduplicate on the full triple (exact match, not just epoch) with `sort -u` or `awk '!seen[$0]++'`, and finally `sort -t, -k1,1n` to order chronologically before prepending the CSV header. Make the script executable and test it end-to-end against the actual input file, not just individual lines.
