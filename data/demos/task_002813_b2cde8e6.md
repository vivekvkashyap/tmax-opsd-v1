Structure the script around three concerns kept cleanly separate: parallel I/O, robust decoding, and deterministic merge/sort. Use `concurrent.futures.ProcessPoolExecutor` (or ThreadPoolExecutor, since this is I/O-bound) to map a per-file parse function across the four log paths. In that function, open each file in binary mode (`'rb'`) and decode with `raw_bytes.decode('utf-8', errors='replace')` — this is the key step naive attempts skip, since plain `open(path)` in text mode will raise a UnicodeDecodeError on the injected invalid bytes rather than substituting U+FFFD. Parse each resulting line with `json.loads`. After collecting all records from all files, deduplicate into a dict keyed by `event_id`, keeping whichever candidate has the higher `retry_count` when a key repeats. Convert the deduplicated values to a list, sort with a key of `(timestamp, event_id)` ascending. Write output with `json.dumps(record, sort_keys=True)` one line per record, and end each line with a single newline — no trailing blank line inconsistencies, since the grader checks a sha256 hash of the exact file bytes, so formatting must be exact (no extra whitespace, consistent line endings).
