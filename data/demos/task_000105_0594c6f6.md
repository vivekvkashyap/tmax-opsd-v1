Read `processHandler` in `/home/user/service/main.go` closely: it spawns a goroutine that always tries to send on an unbuffered `resultChan`, but if the `select` returns via the `<-r.Context().Done()` case (client disconnected/timed out), the handler returns immediately while the background goroutine is still sleeping — when it later tries `resultChan <- "Success"` on an unbuffered channel with no receiver left, it blocks forever, leaking that goroutine permanently. Fix it by making the send non-blocking from the goroutine's side, e.g. give `resultChan` a buffer of size 1 (`make(chan string, 1)`) so the goroutine's send always succeeds and it can exit even if nobody is listening anymore, or alternatively pass the request's context into the worker so it can select on `ctx.Done()` around its own send.
