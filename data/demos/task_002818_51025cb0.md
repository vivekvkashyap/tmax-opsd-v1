This is a self-contained shellcode-execution harness — get the mmap mechanics right rather than trying to hand-decode the machine code by inspection. In test_fixture.c: open payload.bin, read its bytes into a buffer, then call `mmap(NULL, size, PROT_READ | PROT_WRITE | PROT_EXEC, MAP_PRIVATE | MAP_ANONYMOUS, -1, 0)` to get an executable page, `memcpy` the file bytes into that mapped region (you cannot exec directly from a plain malloc'd buffer since it lacks PROT_EXEC), then cast the mapped pointer to a function pointer of type `void (*)(char *, unsigned long)` and call it. Prepare a local buffer containing the 6 bytes of "AgentX" (no null terminator needed within the length argument, though a mutable non-const buffer is required since the function likely mutates the bytes in place) and pass its address plus length 6. After the call returns, read the (now mutated) 6 bytes back out of that buffer and format each as two uppercase hex digits separated by single spaces, writing the resulting line to /home/user/test_output.txt. Compile with `gcc test_fixture.c -o test_fixture` and actually run the resulting binary to produce the output file — don't just compile and stop.
