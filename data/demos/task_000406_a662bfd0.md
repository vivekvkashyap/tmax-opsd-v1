This is black-box reverse engineering — don't assume the obvious "print every FILE/STATUS pair" behavior without testing it. Feed the binary a series of targeted stdin probes and diff its stdout each time: (a) a single well-formed record with one FILE and one STATUS, to confirm the base output format; (b) a record with STATUS exactly "OK" vs. any other value, to check if "OK" is suppressed; (c) a record with multiple FILE: or STATUS: lines in the same block, to see whether it uses the first or last occurrence; (d) records separated by one blank line vs. multiple consecutive blank lines vs. no trailing blank line at EOF; (e) lines with leading whitespace before "FILE:"/"STATUS:", and unrelated lines interspersed (like ID: fields), to see what's ignored; (f) a block missing FILE or missing STATUS entirely. Build your bash script (awk/sed or a read-loop with a state machine) incrementally, re-testing against the binary after each probe, rather than writing the whole thing from assumptions. Only finalize once your script matches the binary on all these edge cases plus a batch of random inputs.
