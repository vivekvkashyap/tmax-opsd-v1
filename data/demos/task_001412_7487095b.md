Build and sanity-check each stage in isolation before chaining them into `run_analysis.sh`. Extract frames with `ffmpeg -i /app/microarray_timelapse.mp4 -vf fps=1 -pix_fmt gray /home/user/frames/frame_%03d.pgm` and confirm you get one grayscale PGM per second of video. For per-quadrant means, remember PGM (P5 binary) has a text header (magic, width, height, maxval) before the raw pixel bytes — parse the header to get width/height, split rows into top/bottom halves and columns into left/right halves, and average pixel values per quadrant per frame with awk (reading raw bytes via `od`/`xxd` piped into awk, or a small helper). Log intensities per frame to `analysis.log` as you go. For convergence, compare each frame's four quadrant means against the *previous* frame's (not frame 1), and only declare convergence at the first frame index where all four absolute deltas are simultaneously `< 2.0` — iterate frame-by-frame rather than trying to vectorize, since you need to stop at the first qualifying frame. For the server, implement the classic `nc -l -p 8080` (or `socat TCP-LISTEN:8080,reuseaddr,fork` for concurrent connections) loop that reads the HTTP request line, routes on `/convergence` vs `/intensities`, and writes back proper `HTTP/1.1 200 OK\r\nContent-Length: N\r\n\r\n<body>` — a common bug is missing `Content-Length` or the blank line, which breaks HTTP clients. Leave it running in the background.
