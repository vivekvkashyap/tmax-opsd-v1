For dependency resolution: parse every JSON file in `/home/user/release/services` with `jq` to collect `(name, version, requires)` tuples, group by service name (each service may have multiple candidate versions), and search for a selection (one version per service) satisfying all `requires` version constraints (parse comparator strings like `>= 1.0.0` and compare with `sort -V` or manual numeric comparison of major.minor.patch) while also forming no dependency cycle among the *selected* versions — build a directed graph of the selected set's requires edges and check for cycles (e.g. via topological sort / DFS with a visiting-set) rather than just checking version constraints alone, since a set can satisfy version bounds yet still be circular. If multiple versions of a service could individually satisfy constraints, you likely need to try combinations (e.g. prefer lowest/highest valid version per service, or exhaustively search since the service count is small) until you find one with zero cycles. Make both scripts executable.
