Do this entirely with Python's `tarfile` and `gzip` modules operating on file-like objects, not by extracting to disk. Open the outer tar with `tarfile.open('/home/user/backup_archive.tar')`, iterate members, and for each `.tar.gz` member call `tarfile.extractfile(member)` to get a stream, then wrap it: since the inner tarball is itself gzip-compressed, open it as `tarfile.open(fileobj=outer_extractfile, mode='r:gz')` — this avoids writing any intermediate file. Iterate that inner tar's members (the `.gz` log files), get each one's stream via `extractfile()`, and wrap that stream with `gzip.GzipFile(fileobj=...)` to decompress the log text in memory; read/decode it and scan line by line. Match metadata lines with a regex like `r'\[METADATA-EXPORT\] file=(\S+) checksum=(\S+)'` and accumulate results into a single dict (later entries overwrite earlier ones only if paths repeat — but here they shouldn't). Write the merged dict as JSON to `/home/user/metadata_report.json` with `json.dump`. The common pitfall is trying to call `tarfile.open` on a `.tar.gz` extractfile object without specifying `mode='r:gz'`, or accidentally writing temp files — avoid both to satisfy the "no extraction to disk" constraint.
