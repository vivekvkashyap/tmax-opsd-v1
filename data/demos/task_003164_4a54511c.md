Do this as four sequential passes with intermediate variables you can inspect, not one giant pipeline. Pass 1: parse `syslog.log` per (user, IP), tracking consecutive "Failed password" counts; a pair qualifies only when >=3 such failures are immediately followed by an "Accepted password" for that same user/IP — use `awk`/`grep` rather than one regex. Pass 2: for each qualifying user, look up their hash in `shadow.bak`, then dictionary-attack it by hashing every word in `wordlist.txt` (`echo -n "$word" | md5sum`) until one matches. Pass 3: map username to UID via `passwd.bak` (field 3), then grep `netstat.txt` for LISTEN lines whose user/UID column equals that UID, extracting the port. Pass 4: keep only users who cleared all three checks, then emit the CSV sorted alphabetically with the exact header — sanity-check against a user who fails one criterion (e.g. only 2 failed attempts) to confirm your logic excludes them correctly.
