Implement the four checks in `audit_filter.go` as independent, short-circuiting gates, each of which can reject outright. (1) Permission check: `os.Stat` the file and compare `info.Mode().Perm()` against exactly `0600` or `0400` — reject anything else. (2) Parse the key with `golang.org/x/crypto/ssh.ParseAuthorizedKey`; inspect the returned key type string to reject `ssh-dss`, and for RSA keys extract the underlying `*rsa.PublicKey` (via type assertion on the crypto key) to check `N.BitLen() < 2048`. (3) Extract the trailing comment from the parsed key (the third field of the authorized_keys line) and shell out to `/app/decoder` with that comment string as `os.Args[1]`, capturing stdout via `exec.Command` — treat the decoder as an opaque tool, don't try to reimplement its logic. (4) Check whether the decoder's stdout contains the substring `EVIL` (`strings.Contains`) and reject if so. Any single failure means `os.Exit(1)`; all checks passing means `os.Exit(0)`. Test against every file in `/app/corpus/clean` and `/app/corpus/evil` before considering it done, since the grader requires 100% correct classification on both sets.
