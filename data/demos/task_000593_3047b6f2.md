Structure this as three separable concerns: safe extraction, encoding conversion, manifest generation. For safe extraction, open the zip with `archive/zip`, and for each file entry compute the intended destination via `filepath.Join(targetDir, entry.Name)`, then verify the cleaned absolute path still has `targetDir` (also cleaned/absolute) as a prefix — the standard zip-slip guard is `strings.HasPrefix(filepath.Clean(dest), filepath.Clean(targetDir)+string(os.PathSeparator))`. If the check fails, `continue` to the next entry rather than erroring out or aborting the whole extraction — a common bug is stopping early instead of skipping just the bad entry. For encoding conversion, after extraction, use `filepath.Walk` on the target dir, and for each regular file decode bytes from Windows-1252 using `golang.org/x/text/encoding/charmap.Windows1252.NewDecoder()` transform, then write the resulting UTF-8 bytes back over the file. For the manifest, compute `sha256.Sum256` over the final UTF-8 bytes (not the original cp1252 bytes) as you process each file, hex-encode lowercase, and key the JSON map by the path relative to targetDir with forward slashes (use `filepath.ToSlash` on `filepath.Rel` results). Initialize the Go module first (`go mod init`, `go get golang.org/x/text`), build, then actually run the binary against the given zip and inspect both `/home/user/manifest.json` and that no file leaked outside the target directory.
