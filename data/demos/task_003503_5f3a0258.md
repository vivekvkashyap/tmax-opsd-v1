Implement this as a single gawk program invoked from the bash wrapper script, since plain POSIX awk lacks the sorted-iteration primitives you need for determinism. Read the whole TSV first into a 2D associative array `edges[src][dst]=weight` (gawk true multidimensional arrays) plus a set of node names, then in an `END` block run the 1000-step loop. Call `srand(seed)` exactly once at the start (in a `BEGIN` block, using the seed passed via `-v seed=...`), and never reseed inside the loop. On each step, if the current node has outgoing edges, use `asorti()` to get destination names in alphabetical order, then call `rand()` exactly once per destination *in that sorted order* — this ordering is critical because `rand()` is a sequential PRNG stream, so processing edges in any other order (e.g. array traversal order, which gawk doesn't guarantee) produces a different, non-matching sequence. Track the max score with `>` (not `>=`) and only update `best` on strict improvement so alphabetically-earlier ties naturally win. If there are no outgoing edges, the node stays put but still counts as a visit for that step. Build the histogram with integer division (`int(count/10)`) over nodes sorted alphabetically via `asorti` on the visits array, and make the script executable with a `#!/usr/bin/awk` wrapper or `#!/bin/bash` calling `gawk`.
