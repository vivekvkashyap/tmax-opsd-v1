Build the Go service in stages, verifying each before moving on. First, parse `/app/data/historical.csv`, group amounts by category, and for each category run bootstrap resampling: draw 1000 samples (same size as that category's row count, sampled with replacement), compute each sample's mean, sort the 1000 means, and take the 2.5th/97.5th percentile positions as CI bounds — store them as strings in Redis under `ci_lower_<category>`/`ci_upper_<category>` before serving traffic. Then implement `/upload`: parse the multipart `file` field, validate every row strictly against the schema (column count, `timestamp` as int64, `amount` as float64) and return 406 on the first failure. If schema passes, compute per-category means of the upload, fetch the matching Redis bounds, and return 406 if any category is missing from Redis or its mean falls outside `[ci_lower, ci_upper]`; otherwise 200. Run the compiled binary in the background, confirm Nginx on 8080 proxies to it, then write `READY` to `/home/user/ready.txt` only once Redis is populated and the listener is up.
