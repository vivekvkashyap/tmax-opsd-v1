Use Python's `tarfile` module opened in streaming mode (`tarfile.open(path, 'r|gz')`) to iterate members without extracting the whole archive to disk, and pull out `meta.txt` via `extractfile()` to read the `project=<name>` line. Look up that project in `/home/user/rules.ini` with `configparser`. For `keep`, just copy the original `.tar.gz` bytes into curated using the atomic pattern below. For `extract_logs`, note a key gotcha: ZIP format needs random-access/seekable data (its central directory lives at the end of the file), so you can't stream into `zipfile.ZipFile` directly from a tar stream — instead read `payload.zip`'s bytes fully into a `io.BytesIO` buffer via the tar member's `extractfile().read()`, then open that buffer with `zipfile.ZipFile` and `.read('logs.tar.bz2')` to get the nested file's bytes. For `drop` or unknown projects, skip entirely and don't log them. For every file you write into `/home/user/curated/`, always write to a `.tmp` sibling path first (`open(tmp, 'wb')` / `shutil.copyfile` to tmp) then `os.rename`/`mv` it to the final name — never write directly to the final path. Append one log line per processed (non-dropped) artifact to `/home/user/curation.log` in the exact `[RULE] Processed <filename>` format.
