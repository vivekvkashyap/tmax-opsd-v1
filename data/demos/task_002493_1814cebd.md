Process the JSONL file as a true stream: open it and iterate line-by-line (e.g. `for line in f: json.loads(line)`), never `readlines()`/load-all-into-memory. Maintain a deque/list of recently seen (timestamp, bigram_set, id) tuples as the rolling window; before comparing a new message, evict entries whose timestamp is `<= current.timestamp - 60` (strictly greater than current-60, and strictly less than current, per the spec) — get this boundary exactly right, since off-by-one on either bound is the most common bug. For normalization: lowercase, apply `unicodedata.normalize('NFKD', text)`, then strip everything that isn't `a-z0-9` via a regex like `re.sub(r'[^a-z0-9]', '', s)` — this is what makes accented/Unicode lookalikes match their ASCII counterparts. Build bigrams as a set of adjacent 2-char substrings; empty/1-char normalized text yields an empty set, and two empty sets should score 0.0 by the spec (not division-by-zero). Compute Jaccard as `len(a&b)/len(a|b)`, track the best-scoring prior match (ties broken by smallest timestamp), and only append a CSV row when the best score is `>= 0.75`, formatting the score with exactly 3 decimals (e.g. f"{score:.3f}"). Add each message to the window after comparing it against the window, not before.
