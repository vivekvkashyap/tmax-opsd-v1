Start by running the init script and confirming MongoDB and the seed data are live before touching code. Locate the aggregation pipeline in `src/queries.rs`: the fix is to reorder stages so `$match` on `customer_id` is the FIRST stage (not last), so Mongo doesn't unwind/scan the whole 200k-document collection before filtering — a naive fix that only adds an index without reordering the pipeline will still be slow because `$unwind` on all documents happens before filtering. Replace any `format!("{{...}}")` + `bson::from_str(...)` string-built queries with the `doc! {}` macro for safe, fast BSON construction. Create a supporting index via `mongosh`: `db.orders.createIndex({customer_id: 1})` so the early `$match` can use an index scan instead of a collection scan. Rebuild with `cargo run --release` (not debug — release mode matters for the latency threshold) and keep the server running in the background. Verify iteratively: hit `/export?customer_id=...` with `curl` and time it, then run a small concurrent load test yourself before trusting the final P95 check.
