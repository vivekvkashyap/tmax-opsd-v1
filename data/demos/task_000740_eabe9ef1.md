Only then check `/etc/nginx/conf.d/api.conf` for its `proxy_pass` target — if it points to the wrong port/host than where Flask is actually listening, fix it and reload/restart nginx. Confirm `curl http://localhost:8080/score?...` succeeds end-to-end before writing the bash script.

For `pipeline.sh`: since both CSVs are sorted by `id`, use `join -t, -1 1 -2 1` on the input and `/app/data/embeddings.csv` to combine `id,text_data,cluster_id` per row (skip headers first with `tail -n +2` or filter them, then re-add the header to output). For each joined row, call the API with `curl -G --data-urlencode "cluster=$cluster_id" --data-urlencode "text=$text_data" http://localhost:8080/score`, parse the JSON status (e.g. with `jq -r .status` or a grep/sed extraction), and append the row to output only if status is "clean". Test against a few known evil/clean strings manually first.
