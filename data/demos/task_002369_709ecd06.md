Group surviving rows by `experiment_id`, then for each group sort by `epoch` ascending BEFORE computing the EMA — a common bug is computing EMA in file/line order instead of epoch order, which silently corrupts the result when logs interleave experiments or arrive out of order. Apply EMA with alpha=0.2, seeding EMA0 with the first (lowest-epoch) valid val_loss, then iterating forward. Round the final EMA per experiment to exactly 4 decimals and dump as a flat `{experiment_id: value}` JSON to `/home/user/artifacts/summary.json`. Wrap this in `run_pipeline.sh`: mkdir -p the artifacts dir, create/activate a venv at `/home/user/venv`, install deps, then invoke the python script — make the script executable and actually run it to produce the file.
