Build the pipeline as: bash script that sets up a venv, then delegates the actual CSV parsing/validation/tokenization to an embedded Python block (via `python3 <<'EOF' ... EOF` or a heredoc-written temp script) rather than trying to do CSV-quote-aware parsing with pure `awk`/`sed` — raw_text fields may contain commas inside quotes, and naive `cut -d,`/`awk -F,` splitting will silently corrupt rows. In Python, use the `csv` module to read rows correctly, then apply the schema filters in order: `item_id.isdigit()` (rejects `A3`), and a strict date regex like `^\d{4}-\d{2}-\d{2}$` (rejects `2023/02/10`) — drop any row failing either check, do not try to "fix" it. For tokenization: lowercase the raw_text, strip everything except `[a-z ]` with a regex substitution, collapse repeated spaces (`re.sub(r' +', ' ', ...)`), then `.strip()`. Write output with `csv.writer` (not manual string joins) so the header and rows are quoted properly, and separately count and write the valid row total. Actually run `pip install csvkit` inside the venv, not just note that you would.
