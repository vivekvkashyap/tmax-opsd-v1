Build this as a recursive resolver, not a single pass. First parse `app.json` with `jq` to get the initial `{plugin: constraint}` pairs, and load `registry.csv` (PluginName,Version,Requires,Sha256) into something awk/grep can filter by name. For a given plugin+minimum-version constraint: list all registry rows for that plugin, filter those whose version satisfies `>=`, sort candidates with `sort -t. -k1,1n -k2,2n -k3,3n` or `sort -V` descending, then walk from highest to lowest — for each candidate, verify `sha256sum downloads/<Name>-<Version>.tar.gz` matches the registry's Sha256 field; if it doesn't match, discard that version and try the next-highest candidate instead of failing outright. Once a version passes checksum, recursively resolve its `Requires` field (format `OtherPlugin>=Version`) the same way, propagating tightened constraints (e.g. a transitive dependency may demand a higher minimum than the top-level app.json did — always resolve against the strictest constraint seen for that plugin). Accumulate final resolved (name, version, sha) triples in an associative structure to avoid duplicate/conflicting entries, then emit them sorted alphabetically by name to `lockfile.csv`. Make the script executable and run it to produce the file.
