Build this bottom-up and verify each layer before adding the next, since "silent binding failures" usually mean a wrong bind address, a cert/key mismatch, or a supervisor config pointing at the wrong interpreter/path. First generate the self-signed cert/key with `openssl req -x509 -newkey rsa:2048 -nodes -keyout /home/user/key.pem -out /home/user/cert.pem -days 365 -subj "/CN=localhost"`. Write `exporter.py` using `http.server` + `ssl.SSLContext` (wrap the socket with `cert.pem`/`key.pem`) explicitly binding to `('127.0.0.1', 8443)` — test it standalone with `python3 exporter.py &` and `curl -k https://127.0.0.1:8443` before wrapping it in supervisor, so you know the server itself works. Have the handler respond to any GET with status 200, `Content-Type: application/json`, and the exact JSON body specified. Once confirmed, write `supervisord.conf` with a `[program:metric_exporter]` section: `command=python3 /home/user/exporter.py`, `autorestart=true`, `stdout_logfile=/home/user/exporter.log`, `stderr_logfile=/home/user/exporter.log` (or redirect stderr to stdout), plus the required `[supervisord]`/`[unix_http_server]` boilerplate, then launch with `supervisord -c /home/user/supervisord.conf` in the background and re-verify the curl check goes through supervisor-managed process.
