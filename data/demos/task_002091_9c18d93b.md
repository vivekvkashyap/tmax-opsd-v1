Parse graph.txt into an adjacency list, build the N x N transition matrix P where P(i,j) = 1/deg(i) if edge(i,j) exists (remember it's undirected, so each edge contributes to both P(i,j) and P(j,i)), then form the linear system (I - d*P^T)x = ((1-d)/N)*ones(N) with d=0.85 and solve it in Eigen using a decomposition like `.partialPivLu().solve(...)` or `.fullPivLu()` for a small dense N x N matrix. For the MCMC part, maintain current node state, use a fast uniform RNG (e.g. `std::mt19937` with `std::uniform_real_distribution`) inside a tight loop over 10,000,000 steps: with probability 0.85 move to a uniformly random neighbor of the current node (pick from that node's adjacency list, not the whole graph), otherwise jump to a uniformly random node in [0,N); increment a visit counter for whichever node you land on (or for the current node before/after the move — be consistent about what "visiting" means), then divide by total steps at the end to normalize into a probability distribution. Round both exact and empirical values to 4 decimals when writing results.csv, and sanity-check that the two columns agree within about 0.005 before finishing — if MCMC is too slow in a plain loop, compile with -O3 and avoid unnecessary allocations per step.
