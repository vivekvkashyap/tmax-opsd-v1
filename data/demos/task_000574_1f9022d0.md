Extract the tar.gz first (`tar -xzf legacy_configs.tar.gz`) to get system_master.ini, then convert its encoding explicitly with `iconv -f ISO-8859-1 -t UTF-8 system_master.ini > system_master_utf8.ini` — do not just copy the file or assume it's already UTF-8, since ISO-8859-1 bytes for accented characters will corrupt or fail if misdecoded. Write split_config.c to read the UTF-8 file line by line (fgets), and whenever a line begins exactly with the literal prefix "[MODULE_" (use strncmp, not strstr, so it must be the start of the line), close the current output file (if any) and open a new sequentially-numbered module_XX.ini (zero-padded, e.g. %02d) in /home/user/config_archive/, writing that trigger line as the first line of the new chunk; keep appending subsequent lines to the current chunk until the next trigger line or EOF. Compile with gcc and run it. Then for each generated module file, grep for a line that is exactly "status=active" (exact match, not just containing "active", to avoid matching "inactive" or "suspended") and `ln -s` its absolute path into /home/user/active_configs/ under the same filename. Finally, `cat` the symlinked files in alphabetical filename order into /home/user/active_master.ini, checking the result stays valid UTF-8.
