Build bottom-up: exporter binary, then service manager, then deploy script, testing each in isolation before wiring them together. In the C exporter, install a SIGTERM handler with `sigaction` (not naive `signal`, to avoid reentrancy issues) that sets a flag or directly calls `remove()` on the instance-specific metrics file and `exit(0)`; the main loop should write the file fresh each second with `fopen(..., "w")` and check the argv[1] instance_id is used consistently in both the filename and the payload string. In service_mgr.sh, `start` should launch the binary with `nohup ... &`, capture `$!` immediately into the pidfile under /home/user/run/; `stop` should read that pidfile, `kill -TERM <pid>`, then poll (e.g. loop checking `kill -0 $pid` until it fails or a timeout) before removing the pidfile — don't just kill and immediately delete, since the process needs time to clean up its own metrics file. In deploy.sh, compile with gcc, call service_mgr.sh restart for each instance, and busy-wait (with a timeout) on the expected .prom file's existence and content using a polling loop, not a fixed sleep, before declaring success.
