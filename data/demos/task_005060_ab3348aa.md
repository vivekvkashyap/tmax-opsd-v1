Start by transcribing `/app/operator_notes.wav` with a speech-to-text tool (e.g. `pip install openai-whisper` then run whisper on the file, or use any available STT CLI) and extract the maintenance-window start/end hours the operator states — don't hardcode a guess. Use that recovered window when writing `/home/user/classifier.py`. For the CSV logic: read the wide-format CSV, melt it to long format (timestamp, sensor, value) with pandas, sort by sensor and timestamp, then compute a rolling mean with window=5 per sensor. Flag a file as evil if (a) the rolling average changes by more than 50% between consecutive windows for any sensor, or (b) there exist exact duplicate rows/time-blocks whose timestamps fall inside the transcribed maintenance window (this catches ETL retry duplication specifically during that window, not duplicates elsewhere). Exit 1 on any such anomaly, exit 0 otherwise. Validate against `/app/data/sample_clean/` and `/app/data/sample_evil/` before trusting your thresholds — the hidden sets reuse the same rules, so tune conservatively rather than overfitting to the 5 samples.
