Parse `tracker_config.ini` with `configparser` to get `target_dir` and `allowed_exts`. Open the tar with `tarfile.open('/home/user/backup.tar', 'r|')` (streaming mode, not `'r'`) and iterate `for member in tar` without calling `.extractall()` or `.getmembers()` (which would buffer everything); use `tar.extractfile(member)` to get a file-like stream for reading contents on demand, only when needed. For each entry, compute the intended path as `os.path.normpath(os.path.join(target_dir, member.name))` — for names starting with `/`, `os.path.join` will actually just take the absolute name, so explicitly strip/reject a leading `/` and also verify the normalized result starts with `target_dir + os.sep` (not just startswith `target_dir`, which lets a sibling directory like `target_dirs/evil` slip through). Log traversal attempts to `security_alerts.log` using the exact `ALERT: Path traversal detected - <original_tar_name>` format with the original `member.name`, not the resolved path. For entries that pass validation and match one of `allowed_exts`, stream-read via `extractfile(member).read()`, compute `hashlib.sha256(data).hexdigest()`, and add to the manifest dict keyed by the safe absolute path, then dump as JSON.
