For transform.sh, use `jq` to parse the JSON robustly rather than regex/sed on raw JSON — read stdin into a variable, extract `msg_id`, `lang`, and `template` with `jq -r`. For substitution, scan the template for all `{key}` placeholders (e.g. with a `grep -oE` or a loop matching `\{[a-zA-Z0-9_]+\}`), and for each one look up `.vars[$key]` via `jq -r --arg k "$key" '.vars[$k] // ""'`. Treat both "key missing" and "value is empty string" as the same case: substitute the literal word `DEFAULT`. Do substitution with a safe string-replace loop (bash parameter expansion `${template//\{$key\}/$val}` or `jq`'s own string building) rather than naive `sed`, since values may contain characters that break sed. Preserve exact spacing/punctuation from the template — only the placeholders change. Output must be exactly `msg_id|lang|interpolated_template` with a trailing newline, nothing else printed to stdout.

For worker.sh: verify/start `redis-server` (check with `redis-cli ping`, start with `redis-server --daemonize yes` if not running), then loop `redis-cli BLPOP loc_jobs 0`, extract the JSON value (BLPOP returns key then value on separate lines), pipe it into `transform.sh`, parse the piped output to split on `|` (msg_id, lang, rest), and idempotently upsert via `redis-cli HSET loc_results "$msg_id" "$lang|$rest"` — HSET overwrites on retry so duplicates never accumulate. Launch worker.sh with `nohup ... &` and disown it so it survives your shell exiting.
