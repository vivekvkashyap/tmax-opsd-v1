Treat this as a graph-aggregation problem solvable with plain Unix pipelines, not a full language. First build an adjacency structure: an awk associative array mapping emp_id -> manager_id from reporting.csv. For each manager (any id appearing as a manager_id), compute direct_reports_count by counting rows where manager_id matches, and total_descendant_salary by doing a full downstream traversal (BFS/DFS) over the emp_id->manager_id edges reversed (manager_id -> list of emp_ids), summing employees.csv salaries for every node reachable, not just direct reports — this is the step naive attempts skip, undercounting salary to only direct reports. A clean approach: load both CSVs into awk arrays in one pass, build a children[] map, then for each manager do a recursive/iterative traversal (a stack-based loop works in awk) collecting all descendants. Join manager id to name/department from employees.csv. Finally sort numerically by manager_id (`sort -t, -k1,1n`) and prepend the exact header before writing to /home/user/manager_summary.csv.
