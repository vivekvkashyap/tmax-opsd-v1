Install `ffmpeg` and `tesseract-ocr` first. Extract frames at a rate that captures each distinct scroll position of the log (e.g. `ffmpeg -i deploy_logs.mp4 -vf fps=2 frame_%04d.png`), then OCR each frame with `tesseract`. Since the log scrolls, frames will overlap — dedupe consecutive near-identical lines (e.g. keep a running set of already-seen `BEGIN_TX...END_TX` blocks, or diff frame text against the previous frame) rather than concatenating every frame's OCR blindly, or you'll get duplicated/interleaved records. Write the reconstruction as a state machine that only starts a new record on `BEGIN_TX` and closes it on `END_TX`, tolerating OCR noise in labels (fuzzy-match `ACTION:`, `PATH:`, etc. with regex, and strip stray characters). For each parsed action, actually perform the filesystem operation under `/home/user/sys_root/` (mkdir -p as needed for CREATE_FILE with base64-decoded DATA, `ln -s` for symlinks, `ln` for hardlinks, `gzip` + `rm` for COMPRESS_STREAM). Finally walk the tree with `find | sort` and compute `sha256sum` per regular file, symlink target via `readlink`, and hardlink detection via matching inode/`stat -c %i`, writing the manifest in the required `<F|S|H> <path> <hash|target>` format sorted by path.
