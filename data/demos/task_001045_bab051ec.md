Design the C++ daemon around two listener threads sharing protected state. Use one thread (or accept loop) for the raw TCP socket on 9000 that reads newline-delimited records, and a separate thread running an httplib server on 9001 for `GET /stats`; protect the unique-count, duplicate-count, and seen-`record_id` set with a mutex since both threads touch them. Deduplication: keep a `std::unordered_set<std::string>` of seen record_ids for the daemon's lifetime — checking membership before writing, and counting a hit as a duplicate instead of writing it. Masking: replace only the last dotted octet of the IP with `XXX`, and replace everything before `@` in the email with `***` while keeping the domain intact — do this with simple string splitting, not regex overkill. For `/stats`, build the JSON body manually (`{"unique": N, "duplicates": M}`) and set `Content-Type: application/json`. For the cron piece: write `rotate.sh` to `cp` the log then truncate it with `> file` (not `rm`/recreate, which changes the inode), then install a crontab line `* * * * * /home/user/rotate.sh` for the current user via `crontab -l | { cat; echo "..."; } | crontab -`, and confirm the cron daemon is actually running.
