To get the `.text` section's virtual address, shell out from Rust via `std::process::Command::new("readelf").args(["-WS", "/home/user/evidence.elf"])`, capture stdout, and parse the row whose section-name column equals `.text` to pull its `Addr` field (a hex string) — avoid hand-parsing raw ELF headers unless you want the extra complexity, `readelf`'s wide `-S` output is simpler to grep/split reliably. Parse that hex string into a `u64`. Build the payload as exactly 72 bytes of `0x90` followed by the address's 8-byte little-endian encoding (`u64::to_le_bytes()`), and write it to payload.bin with `std::fs::write`. Compute SHA-256 over the payload bytes (e.g. the `sha2` crate's `Sha256::digest`) and format the digest as lowercase hex. Build the JSON manually or with `serde_json`, ensuring `text_address` is the address as lowercase hex prefixed with `0x` (no leading zeros beyond what readelf reports) and `payload_sha256` is the lowercase hex digest, then write audit_log.json. Run via `cargo run` to produce both output files.
