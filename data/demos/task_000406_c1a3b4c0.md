Structure this as three independent, testable stages in Go. First, parsing: read the whole log file and split on `\n\n` to get record blocks, then within each block extract the `File:` and `Status:` values (simple prefix matching/`strings.TrimPrefix` is enough — no need for regex); collect File paths only where Status is exactly "Pending", preserving log order. Second, integrity verification: for each candidate path, open it, wrap with `gzip.NewReader`, then `tar.NewReader` on that, and loop calling `tarReader.Next()` until you get `io.EOF` (success) or any other error (treat as corrupted and skip) — a plain text file masquerading as .tar.gz will fail at the gzip stage, which is exactly what you want. Third, the atomic locked update: open `state.json` with `os.OpenFile`, call `syscall.Flock(fd, syscall.LOCK_EX)`, unmarshal, append the valid pending paths to `valid_backups`, marshal, write to a sibling temp file (e.g. `state.json.tmp`) in the same directory, `os.Rename` it over the original, then `Flock(fd, syscall.LOCK_UN)` and close. Test each stage against the sample archives before combining.
