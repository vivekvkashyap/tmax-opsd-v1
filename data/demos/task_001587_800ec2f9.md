Design the C++ checker around two independent, clearly separated checks so failures aren't conflated. For the socket check, use `stat`/`lstat` on `<path>/app.sock` and test `S_ISSOCK(st.st_mode)` — a missing file and a regular file both count as SocketError, so handle "doesn't exist" and "wrong type" the same way. For the config check, parse `config.meta` line by line into key=value pairs and require `TZ` to be exactly `UTC` and `LANG` to be exactly `en_US.UTF-8`; missing file or any mismatch is ConfigError. To get the real target directory name for logs/backups, resolve the symlink argument with `realpath()` (or `readlink`) and take the basename — don't just use the literal path passed in, since it will always be `.../current`. On any failure, append the exact `ALERT: Target <name> failed. Reason: <SocketError|ConfigError>` line to `alerts.log`, then shell out (e.g. `system("tar -czf ...")`) to archive the deployment directory into `/home/user/backups/failed_<name>.tar.gz`. Compile with `g++ -std=c++17`, then manually run the tool against v2 (bad socket) and v3 (bad config) by flipping the `current` symlink between runs, checking output/log/backups after each.
