Build this as a pipeline of clearly separated bash stages, testing each stage's output before moving to the next. Extract the outer tar into `/home/user/processing` with `tar -xf`. Then, for each nested archive found (`find ... -name '*.tar.gz' -o -name '*.tar.bz2' -o -name '*.zip'`), test integrity *before* extracting: `gzip -t`/`tar -tzf` for `.tar.gz`, `bzip2 -t`/`tar -tjf` for `.tar.bz2`, `unzip -t` for `.zip` — check the exit code and skip (don't extract) any archive that fails its integrity test, since one is intentionally corrupted. Extract only the valid ones into `/home/user/extracted`. Use `find /home/user/extracted -type f \( -name '*.bin' -o -name '*.so' \)` to locate target binaries, and `cp` (not `mv`) each into a flat `/home/user/curated` directory using just the basename, so nested directory structure from different archives collapses without collision. Create the final tarball with `tar -czf /home/user/curated_artifacts.tar.gz -C /home/user curated` so the archive root contains the `curated/` directory. For the manifest, `cd` into `/home/user/curated` and run `sha256sum * | sort -k2 > /home/user/manifest.txt` so hashes and sorting exactly match what a verifier would recompute the same way.
