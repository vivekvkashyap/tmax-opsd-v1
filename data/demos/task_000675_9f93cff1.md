Two phases. Phase 1 (services): start Redis (`redis-server --daemonize yes`, default port 6379), then launch the Flask API at `/app/services/api/app.py` with the environment it expects — export `REDIS_HOST=127.0.0.1` and `REDIS_PORT=6379` before starting it (read app.py to confirm how it reads these). Run it in the background, then trigger tuning with `curl -X POST http://127.0.0.1:5000/tune`. The API writes the tuned parameters to Redis under keys `cv_alpha` and `cv_beta`; retrieve them with `redis-cli get cv_alpha` / `redis-cli get cv_beta`. Do not guess these floats — read whatever values the service actually stored. Phase 2 (C filter): write `/home/user/bayes_filter.c` reading doubles from stdin line by line (`while (scanf("%lf",&x)==1)`), computing `y=(x+alpha)/(beta+1.0)` with the exact retrieved alpha/beta, and printing `%.4f\n` per line. Compile with `gcc -O3 bayes_filter.c -o /home/user/bayes_filter`. Common traps: forgetting the `+1.0` in the denominator, using `float`/`%f` and losing precision, or hardcoding the parameters instead of reading them from Redis at build time.
