Design the C program around three separable concerns: file-format dispatch, UTF-8-aware character counting, and a per-server rolling window. Dispatch by checking the argument's suffix (`strstr`/`strrchr` for ".csv" vs ".tsv") and pick the delimiter accordingly, reading each file with `fgets`/`getline` and splitting on that delimiter (watch for a trailing `\r` or `\n` you must strip before using the last field). For UTF-8 character counting, don't use `strlen` — instead count bytes whose top two bits are not `10` (i.e. `(byte & 0xC0) != 0x80`), since continuation bytes shouldn't be counted as separate characters; iterate the raw byte string this way to get code-point count. For the rolling average, maintain a small per-server-name data structure (a fixed-size array of names each holding the last-up-to-3 values seen, updated as a circular buffer) — since server names repeat across both files, keep this state alive across all input files, not reset per-file, and preserve strict input order for both processing and rolling-average computation. Emit each output line as records are processed (not after aggregating everything), formatting the average with exactly 2 decimal places via `%.2f`.
