Do the filtering in bash for speed but the character-length math in Python for correctness — `awk`/`grep` can efficiently pre-filter rows where the ConfigKey column equals `MOTD`, but hand the surviving lines to Python for anything involving Unicode, since byte-counting in a shell pipeline will silently overcount multi-byte characters. In Python, read the TSV with explicit UTF-8 decoding, use `len()` on the decoded string (not `len(s.encode())`) to get true character counts, and group by Date to compute the per-day mean. Sort the resulting per-day averages by actual calendar date, not by first-appearance order in the file. Walk the sorted list computing `abs(avg[i] - avg[i-1])` for each consecutive pair, track the maximum along with which day (the later of the pair) it belongs to — a naive max over unsorted or unfiltered data will pick the wrong day. Round the winning difference to exactly 2 decimal places and write `YYYY-MM-DD,DiffValue` to the output file with no extra whitespace or trailing content, since the check does an exact substring match.
