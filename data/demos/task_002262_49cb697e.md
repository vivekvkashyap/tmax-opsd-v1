Build and test the C++ sanitizer standalone before touching the pipeline integration. Parse the binary stream record-by-record: read 4-byte magic (validate it's `0xDEADBEEF`), then a little-endian `uint32_t` payload size, then that many payload bytes — use `std::ifstream` in binary mode and read raw bytes, not text-mode line reading. Detect maliciousness by checking (a) the payload's *second* line (split on `\n`) equals `STATUS: FATAL_BLOAT`, and (b) whether the substring `[CORRUPT]` appears anywhere in the payload — drop the whole record if either is true, don't attempt to partially write it. For valid records implement RLE by scanning for runs of length >= 3 of the same character and emitting `~<char><count>`, leaving shorter runs untouched — test this against small strings by hand before trusting it. Write the `.zarc` output as the literal header `ZARC001\n` followed by binary length-prefixed compressed payloads. Only after `./sanitizer input output` works correctly on hand-crafted evil and clean samples should you edit `/app/archiver.sh` to pipe its Redis dump through the sanitizer into `/home/user/logs/archive/<timestamp>.zarc`.
