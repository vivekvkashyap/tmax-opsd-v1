For the video part: extract frames with `ffmpeg -i /app/backup_dash.mp4 -vf fps=10 frame_%03d.png` (or similar), then run `tesseract` on the frames around the 1-second mark (since the text scrolls, check several consecutive frames, not just one) to OCR the console error line. Read the exact archive name out of the recovered `FATAL: Disk space exhausted in /var/backups/<name>` text, then `mkdir -p /home/user/<name>` using that recovered value verbatim — do not guess or hardcode a name.

For the parser: model each FILEPATH as a node; symlinks (`S`) point to another node's path (which may be missing), files (`F`) are terminal. For every line, walk the chain of symlink targets using a visited-set keyed by path; if you revisit a path already in the current chain, it's a cycle → `BROKEN`; if a target path isn't found in the manifest at all, it's also `BROKEN`; if the chain terminates at an `F`, output that file's hash. Memoize resolutions (e.g., an associative array of path→result) so repeated targets aren't re-walked, since inputs can have up to 500 lines and will be fuzzed against a reference implementation many times — an unmemoized/naive approach risks both incorrect cycle handling and slow performance. Read manifest with `awk -F'|'` or a `while IFS='|' read` loop, then emit `FILEPATH|RESOLVED|HASH` or `FILEPATH|BROKEN` for every original line, sorted with `sort`.
