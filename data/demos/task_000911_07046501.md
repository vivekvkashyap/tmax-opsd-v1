Start by running `tesseract /app/architecture.png stdout` (try different `--psm` values if text is garbled) to recover the ACL group memberships and each group's target backend port — do not hardcode guessed port numbers. Establish the SSH tunnels as background local-forwards using the given key: `ssh -i /home/user/.ssh/id_rsa -f -N -L 8001:127.0.0.1:<admins_port> user@127.0.0.1 -p 2222` and similarly for 8002 with `<devs_port>`; verify with `ss -tln` that both local ports are actually listening before moving on, since a failed tunnel silently leaves the router unable to deliver. In mail_router.py, read exactly one line of JSON from stdin, parse the `sender` field, look it up against the ACL you extracted (build a dict from the OCR'd group lists), open a plain TCP socket to 127.0.0.1:8001 or 8002 accordingly, send the *exact* JSON string as received (don't re-serialize with different key ordering/whitespace, since the verifier does a substring match on the raw payload), close the socket, and exit 0. Unknown senders must exit 1 without opening any socket. Test end-to-end with `nc` before considering it done.
