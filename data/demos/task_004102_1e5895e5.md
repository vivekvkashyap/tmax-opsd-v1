Work in three stages, each verifying the previous. First, find the crash payload via static string extraction — `strings /app/crash.core | grep -i sync` to locate the exact text buffer the process was holding when it died, rather than single-stepping a stripped binary in gdb; you can corroborate with `gdb -c crash.core <binary>` for the crash frame, but the payload itself is plain text in the dump. Second, write `reproducer.cpp` as a minimal TCP client that connects to a host:port from argv and sends that exact payload followed by `\n`, so it reproduces the crash against the legacy binary on any port. Third, build `ts_server.cpp` as a blocking single-connection line-protocol server: accept once, loop reading lines, dispatch on the leading token (PUT/GET/SYNC), keep an in-memory map, and for SYNC compute the offset modulo 86400 using C++'s native `%` on a wide integer type (e.g. `long long`) so it never overflows like the legacy 32-bit downcast — verify your sign convention matches native `%` semantics rather than assuming Python-style modulo. Compile and leave it running in the background on 127.0.0.1:9000.
