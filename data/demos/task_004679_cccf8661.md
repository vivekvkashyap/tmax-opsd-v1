Model the CSV as a directed graph (adjacency list keyed by tx_id) and find nodes that lie on a cycle, not just nodes that can reach one. A node is part of a cycle iff it can reach itself by following directed edges — the cleanest correct approach is to compute strongly connected components (e.g. Tarjan's or Kosaraju's algorithm) and mark every node whose SCC has more than one member, plus any single-node SCC that has a self-loop edge (tx_id -> itself); nodes merely upstream of a cycle (pointing into it without being reachable back from it) must be excluded. Collect the qualifying tx_ids into a `HashSet`, convert to a sorted `Vec<i64>` ascending, then apply `.iter().skip(offset).take(limit)` for pagination. Parse the three positional CLI args with `std::env::args()`, read the CSV with a plain line-split (no header) using the `csv` crate or manual parsing, and write one ID per line to the output file with `std::fs::write`. Build with `cargo build --release` or `rustc`, then run with the exact given limit/offset before checking the output file.
