Build `gateway.sh` as an orchestrator plus a per-request proxy loop. Launch both backends (`python3 /app/aligner.py &` and `python3 /app/fft.py &`) first and give them a moment to bind, then implement an HTTP listener on 8080 — a simple approach is `socat TCP-LISTEN:8080,fork,reuseaddr EXEC:handler.sh` or a `nc -l` loop feeding into a handler function that parses the raw HTTP request to extract the method/path (`/analyze`) and body. For each request body, `curl -sw '%{http_code}' -X POST --data "$body" http://127.0.0.1:5000/align` and separately to `:5001/spectrum`, capturing both the response body and the HTTP status code (append `-w` and split on a delimiter, or use `-o`/`-w` to a temp variable) rather than assuming success. Check the FFT service's status code explicitly: if it's 500, build the fallback JSON `{"error": "near-singular sequence detected", "alignment": <alignment_json>}` using the alignment result you already fetched; otherwise merge both JSON bodies into `{"alignment":..., "spectrum":...}`. Always respond with `HTTP/1.1 200 OK` and correct `Content-Length`/headers regardless of which branch triggered, and make sure the whole script keeps running (don't let it exit after handling one request) — test manually with `curl -X POST -d "AAAAAAAA" http://127.0.0.1:8080/analyze` and a normal sequence to confirm both branches work.
