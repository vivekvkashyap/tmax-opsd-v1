In your `detector.c`, use `popen`/`system` (or `fork`+`exec`) to invoke the whisper.cpp CLI binary against the input WAV with the base.en model, capturing its transcript output; since adversarial files hide commands under noise, experiment with decoding parameters (larger beam size, e.g. `--beam-size 5`, and/or `--temperature 0` for deterministic decoding, possibly `--best-of` variants) until it reliably surfaces hidden text on files in `/app/corpus/evil/` — verify this empirically by running whisper.cpp manually on a few evil-corpus files first and reading the transcripts before wiring it into your C program. Then in C: lowercase the transcript, split on whitespace, for each token sum the ASCII codes of its characters to get a hash, look that hash up in a table you load from `/app/model_weights.csv` (unmatched tokens contribute 0), sum matched weights, and exit 1 if the sum exceeds 0.5, else exit 0. Compile with something like `gcc -O2 -o /home/user/detector /home/user/detector.c`, then batch-test against every file in both `/app/corpus/clean/` and `/app/corpus/evil/`, checking exit codes, and iterate on the whisper decoding flags if any evil file is misclassified due to a failed transcription.
