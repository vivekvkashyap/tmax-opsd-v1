Work in pure Bash/awk, no Python. First sort each CSV by timestamp with `sort -t, -k1,1n` (or `sort -n`), then join them on the timestamp column using `join -t, -1 1 -2 1` on the two sorted files to keep only timestamps present in both — this naturally gives you the chronologically sorted, inner-joined dataset in one step. Then pipe the joined output through a single `awk -F,` pass that tracks the previous row's cpu value in a variable (initialized empty/unset), and for every row after the first computes `cpu_increase = cpu - prev_cpu`; emit the row only when `cpu_increase >= 50 AND mem_usage >= 90`, formatted exactly as `timestamp,cpu_usage,mem_usage,cpu_increase`. Redirect this to `/home/user/anomalies.csv`. Common pitfalls: forgetting to sort both files identically before `join` (join requires sorted input on the join field), accidentally emitting a row for the very first line (there is no previous row to diff against, so skip it), and using string comparison instead of numeric comparison in awk (always treat the CSV fields as numbers, not text, when comparing thresholds).
