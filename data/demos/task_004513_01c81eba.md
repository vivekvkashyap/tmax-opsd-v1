Write the scanner logic before the routing logic, and test it against a known-safe and a known-malicious archive by hand first. To list paths without extracting, use `tar -tzf <file>` for tar.gz and `unzip -l <file>` for zip (parse the filename column out of unzip's listing with awk, since it has header/footer lines to skip). Classify a path as malicious if it starts with `/` (absolute) or contains `../` — check every line, not just the first, but log only the first offending path found. For archives that pass the malicious check, verify integrity with `tar -tzf` (already run) succeeding without error, or `unzip -t <file>` returning exit code 0; a nonzero exit or stderr output means CORRUPT. Route with `mv` into quarantine/processed as appropriate, and only extract SAFE archives (`tar -xzf` or `unzip`) into a subdirectory named after the archive with its extension stripped (careful: `.tar.gz` needs both suffixes removed, not just the last one). Append one strict-format log line per archive as you process each file rather than batching at the end, so a mid-script failure still leaves partial evidence.
