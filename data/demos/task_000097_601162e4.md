Build bottom-up and test each layer before moving to the next. In `trie.c`, implement a standard array-of-children (or linked list of children) trie node with an `is_end` marker; `trie_insert` walks/creates nodes character by character; `trie_check` walks the trie matching characters of `path` against trie edges and returns 1 as soon as it passes through any node marked `is_end` (that's what makes it a *prefix* match rather than requiring the whole trie string to be consumed) — this is the detail naive implementations miss, since checking only for full-string equality at the end will fail on paths like `/admin.html` that merely start with a blocked prefix `/admin`. Compile it as a position-independent shared library in the Makefile (`gcc -shared -fPIC -O2 -o libtrie.so trie.c`). In `proxy.py`, load the .so with `ctypes.CDLL`, declare `argtypes`/`restype` for each exported function (especially treating `void*` as `ctypes.c_void_p` and strings as `c_char_p`), populate the trie once from `/home/user/blocked.txt` at startup, and in the request handler call `trie_check` before deciding whether to return 403 directly or forward the GET (via `http.client` or `urllib.request`) to `127.0.0.1:9090`, relaying status/headers/body verbatim.
