Parse each `.wal` file as a simple line-oriented state machine rather than a single regex over the whole file: scan line by line, and when you see `[START]` begin buffering a candidate entry (LEVEL, PREFIX, PAYLOAD); only commit it as valid if you subsequently hit `[END]` before either the file ends or another `[START]` appears — if a new `[START]` or EOF arrives while a buffer is open and unterminated, discard that partial entry entirely (this is the key edge case: a trailing truncated record at the end of a file must be silently dropped, not counted or half-processed). For each valid, terminated entry, apply the filter from `config.json` (`LEVEL >= min_level` AND `PREFIX == allowed_prefix` exactly), and for entries passing, base64-decode the payload with `encoding/base64`. Compute the filename as `<PREFIX>_<md5-hex-of-decoded-bytes>.bin` using `crypto/md5`. For the atomic write, write the decoded bytes to a sibling temp file in `/home/user/repo/` (e.g. `<name>.bin.tmp`) with `os.WriteFile`, then `os.Rename` it to the final `.bin` name — never write directly to the final path. Finally write the count of successfully extracted files as a plain integer to `/home/user/summary.txt`.
