(1) Transcribe `/app/telemetry_alert.wav` with a Python speech tool (e.g. `whisper` or `SpeechRecognition`); the speech states repeated "Timestamp <unix>, value <metric>" pairs. (2) Read `/app/system_metrics.csv` (pandas) and melt the wide `server_A_metric/B/C` columns into long rows `timestamp, server_name, metric_value`; keep `server_name` as the original column name. (3) Concatenate the audio rows, sort by `timestamp` ascending. (4) Filter anomalies as strictly `metric_value > 90.0`. (5) Serve with Flask/FastAPI on `0.0.0.0:8080`, endpoint `GET /api/v1/anomalies`, enforcing header `Authorization: Bearer l10n-metrics-secret` (return 401 otherwise). Response JSON must be `{"anomalies": [ {timestamp, server_name, metric_value}, ... ]}` sorted by timestamp. Common pitfalls: emitting timestamps as strings instead of ints, off-by-one on the `>90` boundary, and letting the server block — launch it in the background so the verifier can query it.
