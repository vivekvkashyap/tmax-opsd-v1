Preprocess with shell tools before writing any C: filter the JSONL to only `"interaction_type": "follows"` lines (`grep '"interaction_type": "follows"'` or a small `jq`/`awk` pass), then extract just source and target node names into a simple whitespace- or comma-separated edge list file that's trivial for C to parse (fixed-format lines avoid writing a JSON parser in C). In the C program: read the edge list, intern node names into an array with a lookup (string comparison or a simple hash map) so you can represent the graph as adjacency by integer index, then for directed triangle detection iterate every edge A->B, then every edge B->C, then check whether an edge C->A exists (using an adjacency set/matrix for O(1) lookup rather than rescanning the edge list) — this avoids the O(n^3) naive triple loop over all node combinations. For each found triangle, rotate its three nodes so the lexicographically smallest name comes first (compare the three rotations, keep smallest), then dedupe identical normalized triangles (a triangle can be found from any of its three starting edges), sort the deduped set lexicographically by (node1, node2, node3), and write the first 5 to CSV with no header.
