Handle the audio decoding and graph query as two independent, verifiable sub-problems before wiring up the server. For DTMF: load `/app/endpoints.wav` with `scipy.io.wavfile.read`, isolate each tone segment (split on near-silence), and for each segment run an FFT (or Goertzel algorithm) to find the two dominant frequency peaks, then match those peak pairs against the standard DTMF frequency table (low group ~697/770/852/941 Hz, high group ~1209/1336/1477 Hz) to recover each digit — don't assume the digits, derive them from the spectrum. Use those two recovered digits as source/target node IDs. For the path query, explicitly avoid `vw_edges` (it's an intentional cross-join trap producing wrong/exponential results) — instead read rows straight from the `edges` table into Python, build an adjacency list, and run BFS/Dijkstra (edges appear unweighted here, so BFS suffices) to get the shortest path as a list of node ids from source to target. Finally start a plain Python `http.server`-based (or similar minimal) HTTP server bound exactly to `127.0.0.1:8080`, handling `GET /path` by returning `200` with JSON `{"path": [...]}` built from your computed path, and keep the process running (background it) so the verifier can hit it.
