Design the SQL first in the `sqlite3` CLI against `/home/user/audit.db` before embedding it in C — iterating on raw SQL is much faster than debugging C+SQLite together. Use a recursive CTE seeded with the given manager_id (`SELECT emp_id FROM employees WHERE emp_id = ?1 UNION ALL SELECT e.emp_id FROM employees e JOIN subs s ON e.manager_id = s.emp_id`) to get the manager plus all direct/indirect subordinates, including the manager themselves. Join that against `access_logs`, `GROUP BY emp_id, resource_id` summing `bytes_transferred`, then wrap in an outer query applying `ROW_NUMBER() OVER (PARTITION BY resource_id ORDER BY total_bytes DESC, emp_id ASC)` as rank, filtering `HAVING total_bytes > ?2` before ranking (apply the threshold filter on the aggregated subquery, not after ranking, or ranks will be wrong since filtered-out rows would otherwise occupy rank slots). Final `ORDER BY resource_id ASC, rank ASC`. Once the SQL is validated manually against the known dataset, port it into C using `sqlite3_prepare_v2` with two `?` placeholders bound via `sqlite3_bind_int`, iterate with `sqlite3_step`, and write each row plus a header to `/home/user/audit_results.csv` via plain `fprintf`.
