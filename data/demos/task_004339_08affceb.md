This is a wide-to-long reshape plus forward-fill gap-filling, implemented in C with careful UTF-8 handling. Parse the CSV header to find column offsets rather than hardcoding, and read each row into per-station (Temp, Status) pairs, treating blank fields as "missing" markers (not "0"). Because Status fields contain multi-byte UTF-8 (including a 4-byte emoji), never index or copy them byte-by-byte assuming ASCII — read each field as a raw byte string up to the delimiter/newline and copy it verbatim with memcpy/strcpy-style whole-field copies, never truncating mid-codepoint. Determine min/max Hour present in the input, then iterate every integer hour in that inclusive range. For each station at each hour: if that exact hour existed in the input use its value directly (filling only the missing sub-field via last-known-good), otherwise carry forward the most recent prior known value for both Temp and Status. Maintain the "last known good" Temp/Status independently per station as you scan hours in order, updating it only when an actual (non-blank) value is seen at that hour. Format Temp with `%.1f`, and emit rows sorted by Hour ascending then Station A-before-B. Verify byte-for-byte against the sample output before finishing, since the grading is a strict diff.
