This task chains a recovered secret into a fuzz-tested algorithm, so precision matters more than cleverness. First recover SECRET_KEY: use ffmpeg to extract per-frame data (e.g. `ffmpeg -i /app/ui_test.mp4 -vf "select='1',showinfo" ...` or decode frames and check pixel luminance) and count frames where >=98% of pixels have luminance below 32 — verify your count is stable/reproducible before hardcoding it into `sign_build.sh`, since a wrong key fails every fuzz case. In `sign_build.sh`, process the input string byte-by-byte: for each char, `printf '%d' "'$c"` or a loop with `od`/`awk` to get its ASCII value, XOR against SECRET_KEY, format as 2-digit uppercase hex (`printf '%02X'`), concatenate; separately XOR-accumulate all original ASCII values for the parity byte and append its 2-digit hex at the end — be careful with Bash arithmetic XOR (`$((a ^ b))`) and zero-padding for single-digit hex. Since the verifier fuzzes with millions of random alphanumeric strings against a hidden oracle, test your script against many self-generated random strings and hand-verified XOR math before moving on. For the netcat HTTP server, parse the request line's query string manually since raw `nc` gives no built-in HTTP parsing. Finally configure nginx's `default` site to listen on 8080 and reverse-proxy `/api/` to `http://127.0.0.1:9090/` with the path stripped, then make sure both nginx and your `api.sh` are actually running in the background before finishing.
