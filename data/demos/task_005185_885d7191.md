Build the three pieces in order and test each in isolation before wiring them into the daemon. For the logrotate config, write standard directives (size, rotate count, compress, missingok, nomail) targeting the glob path; since you lack root, logrotate must always be invoked with an explicit `-s <statefile>` argument rather than relying on the default `/var/lib/logrotate/status`. For the storage script, compute directory size with `du -sm` (which already rounds to whole MB) rather than hand-rolling byte math, compare strictly greater than the threshold, and pipe a properly formatted `Subject:`+blank-line+body email into the mock sendmail binary exactly as instructed. For the daemon script, call logrotate with your config and state file first, then the storage script, in that order. Before trusting it, manually create oversized dummy log files under the logs directory and run the daemon yourself to confirm rotation actually compresses/renames files and that the alert email lands correctly in the outbox — logrotate silently no-ops if the state file or permissions are wrong, so don't assume it worked without checking the resulting files.
