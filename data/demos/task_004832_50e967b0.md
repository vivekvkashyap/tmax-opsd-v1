Split into the proxy/mTLS fix and the filter tool, and validate each independently. For Nginx: edit `/app/nginx/nginx.conf` so the 8080 server block does `proxy_pass https://127.0.0.1:8443;` and, since the Go API requires mutual TLS, add `proxy_ssl_certificate`/`proxy_ssl_certificate_key` pointing at the provided client cert/key so Nginx authenticates itself as a client, plus `proxy_ssl_trusted_certificate` (CA cert) with `proxy_ssl_verify on` to validate the API's server cert — a plain reverse proxy directive without these will fail the TLS handshake even though the ports are reachable. Start with `nginx -c /app/nginx/nginx.conf` and confirm `curl http://127.0.0.1:8080/ping` returns the expected status body before moving on. For `/app/filter.go`: read JSONL from stdin line by line, url-decode the `request_payload` field's string value, then case-insensitively check the decoded text for path-traversal markers (`../` or `..\`) or the exact substrings `UNION SELECT`/`1=1` (case-insensitive) — if any match, drop the line entirely (print nothing, don't error out); otherwise apply a regex substitution on `response_body` for the pattern `CRED-[A-Z0-9]{4}-[A-Z0-9]{4}` replacing with `CRED-XXXX-XXXX`, then re-marshal and print the JSON line. Skip (don't crash on) malformed input JSON. Test both directions yourself with a few hand-crafted lines through `go run /app/filter.go` before trusting it against the corpora.
