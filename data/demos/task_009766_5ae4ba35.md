Build the decoder as a straightforward byte-pair loop: read the file in binary, iterate two bytes at a time (count byte, char byte), and write `count` copies of the char to an output buffer — handle EOF/odd-length defensively. For atomicity, do NOT write directly to the target path: use `mkstemp`/a `tmp_XXXXXX` name in the same directory as the destination (same filesystem, so `rename()` is atomic), write and `fclose()` the temp file fully, then call `rename(tmpname, output_path)` — writing to a temp file in a different directory or skipping the fsync/close-before-rename ordering is the common bug. For the file-selection step, use `find /home/user/datasets -name '*.rle' -mtime -7` (or compare `stat` mtime against `now - 7*86400` in your driver script) rather than eyeballing dates — mtime-based filters are easy to get off-by-one on, so double check the boundary with `-mtime -7` semantics. Drive the decoder over each matching file with a shell loop, capture the produced basenames, then `sort` them into `processed_log.txt`.
