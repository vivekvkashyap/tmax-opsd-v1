This is a two-part inverse-problem task: first empirically recover the hidden driving frequency omega from a noisy black-box binary, then write a separate deterministic script that reproduces the noiseless expectation using that recovered omega — don't hardcode any omega you haven't actually derived from data. To estimate omega: build a few small test graphs (a single node, a small chain/star), run `/app/simulate_network` on each many times (dozens of runs) to get many noisy x_0(t) time series, then either average the time series across runs (noise cancels, revealing the underlying sin(omega t) drive) or average their FFT magnitude spectra and take the argmax frequency bin (excluding t=0/DC) — averaging spectra is usually more robust to phase-dependent cancellation than time-domain averaging. Convert the FFT bin index to angular frequency using the known dt=0.01 and N=1000, and round to 3 decimals; sanity-check by comparing against a couple of "nice" candidate values. For the deterministic re-implementation, note the SDE has NO noise term when you set sigma=0 (pure Euler forward integration of drift only): x_{t+dt} = x_t + dt*(-L x_t + sin(omega*t)*e_0), computed for exactly 1000 steps of dt=0.01 starting from all-zero. Read the graph edge list, build the Laplacian L = D - A yourself (don't assume a library is available), and verify your script's output against your own multi-graph testing before trusting it on the hidden validation graph.
