Sequence the setup so each layer can be tested in isolation. Create the five directories with `mkdir -p`, append `export ALERT_DIR=...` / `export ALERT_PORT=8080` to `~/.bashrc`, and export them in the current shell so later spawned processes inherit them. For `alert_daemon.c`: read env vars with `getenv()`, bind a TCP listener on localhost at the given port, and loop `accept()`ing connections; per connection `read()` the request, check it's exactly `TRIGGER\n`, and if so append `ALERT_FIRED\n` to `$ALERT_DIR/alerts.log` (path built from the env var, not hardcoded) before replying `OK\n` and closing — keep looping around `accept()`, don't exit after one client. Compile to `/home/user/bin/alert_daemon`. Generate a self-signed cert/key non-interactively with `openssl req -x509 -newkey rsa:2048 -nodes -keyout server.key -out server.crt -days 365 -subj "/CN=localhost"`, then write `serve_logs.sh` that backgrounds `openssl s_server` on port 8443 with that cert/key, serving the alerts log. `backup.sh` should `tar -czf /home/user/backups/alerts_backup.tar.gz -C <dir> alerts.log`. Finally start the daemon, fire a real `TRIGGER\n` via `nc`, run the backup script, and leave the daemon running.
