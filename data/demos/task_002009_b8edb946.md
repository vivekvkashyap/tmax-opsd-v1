Start by reading `/app/archive_spec.png` closely to recover the exact binary layout — magic bytes, endianness, field widths and order for the header and each entry; transcribe precisely, since one off-by-one in field size or byte order breaks every test case. Scaffold with `cargo new barc2tar` and add the `tar` crate. Design the parser as a streaming state machine over `std::io::stdin().lock()`: read the fixed-size header, validate magic bytes (exit 1 with an stderr message on mismatch), read the entry count, then per entry read the filename and size fields and stream exactly that many payload bytes into a `tar::Builder` wrapping `std::io::stdout().lock()` — use `Builder::append_data` with a populated `Header`, feeding a bounded reader (`Read::take(size)`) rather than buffering the whole payload. Treat any short read before the expected byte count as corruption and exit 1 with a descriptive stderr message. Build with `cargo build --release` and manually test round-tripping a small hand-crafted BARC file before trusting the fuzz comparison.
