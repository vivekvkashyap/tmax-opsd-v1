This is a topological sort with weight-based tie-breaking (a variant of Kahn's algorithm), not a plain DFS-based topo sort — implement it that way. Load the JSON with `jq` and build two structures: a reverse adjacency/in-degree count (how many other libraries list this one as a dependency, since "depends on" means the dependency must come *after* the dependent per the task's ordering rule) and a weight lookup. Initialize a ready-set of all libraries currently at in-degree 0. At each step, among the ready-set, select the library with the highest weight (not first-encountered or alphabetical — read the weight field), append it to the output order, then decrement the in-degree of everything IT depends on, adding any that newly reach 0 into the ready-set. Repeat until all libraries are placed. Be careful about the direction of the graph: "A depends on B" means A must be output before B, so you're effectively doing a reverse topological sort relative to the natural dependency-edge direction — track in-degree as "number of not-yet-output dependents," not "number of dependencies."
