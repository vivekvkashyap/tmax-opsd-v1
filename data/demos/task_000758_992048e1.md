Parse the CSV yourself in C (simple split on commas per line, skipping the header), treating the literal string "NULL" in the parent_id field as a SQL NULL by binding SQLITE_NULL rather than inserting the text "NULL". After inserting all rows, run `CREATE INDEX idx_parent ON bom(parent_id);` before your recursive query so the index is actually used — creating it after the query runs would defeat the purpose. Use `sqlite3_exec` with `EXPLAIN QUERY PLAN <your recursive query>` and dump the returned rows/text to query_plan.txt; verify manually that the plan text mentions using the index (adjust query shape, e.g. join order or WHERE clause on parent_id, if it doesn't). For the cost aggregation, write a `WITH RECURSIVE` CTE that walks from part_id=1 down through children via parent_id, then aggregate bottom-up (recursive queries in SQLite naturally build the row set top-down, so compute the accumulated cost either via a second aggregation pass grouping by depth, or restructure as a recursive CTE that sums as it goes). Build the final row with `json_object('top_part_id', ..., 'total_assembly_cost', ...)` and write the single returned string exactly to bom_result.json. Compile with `gcc analyze_bom.c -o analyze_bom -lsqlite3` and run it to produce both output files.
