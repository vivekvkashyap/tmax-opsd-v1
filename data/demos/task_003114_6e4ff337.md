Build this as a straightforward stateful stream processor in C++; the subtlety is entirely in the rate-limit window semantics, not the crypto/encoding.

1. Set up `/home/user/Makefile` with a `default` target invoking `g++ -std=c++17 api_tester.cpp -o api_tester -I.` (json.hpp is header-only, already provided).
2. Parse `/home/user/requests.json` with the provided nlohmann/json header. Process requests in file order (already chronological).
3. For rate limiting, maintain a per-`user_id` sliding window: keep a deque/list of accepted timestamps for that user, drop entries older than `timestamp - 10` (strictly, only count timestamps within the last 10 seconds inclusive of the boundary as described in the example), and ACCEPT only if fewer than 2 timestamps remain in that window; otherwise RATE_LIMITED. Re-verify against the worked example in the description (100,105,109→ACCEPT,ACCEPT,LIMIT; 111→ACCEPT) to get the window boundary exactly right.
4. For accepted requests: base64-decode `payload`, XOR every byte with `0x42`, then hex-encode (lowercase, no separators) the result.
5. Write one JSON object per line to `/home/user/output.jsonl`; omit `processed_payload` entirely (not null) when status is RATE_LIMITED. Build with `make` and run `./api_tester` to produce the file.
