Solve this as a single streaming pipeline, not extract-then-process: use `tar -xOzf /home/user/incoming_backup.tar.gz data.json` to stream `data.json`'s contents to stdout without writing it to disk, pipe that into `jq -r '.[] | [.server_id, .timestamp, .backup_size] | @csv'` to get CSV rows (note `@csv` quotes fields, so strip the quotes with `tr -d '"'` since the target fields don't need quoting), and pipe the result into the locked append. For the locking requirement, use a subshell that opens fd 200 on the target file and takes the lock before writing, e.g. `( flock -x 200; cat >> /home/user/backup_summary.csv ) 200<>/home/user/backup_summary.csv` (the `<>` open mode is required so the fd exists for `flock` even though you're only appending). Do not write a plain `>>` without the flock wrapper — that defeats the whole point of the locking requirement.
