Structure the proxy as: accept on 8080 (use `fork()` or a `select`/`poll` loop per connection for concurrency), read and parse the HTTP request line and headers, find the `Cookie:` header and extract the `SecureSession=<hex>` value. Hex-decode it, split into the first 16 bytes (IV) and the rest (ciphertext), then use OpenSSL's EVP interface (`EVP_DecryptInit_ex` with `EVP_aes_128_cbc()`, the fixed 16-byte key, and that IV) to decrypt with PKCS#7 padding — a decrypt/padding failure means reject. Parse the resulting plaintext as `user_id|role|timestamp` by splitting on `|`; treat any parse failure as invalid. Apply the four rejection rules in order (missing cookie, decrypt/format failure, role==admin with user_id!=1000, timestamp>1800000000), returning a literal `HTTP/1.1 403 Forbidden\r\n\r\n` response and closing the connection on any hit. For valid requests, open a fresh TCP connection to 127.0.0.1:8081, forward the original raw request bytes unchanged, relay the backend's response back verbatim, then close. Compile with `-lcrypto`, run it in the background, and manually test a few forged/valid cookies before considering it done.
