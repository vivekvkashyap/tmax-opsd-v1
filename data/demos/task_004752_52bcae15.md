Start by reproducing the build failure: `cd /app/freq_optimizer-1.0 && pip install .` and read the linker error — it will complain about undefined references to math functions (exp, log, etc.) coming from the C extension. Open the `Makefile` and check the compile/link command; a missing `-lm` flag is a classic cause of exactly this symptom (the .so links but leaves libm symbols unresolved at import time). Add `-lm` to the compilation/link line, rerun `pip install .`, and smoke-test with `python -c "import freq_optimizer; freq_optimizer.denoise([0.0, 1e10, 1.0])"` to confirm no NaN/OverflowError before moving on.
