Build this as a single idempotent Bash script rather than doing steps by hand. Extract the tarball, then use `find` with case-insensitive matching (e.g. `find . -iregex '.*\.\(txt\|text\)$'`) to catch `.TXT`/`.Txt`/`.text` variants — a naive `*.txt` glob will silently miss the mixed-case ones. For each match, pull the digits out of the filename with a regex (e.g. `grep -oE '[0-9]+'`) and rename via `cp` to `doc_<digits>.md`, preserving the digits exactly as found (don't zero-pad or strip leading zeros — `04` must become `doc_04`, not `doc_4`). Append the literal two-line suffix (a real newline then `[END OF DOCUMENT]`) to every resulting `.md` with `printf` or `echo`, not a substring replace. Separately, find all `.b64` files and decode each with `base64 -d`, writing binary output to a same-named `.bin` file. Finally `tar -czf` the processed directory's *contents* (not the directory itself) — use `-C /home/user/docs_processed .` or `tar -czf ... -C dir $(ls dir)` so paths inside the archive have no leading `docs_processed/` component.
