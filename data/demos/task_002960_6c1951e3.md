Build this as a single pipeline of standard shell utilities rather than a script that writes intermediate files. Use `find /home/user/dataset_raw/ -type f \( -name '*.iso' -o -name '*.u16' \) | sort` to get files in strict alphabetical order by full path — sorting matters because merge order must be deterministic. Loop over the sorted list, dispatching each file to `iconv -f ISO-8859-1 -t UTF-8` or `iconv -f UTF-16LE -t UTF-8` based on its extension, and pipe all the converted output into one continuous stream. Feed that stream directly into `split -l 100 - /home/user/tmp_processing/clean_data_` so chunks land in the temporary directory first (this is the atomicity requirement — never split straight into the final destination, since a reader could see a partially-written directory mid-operation). Only after `split` fully completes, `mv /home/user/tmp_processing/clean_data_* /home/user/dataset_clean/` to publish the chunks atomically, then report the count of moved files. Watch for common mistakes: converting files out of alphabetical order, forgetting `-l 100` produces alphabetic suffixes by default, and leaving stray files behind in the tmp directory.
