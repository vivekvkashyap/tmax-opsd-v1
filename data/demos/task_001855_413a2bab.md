This is a multi-stage pipeline: decode audio, look up data, run analysis, then serve results over HTTP with auth gating on the decoded value. For DTMF decoding, don't guess — use a proper approach: read the WAV samples (e.g. with Python's `wave`/`scipy` or `sox`), apply a sliding-window FFT/Goertzel filter over each tone segment, and match detected frequency pairs against the standard DTMF frequency table (697/770/852/941 Hz rows x 1209/1336/1477/1633 Hz columns) to recover each digit — do this for the whole file to get the full digit sequence. Use that recovered sequence (not a guess) as the `REFERENCE_ID` for both the FASTA header lookup and the Bearer token. Grep the FASTA file for the exact header `>seq_<REFERENCE_ID>` and take the sequence on the following line(s). Compile the aligner with `gcc aligner.c -o /app/aligner -lm` since it needs libm. Run the aligner against every query in queries.fasta, parse the printed score for each, and keep the header with the max score. Finally implement the HTTP server (e.g. Python `http.server` or a small Flask app) bound exactly to 127.0.0.1:9090, checking the Authorization header before returning the plain-text `header:score` body, and leave it running as a background process.
