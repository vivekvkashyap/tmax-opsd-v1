Build bottom-up: C++ library, then Makefile linking it into Go via Cgo, then the service composition. Write `sanitiser.cpp` with `extern "C" bool is_safe_payload(const char* input)` that walks the null-terminated string once checking each character against the forbidden set `| ; & $ > < `` (backtick)` — return false on the first match found, true otherwise — and write a matching `sanitiser.h` declaration since Cgo needs a real header to include. In the Makefile, compile the shared library with `g++ -shared -fPIC -o build/libsanitiser.so src/sanitiser.cpp`, then build the Go gateway with `CGO_LDFLAGS="-L$(PWD)/build -lsanitiser" CGO_CFLAGS="-I$(PWD)/src" go build -o build/gateway src/gateway.go` — getting these include/link paths wrong is the most common failure point, so test the shared lib loads correctly with `ldd build/gateway` before moving on. For `start.sh`, background redis-server, then the Flask backend, then the gateway with `LD_LIBRARY_PATH=$(PWD)/build` so the dynamic linker can find `libsanitiser.so` at runtime (this is the second common failure point — a correctly built binary that can't find its .so at launch), with short sleeps between each to let ports bind. Validate end-to-end manually with curl against both a clean and an injection-laced payload before trusting the hidden corpora.
