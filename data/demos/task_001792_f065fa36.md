Build the pipeline in clearly testable stages before wiring up the web framework. First write the cleaning function standalone: load the uploaded CSV with pandas, and for each of the 5 columns compute `median(skipna=True)`, round it to the nearest integer using banker's rounding (Python's built-in `round()` or `np.round` do half-to-even by default — do NOT use `int(x+0.5)` which rounds half-up), fill NaNs with that rounded median, then cast the whole column to `int64` (only after imputing, since int columns can't hold NaN). Second, write the cleaned 5 columns as a headless CSV to a temp file and shell out to `/app/prop_scorer <path>` via `subprocess.run`, capturing stdout and parsing one float per line as `prop_score`, in the same row order as the input. Third, compute `scipy.stats.pearsonr` between cleaned `f3` and `prop_score`; separately, standardize the 5 int columns with `sklearn.preprocessing.StandardScaler` (sample std, ddof handling matters), fit `sklearn.decomposition.PCA(n_components=1)`, and get PC1 values; compute the 95% CI for the mean of PC1 as `mean ± 1.96 * (std/sqrt(n))` using sample std. Wire all this into a Flask/FastAPI `POST /api/analyze` endpoint on `127.0.0.1:8000` accepting `multipart/form-data` with field `file`, returning the three values rounded to 4 decimals, and run the server in the background so it stays listening.
