Work through this as independent, testable stages rather than one big script. Compile the server first (`g++ /home/user/server.cpp -o /home/user/server`) and confirm it runs and answers on port 8080 with plain `curl` before adding TLS. Generate the cert/key with a single non-interactive openssl command that sets CN=localhost, e.g. `openssl req -x509 -newkey rsa:2048 -nodes -keyout /home/user/key.pem -out /home/user/cert.pem -days 365 -subj "/CN=localhost"` — the `-subj` flag avoids interactive prompts and the `-nodes` flag avoids an encrypted key that would block automated startup. Start the server in the background with output redirected (`/home/user/server > /home/user/app.log 2>&1 &`), then start socat as a TLS listener forwarding to the plain server: `socat openssl-listen:8443,cert=/home/user/cert.pem,key=/home/user/key.pem,verify=0,fork tcp:127.0.0.1:8080 &` — note the combined cert+key requirement of socat's openssl-listen address (some socat builds want a single PEM with both cert and key concatenated, so test with `curl -k https://127.0.0.1:8443` and fix accordingly if it fails). Finally write the logrotate config with `size 1k`, `rotate 3`, `compress`, and `missingok` directives, and leave all background processes running rather than exiting them.
