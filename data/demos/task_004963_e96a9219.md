Run `tesseract /app/system_tag.png stdout` (or equivalent OCR) and strip whitespace/newlines from the result to get the Zone ID — don't hardcode a guess, verify by eyeballing the image too if OCR output looks noisy. Use that extracted value to build a glob like `raw_<ZoneID>_*.log` under /app/logs/ and rename only matches (e.g. with a loop over `mv "$f" "${f/raw_/processed_}"`), leaving files for other zones untouched. Archive with `tar -czf /app/archive_<ZoneID>.tar.gz -C /app/logs processed_<ZoneID>_*.log` so paths inside the tarball are relative, not absolute. For the HTTP server, use Python's http.server or a small socketserver-based handler bound to 127.0.0.1:8888: GET /info returns a static-ish JSON built from the zone id and the count you archived; POST /register must read the JSON body, then update /app/registry.json safely under concurrent load — open the file, take an exclusive `fcntl.flock(fd, LOCK_EX)`, read-modify-append the array, write the full result to a sibling temp file, `os.replace()` it over the target (atomic on the same filesystem), then release the lock. Test the concurrency path yourself by firing several parallel curl POSTs and confirming no entries are dropped or corrupted before considering it done.
