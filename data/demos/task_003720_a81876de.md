Build the virtual environment first with `python3 -m venv /home/user/perf_env`, then install numpy and scipy using that venv's own pip (e.g. `/home/user/perf_env/bin/pip install numpy scipy`) so the verifier can confirm the env was actually used — do not just install into the system/global Python. Write `analyze_perf.py` to load both CSVs (a single float column, no header — `numpy.loadtxt` works), compute `scipy.stats.wasserstein_distance(v1, v2)` and the p-value from `scipy.stats.ks_2samp(v1, v2)`, round both to 4 decimal places, and dump them to `/home/user/perf_report.json` with exactly the two keys `wasserstein_distance` and `ks_p_value`. Run the script using the venv's interpreter, not the system one. Common pitfalls: computing a one-sample KS test or comparing against a theoretical distribution instead of the proper 2-sample test; rounding only for display but writing unrounded floats to JSON; or forgetting scipy's `ks_2samp` returns a tuple/object where the p-value is a named field, not the statistic.
