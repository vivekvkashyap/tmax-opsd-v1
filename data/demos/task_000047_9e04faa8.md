Write a Go binary reader over `dataset.bin`: verify the 4-byte `DATA` magic, then loop reading a little-endian uint16 path length, that many path bytes, a little-endian uint32 content length, then that many content bytes, until EOF. For each record, compute the destination by joining the target root with the record's path via `filepath.Join`, then `filepath.Clean` it and get its absolute form with `filepath.Abs`; the safe check is to confirm the cleaned absolute destination has the target root (also made absolute) as a strict path prefix (compare with a trailing separator, not a naive `strings.HasPrefix` on raw strings, which can be fooled by sibling directories sharing a prefix like `extracted2`). If it fails that check, append the *original, unmodified* path string from the archive (not the cleaned one) to `malicious.log`. If it passes, `os.MkdirAll` the parent directories and write the content bytes to the resolved path. Watch out for paths like `nested/dir/../safe.txt` — these resolve safely to inside the target after cleaning and must be extracted, not rejected, since `..` alone doesn't mean escape.
