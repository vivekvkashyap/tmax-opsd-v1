This is best done with awk (or python inside the shell script) since it needs per-region stateful aggregation. Steps: (1) parse each row, compute `minute_epoch = epoch_sec - (epoch_sec % 60)`; (2) group by (region, minute_epoch), accumulating a running sum/count for latency and a running character count for error_message — count UTF-8 *characters* not bytes, so use a locale-aware length function (e.g. python's `len(str)` on a decoded UTF-8 string, or awk's multibyte-aware length under a UTF-8 locale) rather than byte length, which will overcount multi-byte scripts like Japanese; (3) for each region independently, determine min/max minute_epoch and walk minute-by-minute filling any missing minutes by forward-filling the last known aggregated latency and setting error-char-count to 0 for filled minutes — do this per region, not globally, since regions have independent timelines; (4) after gap-filling, compute the trailing 5-minute rolling average per region over the (possibly filled) per-minute latency series, averaging however many of the last 5 minutes exist near the start; (5) round both latency fields to exactly 2 decimals and format with fixed decimal places (avoid scientific notation); (6) sort final rows by region then minute_epoch before writing, with no header. Build this incrementally and check intermediate per-region series by hand against a couple of expected minutes before trusting the full script.
