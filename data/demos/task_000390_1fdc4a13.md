Use delta-debugging (bisection), not linear scanning, to minimize the leak trigger. First confirm the full `input.txt` triggers a leak: `valgrind --leak-check=full ./service < input.txt` and grep for "definitely lost" with a nonzero byte count. Then binary-search: split the file in half with `head -n`/`tail -n`, test whether each half still leaks (a leak may depend on the boundary between two specific consecutive lines, e.g. one weird value followed by another, so also test windows that straddle the midpoint, not just clean halves). Repeatedly shrink from both ends until removing any further line from the sequence eliminates the leak — that gives the exact minimal contiguous window. Write the surviving lines verbatim, in order, to `/home/user/minimal_leak.txt`. For the regression script, run `valgrind --leak-check=full --error-exitcode=... ` (or parse output) against `./service < "$1"`, grep the valgrind stderr for "definitely lost: " and check the byte count is `> 0` rather than just checking for the presence of the string (some leak-free runs still print "0 bytes"); exit 1 if bytes>0 else exit 0. Make the script executable with `chmod +x`.
