Build this as a pipeline of small, checkable stages rather than one monolithic script. First write a stage that concatenates all `.txt` files, lowercases with `tr`, replaces every non-alphanumeric character with a space (`tr -c 'a-z0-9' ' '`), then splits on whitespace, filters tokens with length > 2, and pipes into `sort | uniq -c | sort -k1,1nr -k2,2` so ties break alphabetically ascending (watch out: `sort -rn` alone reverses secondary alphabetic order too — sort by count descending and word ascending separately, e.g. `sort -k1,1nr -k2,2`). Take the top 5 to build a word→weight lookup (rank 1→5 ... rank 5→1). Second, write a per-file scoring stage: for each file, reuse the identical tokenization logic (extract into a shared awk/shell function so file-level and global tokenization can't drift), count occurrences of only the 5 chosen words, and compute the weighted sum with `awk`/`bc`. Third, route files by score threshold and append `basename:score` lines to the log, finally sorting the log by filename. Test each stage's intermediate output independently before wiring them into `organize.sh`, since silent tokenization mismatches between the global word-count pass and the per-file scoring pass are the most common source of wrong scores.
