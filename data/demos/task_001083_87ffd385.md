Design the traversal and archive writer as separate concerns before coding. For cycle detection, do not rely on a simple "visited" set of inode numbers alone — track the chain of ancestor directories currently on the recursion stack (e.g., by comparing realpath()/device+inode of each symlink target against every directory currently being descended into), and skip a symlink if resolving it lands inside that active ancestor chain; a naive approach that just checks depth or a global visited set will falsely skip legitimate repeated directory names or fail to catch loops that point to a grandparent. For chunking, split any file >50KB into consecutive 50KB chunks (final chunk shorter), and for each chunk write the header fields in big-endian order using `htonl`/`htons` or manual byte-shifting — get the byte order right since verification parses the format strictly. Wrap the header+payload write for a single chunk between `flock(fd, LOCK_EX)` and `flock(fd, LOCK_UN)` so concurrent writers don't interleave records.
