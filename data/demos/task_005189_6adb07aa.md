This is a classic cross-validation data-leakage bug: `SelectKBest.fit_transform` is called on the full `X, y` before `cross_val_score` splits the folds, so validation data influences feature selection. Fix it by wrapping the selector and model in a single `sklearn.pipeline.Pipeline` (e.g. `make_pipeline(SelectKBest(score_func=f_regression, k=10), Ridge())`) and pass that pipeline object — not the pre-selected `X_selected` — into `cross_val_score` along with the original full `X`. This way `SelectKBest` is refit on only the training fold each iteration. Keep the exact same `np.random.seed(42)` data generation, `KFold(n_splits=5, shuffle=True, random_state=42)`, `Ridge()` defaults, and `k=10` — only change the leakage point. Since the data is pure random noise, expect the corrected mean R² to be negative (a properly cross-validated model performing worse than a horizontal-line predictor is the correct signature that leakage was removed). Write `{"mean_r2": ...}` to `/home/user/metrics.json` and actually execute the script so the file is produced.
