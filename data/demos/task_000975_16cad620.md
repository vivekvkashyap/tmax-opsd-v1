Build the C parser around POSIX `regcomp`/`regexec`: use a regex that captures `msgid` and `msgstr` string values (e.g. matching `"msgid"\s*:\s*"([^"]*)"` and similarly for msgstr), remembering POSIX ERE has no `\d`/non-greedy — use character classes and `[^"]*` instead. The trickiest part is the escape-sequence validator: scan the extracted msgstr string yourself (not via regex alone) looking for every occurrence of a backslash followed by `u`; for each one, check that it's followed by exactly 4 hex digits (`[0-9a-fA-F]{4}`) — if the 4 chars after `\u` aren't all hex digits (including running past end of string), the whole line is corrupted and must be dropped silently (no stdout for that line). A line with zero `\u` occurrences is automatically valid. For the shell script, first compile the C program, then fan out over all `*.jsonl` files in the locales directory using `ls`/`find | xargs -P 4 -I{} ./loc_parser < {}` (or `parallel`), redirecting/concatenating all children's stdout into the CSV body, and prepend the literal header line `msgid,msgstr` before that combined output.
