Work bottom-up with a loop, not a fixed-depth script: repeatedly scan the datasets directory for `.zip`/`.tar.gz` files, and for each one first run an integrity check (`unzip -t file.zip` or `zip -T`; `tar -tzf file.tar.gz` or `gzip -t`) before extracting — if the check exits non-zero, delete/skip that archive and never call extract on it (a naive `tar xzf` on garbage bytes will throw a Python/shell exception and abort the whole run, or worse, extract a partial garbage tree). After each pass, re-scan for newly-extracted archives (nested archives only appear once their parent is unpacked) and keep looping until a full pass finds nothing new to extract. Once extraction is stable, `find` all `.gcode` files recursively and, for each, grep the line beginning with `;TIME:` and parse the trailing integer (don't assume it's the first or last line, and don't accidentally match similar prefixes elsewhere in the file). Accumulate the sum in a variable, and write only the bare integer (optionally with a trailing newline) to the specified output path — no extra text or formatting.
