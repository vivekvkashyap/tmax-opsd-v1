Handle the zip extraction manually rather than trusting `ZipFile.extractall`, which is exactly what's vulnerable to zip-slip. For each entry in the zip, resolve what the final on-disk path would be (`os.path.normpath(os.path.join(target_dir, entry.filename))`) and check that this resolved path still starts with the target directory's absolute path (and also reject any entry name that is itself absolute); if it fails that check, skip extraction and log the entry's original (unmodified) name to the warnings file — don't log the resolved/sanitized path. For GCode, process comment-stripping line by line: split on the first unescaped `;`, and drop the line entirely if the remainder (after stripping trailing whitespace) is empty; then chunk the cleaned lines into groups of exactly 500, writing each with a zero-padded 3-digit suffix. For WAL merging, sort the safely-extracted `.wal` filenames alphabetically (not extraction order) and concatenate their raw bytes with a single `open(...,'ab')` loop. Compute the summary counts (blocked files, total clean GCode lines summed across all chunks, merged WAL byte size via `os.path.getsize`) directly from what you actually wrote, not from assumptions, so the numbers stay self-consistent.
