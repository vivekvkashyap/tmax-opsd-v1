Build this as a three-layer pipeline: a SPARQL query, a thin Python runner that executes it, and a C orchestrator that shells out. First set up `python3 -m venv /home/user/venv && ./venv/bin/pip install rdflib`. Write `query_runner.py` to take `(query_file, ttl_file)` as argv, load the graph with `rdflib.Graph().parse(ttl_file, format="turtle")`, run `graph.query(open(query_file).read())`, and print each result row's local names (strip the namespace prefix from each URI, e.g. via `.split('#')[-1]` or `.n3()` post-processing) space-separated per line. In `auditor.c`, write out a SPARQL query file expressing the graph pattern: a `PublicFacing` node connected via `connectsTo` to an `API` node whose `hasEncryption` literal is `"false"`, which itself `connectsTo` a `PIIDatabase` node — select public/api/db names in that order. Use `popen()` to invoke `venv/bin/python query_runner.py query.rq system_graph.ttl`, read each output line, and format/append `VIOLATION: ... connects to ... connects to ...` lines. Collect all violations, sort by the public node name (`qsort` with `strcmp`), then write them out. Compile and run to populate the log.
