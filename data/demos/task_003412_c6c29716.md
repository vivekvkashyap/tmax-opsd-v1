Build this as two scripts: `server.sh` that launches `socat TCP-LISTEN:8080,reuseaddr,fork EXEC:/home/user/handler.sh` in the background, and `handler.sh` that socat invokes per-connection with the raw HTTP request available on stdin. In the handler, parse the request line to extract the `ts` query parameter, then validate it strictly with a regex allow-list (e.g. `^[0-9]+(\.[0-9]+)?$`) before it ever touches a shell command — reject anything else with 400 immediately, since this is the primary injection vector under test. Use `ffmpeg -ss <ts> -i /app/auth_events.mp4 -frames:v 1 out.png` (pass `ts` only after it passed validation, and still prefer an array-style/quoted invocation over string interpolation) then `zbarimg` to decode the QR payload. Critically, validate the *decoded QR string* the same way — allow-list only digits, spaces, and `+-*/` — before evaluating it arithmetically (avoid `eval`/`bc` on unsanitized input; a decoded payload like a shell command must be rejected as 400, not executed). On success, compute the integer result and `base64` it into the response body, writing correct HTTP/1.1 status lines and headers for both the 200 and 400 cases.
