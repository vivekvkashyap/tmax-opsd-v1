Start with the network layer: use `scapy` (or `tshark -r traffic.pcap -Y "udp.port==5000" -T fields -e data`) to pull raw UDP payloads destined for port 5000 out of the pcap, then hex-encode each candidate payload. Try each payload against `python aggregate.py <hex>` at HEAD to see which one raises an exception (likely a ZeroDivisionError or math domain error) — that's your poison payload, save its hex string as-is. For bisection, run `git bisect start`, mark HEAD `bad` and `v1.0` good, then in `git bisect run` use a small script that checks out each commit and runs `python aggregate.py <poison_hex>`, exiting nonzero only when the process crashes (not on unrelated errors) — this pins down the exact first-bad commit hash automatically rather than eyeballing a log. Once found, read that commit's diff to understand exactly what expression divides by (or takes an invalid domain of) something that can be zero/negative for this payload, then write `/home/user/fixed_aggregate.py` guarding just that computation to return 0.0 when the denominator is zero, keeping all other behavior identical to the bad version.
