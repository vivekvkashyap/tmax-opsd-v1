Build this as four independently-testable pieces glued together by one idempotent script. Write `app.py` as a minimal `http.server` handler returning `Deploy Success` with status 200 on any GET, bound to 127.0.0.1:8080. In `deploy.sh`, before starting anything, actively check current state rather than assuming a fresh environment: use something like `ss -ltn | grep -q ':8080 '` (or `lsof -i :8080`) to decide whether to skip starting the app, and similarly check port 9090 before launching `socat TCP-LISTEN:9090,fork,reuseaddr TCP:127.0.0.1:8080 &`; start both with `nohup ... & disown` so they survive script exit. For the log analysis, use `grep ' 404$'` (or awk field check) on `historical.log` to isolate 404 lines, `awk` to extract the IP field, `sort | uniq -c`, then `sort -rn -k1,1 -k2,2` (count descending, IP ascending on ties) and `head -3`, reformatting to `<IP> <COUNT>` per line. For `firewall.conf`, truncate/overwrite it (`>`) each run and append one `DENY_IP <ip>` line per entry read back from `blocked_ips.txt`. Test idempotency yourself by running `deploy.sh` twice and checking process counts before declaring done.
