Start by reading the values out of the image rather than assuming them: run OCR (`tesseract /app/reference_spectrum.png stdout`) or otherwise view the plot title and extract the stated `f0` and `Gamma` numerically — these are the parameters the whole pipeline depends on, so get them programmatically/precisely, don't eyeball or guess. Next fix `simulator.c`: replace the oversized fixed `dt` with `0.01` (or smaller) and recompute `steps = total_time / dt` so total simulated time stays exactly 200s — an Euler integrator with too-large dt is unstable/aliases, which is the root cause of bad spectra; recompile with gcc and link `-lm`. Run the simulator once per seed 1..100 with the extracted `f0`/`Gamma`, capturing each trial's time series to a file or in memory. For each trial, compute the PSD via FFT (e.g. `numpy.fft.rfft` on the series with a known sample rate `1/dt`), find the frequency bin of max power while explicitly excluding the DC (0 Hz) bin, and collect these 100 peak frequencies. Bootstrap resample (≥1000 resamples with replacement) the 100 peaks to build a distribution of means, then take the 2.5/97.5 percentiles for the 95% CI. Write `mean`, `ci_lower`, `ci_upper` to `/app/metrics.json`.
