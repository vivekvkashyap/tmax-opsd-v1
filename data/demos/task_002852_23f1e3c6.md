Build this as four independent, testable pieces before wiring them together. First, transcribe the spoken 4-digit number in the audio file: convert/inspect it with `ffmpeg` and feed it to a speech-to-text tool (or a lightweight ASR you install), then parse the digits it reports into an integer for `run_id` — don't hardcode a guessed number. Second, write the JSONL filter as a pure function you can unit-test against small hand-crafted good/bad lines before running it on the real corpus: reject on `duration_sec <= 0`, `confidence` outside `[0,1]`, empty transcript, or case-insensitive substring matches for the injection markers — check substrings against a lowercased copy of the transcript. Third, implement rolling average with a small fixed-size ring buffer (or slice) of the last 5 valid confidences, recomputed after each accepted record — don't average over all valid records ever seen. Fourth, use `bufio.Scanner`/`bufio.Reader` on stdin line-by-line (never `ioutil.ReadAll`) so memory stays flat, and batch inserts into SQLite via a prepared statement inside a transaction using `database/sql` + `mattn/go-sqlite3`. Test the whole pipeline against small synthetic clean and evil samples before trusting it on the real corpus.
