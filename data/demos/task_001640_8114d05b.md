Treat this as three concerns that must each be correct independently: locale/timezone environment, black-frame counting via ffmpeg, and a raw-socket HTTP server. For counting black frames, don't try to reimplement pixel analysis in C — shell out from your C program (or precompute and cache) to ffmpeg's blackframe filter, e.g. run `ffmpeg -i /app/edge_feed.mp4 -vf "blackframe=amount=100:threshold=0" -f null -` and count the `Parsed_blackframe_` lines in stderr; capture that count once (at startup or on first request) and reuse it for `/metrics` rather than re-running ffmpeg per request. For `/health`, set `TZ=Asia/Tokyo` and call `setlocale(LC_TIME, "ja_JP.UTF-8")` (verify the locale is actually installed with `locale -a` first) before using `strftime` with `%c`, and use `tzset()` so localtime reflects Tokyo. Build the HTTP server with POSIX sockets (`socket`/`bind`/`listen`/`accept`), parse just the request line to route `GET /health` vs `GET /metrics`, and always send correct `HTTP/1.1 200 OK` status lines and headers before the body. In `deploy.sh`, export the env vars, compile with gcc, and launch the daemon with `&` so it stays running in the background.
