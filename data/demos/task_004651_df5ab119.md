Work top to bottom through the four deliverables, verifying each before moving on. First `mkdir -p` all six directories in one command. For the C daemon, keep it minimal: an infinite loop that opens the log file in append mode (or reopens each iteration), writes "SERVICE_OK\n", calls `fflush` (or use `setvbuf`/close-reopen) so output isn't buffered, then `sleep(1)`; compile with `gcc -o /home/user/bin/microservice /home/user/src/microservice.c`. Write the systemd unit file exactly matching the required `[Unit]`/`[Service]` sections and field values (Description, ExecStart, Restart=always) — malformed ini syntax is the easy way to fail here, so keep it byte-for-byte close to what's expected. Run `systemctl --user daemon-reload` and `systemctl --user start micro.service`, but don't let a failure there abort your task (container environments often lack a user systemd session). For logrotate, write the generator script so it always overwrites the same target path with the same fixed content (a heredoc redirected with `>`, not `>>`) — that's what makes repeated runs idempotent and keeps line count low; `chmod +x` it and run it once at the end.
