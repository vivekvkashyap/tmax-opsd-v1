Use `net/http.NewServeMux()` (or Go 1.22+ method+pattern routing like `mux.HandleFunc("POST /job/{platform}/{job_id}", ...)`) and pull path segments with `r.PathValue(...)` or manual `strings.Split(r.URL.Path, "/")` if targeting an older Go version — check `go version` first. Guard the in-memory metrics map with a `sync.Mutex` (or `sync.RWMutex`) around every read/write, including inside the summary handler. For the build-tag split, put `//go:build linux` (and the corresponding `//go:build windows`) as the very first line of each file followed by a blank line — a common mistake is placing it after package declaration or forgetting the blank line, which silently disables the constraint. Write logs with `os.OpenFile(path, os.O_APPEND|os.O_CREATE|os.O_WRONLY, 0644)` and a trailing `\n`, matching the exact string format required. Cross-compile with `GOOS=linux GOARCH=amd64 go build -o api-linux-amd64` and `GOOS=windows GOARCH=amd64 go build -o api-windows-amd64.exe` inside `build_cross.sh`; test locally by starting the linux binary, curling each endpoint with `curl -s -X POST ... -d '{"status":...,"duration":...}'`, then killing the background PID (`kill $!`) before the test script exits.
