Build this as a two-stage index-then-query pipeline in pure Bash/coreutils, not a single script that re-scans the TSV every query. For `setup_index.sh`: read `/home/user/network.tsv`, get unique Source values (`cut -f1 | sort -u`), and for each one filter matching rows and write sorted, unique Target values to `graph_index/<Source>.txt` (use `awk -F'\t' -v s="$src" '$1==s{print $2}' | sort -u`). For `query_traversal.sh`: read the starting node's index file to get distance-1 neighbors; for each neighbor, read ITS index file to collect distance-2 candidates, tracking how many distinct intermediate nodes reach each candidate (e.g. accumulate all candidate lines into one stream, then `sort | uniq -c`). Exclude any candidate equal to the start node or present in the distance-1 set — do this exclusion as a final filter step, not by trying to avoid generating them. Sort results by count descending then name ascending (`sort -k1,1nr -k2,2` on swapped fields, or `sort -t, -k2,2nr -k1,1`), then take the top 5 with `head -5`, formatting each line as `Node,Count`. Test with a small hand-built network.tsv first, and chmod +x both scripts.
