If any run yields fewer than 1000 lines, exit 1 (bad); only exit 0 (good) if every run hits exactly 1000. Make the script self-contained (cd into the repo, rebuild each time since bisect checks out different commits) and idempotent. Then run `git bisect start`, mark `HEAD` bad and `v1.0` good, and hand control to `git bisect run <your-script.sh>`. Let bisect finish and print the first bad commit automatically rather than trying to binary-search manually — this avoids false positives from a single lucky/unlucky run. Capture the resulting commit hash (`git rev-parse` on the identified commit, or parse bisect's output) and write only the 40-char hash to the required file.
