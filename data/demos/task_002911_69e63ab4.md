Two independent pieces: OCR a small parameter from an image, and a bit-exact LU-decomposition C program. Run `tesseract /app/config.png stdout` (or similar OCR) to read off the integer k printed in the image — don't guess it, verify the OCR output looks like a clean digit and use that value in your C code. For the C program: read 100 doubles (little-endian, row-major 10x10) from stdin with `fread`, implement a standard Doolittle LU decomposition without pivoting (the task guarantees inputs won't need it), then truncate L to its first k columns and U to its first k rows, multiply them back together (A' = L[:, :k] @ U[:k, :]), and `fwrite` the resulting 100 doubles to stdout with zero extra output — no printf debug text, no trailing newline text mixed into the binary stream. Compile with plain gcc (no LAPACK needed for a 10x10 case), and test manually with a small known diagonally-dominant matrix piped through the binary to confirm the byte counts and rough correctness before relying on the fuzz-equivalence check against the hidden oracle.
