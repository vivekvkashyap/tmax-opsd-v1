Build this as a single streaming pipeline, not a script that slurps files into memory. Use `find /home/user/project_logs -name '*.log' -print0 | xargs -0 awk '...'` (or `cat` the file list through one awk invocation) so awk sees a continuous stream across all files. In the awk program, maintain a "current record" buffer: when a line starts with `[` (a new entry), first flush/print the previous buffered record if it matched `[ERROR]` or `[CRITICAL]`, then start a new buffer from the current line; when a line does NOT start with `[`, append it to the buffer joined with a literal tab (`buffer = buffer "\t" $0`) instead of a newline — remember to flush the last buffered record in an `END` block, since the final entry has no trailing new-record line to trigger its flush. After extraction, transform the `[YYYY-MM-DD HH:MM:SS]` and `[LEVEL]` prefix into `YYYY/MM/DD-HH:MM:SS<TAB>LEVEL<TAB>` — a `sed` regex with capture groups on the already-tab-joined line works well. Finally pipe through `sort` (the reformatted timestamp sorts correctly as plain text) and write to `/home/user/organized_errors.tsv`.
