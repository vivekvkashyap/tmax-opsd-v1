Build the three pieces in order and test incrementally rather than writing everything then testing once. First get Nginx up: write `/home/user/nginx.conf` as a minimal standalone config (include `events {}` and an `http { server { listen 127.0.0.1:8000; location / { proxy_pass http://127.0.0.1:8080; proxy_set_header Host $host; } } }` block, plus `pid`/error/access log paths writable by the user), then start it with `nginx -c /home/user/nginx.conf` and confirm with `curl` that 8000 reaches 8080 once the WAF is up.

For the WAF, use Python's `ast` module, not regex, to validate `expression`: `ast.parse(expr, mode='eval')`, then walk the tree and reject on any `ast.Call`, `ast.Attribute`, `ast.Import`, `ast.Subscript`, or other non-literal/non-arithmetic node — allow only `ast.Expression`, `BinOp`, `UnaryOp`, numeric/string `Constant`, and `Name` nodes with safe operators. A naive attempt that just blacklists keywords like `__import__` will miss disguised RCE payloads and other node-based tricks.

For `patch`, parse each `---`/`+++` diff header line, extract the filename token, and reject if it starts with `/` or contains `../` anywhere in the path (not just as a prefix) — check path components after normalizing, since evil patches will try things like `a/../../etc/passwd`.

On any violation return 403 immediately without contacting the backend; otherwise forward the exact JSON via `requests.post` to `http://127.0.0.1:8081/build` and relay its status/body verbatim. Test both corpora directly against the WAF (port 8080) before testing through Nginx (port 8000) to isolate failures.
