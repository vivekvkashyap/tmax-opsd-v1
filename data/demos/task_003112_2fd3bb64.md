Build this as a small stateful web service, and pay special attention to the two easy-to-miss requirements: exact validation semantics and correct rate-limit accounting. Use Flask (or FastAPI+uvicorn) bound to `127.0.0.1:8080`. On startup, open a connection to `/app/metrics.db` and run `CREATE TABLE IF NOT EXISTS metrics (id INTEGER PRIMARY KEY, frame INTEGER, brightness REAL)`. In the `POST /analyze` handler: parse JSON, return 400 if `frame` is missing, not an int (reject bools/floats/strings too — check type strictly), or negative — do this check *before* touching the rate limiter, since invalid requests must not consume it. For valid requests, maintain a simple in-memory sliding/fixed window counter (e.g., a list of timestamps of successful requests in the last 60s); if a 6th valid request arrives within the window, return 429 without processing it further. For accepted requests, open `/app/video.mp4` with `cv2.VideoCapture`, seek to the exact frame index (`cap.set(cv2.CAP_PROP_POS_FRAMES, frame)`), read it, convert with `cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)`, compute `float(numpy.mean(gray))`, insert `(frame, brightness)` into `metrics`, commit, and return `{"brightness": brightness}` as 200. Start the server in the background (e.g. `nohup python3 server.py &`) and confirm it's listening before finishing.
