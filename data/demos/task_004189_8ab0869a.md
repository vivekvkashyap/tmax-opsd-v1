Two independent deliverables here. For the subtitle extraction, first inspect the video's streams (`ffprobe -show_streams` or `ffmpeg -i`) to confirm which stream index carries subtitles, then extract with `ffmpeg -i input -map 0:1 -c copy output.srt` (adjust the map index to whatever you actually find, and use a text codec like `srt` if copy fails for the container's subtitle format) — verify the output file is non-empty and looks like valid SRT before moving on. For the stream parser, build the regex to match the format precisely, component by component: fixed-width digit groups for HH/MM/SS/mmm separated by exact literal characters, a speaker-ID token from an explicit character class with a length range, and the remaining text as a capture group — anchor the pattern at both ends so partial/loose matches are rejected. Read stdin line-by-line (not all at once, to support streaming), strip only the trailing newline, apply `fullmatch`/anchored `match`, and print either the templated `<trans .../>` line or the literal `<error>MALFORMED</error>` line for every input line, flushing/printing incrementally rather than buffering all output. Test against edge cases yourself — missing spaces, lowercase speaker IDs, malformed timestamps — to be confident before assuming correctness against the hidden fuzz oracle.
