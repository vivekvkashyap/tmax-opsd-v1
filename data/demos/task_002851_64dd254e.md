Write setup.sh to be idempotent: use `mkdir -p` for directories, and for the symlink check if it already exists/points correctly before creating (`ln -sfn target link` is safe to re-run), then write the config file with a single `IMAGE=/app/target_image.png` line. For the C HTTP server, you don't need a full HTTP library — a minimal server that `bind`/`listen`/`accept`s on 127.0.0.1:8080, reads the request line, checks if it starts with `GET /extract`, and if so runs tesseract via `popen("tesseract <path> stdout", "r")` (reading the IMAGE path out of config.txt at request time or startup) and streams its output back with a `200 OK` header is sufficient; anything else should get a `404 Not Found` response with correct HTTP status line and headers so curl-based checks pass. Test the server locally with `curl http://127.0.0.1:8080/extract` before setting up the tunnel, to isolate OCR/server bugs from tunnel bugs. For the SSH tunnel, since passwordless SSH to `user@localhost` is already configured, run `ssh -N -L 9090:127.0.0.1:8080 user@localhost &` in the background (the `-N` flag avoids opening an interactive shell) and verify with `curl http://127.0.0.1:9090/extract` that the forwarded request reaches your server end-to-end.
