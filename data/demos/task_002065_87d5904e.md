The core technique is path canonicalization before any filesystem write, not string-pattern matching on `..`. For each manifest entry, compute the fully resolved absolute path of the destination by joining it onto `/home/user/project_dir/organized_workspace/` and normalizing (e.g. Python's `os.path.normpath`/`pathlib.Path.resolve()` semantics, or realpath-style logic) — do this even though the parent directories may not exist yet, so resolve syntactically rather than relying on a real filesystem lookup that would fail on non-existent paths. A resolved path is valid only if it is strictly inside the workspace directory (compare with a trailing-separator-aware prefix check, and reject if it resolves to the workspace root itself). For valid entries, `mkdir -p` the parent directories first, then create a symlink with an absolute target path (`os.symlink(src_abs, dest)`) or a hardlink (`os.link`) per `link_type`. For invalid entries, skip the link entirely and append `Invalid path escape attempt: <dest>` (using the raw JSON string, unmodified) to the log, preserving manifest order. Double-check nothing ever gets written outside `organized_workspace/` — a naive `..`-substring check is insufficient since `conf/../../etc/passwd` is safe-looking mid-string but still escapes.
