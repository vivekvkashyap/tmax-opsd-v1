In the C filter, read `stdin` line by line, parse the fixed `[timestamp] [LEVEL] [IP] message` format with `sscanf`/manual tokenizing on `[`/`]`/space, drop any line whose LEVEL isn't exactly `ERROR` or `CRITICAL`, and for kept lines locate the IP's last octet (find the third `.` in the address and replace everything after it up to the closing `]` with `XXX`) while re-emitting the line in the identical original format. Compile it once (`g++`/`gcc -o log_filter log_filter.cpp`) and test it standalone with a piped sample before wiring up the shell script. In `process_logs.sh`, first capture the list of old files with `find /home/user/logs -name '*.log' -mtime +7` into a variable/array — do this *before* any deletion — then for each file `cat` it through `./log_filter >> /home/user/critical_archive.log`, and only after all files have been successfully piped through, loop back over that same captured list and `rm` exactly those files. Don't use a single `find ... -exec` combining process-and-delete in one pass, since a late failure could delete files whose content wasn't archived, and don't re-run `find -mtime +7` for the delete step since file mtimes/results could differ.
