Split this into independent, testable stages rather than one monolithic script. First extract frames from the video with `ffmpeg -i /app/monitoring_feed.mp4 out_%04d.ppm` (PPM is trivial to parse: a header line then raw RGB triples), then write a small C (or even shell/python for a quick sanity check first) program that reads pixels (0,0)-(15,15) from each frame and counts frames where every pixel in that block satisfies R>200, G<50, B<50 — count a frame only if the whole block matches the high-load color, and verify against a couple of frames manually before trusting the total. Write that count into `~/server.conf` alongside `PORT=8080`. For the HTTP server, hand-roll the socket loop with `accept`/`read`/`write`, parse just the request line to route `/health` and `/report`, and build responses with explicit `\r\n\r\n` between headers and body plus a correct `Content-Length`. For the supervisor, use a `while true; do ./server; done`-style loop backgrounded with `nohup ... & disown`, redirecting stdout/stderr to `~/server.log`, and confirm via `ps` that `server` is a child of `start.sh` and that killing it triggers a restart within a couple seconds.
