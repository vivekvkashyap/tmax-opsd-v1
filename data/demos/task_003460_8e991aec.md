Set up the Rust project with `cargo new config_tracker` and add `serde`, `serde_json` (with `derive`), and `csv` to `Cargo.toml`. Stream the JSONL file line-by-line using a buffered reader (`BufReader::new(File::open(...)).lines()`) rather than reading the whole file into a `String`, deserializing each line into a struct with `serde_json::from_str`. Group events into a `HashMap<String, Vec<Event>>` keyed by `service` (this needs all events in memory grouped by key, but not the raw file — that's fine since grouping-then-sorting is inherent to the task). For each service's Vec, sort by `timestamp` ascending, then walk consecutive pairs comparing `curr.timestamp - prev.timestamp`: strictly less than 1000 is a violation (note "strictly less than" means exactly 1000ms difference is NOT a violation — an easy off-by-one to get wrong). Collect violations, then produce the final CSV by sorting all violation rows first by `service` using Unicode scalar value ordering (Rust's default `String`/`&str` `Ord` already compares by Unicode codepoint, so a plain sort works) and then by `timestamp`, writing with the `csv` crate ensuring UTF-8 output and the exact header `service,timestamp,change_id`.
