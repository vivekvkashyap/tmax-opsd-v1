Use Go's `encoding/csv` reader, not a naive line-splitter — it natively handles quoted fields with embedded newlines, so read the whole file with `csv.NewReader` rather than `bufio.Scanner`. Process rows in this order: (1) dedupe first by `(timestamp, uppercase(sensor_id))` keeping only the first occurrence in file order; (2) then normalize `sensor_id` to uppercase and collapse all whitespace runs (including embedded newlines/tabs) in `notes` to a single space via a regex like `\s+`, trimming ends; (3) compute per-sensor mean of valid (non-empty) `reading` values *after* dedup, and impute missing readings with that mean (or `0.00` if the sensor has zero valid readings); (4) sort rows by timestamp string, tie-broken by normalized sensor_id; (5) write with Go's `csv.Writer` (which applies standard quoting only when needed) formatting readings with `%.2f`. Watch two common failure points: doing imputation before dedup (wrong mean) and using a scanner-based parser that mis-splits multiline quoted notes.
