Write `run_pipeline.sh` as a sequence of shell steps, not a Makefile: `mkdir -p` the `src`, `bin`, `results` subdirectories, `mv` the `.c` files into `src/`, then compile with gcc. The key pitfall is the linking error mentioned in the prompt — functions like `sin`/`cos` from `<math.h>` require the `-lm` flag placed *after* the source/object files on the gcc command line (`gcc src/main.c src/math_ops.c -o bin/calc -lm`); putting `-lm` before the sources or omitting it reproduces the reported failure. For benchmarking, invoke `/usr/bin/time -p` around the `calc 5000000` execution and redirect only stderr (where `time -p` writes its real/user/sys lines) to `results/benchmark.txt`, while separately capturing calc's stdout, piping it through `base64`, and redirecting that to `results/encoded_out.b64` — you'll likely need to run the timed command and the base64 capture as a single pipeline (e.g. `{ /usr/bin/time -p bin/calc 5000000 2>results/benchmark.txt; } | base64 > results/encoded_out.b64`) to get both outputs correctly separated. Make the script executable but do not execute it yourself.
