Work in dependency order: fix the Rust source first, then build, then the proxy, then wire the servers together. Apply the patch with `patch -p0 < fix_math.patch` (or `-p1`, check the diff headers) from inside `/home/user/rust_backend`, then verify the change actually landed by re-reading `src/main.rs` before building — a patch that silently fails to apply (wrong strip level, already-modified file) is the most common trap. Run `cargo build --release`, and locate the resulting binary under `target/release/`. For the Python 2→3 migration, don't just eyeball syntax: `BaseHTTPServer` becomes `http.server` (`HTTPServer`, `BaseHTTPRequestHandler`), `urllib2.urlopen` becomes `urllib.request.urlopen`, `except Exception, e:` becomes `except Exception as e:`, and `print "x", e` becomes `print("x", e)`. Start the Rust backend and the Python proxy as background processes (e.g. `nohup ... &` or `&` with disown), then poll with `curl` on both ports to confirm they're actually listening before running the integration test — starting a background process doesn't guarantee it bound the socket yet. Only then run `test_integration.py` and inspect `/home/user/test_results.txt`.
