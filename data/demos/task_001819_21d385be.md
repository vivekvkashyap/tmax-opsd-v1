Build and test the two pieces (C monitor, bash orchestrator) independently before wiring them together. In monitor.c, use socket()/connect() with AF_INET/SOCK_STREAM to 127.0.0.1 on each port; treat a failed connect() (non-zero return / ECONNREFUSED) as down and append exactly "ALERT: PORT <port> DOWN\n" to /home/user/monitor.log by opening the log in append mode — do not overwrite it between runs, since the script calls monitor twice and only the second run's failure should appear (make sure a successful first run writes nothing, so the log ends up with exactly one line). On success, close the socket and log nothing. In launch.sh, use `set -e`, launch each `python3 -m http.server` in the background with `&` bound explicitly to 127.0.0.1 (not 0.0.0.0), `sleep 2`, compile monitor.c with gcc, run monitor once (should be silent), then kill only the port-9002 server (find its PID, e.g. via the variable you saved when backgrounding it, not by scanning for "python3" broadly which could kill port 9001 too), then run monitor again to produce the alert. Make sure the port-9001 server process survives after the script exits (avoid `wait` on it, and don't let it be a child that dies when the script's subshell ends).
