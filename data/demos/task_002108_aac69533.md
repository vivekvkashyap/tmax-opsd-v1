Treat this as two-stage graph aggregation across heterogeneous stores, not a single query. First load both sources into Python: read `org` from SQLite via `sqlite3`, and `permissions.json` via `json.load`. Build a children map (`manager_id -> [emp_id,...]`) from the org table so you can walk the hierarchy top-down or recurse bottom-up per employee. For each employee, their resolved access = their own direct permissions.json grants UNION the resolved access of every direct/indirect report — compute this recursively (memoize on emp_id, or do a post-order traversal from leaves upward) rather than re-walking the whole subtree for every node, which is correct but wasteful for larger hierarchies. Use a `set()` per employee to naturally dedupe resource_ids, then `sorted(...)` before writing. Watch two common mistakes: (1) permissions.json keys are strings while org emp_ids are ints — normalize consistently when looking up; (2) an employee with no direct grants and no reports must still appear in the output with an empty list, since every org-table employee needs an entry. Finally `json.dump` the emp_id(string)->sorted resource list mapping to `/home/user/access_graph.json`.
