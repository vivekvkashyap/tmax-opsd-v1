Build `filter.sh` as a sequence of independent, always-cleaned-up checks that short-circuit on the first violation. For symlink loops, don't naively `find -L` the whole tree (it can hang) — instead walk with a visited-inode/path set (e.g. `readlink -f` with a loop counter, or `find -L ... -maxdepth N` combined with detecting `ELOOP` from `stat`/`realpath`, which reliably errors out on a real loop rather than hanging). For the archive, first confirm `payload.tar` exists, then run `tar -tf payload.tar >/dev/null 2>&1` and treat a nonzero exit as corruption. Extract into a `mktemp -d` sandbox, locate the `.mp4`, and pull its first frame with `ffmpeg -i video.mp4 -vframes 1 frame.jpg`; read the first 3 bytes with `head -c3 frame.jpg | xxd -p` and compare against the JPEG magic `ffd8ff`. Separately, use `ffprobe -count_frames` (or equivalent) on `/app/reference_feed.mp4` once to determine its exact frame count, then compare that same frame-count method against each package's `.mp4` to catch the recurring corrupted feed. Always `rm -rf` the temp dir on every exit path (trap on EXIT is cleaner than manual cleanup at each return).
