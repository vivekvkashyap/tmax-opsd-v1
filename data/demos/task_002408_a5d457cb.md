This is a debugging exercise across three layers: log correlation, build repair, and a numerical-stability fix. First, correlate the two logs despite clock drift: in `pricing-engine.log`, find the transaction that logs "Started calculating yield" but has no matching "Finished" entry — that's the hang; match its `tx_id` back to `api-gateway.log` to read off its `rate` parameter and confirm that gateway entry shows a timeout/504 status. Write exactly that `tx_id` and `rate` (nothing else) as the two lines of `bug_report.txt`. Second, `cd pricing-engine && go build ./...`, read the compiler error, and fix the actual syntax/build defect it points to (don't just delete files blindly — check what's genuinely broken). Third, in the Newton-Raphson yield calculation, look for two related bugs: a narrow floating-point type causing the convergence check to never reach its tight epsilon threshold (precision loss) — widen it — and a termination condition with no iteration cap, so also add a bounded max-iterations safety fallback. Finally write `run.sh` to `go build` and launch the binary backgrounded on port 8080, then confirm `curl 'localhost:8080/calculate?rate=<the crashing rate>'` returns promptly instead of hanging.
