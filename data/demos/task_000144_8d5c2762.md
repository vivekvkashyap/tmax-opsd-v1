Install `libeigen3-dev` and `g++` first (`apt-get update && apt-get install -y libeigen3-dev g++`). Structure the C++ program around Eigen types: load the spectrum into an `Eigen::VectorXd` (x, y), the covariance into an `Eigen::MatrixXd` (parse the CSV row by row, splitting on commas), then compute `Eigen::LLT<Eigen::MatrixXd> llt(Sigma)` once outside the MCMC loop — reuse the factorization for every likelihood evaluation instead of re-decomposing per proposal (that's the main naive mistake, and also don't invert Sigma explicitly). For each proposal, compute the model vector f = A*exp(-(x-mu)^2/(2*sigma^2)) elementwise, get diff = y - f, solve `llt.solve(diff)` to get Sigma^{-1}*diff, then LL = -0.5 * diff.dot(solution). Follow the Metropolis rule exactly as specified: P = min(1, exp(LL_prop - LL_curr)), guarding against overflow when LL_prop - LL_curr is large positive (clamp to 1.0 rather than calling exp on a big number). Read proposals.csv and uniform.txt in the same order as iteration i=0..999, updating current state only on acceptance. Write the final state with high precision (e.g. `printf`/`std::setprecision(6)`) to /home/user/final_state.txt, and compile with `-O2` linking against Eigen (header-only, just needs the include path `-I/usr/include/eigen3`).
