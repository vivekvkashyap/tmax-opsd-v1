This is fundamentally a robust-directory-walker + file-hashing tool with concurrency safety, not a service-configuration task — the running Nginx/Redis/DocGen services are context, not something you need to reconfigure unless permissions are actually blocking reads. Implement the traversal yourself with explicit cycle detection: maintain a set of visited real paths (`os.path.realpath`) for the *current recursion path* (or globally, matching what "already visited" should mean) and skip a symlinked directory if its resolved target is already in that set, rather than relying on `os.walk`'s default behavior which doesn't protect against this. For each regular file, open it, then `fcntl.flock(fd, fcntl.LOCK_SH | fcntl.LOCK_NB)` before reading — catch the `BlockingIOError`/`OSError` if the lock is already held exclusively, log a warning to stderr, and skip that file entirely rather than crashing or blocking. Compute SHA-256 over the full file content, record size, and record whether the manifest entry is `"file"` or `"symlink"` type. Build the manifest dict with relative POSIX-style paths as keys, then serialize with `json.dumps(manifest, indent=2, sort_keys=True)`. Test it manually against a directory you construct with deliberate symlink loops and a background writer holding a lock, since the real verifier fuzzes exactly that scenario against a hidden oracle for byte-identical output.
