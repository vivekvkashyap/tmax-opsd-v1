Start with static analysis of the ELF to find the hidden key without executing untrusted code: `readelf -S /home/user/scanner.elf` to list sections and confirm `.secret_key` exists, then dump its bytes directly with `objdump -s -j .secret_key /home/user/scanner.elf` (or compute the section's file offset from `readelf -S` and extract with `dd`/`xxd`) to read the single key byte in hex. Use that byte to XOR-decrypt `/home/user/cert.enc`: a simple loop in bash/python (`xxd` + shell arithmetic, or a one-line `python3 -c` since python is standard) applying `byte ^ key` to every byte, writing the result to `/home/user/decrypted.crt`; verify it's a valid PEM cert with `openssl x509 -in /home/user/decrypted.crt -noout -text` before trusting the key you extracted — if openssl errors, recheck you read the correct section/offset (e.g., an off-by-one on file vs. virtual address). Then pull the CN with `openssl x509 -in decrypted.crt -noout -subject` and the fingerprint with `openssl x509 -in decrypted.crt -noout -fingerprint -sha256`, uppercase and colon-separated as openssl already formats it. Write the three-line report to `/home/user/report.txt` in exactly the specified `KEY:`/`SUBJECT_CN:`/`FINGERPRINT:` format, matching field labels and value formatting precisely (e.g. `0xXX` hex casing) since the grader does exact string matching.
