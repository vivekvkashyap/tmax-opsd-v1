Build this as a two-phase C++ project: a data-loading/indexing phase, then a query phase. First read nodes.csv and edges.csv, building a `std::unordered_map<std::string,int>` from node_id string to a dense integer index as you encounter new IDs; store the adjacency list as `vector<vector<pair<int,double_distance,double_cost>>>` (or parallel arrays) indexed by that integer, since string comparisons in a hot Dijkstra loop are slow. Implement Dijkstra with a `priority_queue` of (distance, node) using strict `>` comparisons so ties don't matter (the task guarantees uniqueness), and track a `prev[]` array to reconstruct the path by walking backwards from target to source, then reversing. Separately accumulate the total cost by summing edge costs along the reconstructed path (do NOT run a second cost-based Dijkstra — cost is just accumulated along the chosen distance-shortest path). Handle unreachable targets by checking if the target's distance is still infinity and emit the `-1,-1,NONE` line. Use `std::setprecision(1) << std::fixed` for all float output, and be careful to open both output files with directory creation (`mkdir -p` equivalent) before writing. Write run.sh to compile with `g++ -O3 -std=c++17 main.cpp -o app && ./app`. Test with small dummy CSVs first before trusting correctness on the real data.
