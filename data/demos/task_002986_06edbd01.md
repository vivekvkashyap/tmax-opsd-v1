Treat the legacy binary as an opaque black box you must call once per request, never keep it open as a long-lived pipe. In your HTTP handler: parse the JSON body, pack `source` and `destination` each as an unsigned 32-bit little-endian integer (8 bytes total, e.g. Python `struct.pack('<II', source, destination)`), spawn `/app/legacy_router` as a fresh subprocess per call, write those 8 bytes to its stdin, close stdin (or use `communicate()`), read exactly 4 bytes back from stdout, and unpack as little-endian uint32 (`struct.unpack('<I', ...)`) to get `next_hop`. Bind the server strictly to `127.0.0.1:9000` and keep it running in the background (e.g. `nohup ... &` or a daemon process) so the verifier can hit it later. Watch for two common mistakes: byte order (must be little-endian, not network/big-endian) and reusing one subprocess across requests (spawn-per-request is required since streaming mode leaks memory).
