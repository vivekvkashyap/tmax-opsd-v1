Compile libcwav.c with `gcc -shared -fPIC -o /app/lib/libcwav.so libcwav.c` inside build.sh, and in the same script pip-install `SpeechRecognition` plus `pocketsphinx` (and any OS packages like `swig`/`portaudio19-dev` needed for pocketsphinx to build). In your FFI module, load the library with `ctypes.CDLL`, and explicitly set `.argtypes = [c_char_p, c_char_p]` and `.restype = c_int` for `extract_wav` — a missing signature causes silent pointer/return corruption. In audio_detector.py, iterate every `.cwav` in the given directory, call `extract_wav(cwav_path, tmp_wav_path)`, check the return code, then transcribe the resulting wav with `sr.Recognizer().recognize_sphinx(audio)`. Lowercase the transcript and check for the substring "launch"; wrap recognition in try/except (e.g. `UnknownValueError`) so unrecognized audio doesn't crash the pipeline — default such cases to ACCEPT unless another signal says otherwise. Write `results.json` mapping each basename to REJECT/ACCEPT. Validate iteratively against `/app/corpora/clean` and `/app/corpora/evil` before trusting the pipeline, since pocketsphinx accuracy varies and you may need tuning (e.g. grammar/keyword-spotting mode) to reliably catch "launch".
