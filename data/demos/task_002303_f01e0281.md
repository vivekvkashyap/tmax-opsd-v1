First start `/app/start_services.sh` and confirm the three simulated services are actually listening on 9001/9002/9003 (`ss -ltnp` or `curl`). For the port remap, generate an SSH keypair, append the public key to `~/.ssh/authorized_keys`, make sure sshd is running and passwordless localhost SSH works, then establish background local forwards (`ssh -f -N -L 8002:127.0.0.1:9002 -L 8001:127.0.0.1:9001 user@localhost`, or two separate `-L` invocations) — verify end-to-end by curling `127.0.0.1:9003` and checking you get the chained response, not just that ssh didn't error. For storage, `mkdir -p /app/logs` then `sudo mount -t tmpfs -o size=50M tmpfs /app/logs` and confirm with `mount | grep /app/logs`. For the C parser, write straightforward, allocation-light code (no per-line regex libraries — plain string scanning for `txid=` and `time=` fields is far faster) since it will be timed against a 1M-line file at a 0.2s budget; read the whole file with buffered I/O rather than line-by-line `fgets` if profiling shows that's the bottleneck. Set up the bare repo, clone, and write the `post-receive` hook as a script that checks out the pushed commit into a work tree and runs `gcc -O3 alert_parser.c -o /app/bin/alert_parser`, then commit+push your C source to trigger it and confirm the binary appears. Finally run the compiled binary against `/app/historical.log` redirecting output to `/app/logs/alerts.txt`.
