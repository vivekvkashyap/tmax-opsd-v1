Use `file` (or raw magic-byte inspection via `xxd`/`od` on the first bytes, since ELF starts with `\x7fELF` and the e_type field at offset 16 distinguishes ET_EXEC/ET_DYN executables from ET_REL relocatables) to classify each file in `/home/user/project/artifacts` without relying on file extensions, since names are arbitrary. For each file, run `file <path>` and pattern-match its output string for "executable" vs "relocatable" (object file) to decide the copy+rename target into `/home/user/project/organized/` as `exe_<name>.bin` or `obj_<name>.o`; skip anything `file` doesn't identify as ELF. Then determine the largest `*.bin` file in `organized/` with `ls -S` or `stat`/`du` comparisons (not by trusting original file sizes, since only ELF executables count and copies may differ), and use `split -b 1024 -d` or `split -b 1024` with proper suffix length to produce `part_aa`, `part_ab`, etc. under `/home/user/project/chunks/`. Verify correctness yourself with `cat chunks/part_* | cmp - organized/exe_<largest>.bin` before finishing — this is exactly how the grader checks reconstruction.
