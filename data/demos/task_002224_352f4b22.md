Build this as a small FastAPI/uvicorn app listening on `127.0.0.1:8080` with a `/profile/{frame_index}` GET handler. Steps inside the handler: (1) extract/read the requested frame from `/app/rendering.mp4` (e.g. via OpenCV `cv2.VideoCapture`, seeking to the given 0-indexed frame) and convert to grayscale, flattening to a 1D pixel array; (2) generate the theoretical comparison sample fresh on each request (or cache it, but with the fixed seed) using `np.random.seed(42)` then `scipy.stats.beta.rvs(2.0, 5.0, size=100000) * 255` — the seed must be set immediately before this call so it's reproducible regardless of what happened earlier in the request; (3) compute `scipy.stats.wasserstein_distance` between the frame pixels and the samples; (4) compute `scipy.stats.ks_2samp` for the p-value; (5) apply the stability rule exactly: `stable = (p_value > 0.01) and (wasserstein < 10.0)`; (6) return the JSON with keys `frame`, `wasserstein`, `p_value`, `stable`. Install dependencies first (`pip install scipy numpy opencv-python-headless fastapi uvicorn`), launch the server in the background (e.g. `uvicorn ... &` or nohup), and self-test with `curl http://127.0.0.1:8080/profile/0` before finishing, checking a few different frame indices to confirm the endpoint responds correctly under repeated requests.
