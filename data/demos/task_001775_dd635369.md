Implement `scan_content` in C doing a simple substring search for the two threat markers from the spec, returning `is_safe`/`threat_score`/`file_hash` (leave hash empty) by value; compile the shared library with aggressive optimization (`-O3 -fPIC -shared`) into `/app/libscanner.so` — this is where most of your speedup over the pure-Python baseline comes from, since substring scanning in C beats Python's overhead-heavy per-file loop with its simulated extra work. In `/app/fast_organizer.py`, load the library with `ctypes.CDLL`, define a matching `ctypes.Structure` subclass, set `argtypes`/`restype` on the function, iterate `/app/uploads/`, read each file's bytes, call into C, and move files to `/app/safe/` or `/app/quarantine/` based on the result, finally writing counts to `/app/results.json`. Benchmark both scripts yourself (`time python3 ...`) before finishing — you need at least a 5x wall-clock speedup versus `/app/slow_organizer.py` while producing identical classification counts.
