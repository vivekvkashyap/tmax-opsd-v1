Use `filepath.Walk` to find `.gcode` and `.elf` files recursively. For gcode files, decode from ISO-8859-1 (each byte maps 1:1 to the same Unicode code point 0-255, so you can convert byte-by-byte to `rune`, or use `golang.org/x/text/encoding/charmap.ISO8859_1`) before scanning lines — don't treat the raw bytes as UTF-8 or you'll mangle non-ASCII bytes. Collect the first 3 lines whose trimmed text starts with `G0` or `G1` per file, and sort files alphabetically by base filename before writing the summary block (`[filename]` header then the lines). For ELF files, use `debug/elf.Open` and read `f.Entry`, formatting as lowercase hex with a `0x` prefix — match the exact spacing/format the task specifies since the verifier reconstructs its own expected string dynamically via `readelf -h`, so don't hardcode entry-point values, they vary per compiler/build. Concatenate gcode summary + one blank line + elf summary exactly, then compress with `compress/zlib` and write the raw compressed bytes (not base64) to the output file.
