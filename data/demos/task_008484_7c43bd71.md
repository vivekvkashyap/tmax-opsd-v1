For idempotent port forwarding, before launching `socat`, check whether a forwarder is already listening — e.g. `pgrep -f "socat.*TCP4-LISTEN:8080"` or attempt a bind test — and only start a new background `socat TCP4-LISTEN:8080,bind=127.0.0.1,fork,reuseaddr TCP4:127.0.0.1:9090 &` if none is found; this is the main place naive scripts fail idempotency by spawning a duplicate every run. After launching, poll with a loop like `for i in $(seq 1 10); do nc -z 127.0.0.1 8080 && break; sleep 1; done` (or `curl -s -o /dev/null`) up to the 10-second budget rather than a fixed sleep. For the cron entry, read the existing crontab, filter out any line already containing `/home/user/record_capacity.sh`, then append exactly one fresh `* * * * * /home/user/record_capacity.sh` line and pipe back into `crontab -` — e.g. `(crontab -l 2>/dev/null | grep -vF 'record_capacity.sh'; echo '* * * * * /home/user/record_capacity.sh') | crontab -`, which guarantees exactly one instance no matter how many times the script runs.
