Split this into extraction, metadata parsing, hashing/renaming, and serving — validate each stage's output before moving to the next. Extract the tarball with `tar xzf`, then find all `.txt` files recursively (`find /home/user/extracted -name '*.txt'`). For metadata, since some files may be huge, never read a whole file — open it, seek to the start, and read only the first 500 bytes (Python `mmap` with a slice, or `f.read(500)` on a buffered stream both work; the key constraint is avoiding loading gigabyte files fully) and locate the line beginning exactly `Doc-Title: `, extracting the rest of that line as the title, then replace spaces with underscores. For hashing, treat `/app/doc_hasher` as an opaque tool — just invoke it as a subprocess per file and capture its 16-hex-char stdout; don't try to reimplement or guess its algorithm. Move/rename each file into `/home/user/organized_docs` as `<hash>_<title>.txt` exactly. For the dual-protocol service, run an HTTP server (e.g. `http.server` subclass or a simple socket-based one) bound to 0.0.0.0:8080 serving files from the organized directory by filename, and separately a raw TCP socket server on 0.0.0.0:9090 that, per connection, reads a line, looks up the matching `<hash>_...` filename (build a hash-to-filename dict once from the organized directory), writes it back with a trailing newline, and closes the connection — run both concurrently (threads or async) in one persistent background process.
