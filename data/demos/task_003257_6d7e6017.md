Approach this as a graph-chain-building problem, not a flat filter: first isolate all rows for the target db_name (with `awk -F, -v db="$1" '$2==db'`), find the `full` backup with the maximum `timestamp` among rows where `type=="full"` — that's your chain root. Then grow the chain iteratively: starting from a "valid IDs" set containing just the base backup's id, repeatedly scan the incremental rows for that db and add any `inc` row whose `parent_id` is already in the valid set (and isn't already added), appending it to an ordered chain list; repeat this scan until a full pass adds nothing new, since a single linear scan may miss a valid incremental if its parent appears later in the (numerically ordered) IDs but the file isn't guaranteed sorted by dependency order — this fixed-point iteration also naturally excludes orphaned incrementals whose parent chain doesn't trace back to the base (e.g. `missing_parent` or a parent outside the chain). Sum `size_bytes` for the base plus every chain member you collected.
