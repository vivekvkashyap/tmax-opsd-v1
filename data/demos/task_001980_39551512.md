In the C program, use `mmap` (`PROT_READ, MAP_PRIVATE`, with `fstat` to get file size) to map the whole archive read-only, then walk the buffer manually with a byte offset: read the 12-byte header as three little-endian `uint32_t`s (verify the magic equals `0xBACCBACC` as a sanity check), branch on `Type`, and for `Type == 2` write exactly `Length` payload bytes (the payload is NOT null-terminated, so use `fwrite(ptr, 1, length, out)` rather than treating it as a C string) followed by `\n`, then advance the offset by `12 + Length` to reach the next record — an off-by-one here is the most common bug. After running the compiled `extract_metadata`, use `awk -F'[:|]'` (or `sed` with capture groups) to pull out the ID, DATE, and STATUS fields from lines shaped `ID:XXXX|DATE:YYYYMMDD|STATUS:Z` and emit `XXXX,YYYYMMDD,Z` per line into the CSV. Finally `cd /home/user && sha256sum clean_metadata.csv > manifest.txt` so the checksum manifest records just the bare filename, not a full path.
