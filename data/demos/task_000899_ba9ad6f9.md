Treat this as a relational join-and-aggregate pipeline built from CLI tools, not a general-purpose script. Steps: (1) sort both CSVs on the join key and use `join` (or an `awk` associative-array lookup keyed by node_id -> department) to attach source_department and target_department to each edge; (2) filter out rows where source_department == target_department (internal collaboration); (3) group the remaining rows by source_department, summing weight per (source_department, target_department) pair; (4) for each source_department, pick the target_department with the highest summed weight as top_partner (break ties by any deterministic rule, e.g. first-seen or alphabetical) and also sum all its external weight as total_external_weight; (5) emit the final per-department objects sorted alphabetically by department name, using `jq -n`/`jq -s` or careful `awk` printf to build valid JSON — verify with `jq .` that the output parses. Watch for directionality: edges are source->target, so only aggregate collaboration weight going *out* of each department, and don't double count the reverse direction.
