Work file-by-file: for each language, first normalize to UTF-8 with `iconv -f <src-encoding> -t UTF-8`, redirecting into a temp file rather than trying to process the original encoding in place. Set `export LC_ALL=C.UTF-8` before any character counting so multi-byte characters count as one character each, not as bytes — this is the classic mistake (using `wc -c` or default locale `wc -m`/`awk length()` gives wrong results for non-ASCII text). Use `awk -F, 'NF>=2{print $1, length($2)}'` (with LC_ALL set) to build KEY→length maps for en.csv and each translated file. Then join en's length against fr's and jp's length per KEY (e.g. via `join` on sorted files, or an associative array in a single awk pass reading en first into an array then scanning the translated file), and for each row check `translated_len > 2*en_len` strictly. Emit matching rows as `lang,KEY`, concatenate the fr and jp results, and pipe through `sort` before writing to the output file. Verify by inspecting byte lengths of your intermediate files with `iconv` output to make sure conversion didn't silently drop or mangle bytes.
