Build the awk/jq pipeline in stages and inspect intermediate output at each step rather than writing one giant one-liner blind. Step 1: `tail -n +2 transactions.csv | awk -F, '$3 > 10'` to drop the header and filter low-value transactions. Step 2: aggregate with awk into a `sender,receiver,sum` table by keying on `sender FS receiver` in an associative array and summing amounts, printing at END. Step 3: from that aggregated edge list, compute per-node stats — a node's `out_deg`/`total_out` come from grouping edges where it's the sender (count of distinct receivers, sum of weights), and `in_deg`/`total_in` from edges where it's the receiver; make sure to enumerate the full node set (union of senders and receivers) so nodes with only incoming or only outgoing edges still appear with 0 for the missing side. Step 4: sort with `sort -t, -k4,4nr -k1,1` (numeric descending on total_out, then alphabetical) and take the top 3 with `head -n 3`. Step 5: pipe the final rows through `jq` using `-R`/`--arg`/`-s` to build strictly-typed JSON objects (numbers as numbers, not strings) and write pretty-printed with `jq .` to `top_nodes.json`.
