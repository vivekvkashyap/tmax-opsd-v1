Write a single Python script that does the whole pipeline in one pass. Load the config JSON first to get `output_dir` and the component->format mapping. Use `glob`/`os.walk` to recursively find files under the logs directory whose basename starts with `app` and contains `.log` anywhere. For each file, read line by line, and use a regex anchored at the start of the line like `^\[(\d{4}-\d{2}-\d{2} \d{2}:\d{2}:\d{2})\] \[(\w+)\] \[(\w+)\] (.*)$` — lines that don't match (blank, malformed, truncated by the race condition) must be silently skipped, not raise an exception. Group parsed entries by the component field, but only keep components that appear as keys in `component_formats`; drop everything else (e.g. UNKNOWN). Since the timestamp string format is already lexicographically sortable (ISO-like `YYYY-MM-DD HH:MM:SS`), sort each component's entries by the raw timestamp string. Create the output directory with `os.makedirs(..., exist_ok=True)`. For CSV output, use Python's `csv` module (not manual string joins) so quoting of embedded commas/quotes in messages is handled correctly per RFC 4180 — writing it by hand is the most common way this task goes wrong. For JSON, use `json.dump` with a list of dicts using keys `timestamp`, `level`, `message`. Name files exactly `<component>_logs.<ext>` in lowercase.
