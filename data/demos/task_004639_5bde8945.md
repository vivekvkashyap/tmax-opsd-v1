Build this as a clear sequential pipeline in Python (pandas is a natural fit for the resampling/rolling steps). Step 1: use a regex like `\[(\d{4}-\d{2}-\d{2} \d{2}:\d{2}:\d{2})\] \[UPDATE\] (\S+) : (\S+) -> (\S+)` applied line-by-line (or via `re.findall`) to pull only UPDATE lines, ignoring noise lines. Step 2: for each parsed entry compute `hashlib.sha256(f"{server_id}{config_key}{config_value}".encode()).hexdigest()`, and keep only the first occurrence per hash in file order (a dict keyed by hash, only inserting if absent, preserves "first wins"). Step 3: parse timestamps, floor to the minute with `dt.replace(second=0)` (or pandas `.dt.floor('T')`), group by (server_id, minute) taking the mean, then reindex each server's series to a complete per-minute range and forward-fill — do this per server_id independently, not globally. Step 4: apply a rolling mean with `min_periods=1` and window=3 on each server's filled series so early points average over whatever history exists. Step 5: for each deduplicated config change, floor its timestamp, look up the rolling value for that server/minute, and drop it if the server has no CPU data at or before that minute (i.e. the lookup would require backward-extrapolation, not just forward-fill). Round to 2 decimals, sort by original timestamp, and dump as JSON. Watch for the trap of applying rolling/ffill across all servers as one dataframe instead of grouping first — that will corrupt boundaries between servers.
