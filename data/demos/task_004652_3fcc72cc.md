Separate concerns: the bash script should only handle the raw HTTP socket protocol, and a Python helper should do all numeric work. Use `nc -l -p 8080` (or `socat TCP-LISTEN:8080,reuseaddr,fork`) in a loop that reads the request line, extracts `frame` and `rtol` from the query string with a regex or `cut`/`sed`, calls a Python script passing those as arguments, captures its stdout, and writes back a well-formed `HTTP/1.1 200 OK` response including `Content-Length` and a blank line before the body — a common failure is omitting required headers or the CRLF terminators so curl hangs or clients reject the response. In the Python helper, use OpenCV (`cv2.VideoCapture`, `set(cv2.CAP_PROP_POS_FRAMES, N)`) to grab the exact frame, convert to grayscale, and slice out the central 50% using `h//4:3*h//4, w//4:3*w//4` (not a naive percentage that rounds differently). Feed the mean intensity as the initial condition into `scipy.integrate.solve_ivp` for `dx/dt = -0.05x + 10*exp(-t)` over `[0, 5.0]`, passing the request's rtol and atol=1e-6, then format `res.y[0][-1]` to exactly 4 decimal places as the entire response body. Test end-to-end with `curl` before backgrounding the server with `&` or `nohup`.
