Replace the naive O(E^3) nested-loop triangle search with an indexed approach: after reading `edges.csv` with the `csv` crate, build a `HashMap<u32, HashSet<u32>>` (or `Vec` sorted per node) representing the outgoing-adjacency list, and separately a `HashMap<u32, usize>` (or reuse the adjacency set lengths) for out-degree counts. To detect directed triangles A->B->C->A efficiently, iterate each edge (A,B), then for each neighbor C of B (i.e. B->C edges), do an O(1) hash lookup to check whether C->A exists in the adjacency map — this turns the triangle search into roughly O(E * avg_degree) instead of checking all edge triples. Collect every node that appears in at least one such triangle into a HashSet to dedupe. Then, using the full-graph adjacency map (not just triangle-involved edges), look up each candidate node's total out-degree, sort candidates by out-degree descending with ties broken by node ID ascending, take the top 5, and serialize as a plain JSON array of integers via `serde_json` to `/home/user/results.json`. Build and time with `cargo build --release` to confirm it finishes well under 2 seconds on the 20k-edge graph.
