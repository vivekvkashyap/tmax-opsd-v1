Start by transcribing `/app/instructions.wav` (convert with `ffmpeg` if needed, then any available offline transcription tool) to recover the exact 7-letter motif and k-mer size — sanity-check the motif length is 7 and k is a small sane integer before hardcoding them. Filter the FASTA sequences by substring search for the motif first. For each surviving sequence, build a de Bruijn graph (k-mers as nodes, edges from overlaps), then get its stationary distribution via power iteration on the transition matrix (normalize rows, iterate `v = v·P` to convergence) rather than a full eigendecomposition, which is slower. For the JSD matrix, precompute all per-sequence distributions first, then parallelize only the O(N²) pairwise-JSD double loop with `#pragma omp parallel for` on the outer index — each (i,j) writes a distinct cell so there's no race and no critical section needed. Benchmark `OMP_NUM_THREADS=1` vs `=4` to confirm real speedup before finalizing.
