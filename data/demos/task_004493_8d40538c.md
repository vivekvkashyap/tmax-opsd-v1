Treat `/app/ts_aggregator` as a black box you must protect against, not something to reverse-engineer — you're only told it crashes on orphaned/invalid unicode escapes in JSON string values and on missing keys, so build defenses for exactly those failure modes rather than trying to fully understand the binary. Also verify all three required keys (`ts`, `sensor_id`, `msg`) are present, dropping lines missing any. For surviving lines, normalize `msg` to lowercase and strip everything except alphanumerics and spaces (e.g. `re.sub(r'[^a-z0-9 ]', '', msg.lower())`). Sort the survivors by `ts` (string or parsed datetime, consistent with ISO-8601 lexical order) and print one compact JSON object per line to stdout only — no debug prints, since this stream is piped straight into the aggregator.
