Structure the watcher as a polling loop: each second, list `/home/user/repo/incoming/`, and track a local set of filenames already handled so you never reprocess one. On seeing a new entry, first check if it is exactly `SHUTDOWN` — if so, break the loop immediately without touching the manifest or moving it. Otherwise compute `hashlib.sha256(open(path,'rb').read()).hexdigest()`, load the current `manifest.json`, insert the new key/value, then perform the atomic write pattern: write the full updated JSON to a temp file created in the *same directory* as `manifest.json` (e.g. via `tempfile.NamedTemporaryFile(dir=..., delete=False)`), flush/fsync it, then call `os.replace(tmp_path, manifest_path)` — never write directly to `manifest.json` in place, since a crash mid-write would corrupt it. Only after the manifest write succeeds, `shutil.move` the file into `processed/`. Be careful about read timing: a file may still be mid-write when first observed by your poll — consider checking file size stability across two polls before hashing, though for this test's simple `echo`-style writes a straightforward hash-on-first-sight will typically work.
