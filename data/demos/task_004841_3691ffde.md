First create the three directories and append the `export SOCKET_MAPPING_DIR="/home/user/proxy/run"` and `PATH` update lines to `/home/user/.bash_profile`, then also export them directly in your automation script (don't rely solely on `.bash_profile` sourcing, since non-login shells running the script may not pick it up). Compile `daemon.c` with `gcc /home/user/_service/daemon.c -o /home/user/bin/daemon`. In `start_stack.sh`: use `df -k /home/user | awk 'NR==2{print $4}'` (or similar) to get available KB and exit 1 if under 50000; extract the socket path from the `#define SOCKET_PATH "..."` line dynamically with `grep`/`sed` (e.g. `sed -n 's/.*SOCKET_PATH "\(.*\)".*/\1/p' daemon.c`) rather than hardcoding it; start the daemon in the background, `sleep 1` so it has time to bind/create the actual socket file before you symlink to it; then `ln -sf <extracted_path> "$SOCKET_MAPPING_DIR/upstream.sock"`; finally run `check_upstream.sh > /home/user/proxy/status.log`. The main pitfall is symlinking before the daemon has actually created the socket, or hardcoding the path instead of parsing it.
