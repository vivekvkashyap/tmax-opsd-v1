Build this bottom-up rather than jumping straight to `/solve`. First create the venv (`python3 -m venv /home/user/venv`) and install numpy/h5py/flask/requests/scipy into it; edit `/app/startup.sh` to `source /home/user/venv/bin/activate` before launching each service, and start both `data_server.py` and `processor_api.py` in the background with `&` (and `disown`/redirect output) so the script itself exits while the servers keep running — test this by re-running startup.sh and confirming both ports are still listening afterward. For the `/solve` handler: fetch the bytes from `http://127.0.0.1:8081/data` with `requests.get`, write them to a temp `.h5` file, then open it with `h5py` and read the `spectroscopy/raw` and `spectroscopy/vector` datasets. Compute `numpy.linalg.svd(M, full_matrices=False)`, zero out all singular values past the top 3 (build a filtered diagonal, not just slicing U/Vt), reconstruct `M_filtered = U @ diag(S_filtered) @ Vt`, then solve with `numpy.linalg.lstsq(M_filtered, b, rcond=None)` and return `x.tolist()` as JSON. A common mistake is truncating matrix dimensions instead of zeroing extra singular values — reconstructed M must stay the original shape.
