Write ci_build.py as a sequence of subprocess calls, checking the return code of each step and exiting non-zero immediately (fail fast) if any step errors—do not swallow exceptions silently. Steps in order: (1) os.makedirs for the build dir, (2) compile with something like `gcc -shared -fPIC -o build/libalgo.so src/algo.c`, checking subprocess.run(...).returncode, (3) create the venv with `python3 -m venv /home/user/tool/venv`, (4) install requirements using the venv's own pip (e.g. `/home/user/tool/venv/bin/pip install -r requirements.txt`)—never the system pip, since the whole point is isolation, (5) run main.py with the venv's python (`/home/user/tool/venv/bin/python src/main.py`), passing an env dict that copies os.environ and sets/prepends LD_LIBRARY_PATH to include the build directory (common bug: overwriting LD_LIBRARY_PATH instead of prepending it, breaking other library resolution). Check subprocess exit codes at every step and propagate failure via sys.exit(non-zero). Finally verify output.txt was actually created before declaring success.
