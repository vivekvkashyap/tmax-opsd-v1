Build the normalizer first, since the service pipeline depends on it. Read stdin as bytes; validate UTF-8 strictly (reject overlong encodings, unpaired surrogates, truncated sequences) before anything else, printing the exact error and exiting 1 on failure — this is where naive attempts fail most. Parse line by line, only accepting lines starting with literally "+ " or "- "; skip everything else. Lowercase keys with ASCII-only rules. For "+" lines, trim ASCII whitespace and enforce the byte-length limit on the trimmed value. Use an ordered map keyed by lowercased key so the last operation wins, then emit entries sorted alphabetically in the required "+ key=value" / "- key" format. For Part 2, configure nginx's proxy_pass from 0.0.0.0:8080/patch to 127.0.0.1:5000/ingest, start redis-server with --maxmemory 10mb, and launch Flask so it shells out to your compiled /home/user/normalizer and pushes its stdout into Redis. Test end-to-end with curl before considering it done.
