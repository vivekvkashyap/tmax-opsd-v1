Build the safety checks before writing any extraction or upload code, and test them against both a hand-made "evil" archive set and a "clean" set of your own so you can iterate quickly before the hidden corpus runs. For zip: use `zipfile.ZipFile` and inspect `ZipInfo` entries — reject any name starting with `/` or containing `..` after normalization, reject entries whose external_attr encodes a symlink mode, and compare `file_size` (uncompressed) vs `compress_size` per-entry and in aggregate to catch bomb ratios (>50x) and absolute-size blowups (>100MB) — check this BEFORE calling `extractall`, never after, since a bomb will exhaust disk/memory during extraction. For tar.gz: use `tarfile.open` and iterate `getmembers()`, checking `.isabs()`/`..` in `.name`, `.issym()`/`.islnk()` and whether `.linkname` resolves outside the extraction root, plus `.size` vs the compressed stream size. Only extract after every member passes. For accepted archives, stream files into 1MB chunks with `hashlib.sha256` computed per chunk, build the manifest dict, `json.dumps` it into Redis via `redis-py`, and upload chunks to MinIO with `boto3` or the `minio` SDK using the given credentials/bucket. Always print exactly "REJECTED"/"ACCEPTED" and use matching exit codes.
