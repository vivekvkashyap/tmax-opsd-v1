Work through this in order rather than jumping straight to code. First provision secrets: `mkdir -p /home/user/.secrets`, generate 32 random bytes (e.g. `openssl rand -base64 32` or Go's `crypto/rand`), write base64 output to `new_aes.key`, then `chmod 600` it. Separately run `ssh-keygen -t ed25519 -f /home/user/.ssh/new_service_key -N rotated_pass_2024` and `chmod 600` the private key. Next, write the Go program: read `legacy.enc`, hex-decode it, split off the first 12 bytes as the GCM nonce and the rest as ciphertext, then open an AES-GCM cipher with the literal old key string and call `Open` to recover the plaintext — treat this key as fixed, task-given data, not a secret to protect. Use `regexp` with a standard IPv4 pattern (four dot-separated 0-255 octets) against the decrypted text, dedupe matches, and join with commas — don't hardcode any IPs, derive them from decryption. For re-encryption, base64-decode your own new_aes.key, generate a fresh random 12-byte nonce, GCM-seal the IP string, prepend the nonce, hex-encode, and write to `ips_secured.enc`. Finally write `run.sh` that does `go build`/`go run` on the program with zero secrets passed as CLI args — read keys from their files inside the Go code instead.
