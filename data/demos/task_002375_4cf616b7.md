Two independent pieces: transcription and a Bash-only HTTP API. For transcription, install a local ASR tool (e.g. `pip install openai-whisper`, which needs `ffmpeg`) and run it on `/app/interactions.wav` to get raw text; then parse the transcript with a script that scans for the pattern `[Protein] [relation] [Protein]` per sentence, skipping boilerplate lines like "Dataset recording begins/End of recording" — store results as flat lines (e.g. `source|relation|target`) in a data file the Bash server can query with `awk`/`grep`. For the server: use `socat TCP-LISTEN:8080,fork,reuseaddr EXEC:./handler.sh` (or a `nc`-based accept loop) so each connection invokes a Bash script that parses the HTTP request line to get the path and query string, then answers with `awk -F'|'` filtering by protein and sorting with `sort` — do not shell out to Python/Node for the filtering/sorting logic itself, only for TCP plumbing if needed. Ensure JSON is built manually or via `jq -n` from the parsed fields, sorted correctly (`/interactions` by target, `/export` by source then target), with valid `HTTP/1.1 200 OK` and `Content-Type: application/json` headers. Only touch `/app/ready.txt` after the listener is confirmed up (e.g. verify with a local curl loop) to avoid a race with the verifier.
