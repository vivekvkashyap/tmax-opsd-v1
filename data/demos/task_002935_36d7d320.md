Build this as three loosely coupled components that you test individually before wiring together into one running service. First, get transcription working standalone: load the "tiny" Whisper model once at startup (loading it per-file is slow and wasteful) and confirm on `/app/audio_sample.wav` that you can produce a stripped, lowercased transcription string before hooking up any file watching. Second, implement the watchdog handler (using `watchdog`'s `Observer` with a recursive handler on the dropzone) so that on file-creation/move events for `.wav` files it transcribes, creates `organized/<transcription>/` if needed, and makes a hard link with `os.link` (not `shutil.copy` or a symlink — the verifier checks inode equality via `stat`) into that directory, preserving the original filename. Maintain an in-memory inventory dict (category -> list of absolute paths) that both the watcher and the API read/write, guarding it appropriately since the watcher and HTTP server run concurrently. Third, expose `/inventory` via Flask/FastAPI on 127.0.0.1:8000 returning that dict as JSON. Run everything as one long-lived background process (e.g. via `nohup ... &` or a systemd/tmux session), and manually copy the sample file into the dropzone subfolder to confirm end-to-end before leaving it running.
