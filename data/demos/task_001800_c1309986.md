This is a multi-stage task — solve Stage 1 fully before writing anything that depends on its output. For Stage 1, extract frames with `ffmpeg -i /app/san_monitoring.mp4 frame_%04d.ppm` (or similar), then write a small C program that reads each PPM (parse the header for width/height/maxval, then the raw pixel triples) and counts pixels satisfying R>200, G<50, B<50 per frame; find the 1-indexed frame with the maximum count — that integer is your `PEAK_USAGE_OFFSET`, derived from the video, not assumed. For Stage 2, write `quota_checker` to parse the simple `.alloc` key:value format, then apply the three rules in order: (a) exact-line membership check of USER against `/app/acl.txt`, (b) MOUNT must start with `/mnt/storage/` and must not contain `..` as a path component (careful with substring vs component matching), (c) SIZE must be `<= 100 - PEAK_USAGE_OFFSET` using the value you computed, not a guessed constant. Validate against both `/app/corpora/clean/` (must all pass) and `/app/corpora/evil/` (must all fail) before moving on. For Stage 3, init a bare repo, write a `pre-receive` hook that extracts pushed `.alloc` files (via `git archive` or reading blobs from the ref) and runs `quota_checker` on each, rejecting the push (non-zero exit, print to stderr) on any failure; configure nginx as a `git-http-backend` CGI proxy on port 8080 with `GIT_PROJECT_ROOT` and `GIT_HTTP_EXPORT_ALL` set, then test an actual clone/push over HTTP.
