Two distinct bugs — a link error and a runtime overflow. First try to compile both translation units together (`g++ analyzer.cpp utils.cpp -o /home/user/analyzer`) and read the linker error: it reports an undefined reference to `extract_bytes`. The cause is a signature mismatch between the forward declaration in `analyzer.cpp` and the real definition in `utils.cpp` (one takes `std::string` by value, the other `const std::string&`) — C++ name mangling makes these different symbols. Make the declaration exactly match the definition so they link. Second, once it builds, the total is wrong because the accumulator that sums the per-line byte counts is a 32-bit `int` and the true total exceeds 2^31-1, so it overflows. Change that accumulator's type to `long long` (or `unsigned long long`/`uint64_t`). Then run the binary on `/home/user/system.log`, and write only the printed total (digits, optional trailing newline) to `/home/user/result.txt`. Do not hardcode the number — let the fixed program compute it. Note the extractor reads the last space-separated field per line.
