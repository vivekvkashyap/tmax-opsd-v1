Use `pexpect` to drive the interactive shell rather than a plain `subprocess.run` (a raw pipe will not behave like a real TTY prompt loop, and `unshare` needs an interactive shell). Spawn the exact `unshare` command given, then after each command sent, call `child.expect()` against a prompt pattern (e.g. a regex matching `#` or `\$` at end of line, or better: set `PS1` to a unique marker string right after spawning and match on that) instead of relying on fixed `sleep()` delays — this is the most common cause of silent failures, since commands can race ahead of shell readiness. Send each setup step (`ip link set lo up`, create+configure `net0`, `mkdir`, `mount -t tmpfs`, `ip -j route > ...`) one at a time, waiting for the prompt between each. To get data out of the namespace, don't rely on `cp` sharing the same mount namespace — since `net0`/tmpfs live only inside the unshared namespace, either write the file directly to a path also visible on the host (bind-mount considerations aside, `/mnt/ramdisk` is only visible inside the container) or have the in-shell command `cat` the file and capture pexpect's output buffer to write it to `/home/user/routes_output.json` from the Python side. Finally send `exit` and wait for EOF/close before validating.
