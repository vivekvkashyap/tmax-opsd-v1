Use `ffmpeg`'s own filter chain to do extraction, grayscale conversion, and resize in one pass rather than hand-rolling frame math in Python — this avoids subtle interpolation/timing mismatches with the reference. A reliable approach: run `ffmpeg -i /app/experiment_feed.mp4 -vf "fps=4,scale=128:128:flags=bilinear,format=gray" frame_%04d.png` into a temp dir, then load the PNGs in chronological (sorted filename) order with PIL/numpy into a uint8 array of shape (N,128,128), and save with `np.savez_compressed('/home/user/dataset.npz', frames=arr)`. Double-check the scale filter explicitly requests bilinear (ffmpeg's default scale algorithm may differ) and that grayscale conversion happens via `format=gray` so pixel values match single-channel luma, not just channel-averaging. For chunking, read the saved npz back, slice `frames[i:i+50]` in a loop, and write each chunk with the same `frames` key, zero-padding chunk indices to three digits.
