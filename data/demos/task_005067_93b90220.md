Implement the Go tokenizer as a streaming reader over stdin: read in fixed-size chunks of exactly `320*240*3` bytes (use `io.ReadFull` so partial reads are retried, and treat `io.ErrUnexpectedEOF`/`io.EOF` on a short final read as "discard trailing partial frame, stop"). For each complete frame, index pixels as `idx = (y*320 + x) * 3` for R, with G and B at +1/+2, and sum only over the center region y in [60,180) and x in [80,240) (60 rows/80 cols margin on each side, giving the 160x120 center) — off-by-one errors in the margin bounds are the most common mistake here, so double check the half-margins (80 and 60) are subtracted symmetrically. Compute integer averages via truncating integer division (Go's `/` on ints already truncates toward zero for positive values, which is fine here since RGB bytes are unsigned). Build the 16-bit token by taking the top 4 bits of each averaged channel and packing as `R<<8 | G<<4 | B`, then write it as little-endian via `binary.Write(os.Stdout, binary.LittleEndian, token)`. Since this is checked by fuzzing against a hidden oracle across many random byte lengths, test your binary yourself with truncated/empty/multi-frame random inputs to confirm frame-boundary handling before wiring up the ffmpeg pipeline script.
