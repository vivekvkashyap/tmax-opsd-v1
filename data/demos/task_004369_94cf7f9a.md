Install the C protobuf toolchain first (`apt-get install -y protobuf-c-compiler libprotobuf-c-dev`), then generate C bindings for both schemas with `protoc --c_out=. -I/home/user/schemas /home/user/schemas/v1.proto` and the same for `v2.proto` — this gives you `TelemetryV1`/`TelemetryV2` structs and `_unpack`/`_pack`/`_init` functions to call from `migrate.c` rather than hand-rolling wire-format parsing. In your program: read the raw bytes of `/home/user/data/input_v1.bin`, unpack with `telemetry_v1__telemetry_v1__unpack`, copy `device_id` and `temperature` straight across, and map `status_code` with a simple string comparison chain (`strcmp` against `"OK"` and `"ERR"`, default to the `UNKNOWN` enum value) into the v2 struct's `status` field, and hardcode `timestamp = 1710000000`. Pack the v2 struct to a buffer sized via `..._get_packed_size` and write those exact bytes to `output_v2.bin`. Write `migration.log` as a single line with the device id and the integer value of the status enum you assigned (not the string). Compile with `gcc migrate.c -o migrate $(pkg-config --cflags --libs libprotobuf-c) $(protoc-c) ...` (link the generated `.pb-c.c` files too), run it, and spot-check `output_v2.bin` with `protoc --decode=TelemetryV2 -I/home/user/schemas v2.proto < output_v2.bin` to confirm the fields match before finishing.
