This is graded by fuzz-equivalence against a hidden reference, so implement the spec's exact arithmetic order rather than reusing scikit-learn's batch TF-IDF (which is precisely the leaky behavior you're asked to replace). Write `online_tfidf.py` to read stdin line by line, maintaining persistent state across lines within one process run: a total-document counter N and a dict of document frequencies DF. For each line: lowercase and whitespace-split into tokens; if empty, print `0.0000` and move on without touching N or DF. Otherwise increment N first, then for every *unique* token in this document increment its DF by 1 (this happens before computing TF-IDF for this same document — the current document counts toward its own IDF, which is what makes it causal/online rather than a leak). Compute per-token TF as raw count/doc length, IDF as `ln(N/DF(t)) + 1.0` using `math.log`, multiply, square each weight, sum, sqrt for the L2 norm, and print with `"{:.4f}".format(...)`. For the audio phase, run `whisper /app/dataset/interview.wav --model tiny` and pull the plain transcript text (ignore the timestamped/JSON variants), splitting into one line per segment for transcription.txt; then pipe that file through your script into features.txt.
