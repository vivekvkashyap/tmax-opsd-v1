Since `/app/bin_to_json` is a stripped opaque tool, don't try to reverse its internals — just treat it as a black-box CLI you invoke per file and consume its stdout. Extract `raw_logs.tar` first, then in `compact_logs.py` iterate the resulting `.binlog` files, run each through `subprocess.run(["/app/bin_to_json", path], capture_output=True)`, split stdout on newlines, `json.loads` each line, and skip any record where `record.get("level") == "DEBUG"`. Stream the surviving lines into a gzip file rather than buffering everything in memory unnecessarily, and critically open the gzip writer with the maximum compression level explicitly (e.g. `gzip.open(path, "wt", compresslevel=9)`) — the default compression level is not maximal and will leave the file noticeably larger, which is what the size-threshold check is actually testing. Write each JSON record as its own line (newline-delimited) inside the gzip stream so the output remains valid NDJSON when decompressed. After running, sanity-check by decompressing and confirming no DEBUG records slipped through and that `gzip.open(...).read()` parses cleanly line by line, then check the resulting file size directly with `ls -la` or `os.path.getsize`.
