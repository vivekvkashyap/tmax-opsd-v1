Build this as four independent, testable pieces before wiring them together. First write the mock SMTP server (Python's `smtpd`/`aiosmtpd`, or a minimal socket-based SMTP responder that just accepts DATA and appends the raw message to `/home/user/email_out.log`) listening on 127.0.0.1:10025 — test it alone with `sendmail`/`smtplib` before anything else depends on it. Next write `monitor.py` as a polling loop (sleep 1s) that reads the integer from `/home/user/cpu.txt` (default 0 if missing), compares against `ALERT_THRESHOLD` read from the environment, and tracks an `already_alerted` boolean so it only fires once per threshold-crossing (reset the flag when the value drops back to <= threshold) — send mail via `smtplib.SMTP('127.0.0.1', 10025)` with `sendmail`. Then `.monitor_profile` just exports the two env vars. Finally `supervisor.sh` sources the profile, backgrounds the SMTP server and `monitor.py`, then loops checking `kill -0 $pid` (or `wait`) on the monitor process and relaunching it within 2 seconds if it's gone — this restart loop is what the verifier specifically tests by killing monitor.py mid-run, so make sure the supervisor's watch loop has a short poll interval (well under 2s) and correctly re-captures the new PID after each restart.
