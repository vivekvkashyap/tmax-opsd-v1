First load `nodes.csv` and `edges.csv` into in-memory structures (an adjacency list keyed by node id, since edges are directed — don't add reverse edges). Implement Dijkstra's algorithm from the given start node over that adjacency list to get shortest-path cost to every reachable node; a naive attempt often forgets edges are directed or mishandles unreachable nodes (they should simply not appear in the candidate set). Then apply the filters in this exact order: exclude the start node itself, exclude nodes whose `type` matches the excluded type, and keep only nodes with cost <= max_cost (inclusive of equality). Sort survivors strictly by `name` descending (alphabetical, not by cost). Paginate with 1-indexed page_number using zero-based array slicing (`page_number - 1`) times `page_size`. Wire the specific query values (start node, max_cost, exclude type, page_size, page_number) as either CLI args or hardcoded parameters, run the binary, and write the resulting page as a single-line, non-pretty-printed JSON array of `{id, name, cost}` objects to `/home/user/output.json`. Double check the JSON is compact — no extra whitespace — since verification does exact string comparison.
