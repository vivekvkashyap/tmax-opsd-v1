Split this into extraction, transform, and serving steps. Extraction: open the video with cv2, iterate frames in order, convert each to grayscale and take the mean pixel value as brightness; derive each frame's timestamp as the base timestamp plus `frame_index * 40ms`, formatted ISO8601 with millisecond precision and a trailing `Z`. Transform: compute the rolling average of brightness with a window of 5, but for indices 0-3 average only the frames seen so far (not a fixed-size window with NaN padding) — this is where naive pandas `.rolling(5).mean()` gets it wrong, since it produces NaN for the first frames instead of a partial-window mean; use `min_periods=1`. Round every output value to exactly 2 decimals with standard rounding. Reshape to long format: two records per timestamp (`brightness`, `rolling_avg`), sorted by timestamp then metric name alphabetically. Serving: run a Flask/FastAPI app bound to `127.0.0.1:8080` at `GET /api/v1/telemetry`, reject requests missing the exact `Authorization: Bearer TRL-992-DATA` header with 401, filter by inclusive `start`/`end` query params, and launch it as a background/daemon process (not blocking foreground) so it stays reachable for the verifier's HTTP requests.
