Work in stages and verify each before moving on. First, write the C++ Base64Url decoder: split the JWT on `.`, take the payload segment, replace `-`→`+` and `_`→`/`, pad with `=` to a multiple of 4 length, then decode using a standard 64-char alphabet lookup table you build yourself (index each of the 4 base64 chars in a group, combine 6-bit values into 3 bytes). Do the same for the header if needed. Once you have the decoded JSON payload as a string, locate the substring `"exfil_path":"` and extract the value up to the next unescaped `"` — a manual scan is fine since you're told to avoid JSON libraries. Print that path so you can inspect it. Compile with a plain `g++ decoder.cpp -o decoder`, run it, and capture the printed path. Then run `openssl x509 -in <that path> -noout -fingerprint -sha256` and redirect the exact output line into the fingerprint file — do not reformat or strip the `SHA256 Fingerprint=` prefix. Watch for the empty signature segment after the second `.` in the JWT (alg=none tokens have no signature bytes) — don't let your splitting logic choke on a trailing empty field.
