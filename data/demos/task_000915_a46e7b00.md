Use `find /home/user/configs -type f -mtime -1` (or `-newermt "24 hours ago"`) to select recently modified files rather than checking all files — this is what separates the *_new files from the *_old ones. For each matched file, dispatch by extension: for .json use a JSON parser (`jq -r .version file` or python -c) rather than a fragile grep, since the version key may not always be on its own line; for .csv, match the row whose first column is exactly `version` (not a substring) and take the second field, e.g. `awk -F, '$1=="version"{print $2}'`; for .xml extract text strictly between `<version>` and `</version>` with a targeted regex/sed, not a generic tag-stripper that could grab nested content. Append `basename:version` lines using `flock` around the write, e.g. `( flock -x 200; echo "$name:$version" >> /home/user/config_inventory.txt ) 200>/home/user/config_inventory.txt.lock` — using a separate lock file (not the target file itself) avoids truncation/append races. Chmod +x the script and run it once, then check the inventory file only contains lines for the freshly modified files, not the old ones.
