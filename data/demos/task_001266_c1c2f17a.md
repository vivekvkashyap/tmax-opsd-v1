For shortest path, write a recursive CTE that builds candidate paths from the source, tracking the accumulated path (list of ids) and depth, stops as soon as it hits the target, and orders by path length so you take the minimum — a naive CTE without a stop condition or ordering may explore cycles or return a non-shortest path when decoy edges exist. Reconstruct the ordered id sequence and join back to `papers` for titles. For the HTTP service, use Python's built-in `http.server` bound to `127.0.0.1:8080`, compute the path once (or per-request) and serve it at `GET /citation-path` as JSON matching the exact schema (step starts at 1, path_length = edges = nodes-1). Launch it so it keeps running (e.g. background with `&` or `nohup`) rather than exiting after one request.
