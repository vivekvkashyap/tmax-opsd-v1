This is a dynamic-linking interposition problem — solve it by overriding only the one symbol you need, letting the linker resolve everything else normally. Write a small `mock.c` that defines just `const char* sm_get_version()` returning `"2.1.0"` (do not redefine `sm_process`, since that must still come from the real `libstatemachine.so`), compile it as a shared object with `gcc -shared -fPIC -o libmock.so mock.c`, and confirm via `nm -D` or by running the app that your mock's symbol takes priority. Run the target with `LD_PRELOAD=./libmock.so ./app` first without valgrind to confirm the app prints that it's taking the fast/V2 path — if it still shows the slow path, your preload isn't being picked up (check the rpath/library search order) before wrapping it in valgrind. Then run `LD_PRELOAD=./libmock.so valgrind --leak-check=full ./app`, capturing full output, and read the "definitely lost" summary line carefully — it reports both a byte count and block count, you want the byte count. Write exactly that integer (no extra text, trailing whitespace is fine) to `/home/user/project/leak_size.txt`.
