The key insight is that `/app/artifact_packer` deduplicates data by inode, not by symlink target or filename, so your dedup step must physically replace duplicate files with hard links (same inode) rather than just noting they're identical. In your script: hash every file in `/home/user/raw_artifacts/` (e.g. `md5sum` or `sha256sum`), group by hash, and for each group beyond the first file, `rm` the duplicate and `ln` (hard link, not symlink) it back to the kept original so they share an inode. Next, `gzip` every `*.meta.txt` file to produce `*.meta.gz` (gzip's default behavior renames and removes the original — verify this matches the requirement) and remove the leftover `.meta.txt` if any remain. Then build `/home/user/curated/{binaries,docs,metadata}/` and populate them with symbolic links (`ln -s`) pointing back into `/home/user/raw_artifacts/`, routed by extension. Finally run `/app/artifact_packer /home/user/curated /home/user/final_archive.pack` unmodified and check the resulting size with `stat -c %s` against the threshold — if it's still too large, verify your hard-linking actually reduced distinct inode count with `ls -i` or `stat` before assuming the packer is broken.
