In the Go program, walk `/app/raw_data/` recursively, and for each file read only the first F bytes for the header (avoid loading whole files just for this) plus stream the full file through `sha256.New()` for the checksum. Fan out file processing across goroutines with a worker pool, and protect the shared manifest map (header-hex -> []paths) with a `sync.Mutex` around each insert; serialize/write the manifest JSON once after all workers finish (via `sync.WaitGroup`). For deduplication, group files by full SHA-256: write the actual bytes into the zip for the first file of each hash group, then for every subsequent duplicate add a zip entry whose mode bits mark it as a symlink (`os.ModeSymlink` in the `FileHeader.SetMode`) with its content set to the relative/absolute path of the first file — this is what shrinks the archive far below naive compression. Run the program and confirm both `manifest.json` and `organized.zip` exist and the zip is well under the target size.
