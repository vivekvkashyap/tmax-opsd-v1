Build the pipeline as a sequence of composable text transforms, verifying each stage against the raw input before chaining. Use `sed`/`grep -oE`/`perl -pe` with a regex like `\\\\x([0-9A-Fa-f]{2})` to find each `\xNN` escape and rewrite it as `\u00NN` — do this as a line-oriented substitution (`s/\\\\x([0-9A-Fa-f]{2})/\\\\u00\\1/g`) since a naive approach might also mangle escapes that aren't hex-pair form. For the null `user_id`, pipe through `jq` (or a sed pass before/after JSON parsing) replacing bare `null` specifically in the `user_id` field value with the string `"UNKNOWN"` — target the field by key, not any `null` in the line. For email masking, match `"email":"([^"@]+)@` and replace the captured local-part with `***`, keeping the domain intact — use `jq -r`/`sed` with capture groups rather than blindly truncating. After all three transforms, validate every output line individually with `jq -e . <<< "$line" > /dev/null` before writing it to `clean_logs.jsonl`, so you catch any transform that broke JSON validity, and confirm line counts match input.
