Structure this as a clear pipeline of graph filtering, aggregation, and stable multi-key sorting in C, testing each stage before adding pagination. Steps: (1) parse `nodes.csv` into an array of `{id, year}` and keep only those with `year >= min_year`; build a lookup (e.g. sorted array + binary search, or a simple hash) from id to "is in subgraph". (3) Sort the filtered node list by in-degree descending, tie-broken by id ascending (`strcmp`) — implement this as a custom comparator passed to `qsort`, not two separate sorts, since qsort isn't stable. (4) Paginate: with 1-indexed page_number, the starting index is `(page_number-1)*page_size`; print `min(page_size, remaining)` rows as `id,in_degree`, one per line, and print nothing if the requested page is beyond the array bounds.
