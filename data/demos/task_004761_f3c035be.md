Four separate deliverables, build and test each in isolation. (1) `config_sanitizer.c`: read stdin line by line; for each non-empty line, first reject outright if it contains any shell metacharacter (`;&|$><` etc. — scan raw bytes before any parsing); then match strictly against only two patterns — `useradd <lowercase-alnum, ≤16 chars>` and `ip route add <ipv4>/<cidr> via <ipv4>` — reject anything else, and specifically reject the forbidden gateway `10.99.99.99`; exit 0 only if every line passes, else exit ≥1. Build a small local corpus of clean/evil lines yourself to test this before trusting the hidden adversarial corpus. (2) For `/app/config_decoder`, black-box it: feed sample base64 text via `-i <file>` or stdin and diff output to confirm it's base64-decode-then-ROT13 (or reverse-engineer with `strings`/`objdump` if black-boxing is ambiguous) — you need this exact transform to correctly encode `.enc` test fixtures in the git hook and to understand what your hook must call. (3) Git hook: in `pre-receive`, read stdin refs, use `git diff --name-only` between old/new revs to find `.enc` paths, `git show <rev>:<path>` each blob into `/app/config_decoder`, pipe that into `/home/user/config_sanitizer`, and reject the whole push (non-zero exit, no `git update-ref`) if any file fails. (4) `provision_daemon.sh`: read the given plaintext file line by line and append `[SUCCESS] Applied: <line>` to `/home/user/provision.log` for each valid command line, then `chmod +x` it.
