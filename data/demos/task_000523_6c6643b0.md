Build the pipeline as discrete, testable stages inside your bash script. First extract frames from the video (`ffmpeg -i <video> -vf fps=<N> frames/out%04d.png`) and OCR each frame in order with `tesseract`, concatenating the recognized text while stripping whitespace/newlines and any obviously non-base64 OCR noise characters (keep only `[A-Za-z0-9+/=]`) before joining — OCR errors are the main source of failure, so consider per-frame `tesseract --psm` tuning and only keep frames whose text looks like a valid base64 chunk. Base64-decode the joined string to get the encrypted payload. For safe key extraction, run the malware under `bwrap --unshare-net --ro-bind / / --tmpfs /tmp ...` so it cannot reach a C2 and instead prints `KEY:`/`IV:` to stdout on its network-check failure; parse those two hex values from the captured output, not from any hardcoded guess. Decrypt with `openssl enc -d -aes-256-cbc -K <key> -iv <iv> -in payload.bin -out payload.txt` (adjust flags if the payload lacks the standard salt header). Finally, grep/parse the decrypted HTTP requests for the `Session-Token=` value inside `Cookie:`/`Set-Cookie:` headers and print each extracted token on its own line, since the grading metric rewards exact per-token matches across multiple held-out video/binary pairs.
