Build this as three separate, independently testable stages connected by pipes — debug each stage against sample input before chaining. In `normalize.py`, read JSONL from stdin line by line, apply to `raw_text`: strip leading/trailing whitespace, collapse internal whitespace runs to a single space with `re.sub(r'\s+', ' ', text)`, then translate typographic quotes to ASCII quotes via a small mapping (curly double quotes and guillemets → `"`, curly single quotes → `'`) — do this replacement before or after whitespace collapsing consistently, and preserve all other JSON fields unchanged, emitting one JSON object per line. In `aggregate.py`, group records by `(lang, id)`, and within each group pick the winner by `contributor_score` descending then `timestamp` descending (a simple `max()` with a tuple key does this in one pass); emit rows via Python's `csv.writer` (which handles quote-escaping for you, e.g. doubling embedded `"`), sorted by `lang` then `id` ascending, with header `lang,id,normalized_text`. In `pipeline.sh`, start with `set -e`, then `cat raw_translations.jsonl | python3 normalize.py | python3 aggregate.py > final_translations.csv`; `chmod +x` it and run it.
