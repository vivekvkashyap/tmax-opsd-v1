Fix the ABI issue first: since `libmathops.c` is compiled as plain C, its `compute_mean` symbol has no C++ name mangling, so `processor.cpp` must declare it inside `extern "C" { double compute_mean(double* data, int n); }` (or `extern "C" double compute_mean(...)`) before calling it — a bare C++ declaration will fail to link against the C-compiled symbol. Next, implement the CSV parsing (skip the header line, split on comma, parse the second column as double into a `std::vector<double>`), then for every window of 3 consecutive elements (`i` from 0 to n-3) call `compute_mean(&data[i], 3)` and collect the results, sort descending with `std::sort` and a reverse comparator, and write each with `std::fixed << std::setprecision(2)` one per line to `output.txt`. For `build.sh`, compile the C file with `gcc -shared -fPIC -o libmathops.so libmathops.c`, then compile `processor.cpp` with `g++ -o processor processor.cpp -L. -lmathops -Wl,-rpath,'$ORIGIN'` (or set `LD_LIBRARY_PATH` at runtime) so the resulting `./processor` finds `libmathops.so` in the same directory without manual environment tweaking.
