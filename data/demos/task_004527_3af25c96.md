For the final packaging, don't use Python's zipfile for splitting — shell out to the `zip` CLI's own multi-part support: `zip -r error_logs.zip /home/user/processed_logs -s 20k` (or split an existing zip with `zip -s 20k in.zip --out error_logs.zip`) run from inside `/home/user/archive_out/`, which produces `error_logs.zip` plus `.z01`, `.z02`, etc.
