Write `catalog.proto` with `syntax = "proto3";` and message `Product { int32 id = 1; string slug = 2; bool in_stock = 3; }`, then use `protoc-c --c_out=. catalog.proto` (wire this into the Makefile's `all` target as a prerequisite step) to generate `catalog.pb-c.c`/`catalog.pb-c.h` before compiling `api_endpoint.c`. In the C program, include the generated header, initialize the message with `PRODUCT__INIT` (or the equivalent init macro protoc-c generates), set the three fields, get the packed size with `product__get_packed_size`, allocate a buffer, and call `product__pack`. Print the HTTP header text `Content-Type: application/x-protobuf` followed by a CRLF blank line (`\r\n\r\n`) to stdout using `fwrite`/`printf`, then `fwrite` the raw packed bytes immediately after — be careful to use binary-safe writes for the payload (not `printf("%s")`, which would stop at embedded NUL bytes or misinterpret binary data). In the Makefile, link with `-lprotobuf-c` when compiling both `api_endpoint.c` and `catalog.pb-c.c` into the `api_endpoint` binary. Finally run `make` then `./api_endpoint > response.bin`, and manually inspect `response.bin` (e.g. `xxd`) to confirm the header/body split and binary payload look correct before considering it done.
