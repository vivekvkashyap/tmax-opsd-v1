Start by transcribing `/app/voicemail.wav` — use `whisper` (or ffmpeg to extract audio plus a speech-to-text tool) to get the spoken text as a literal transcript; do not guess at the byte offset, encoding, or replacement rule, they must come from what you actually hear/read in the transcript. Listen carefully for numbers spoken as words (e.g. "five hundred and twelve") and convert them to digits, and for the codec name (spoken like "CP zero three seven" maps to a Python codec name such as `cp037`). Then write `archive_extractor.py` to: open the file in binary mode, `seek()` to the discovered offset, `read()` to EOF, `decode()` using the discovered codec, apply the exact string-replace rule mentioned (case and spelling matter), and `print(..., end='')` so no extra trailing newline is added beyond what's in the payload. Test against a few hand-crafted `.dat` files (encode a string with the same codec, pad the front with junk bytes to the right length) before trusting it — since the grader fuzzes with random binary noise before the payload, off-by-one offset errors or wrong codec choice will show up immediately as garbled output.
