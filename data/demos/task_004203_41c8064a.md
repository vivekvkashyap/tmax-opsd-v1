Work through this as a pipeline with clear checkpoints rather than one long script. First extract the outer tar.gz, then glob for `*.zip` under the extraction root and extract each into a directory named after its stem, deleting the zip only after a successful extract (check the zip's return code). For the log cleanup, open each `.log` file, filter out lines containing the exact substring `DEBUG`, and do a literal (not regex) replace of `WARN` with `WARNING` — be careful that a naive regex replace of `WARN` could accidentally match inside `WARNING` if you run it twice or in the wrong order, so replace before any further processing and only once. Walk the entire extracted tree (os.walk or pathlib.rglob) to hash every regular file with hashlib.sha256, reading in chunks for large files, and build a dict keyed by the path relative to the extraction root (use consistent separators, e.g. forward slashes). For the manifest write, the key requirement is atomicity: write JSON to `final_manifest.json.tmp` in the same directory as the destination, flush and fsync before closing, then use `os.rename` (or `Path.replace`) to atomically move it to `final_manifest.json` — never write directly to the final path or leave the `.tmp` file behind. Verify no zip files remain afterward.
