Use `ln -sfn /home/user/releases/v2 /home/user/app_current` (the `-f` and especially `-n` flags matter: without `-n`, re-running on an existing symlink creates a nested link *inside* the target directory instead of replacing it) and `mkdir -p /home/user/logs` — both naturally idempotent. In `start_worker.sh`, export `TZ=Europe/Berlin` and `LC_ALL=en_US.UTF-8`, `cd /home/user/logs` (so the worker's relative `worker_output.log` lands there), then launch `python3 /home/user/app_current/worker.py` in the background with stdout/stderr redirected to `/home/user/logs/worker_output.log` — note the worker itself may also write that file directly, so redirection and cwd both need to be correct for the content checks to pass. Write `start_proxy.sh` as a simple `socat TCP-LISTEN:8080,fork TCP:127.0.0.1:8081 &` wrapper, executable. Wrap the whole thing in `deploy.sh`, executable, and actually run all three scripts yourself so the log file and other artifacts exist for verification, re-running `deploy.sh` once more to confirm no duplication or errors.
