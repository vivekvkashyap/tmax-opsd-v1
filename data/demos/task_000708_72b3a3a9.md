First determine which files changed: extract `/home/user/configs_old.tar.gz` to a temp/scratch directory (don't touch the original archive) and `diff` or `cmp` each file against its counterpart in `/home/user/configs_current/` by basename — only files present in both with differing content count as "changed" for this task. For each changed file, edit it in place with `sed -i` (e.g. `sed -i 's/DEBUG_LEVEL=1/LOG_LEVEL=debug/; s/SERVER_PORT=8080/SERVER_PORT=9000/' file`), applying both substitutions even if only one pattern is present. Build the bzip2 tarball with `tar -cjf /home/user/configs_patch.tar.bz2 -C /home/user/configs_current file1 file2 ...` so paths are stored flat at the archive root rather than nested — a common mistake is tarring from the parent directory and getting `configs_current/web.conf` paths instead of `web.conf`. Write the sorted basenames (one per line) to `/home/user/patch_log.txt` with `sort`. Diff against the extracted old archive BEFORE editing files in place, since editing first would make everything look "changed."
