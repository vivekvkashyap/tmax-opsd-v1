Work in clear pipeline stages so each transformation is independently checkable. First, sanitize the raw file with `sed` to strip the malformed `\uX` + exactly 3 digits pattern (a regex like `\\uX[0-9]{3}` — remember to escape the backslash for the shell/sed) into a temp file or via a pipe, BEFORE any JSON parsing, since `jq`/`json` will choke on the malformed escape otherwise. Second, use `jq` to pull `time`, `device`, `temp`, `humidity` from each cleaned line, and truncate the timestamp to the top of the hour by replacing the minutes/seconds portion (e.g. a `sub` with a regex targeting `:MM:SSZ$` and replacing with `:00:00Z`) — do this inside jq or with `sed`/`awk` on the extracted field, not by string-slicing blindly, in case timestamp formats vary. Third, reshape wide-to-long: for each record emit two rows, one with metric="temp"/value=temp and one with metric="humidity"/value=humidity, producing CSV via `jq -r ... | @csv` then stripping the quotes jq adds around numbers/strings as needed. Finally pipe through `sort -t, -k1,1 -k2,2 -k3,3` for the three-key sort and prepend the header line. Verify the final file with `cat -A` to catch any hidden quoting/whitespace artifacts.
