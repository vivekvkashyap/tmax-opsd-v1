The last record in the file always gets `end_sec = start_sec+3` with no trailing silence record. Pass 3: emit each record (original + inserted silence, in chronological order) as one compact JSON object per line with the exact key order `start_sec, end_sec, speaker, text`, writing to the specified `.jsonl` path.
