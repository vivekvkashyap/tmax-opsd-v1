Then compute Euclidean distance to every other row in a single awk pass: read the target values as awk variables (pass them in with `-v`), skip the row where `$1=="target_842"`, and for each other row compute `sqrt(sum((target_i - v_i)^2))` across the five columns, printing `distance,item_id`. Pipe that through `sort -t, -k1,1g` (numeric/general sort on the distance field) and `head -n 5`, then use `cut`/`awk` to extract just the `item_id` column in that sorted order into `/home/user/top5_baseline.txt`. Watch two easy mistakes: use general/numeric sort (`-g` or `-n`) not the default lexical string sort, which would misorder floating-point distances, and make sure you exclude the target's own row (distance 0) rather than accidentally letting it rank first.
