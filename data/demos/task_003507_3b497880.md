Compile the .proto with `python -m grpc_tools.protoc -I. --python_out=. graph.proto` (install `grpcio-tools`/`protobuf` via pip first) to generate `graph_pb2.py`, then in your script `import graph_pb2`, instantiate a `Graph()`, and call `.ParseFromString()` on the raw bytes read from `graph_data.bin`. For the ordering logic, implement a standard Kahn's-algorithm topological sort: build an in-degree/dependency map from each node's `depends_on` list, maintain a set of "ready" nodes (all dependencies already emitted), and at each step pop the alphabetically smallest ready node (use a min-heap or `sorted()` on the ready set) rather than just iterating the node list in file order — this tie-breaker is the part naive implementations get wrong. After emitting a node, check whether it unlocks any dependents and add them to the ready set. Write the resulting ID sequence as a single comma-separated line to `/home/user/order.txt` with no extra whitespace.
