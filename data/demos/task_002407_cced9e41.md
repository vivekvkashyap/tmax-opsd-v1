Three independent deliverables feeding into one system — solve them in order since later phases depend on values from earlier ones. Phase 1: transcribe `/app/architect_voicemail.wav` (e.g. with a local speech-to-text tool such as `whisper`) and carefully parse the spoken numbers for the WebSocket PORT and the MEMORY_LIMIT in MB — listen for numbers spoken as words and convert them correctly. Phase 2: write `patch_sanitizer.py` to parse a unified diff's `---`/`+++` file header lines, and for each referenced path, reject (exit 1, stderr reason) if it's absolute, contains `../` that would escape the working root, or touches a dot-directory like `.git/`; otherwise accept only if every touched path normalizes to staying inside `src/` (exit 0). This will be fuzzed against many clean/evil patches, so use real path normalization, not a shallow substring check. Phase 3: build `build_server.py` as a WebSocket server bound to the transcribed PORT, that writes incoming patch text to a temp file, invokes the sanitizer as a subprocess, and reports pass/fail; wrap the message-handling loop with `tracemalloc` to track peak memory, and have `benchmark.py` fire 100 mixed messages then log `Peak Memory: <X> MB` to the report file, keeping X under the transcribed limit.
