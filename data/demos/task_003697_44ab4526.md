Build this as an ordered bash script that both configures and launches everything, since the verifier checks live process/network state, not just file contents. Set up the SSH tunnel as a real background process with `ssh -f -N -L 9091:127.0.0.1:8081 127.0.0.1` (the `-f` backgrounds it after auth so you can still capture its PID via `pgrep -f "9091:127.0.0.1:8081"` right after, since `-f` forks). Write `/home/user/restore_env.sh` with plain `export DB_PORT=9091` / `export RESTORE_ENV=testing` lines. Launch `/home/user/bin/db-restore.sh &` first and capture `$!`, then `source /home/user/restore_env.sh && /home/user/bin/app-restore.sh &` and capture its `$!` — write both PIDs plus the tunnel PID (found above) to `/home/user/service.pids`, DB first, then app, then tunnel, one per line. Write `/home/user/logrotate.conf` with `/home/user/logs/*.log { size 10k rotate 5 compress missingok }` syntax, then run `logrotate -s /home/user/logrotate.state /home/user/logrotate.conf` explicitly (non-root requires the `-s` state-file flag) at the end of the script, and verify afterward that `.log.1.gz` files were actually produced given the pre-padded log sizes.
