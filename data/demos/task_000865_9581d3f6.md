Set up the Rust project first with `cargo new diffusion_data`, add `csv`, `ndarray`, and `hdf5` crates in Cargo.toml, and confirm `libhdf5-dev` is present before coding. Read the CSV as a matrix of f64 (10 rows x 50 columns), one initial profile per row. For each row, implement the explicit FTCS update `u[j] += alpha*dt/dx^2 * (u[j+1] - 2*u[j] + u[j-1])` for interior points only (j=1..48), always re-clamping the fixed boundary values u[0]=0 and u[49]=0 each step (they should already stay 0 if never updated, but do not let the loop touch them). Compute dx as 1.0/49.0 exactly as specified. Loop until the max absolute difference between the new and old vectors is strictly below the given tolerance, then store that row's converged vector. A common naive mistake is updating boundaries or using dx=1/50 instead of 1/49 — verify convergence per-row (each row converges independently, some faster than others). After collecting all 10 converged rows into a (10,50) f64 array, write it as a single HDF5 dataset named `converged_states` at the file root. Finally write `run_pipeline.sh` to `cargo build --release` and run the resulting binary, then `chmod +x` it.
