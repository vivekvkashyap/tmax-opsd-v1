First recover the secret: run `strings /app/auth_daemon | less` (or grep for likely key-shaped strings) to locate the hardcoded HMAC key embedded in the binary's rodata — don't try to guess it or brute-force it. Then write `detector.c` that: (1) reads the JWT file, splits on `.` into header/payload/signature segments; (2) Base64Url-decodes header and payload (remember to add padding and swap `-_` for `+/` before feeding to a standard base64 decoder); (3) checks the header JSON for `"alg"` case-insensitively equal to `none` — if so, reject immediately without touching the signature; (4) checks the payload JSON for `"role":"admin"` (a simple substring/JSON key check is fine) — reject if present; (5) otherwise recomputes HMAC-SHA256 over `header.payload` using OpenSSL's `HMAC()` with the recovered key, and compares it (constant-time or not, doesn't matter here) to the decoded signature bytes — reject on mismatch. Any parse failure, missing file, or malformed base64 should also exit 1. Compile with `-lcrypto` and test against both a self-forged clean token and known-malicious patterns before finalizing.
