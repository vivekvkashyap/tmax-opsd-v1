This is a DAG cumulative-cost problem: build an adjacency list from `dependencies.csv` (service_id -> list of depends_on_id), then compute cumulative cost per service as base_cost plus the sum of cumulative costs of everything it directly depends on. Since costs of dependencies must be known before a service's own cost can be computed, do NOT do a naive single pass in file order — instead use memoized recursion (DFS with a cache) or a topological sort (e.g., Kahn's algorithm on the reversed edge direction, processing leaves first) so every dependency is fully resolved before being added into a parent's total. Watch two common pitfalls: (1) double-counting or infinite recursion if you don't memoize/cache already-computed costs, and (2) reading the CSV files' header rows as data (skip the first line of each CSV). After computing all cumulative costs, scan for the maximum and write `SERVICE_ID:VALUE` (no extra whitespace) to `/home/user/highest_cost.txt`. Compile with `rustc calculate_costs.rs` and run it to actually generate the output file — verify the number by manually tracing a couple of dependency chains by hand.
