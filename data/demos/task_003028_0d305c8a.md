The core bug is that `TfidfVectorizer.fit_transform` and `TruncatedSVD.fit_transform` are currently called on the full dataset before any split, so test information leaks into training. Fix this by wrapping the three steps — TF-IDF, TruncatedSVD, RandomForestClassifier — into a single `sklearn.pipeline.Pipeline`, and only ever call `.fit`/`.fit_transform` on that pipeline inside a cross-validation loop, never on the raw full dataset ahead of time. Use `StratifiedKFold(n_splits=5, shuffle=True, random_state=42)` together with `cross_val_score(pipeline, X, y, cv=cv, scoring='accuracy')` on the raw text column so the vectorizer/SVD are refit per fold. From the 5 resulting fold scores, compute the mean and use `scipy.stats.t.interval(0.95, df=n-1, loc=mean, scale=sem)` (t-distribution, not normal, since n=5 is small) to get the CI bounds — round both to 4 decimal places before writing `{"ci_lower": ..., "ci_upper": ...}` to `/home/user/ci_results.json`. Also log `cv_mean`, `ci_lower`, `ci_upper` via `mlflow.log_metric` inside an `mlflow.start_run()` under experiment name `"text_classification_fixed"`. Install missing packages with pip, then actually execute the script to produce both the JSON file and the MLflow run.
