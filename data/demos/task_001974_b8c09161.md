First recover the transformation algorithm from the scanned page: OCR `/app/archival_spec_page.png` (tesseract/pytesseract) and read the extracted text carefully — it describes a simple byte-level operation applied uniformly to every byte in order; don't assume anything more complex than what the text literally says. Read `/app/archival_config.json` to get the footer prefix/suffix and hash algorithm name dynamically (e.g. `hashlib.new(config['hash_algorithm'])`, not hardcoded "sha256"). In archive_tool.py: read the input file in binary mode, compute the checksum over the ORIGINAL bytes before transforming, apply the byte transform to produce output bytes, then write the transformed bytes followed by `footer_prefix + hexdigest + footer_suffix` UTF-8 encoded, all in binary mode (text-mode I/O can corrupt transformed byte data). Since grading fuzzes random binary files of varying sizes against a hidden oracle, generate your own random binary test files (small and large) and confirm your checksum matches a manual `sha256sum` of the original input.
