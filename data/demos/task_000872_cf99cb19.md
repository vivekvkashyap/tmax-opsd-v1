Treat this as three separable subproblems: (1) recover the formula, (2) find and fix the concurrency bug via bisection, (3) stand up a correct dual-protocol server. For the formula, don't guess blindly — run /app/oracle_bin with several chosen input pairs (including edge cases like a=b, one value near zero) and fit a hypothesis (check simple combinations of a, b: sum, product, ratio) against the outputs before committing to an implementation. For the bisection, write a small script that fires many concurrent requests at a running instance of the service and checks results against the recovered formula; use `git bisect run` with that script so bisection is automated rather than manually eyeballing commits — the bug is a shared/global mutable variable read after a delay, a classic read-after-write race, so once you spot a global being assigned then used without synchronization, that's the fix target (compute results in fully local variables, no shared mutable state, or add proper locking). For deployment, implement one core computation function and wrap it with two listeners (an HTTP handler and a raw TCP socket loop) that both call it, parsing/formatting per the given wire formats, and launch it as a background process before finishing.
