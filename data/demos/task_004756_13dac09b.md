Build the honeypot as two independent layers: TLS setup and application logic. First generate the self-signed cert/key with `openssl req -x509 -newkey rsa:2048 -nodes -keyout /home/user/certs/key.pem -out /home/user/certs/cert.pem -days 365 -subj '/CN=localhost'`, creating the certs directory first. Then write a Python `http.server`-based handler wrapped with `ssl.wrap_socket`/`SSLContext.load_cert_chain` bound to `127.0.0.1:9090`. In the `POST /auth` handler: read `Content-Length` bytes of body if needed, but the key signal is the `Authorization: Bearer <token>` header — split on the two `.` separators of the JWT, base64url-decode (with padding fixup) the header and payload segments, and hash the *entire raw token string* (not decoded parts) with `hashlib.sha256().hexdigest()`. Only fire the alert/log lines when the decoded header's `alg` is case-insensitively `none` AND the decoded payload's `role` is `admin`; get the log format exactly right (spacing, pipes). Separately check `os.geteuid()` for the privesc audit line. Always respond 200 with the JSON body regardless of detection. Launch the server as a background/daemon process so it stays reachable after your turn ends.
