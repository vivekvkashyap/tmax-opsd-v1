Write the hook's C source separately (e.g. `hook.c`), compile it with `gcc -O2 -o /home/user/user_registry.git/hooks/post-receive hook.c`, and `chmod +x` it — a git hook must be a real executable (or a `#!`-shebang script), a plain text file won't run. Inside the C program, use `system()` calls: to materialize the pushed tree, run something like `git --git-dir=/home/user/user_registry.git --work-tree=/home/user/active_users checkout -f master` (checkout, not archive, is simplest since work-tree already exists), then `tar -czf /home/user/backups/latest.tar.gz -C /home/user/active_users .`, then `chmod 0400` on that tarball via `chmod()` syscall or `system("chmod 400 ...")`. For the timezone: `setenv("TZ", "Asia/Tokyo", 1); tzset();` before calling `localtime()` and `strftime(buf, sizeof(buf), "%Y-%m-%d %H:%M:%S %Z", tm)` — `%Z` will then render `JST` correctly, and append the line wrapped in the exact `[...] Registry updated` format via `fprintf` to an fopen'd `/home/user/sync.log` in append mode. Finally clone the bare repo, add `admin_user.conf`, commit on `master`, and `git push` to trigger the hook — check `/home/user/sync.log` afterward to confirm it actually ran.
