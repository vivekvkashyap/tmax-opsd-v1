Extract first with `tar -xzf /home/user/legacy_assets.tar.gz -C /home/user/processing` (create the dir first), preserving the original subdirectory structure since that's what your script must walk. In `organize.py`, use `os.walk` or `glob('**/*.json', recursive=True)` to find every JSON file regardless of nesting depth. For each file: open it, call `fcntl.flock(fd, fcntl.LOCK_EX)` on the file descriptor before reading, parse the JSON, add `"processed": True`, dump to YAML with `yaml.dump` (install `pyyaml` if missing), write the new file as `asset_<original_basename>.yaml` in the *same directory* as the source (not flattened yet — that happens in the zip step), release the lock, then delete the original `.json`. Be careful with the rename: strip only the `.json` extension before prepending `asset_` and appending `.yaml`, so `settings.json` in a nested subdir becomes `asset_settings.yaml` there, not `asset_settings.json.yaml`. After running the script, build the final zip by walking `/home/user/processing` for `*.yaml` files and adding each with `zipfile.write(path, arcname=os.path.basename(path))` — using `arcname` is the key trick to flatten directory structure into a zip root without losing nested files.
