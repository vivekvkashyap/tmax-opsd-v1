Structure this as: segment -> classify -> dedupe -> verify, and validate each stage before trusting the next. Use ffmpeg's segment muxer (e.g. `-f segment -segment_time 1 -reset_timestamps 1`) with re-encoding (not `-c copy`, which can't cut on arbitrary 1s boundaries cleanly) so each chunk is an independently decodable, playable file — this matters for the later concatenation fidelity check. For motion detection, decode frames per chunk (via ffmpeg piped to raw frames, or a library like opencv/PIL) and compare consecutive frames with a simple metric like mean absolute pixel difference or SSIM against a small threshold; tune the threshold conservatively since it directly affects the fidelity/storage tradeoff score. For dedup, track the first chunk classified static as the canonical file; for every later static chunk, delete it and create a hard link (`os.link`, not a symlink or copy) to the canonical file so `du` sees shared physical space. Keep motion chunks as normal independent files. Write the true/false motion classification per chunk to the JSON log. Sanity-check by concatenating all chunks (ffmpeg concat demuxer) and confirming it plays back correctly, and by running `du -sb` on the archive dir to see real space savings from the hard links.
