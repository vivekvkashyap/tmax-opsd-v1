Write a Python script that does a manual recursive walk instead of relying on `shutil.make_archive` or `tarfile.add(recursive=True)`, since those don't guard against symlink cycles. Implement your own traversal (e.g. `os.walk` with `followlinks=False`, or a manual stack/recursion using `os.scandir`) and maintain a `set()` of real paths already visited, computed with `os.path.realpath()` for every file and directory encountered — before descending into a directory or adding a file, check `os.path.realpath()` against the seen set and skip if already present, then add it. Only add regular files (`os.path.isfile` on the resolved path) to the archive, using the resolved real absolute path as both the source and the `arcname` in `tarfile.open(..., 'w:gz').add()`. For atomicity, open/write the tar at a `.tmp` suffix path and call `os.rename()` (not `shutil.move`) to move it to the final `.tar.gz` name only after the writer closes successfully. After renaming, reopen the final archive with `tarfile.open(mode='r:gz')` and iterate/read its members to confirm it's not corrupt before declaring success. Finally, write every real path you added to the archive to the log file, sorted alphabetically, one per line.
