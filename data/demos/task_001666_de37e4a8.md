Start by fixing the Makefile: a link error when a C program calls math functions (e.g. cos/sin) usually means the math library isn't linked — add `-lm` to the compile/link command so `layout_engine` builds cleanly. Confirm it works standalone first: write a small test input file and run `./layout_engine <file>` to see the CSV output format before wiring it into the API. For the Flask/FastAPI server, the key design point is the cache key: canonicalize the request payload deterministically (e.g. sort dict keys / use a stable JSON serialization or hash of the sorted vertex+edge structure) before using it as the Redis key, since the same logical graph must always hash the same way. On a cache hit, return the stored JSON immediately without touching the subprocess; on a miss, invoke `layout_engine` via `subprocess.run`, parse its CSV stdout into the coordinate list, store the JSON result in Redis, then respond. Test correctness on a single request first, then hit the endpoint many times in a loop and time it — if latency doesn't drop dramatically after the first call, the cache key or hit-path logic is wrong.
