Implement this as clearly separated passes rather than one tangled loop, since the spec has several independent stateful rules that are each easy to get subtly wrong in C. Pass 1: parse each CSV row (timestamp, message, reading), and while reading the message, scan for `\uXXXX` sequences and convert each codepoint to UTF-8 bytes yourself (for BMP codepoints this is a straightforward 1-3 byte UTF-8 encoding based on the codepoint's numeric range — don't assume all are ASCII, some in this task are outside the ASCII range and need genuine multi-byte UTF-8 output). Pass 2: compute djb2 over the *normalized* message bytes and check/insert into a hash set (a simple open-addressing table or linked list keyed by hash+string compare, to reject collisions correctly) — dedupe based on message content only, not the full row, and remember T0 is the timestamp of the first row that survives dedup, not the first row in the file. Pass 3: bucket by `(timestamp - T0) / 60`, and for each bucket in order from T0's bucket up through the bucket containing the last valid row's timestamp, pick the latest valid row landing in that bucket, or forward-fill the previous bucket's values if none landed there — carefully handle the very first bucket already having T0's row. Write with `%.2f` formatting and diff your output against a hand-traced expectation for a small input before trusting the full file.
