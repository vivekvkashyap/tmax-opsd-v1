Inspect `/app/iot-gateway-2.1.0/Makefile` — it installs to `$(PREFIX)/bin` with `PREFIX ?= /usr/local`, which you can't write to. Rather than hardcoding paths, either edit the Makefile default to `PREFIX ?= /home/user/local` or invoke `make PREFIX=/home/user/local install` from your deploy script — either way the binary must land at `/home/user/local/bin/gatewayd`. Write `supervisor.sh` to `export GATEWAY_AUTH_TOKEN=edge2024` then loop: run `gatewayd` in the foreground, and on exit `sleep 1` and restart (a `while true; do gatewayd; sleep 1; done` pattern), trapping signals so it doesn't die permanently on one crash. Use `socat TCP-LISTEN:9090,fork,reuseaddr TCP:127.0.0.1:8080 &` to forward external port 9090 to the internal service on 8080 (install socat via apt if missing). `deploy.sh` should chain: patched `make install`, launch `supervisor.sh &`, launch the `socat` forward `&`, then exit — leaving both backgrounded so the verifier's HTTP GET to `/health` on port 9090 reaches the running `gatewayd` and gets a 200 with the authorized payload.
