This is an HTTP wrapper around a black-box binary plus a signal-processing/statistics pipeline; treat `/app/sim_spectrometer` as opaque and only use its documented CLI contract (`<input.fasta> <output.csv>`). Build a Flask/FastAPI app bound to `127.0.0.1:8000` with `POST /process`. In the handler: parse the incoming `pdb_data` string for `SEQRES` lines, extract and concatenate the three-letter residue codes in order, map them to one-letter amino acid codes, and write a standard FASTA record to a temp file. Invoke the binary 15 separate times (fresh output CSV path each run, since its multithreaded core is intentionally non-deterministic) via `subprocess`. For each of the 15 output CSVs, read the `time`/`amplitude` columns, run `numpy.fft.rfft` (sample rate 1000 Hz implied by the 0.001s time step), compute the magnitude spectrum, and find the frequency bin with peak magnitude among strictly positive frequencies — that's the dominant frequency for that run. Load `/app/reference_freqs.csv`, run `scipy.stats.ttest_ind(your_15_freqs, reference_freqs, equal_var=False)`, and return `{"mean_dominant_freq": ..., "p_value": ...}` as JSON. Start the service and leave it running so requests can be made against it after your turn.
