Build the script as a sequence of idempotent, individually-testable shell commands, then wrap them in a script and chmod +x it. Use `mkdir -p` for all three nested directories in one or three calls. Create the symlink with `ln -sfn /home/user/service/logs /home/user/service/current_logs` (use `-f` so reruns don't fail, and a directory-target symlink not a copy). For the cert, use a single non-interactive `openssl req -x509 -newkey rsa:2048 -nodes -keyout key.pem -out cert.pem -days 30 -subj "/CN=capacity.local"` — `-nodes` is required for an unencrypted key, and `-subj` avoids interactive prompts. For the logrotate config, just heredoc the four required directives (`/home/user/service/logs/*.log { daily rotate 7 compress }`) — no need to actually run logrotate. Generate the dummy log with `yes "dummy log entry" | head -n 1000 > access.log` or a `seq`/loop equivalent producing exactly 1000 lines with no trailing blank line. Compute the size last, after all files exist, with `du -sb /home/user/service | cut -f1 > /home/user/baseline_size.txt` so the number reflects the final state.
