Build and test each of the three components independently before combining them. For the C volume-halving program: copy the first 44 bytes verbatim as the header, then for every remaining 16-bit sample divide by 2 with truncation toward zero — do NOT implement this as an arithmetic right shift (`>>1`), since shifting rounds negative odd values toward negative infinity instead of toward zero, which is a classic off-by-one bug for negative PCM samples; use integer division (C's `/` on signed ints truncates toward zero) instead. For the HTTP server: on each GET, read the wav file's actual size for `Content-Length`, send a minimal correct `HTTP/1.1 200 OK` header block with `Content-Type: audio/wav`, a blank line, then the raw bytes — test it directly with `curl` against port 8080 before adding the tunnel. For the forward, run `socat TCP-LISTEN:9090,fork,reuseaddr TCP:127.0.0.1:8080` in the background. Finally verify end-to-end by curling port 9090 and confirming the downloaded bytes match your own halved-sample computation, not just that a response arrives.
