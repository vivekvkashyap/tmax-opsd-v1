Start with git forensics: run `git log --diff-filter=D --summary` (or `git log --all --full-history -- capture.pcap`) to find the commit that deleted `capture.pcap`, then `git show <commit>^:capture.pcap > capture.pcap` (or `git checkout <commit>^ -- capture.pcap`) to restore it from before the deletion. Analyze the recovered pcap with `tcpdump -A -r capture.pcap` or `tshark -r capture.pcap -Y http` to find the plaintext HTTP request and read the `Authorization: Bearer <token>` header value directly from the packet payload — do not guess it. Patch `parser_service.py`'s token placeholder with the recovered value. For the parsing bug, inspect the `struct.unpack` format string: a format code for temperature that treats it as unsigned (e.g. `I`) will wrap negative milli-Celsius values into huge positive numbers; change that field's format code to the signed equivalent (`i`) while keeping byte order and the other fields untouched. After fixing both issues, launch services via `startup.sh`, then manually send a test UDP packet and query the admin HTTP endpoint to confirm negative temperatures and the ingest auth both round-trip correctly before leaving the services running in the background.
