This is a compile-and-link task more than an algorithmic one — get the toolchain right first. Write `/home/user/analyze.c` using the HDF5 C API (`H5Fopen`, `H5Dopen2`, `H5Dread` with `H5T_NATIVE_DOUBLE`) to read the `matrix_data` dataset into a flat 100x50 double buffer, then hand that to GSL's SVD routine (`gsl_linalg_SV_decomp`, which needs a `gsl_matrix` with rows >= cols — 100x50 already satisfies this) to get singular values in `gsl_vector *S`, sorted descending by construction. Take the first 3, write them to `top_sv.txt` at `%.6f` each on its own line. Compile with both libraries linked, e.g. `gcc analyze.c -o analyze $(pkg-config --cflags --libs hdf5) -lgsl -lgslcblas -lm` (adjust flags/paths if pkg-config isn't set up for hdf5; `-I/usr/include/hdf5/serial -lhdf5_serial` style flags are common on Debian-based images). Then read the 3 pre-existing reference values from `/home/user/reference_sv.txt`, compute mean absolute error against your computed values, and write that single `%.6f` number to `sv_error.txt`. Run the binary and sanity-check both output files exist with 3 and 1 lines respectively.
