Before writing C, confirm GSL is available (`pkg-config --cflags --libs gsl` or install `libgsl-dev`) and check the binary layout you're parsing: the file is a flat sequence of interleaved `double` pairs (X,Y,X,Y,...), so read it with `fread` into one buffer, then de-interleave into two separate arrays (stride-2 extraction) before calling GSL — `gsl_stats_covariance` expects two contiguous arrays with a stride, so either de-interleave manually or pass `stride=2` starting at offset 0 for X and offset 1 for Y directly into the buffer, whichever is less error-prone for you. Compile-link order matters: `gcc etl_covar.c -o etl_covar -lgsl -lgslcblas -lm` (GSL requires the CBLAS and math libs after it on most linkers). Write the result with `fprintf(fp, "%.6f\n", covariance)` to `/home/user/covariance_result.txt`, with nothing else in the file. Sanity-check by computing the covariance a second way (even a quick Python script reading the same binary with `struct.unpack`) to confirm your de-interleaving didn't swap X and Y or introduce an off-by-one in the read loop before trusting the C output.
