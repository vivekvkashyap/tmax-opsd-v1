Start by confirming the background services are up (`/app/start_services.sh`, check ports 8000 and 9090). For the hook: a `post-receive` hook reads lines `<old> <new> <ref>` from stdin; for each, run `git diff-tree -r --name-status --diff-filter=AM <old> <new>` (or against the empty tree hash for the first push) filtered to `*.yaml`, then for each changed path use `git cat-file -p <new>:<path>` to get the blob content at the new commit — do this all from within the C program via `popen`/`fork+exec` of `git`, not shelling out the whole logic. Critically, for the HTTP POST use a **persistent libcurl easy handle reused across all 500 files** (or a multi-handle with connection reuse) rather than spawning a new process or reconnecting per file — that's the main way naive attempts blow through the 1.5s budget. Link with `-lcurl`, compile with `gcc -O2`, and place the binary directly as `hooks/post-receive` with `chmod +x`. For the cron job, use `crontab -e`/`(crontab -l; echo "* * * * * curl -s http://127.0.0.1:9090/sync >> /home/user/sync.log; echo >> /home/user/sync.log") | crontab -` style installation, appending a newline after curl's output as required.
