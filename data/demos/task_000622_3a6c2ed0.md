Build and test each of the four pieces in isolation before chaining them, since bugs compound quickly in a C-program-spawns-shell-script-spawns-daemon pipeline. Write `trigger_alert.sh` first — it must grep `valid_users.txt` for an exact-line match of `$1` (use `grep -qxF "$owner_name" file` to avoid partial-match false positives like "alice" matching "alice2"), exit silently if absent, and otherwise append the exact alert string with `$1`/`$2` interpolated. Write and test it standalone with both a valid and invalid user before touching C. For `check_ports.c`, parse `services.conf` line by line splitting on `:`, attempt a `connect()` to `127.0.0.1:<port>` with a short timeout, and on failure invoke `trigger_alert.sh` via `system()` or `execve` with the owner and port as separate argv entries (not string-concatenated into one arg). Compile with `gcc` and test manually against one open and one closed port before building the init script. For `netmon-init.sh`, background a loop that calls `check_ports` then sleeps 1s, redirect it with `&`, capture `$!` into the PID file for `start`, and for `stop` read that PID and `kill` it then remove the file — verify the loop actually terminates on stop, not just that the PID file is removed.
