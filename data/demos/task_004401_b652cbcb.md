Generate the TLS material first with straightforward openssl commands: `openssl req -x509 -newkey rsa:2048 -nodes -keyout server.key -out server.crt -days 365 -subj "/CN=inspector.local"` (verify the CN with `openssl x509 -noout -subject`). For the C audit, locate the `strcpy(buffer, req)` into a fixed `char buffer[64]` in `parse_request` — this is the classic stack overflow. Replace it with a bounded copy, e.g. `snprintf(buffer, sizeof(buffer), "%s", req)` or `strncpy` plus explicit null-termination, so arbitrarily long input can't smash the stack; do not just enlarge the buffer, since the vulnerability is unbounded copy length vs any fixed size. For the CSP requirement, after decrypting, scan the decrypted buffer for the first line (the HTTP status line), and when writing output insert the exact CSP header string on its own line immediately after that line before writing the rest of the body — don't just append it at the end. Recompile with `gcc ... -lcrypto`, run it, and confirm the output file contains both the payload and the header in the right position.
