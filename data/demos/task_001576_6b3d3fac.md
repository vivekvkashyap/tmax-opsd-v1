Build the pipeline as discrete, testable stages rather than one monolithic script. First parse the segment file (tab-separated start/end/label) and filter to `SPEECH` rows only — use Python's own parsing or `awk -F'\t' '$3=="SPEECH"'` for robustness against float formatting. Second, use a proper audio library (`pydub`, or shell out to `ffmpeg -ss <start> -to <end>`) to extract each speech segment as a temp WAV chunk, keyed by segment index so ordering is preserved, then concatenate them in original chronological order (not filesystem/glob order, which can sort "10" before "2") — concatenate via `pydub.AudioSegment` sum or `ffmpeg concat` demuxer with an explicit ordered file list. Third, convert to 16kHz mono MP3 at 32kbps with `ffmpeg -i merged.wav -ar 16000 -ac 1 -b:a 32k output.mp3` (or pydub's export with matching parameters) — get these exact parameters right since they directly affect the WER/file-size checks. Finally write to a temp file in the same directory as the destination and `os.replace()`/`mv` it to `/home/user/archived_call_001.mp3` for atomicity. Test by listening to or transcribing a sample locally before trusting the pipeline end-to-end.
