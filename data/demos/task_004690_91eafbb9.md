Use raw BSD sockets in C: `socket(AF_INET, SOCK_STREAM, 0)`, `connect()` to `127.0.0.1:8080`, then loop `read()`/`recv()` into a growing buffer and split on `\n` to get complete lines — TCP does not guarantee one `read()` call yields one CSV line, so buffer partial lines across reads until the connection closes (recv returns 0). Parse each line with `sscanf` into id (int), timestamp (long), cpu (float), memory (int). Track seen ids with a simple array/hash-set (id range is small) to skip duplicates — do the dedup check before updating the rolling window. Maintain a small fixed-size circular buffer (or just an array with a count) holding the cpu values of the last up to 3 *unique* records seen so far, and compute the average of whatever is currently in that window each time a new unique record arrives (1, then 2, then always 3). Write header `id,rolling_cpu\n` first, then one line per unique record formatted with `%d,%.2f\n`. Compile with `gcc -o process process.c` and run `./process` after the server is listening on port 8080 so the output file `/home/user/clean_stats.csv` is regenerated correctly.
