Build the pipeline as separate, testable awk passes rather than one monolithic script. First write a shared tokenizer (a small awk/sed snippet using `tr '[:upper:]' '[:lower:]'`, then `sed 's/[^a-z]/ /g'`, then `tr -s ' '` and trim) and apply it identically to all three files — reuse it verbatim so training and target tokenization can never drift apart. Second, compute word counts per corpus with `tr ' ' '\n' | sort | uniq -c`, and derive N_domain, N_generic, and V (the count of *distinct* words across the union of both vocabularies, not the sum of each vocabulary size) — a common bug is double-counting words that appear in both files. Third, load both count tables into a single awk program as associative arrays keyed by word, then stream target.txt line by line, tokenizing each line the same way, summing `log((count_d+1)/(N_domain+V)) - log((count_g+1)/(N_generic+V))` per word using awk's `log()` (natural log), treating unseen words as count 0. Format the score with `printf "%.4f"` and emit `LINE\tSCORE\tCLASS` per line with a strict `>0` threshold. Watch for off-by-one line numbering and empty lines in target.txt.
