Start by getting the Python storage backend running on 127.0.0.1:9000 (check it responds before touching Rust). Extract the embedded hex key from the ELF's `.rodata` section with `objdump -s -j .rodata /app/malware_implant.bin` or `strings -a` piped through `grep -E '^[0-9a-f]{64}$'` — look for a 64-char lowercase hex run, don't guess or hardcode a value. Paste that recovered key into the Rust decryption function's constant. For the path-traversal fix in `/export`, never trust the client filename directly: reject any name containing `..` or `/`, then join it to the fixed `exports/` directory and canonicalize/compare the resolved path stays inside that directory before writing. For checksum verification, decrypt first, compute SHA256 of the plaintext bytes, and compare (constant-time or simple ==) against the `X-Checksum` response header from the storage backend, returning HTTP 500 on mismatch. Set `STORAGE_URL` in `.env` to match the Python service's actual host:port. Build with `cargo build --release`, run both services in the background (e.g. nohup/&), and manually curl each endpoint to confirm behavior before finishing.
