Start by running `tesseract /app/system_layout.png stdout` (try preprocessing with `convert`/`pnmnorm` if OCR quality is poor) to recover the `API_SECRET` token and `SOCK_PATH` value written in the note — do not guess these, read them from the image. Once you have `SOCK_PATH`, `mkdir -p` its parent directory under `/home/user/backend/` and `chmod` it so both the backend process (owner) and nginx (reader/connector) can access the socket — a common mistake is leaving the directory too restrictive for nginx's worker user. Export `API_SECRET` and `SOCKET_PATH` as environment variables and launch the Python backend in the background (e.g. `nohup ... &` or gunicorn binding `--bind unix:<SOCK_PATH>` if it's a Flask app). Edit `/home/user/nginx/nginx.conf` so the `/api/` location's `proxy_pass` points at `http://unix:<SOCK_PATH>:` instead of the broken `127.0.0.1:9000` target — this unix-socket proxy_pass syntax is easy to get wrong (note the trailing colon and no path after it). Restart nginx with `nginx -c ...`, then verify end-to-end with the curl command from the task, checking headers match and body is exact JSON.
