This is a hierarchy explosion problem, so think in terms of repeated joins rather than one clever one-liner. A clean coreutils approach: iteratively expand a working list of (id, multiplier) pairs starting from `(P-01, 1)`. In each pass, join the current frontier against bom.csv on parent_id (using `join` after sorting both by the join key, or `awk` with an associative array loaded from bom.csv) to produce the next frontier's (child_id, multiplier*qty) pairs; repeat until no id in the frontier is a parent_id in bom.csv anymore (i.e., it resolves to a leaf/raw component per components.csv's type column). Accumulate every raw-typed id's multiplier into a running total (awk associative array keyed by id, summed across all passes) rather than only keeping the last pass's numbers — a leaf can be reached through more than one path/depth. Once totals are final, join the (id,total) pairs against components.csv on id to pull in name, then `sort` alphabetically by name and prepend the `name,quantity` header with awk/printf, writing to /home/user/raw_materials_needed.csv. Test incrementally: print the frontier after each expansion pass to confirm multipliers are compounding correctly before trusting the final totals.
