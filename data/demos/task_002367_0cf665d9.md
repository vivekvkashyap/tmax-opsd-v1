The core technique is computing exact seek timestamps from the video's real duration, not hardcoded guesses. In your script, first probe the actual duration with `ffprobe -v error -show_entries format=duration -of csv=p=0 "$1"`, then compute each of the 10 target timestamps as `duration * percent` (10%, 20%, ..., 90%, 99%) in a shell arithmetic context using `bc` or `awk` for floating point (bash integer arithmetic will truncate incorrectly). For each timestamp, use `ffmpeg -ss <timestamp> -i "$1" -frames:v 1 -q:v 2 <tmpfile>.jpg` — place `-ss` before `-i` for fast/accurate seeking, and verify each extracted frame is non-empty before proceeding, since silent extraction failures would tank the similarity score against the golden frames. Rename outputs sequentially to `fig_01.jpg` through `fig_10.jpg` matching the timestamp order. Generate the manifest with `sha256sum fig_*.jpg > manifest.sha256` (run from inside the directory so paths in the manifest are relative/consistent), then `tar czf` the images and manifest together, naming the archive from the video's basename (strip extension, e.g. via `basename "$1" .mp4`) and placing it in `/home/user/processed/`, creating that directory first if needed.
