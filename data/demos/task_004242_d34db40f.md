Start by extracting the rule text from the image yourself: run `tesseract /app/dataset_rules.png stdout` (or similar OCR invocation) and read the resulting text carefully rather than guessing at plausible-sounding rules — the exact thresholds and trigger strings matter and must come from the OCR output. Encode each rule as an explicit, independently-testable check in `organizer.py` (e.g. a substring search, a regex for repeated characters, and a line-count check), since the hidden test corpus is specifically designed to hit edge cases near each rule's boundary. For the CLI, read the whole input file's bytes/text once, evaluate all rules, and `sys.exit(1)` immediately without touching the output path if any rule fails. For the atomic-write requirement on success, write the output into a temp file created with `tempfile.mkstemp` (or `NamedTemporaryFile(delete=False)`) in the *same directory* as the target output path, flush/close it, then call `os.replace(tmp_path, output_path)` — never write directly to the destination or use a temp dir on a different filesystem, since that breaks atomicity and the automated check for `tempfile`/`os.replace` usage.
