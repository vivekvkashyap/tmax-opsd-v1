Treat this as three separate stages glued together in one script. First, fix the CSV: a naive `sed`/`grep` newline-strip breaks because it can't tell "newline inside a quoted field" from "newline ending a record." Use a stateful pass that tracks whether you're inside double quotes (e.g. split the stream on the `"` character with `awk -v RS='"'`, and on every other record — the ones inside quotes — replace embedded `\n` with a space before reassembling the line with its quote delimiters). Verify by comparing line counts before/after against the expected row count. Second, split the cleaned, newline-safe file into N chunks (e.g. `split -n l/N` or by line count) and feed each chunk to `/app/analyzer` concurrently with `xargs -P` or GNU `parallel`, redirecting each chunk's stdout to its own temp file — never modify the analyzer itself. Third, concatenate all chunk outputs and pipe through `sort -t, -k1,1` (numeric/chronological on the timestamp column) into `/home/user/anomalies.csv`. Throughout, append timestamps (`date`) marking when each parallel batch starts and finishes to `/home/user/pipeline.log`. Finally `chmod +x /home/user/process.sh`.
