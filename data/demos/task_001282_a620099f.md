Write the sanitizer in C first and test it standalone via stdin/stdout redirection before wiring it into the pipeline. In `/home/user/sanitizer.c`, read input line by line (e.g. `getline` or `fgets`), split each line on commas expecting exactly 4 fields, then validate: field count must be exactly 4; `feature1`/`feature2` must parse fully as numeric (use `strtod` and check the end pointer consumed the whole field, allowing only digits, one optional leading `-`, and one optional `.`, rejecting things like "NaN" or "drop table"); both must fall within [-1000.0, 1000.0] inclusive; `category` (last field) must be <=10 chars with no trailing extras (watch for a 5th comma-separated column, which should also cause rejection). Only print rows that pass all checks, exactly as received. Compile with `gcc -o /home/user/sanitizer /home/user/sanitizer.c` and test with `echo "..." | ./sanitizer` against your own handmade valid and malformed CSV rows (missing columns, non-numeric values, out-of-range floats, overlong categories) before touching the pipeline. Then in `/app/start_pipeline.sh`, launch `redis-server --daemonize yes`, and set up a netcat listener on port 9000 that pipes each connection's data through the sanitizer and into `redis-cli -x rpush clean_data` (a `while true; do nc -l -p 9000 | ./sanitizer | redis-cli -x rpush clean_data; done` loop is simpler and more portable than relying on `nc -c`/`-e` exec flags, which aren't always available).
