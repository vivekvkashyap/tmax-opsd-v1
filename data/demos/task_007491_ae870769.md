Treat this as a linear pipeline where each stage's output feeds the next, and verify each stage manually before wiring it into `pipeline.sh`. Compile with `g++ -O2 -pg spectro.cpp -o spectro`, run `./spectro` in the same working directory so `peaks.csv` and `gmon.out` land next to the binary (gprof needs both the executable and `gmon.out` in the run directory), then run `gprof spectro gmon.out` and extract the flat-profile row for `refine_mesh` — pipe through `grep refine_mesh | awk '{print $1}'` to isolate the `% time` column, and write only that numeric token (no extra text) as the first line of `metrics.txt`. A common gprof pitfall is running the binary from a different directory than where gprof is invoked, or forgetting `-pg` on the compile line, which produces an empty/missing `gmon.out`. For the second line, just run `python3 density.py` and capture stdout (it already prints the formatted KDE max) — append it with `>>`, don't reformat or recompute it yourself. Make `pipeline.sh` executable and chain these steps with `set -e` so a failure in one stage doesn't silently corrupt `metrics.txt`.
