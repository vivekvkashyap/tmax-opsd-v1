Write a Go program that treats each `.dat` file as an opaque binary blob to be parsed by its header, not by filename. For every file in `/home/user/backups/`: open it, read the first 11 bytes and verify they equal the ASCII magic string `ARCHIVE_BKP` (reject/skip otherwise), read byte 11 as an unsigned 8-bit sequence ID, read bytes 12-15 as a little-endian uint32 payload length, then read exactly that many bytes starting at offset 16 as the payload — do not read to EOF, since files intentionally contain trailing garbage past the declared payload size. Collect (sequenceID, payload) pairs across all files, sort by sequence ID ascending, and concatenate the payloads in that order into `/home/user/restored_data.bin`. Finally write the total number of concatenated payload bytes (sum of the individual payload lengths, not the file sizes) to `/home/user/metrics.log`. The key correctness traps are: relying on filename order instead of the embedded sequence ID, and reading past the declared payload length into the garbage suffix.
