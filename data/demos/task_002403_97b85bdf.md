Treat the WAF as a strict input-validation filter that must be tested against many encoding variants, so canonicalize before deciding. Steps: read stdin fully; parse headers to locate `Content-Disposition` and pull out the `filename=` value; before any traversal check, URL-decode the filename (handle both single and double percent-encoding, e.g. `%2e%2e%2f` and `%252e%252e`) and normalize path separators; then reject (exit 403, print "HTTP/1.1 403 Forbidden") if the decoded/normalized value contains `../`, `..\`, starts with `/`, or resolves outside the upload directory — do this with real path normalization logic, not a single regex, since fuzzers will try nested/mixed encodings. If clean, hash the POST body with `sha256sum`/`openssl dgst -sha256` and print `HTTP/1.1 200 OK` then `OK: [filename] [sha256]`. Separately: `chmod 2775` and `chown :www-data /var/uploads` for the SGID bit; check `server.crt` expiry/chain with `openssl verify`/`openssl x509 -checkend`, regenerating a self-signed cert with `openssl req -x509 -newkey` if invalid; and repoint nginx.conf's proxy target from port 8080 to a FastCGI/socat listener you spawn on port 9000 that runs the WAF script.
