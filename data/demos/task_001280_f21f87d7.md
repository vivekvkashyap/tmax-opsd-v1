Start by extracting the backend port from the image: run `tesseract /app/arch_diagram.png stdout` (or another OCR tool) and read the port number from its output — do not assume a value. Then build the pieces in dependency order: (1) write `/home/user/setup_tunnel.py` to idempotently start an SSH local port-forward (`ssh -N -L <port>:localhost:8000 localhost`) as a background process, checking first whether the tunnel/port is already active so re-running the script doesn't spawn duplicates; (2) edit `/home/user/nginx/nginx.conf` to change the `/api` location's `proxy_pass` target to `http://localhost:<port>` matching the tunnel's local port, then apply with `nginx -s reload` (not a restart) so the running master process picks it up; (3) write `/home/user/load_test.py` using `concurrent.futures` or `threading` to fire 100 concurrent GETs at `http://localhost:8080/api` and print `successes/100` as a float. Test each stage independently (curl the tunnel directly, curl through nginx, then run the load test) before declaring done, since a broken tunnel or stale nginx config will silently fail the load test.
