(1) Extract frames at exact indices 0,25,50,...,1000 with ffmpeg's select filter, e.g. `ffmpeg -i /app/experiment.mp4 -vf "select='not(mod(n\,25))'" -vsync vfr -q:v 2 tmp_%04d.jpg` — verify the frame count and spot-check a couple of frames rather than assuming the filter syntax is exactly right first try, since off-by-one selection will fail the SSIM check entirely. (2) Recursively walk raw_metadata/ (`os.walk` or `find ... -name '*.json'`), json.load each file, and build a map from target_frame -> json path, tolerating files that fail to parse. (3) For each matched (frame, json) pair, hard-link (not copy) both files into curated_dataset/ using `os.link` with names `sample_<frame:06d>.jpg` / `.json` — hard links share inode/data so this preserves them cheaply and lets the archive step detect shared inodes. (4) Build the tar.gz with Python's tarfile module, which by default preserves hardlinks between files added in the same archive (it tracks inodes automatically) — do not follow symlinks/copy content twice; just add each file normally via `tar.add()` in a single pass so tarfile's built-in hardlink detection kicks in.
