This is a two-stage problem: data-cleaning then shortest path. In your C++ program, first read `/home/user/network.csv` (skip header), and for every (source,target) pair keep only the row with the maximum `timestamp` — a `map<pair<string,string>, Row>` where you overwrite only if the new row's timestamp is greater works well; watch out for multiple rows per edge with out-of-order timestamps in the file, don't assume the last occurrence is the latest. After reducing to one row per directed edge, drop any edge whose reliability_score is `< 0.8` (strictly less) — keep `== 0.8` edges. Build an adjacency list from the surviving edges only, then run Dijkstra (or since latencies are likely small positive integers, still prefer Dijkstra/priority_queue for correctness) from S to T tracking latency sums and predecessor pointers to reconstruct the path. Write `Latency: <n>` and `Path: <comma-separated nodes>` to `/home/user/shortest_path.txt` in exactly that two-line format. Compile with `g++ -O2 /home/user/solve.cpp -o /home/user/solve` and run it, verifying by hand-tracing a couple of the CSV rows that your dedup/filter logic picked the edges you expect before trusting the final path.
