First bring up the two dependency services before writing any C code: start Redis with `redis-server --daemonize yes` (verify with `redis-cli ping`), and start the Flask app with `python3 /app/config_service.py &` (verify `curl http://127.0.0.1:5000/window` returns the window size, and check `redis-cli GET fallback_mean` for the fallback). Install `libcurl` and `hiredis` dev packages if missing (e.g. `apt-get install -y libcurl4-openssl-dev libhiredis-dev`). In `normalize_stream.c`, at startup fetch `W` via a libcurl GET to `/window` and parse the integer, then connect to Redis via hiredis and GET `fallback_mean` as a float — do this once, not per-line. Read stdin line by line, and for each number: compute mean/std using ONLY the up-to-`W` previously-seen values (a sliding window buffer), excluding the current value — this ordering is the easy place to get wrong (leaking future/current data). Handle the count==0 case (use fallback_mean, std=1.0) and count==1 or std==0 cases (std=1.0) exactly as specified. Print with `printf("%.4f\n", ...)`. Compile with the correct `-lcurl -lhiredis` flags to `/home/user/normalize_stream`, then test against a small manually-computed sequence before trusting it, since verification is bit-exact fuzzing against a hidden oracle.
