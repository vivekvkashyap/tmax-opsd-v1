Read `corpus.txt` and split on single spaces exactly (don't use `.split()` with no args, which collapses whitespace differently) to get the true token count, then map each word through `vocab.json` with `vocab.get(word, vocab['<unk>'])`. Create the memmap with `np.memmap(path, dtype=np.int32, mode='w+', shape=(len(tokens),))`, assign the token IDs into it, and call `.flush()` so the file is fully written to disk (a common bug is forgetting to flush, leaving a truncated/zero-filled file). Define the model as an `nn.Module` executing Embedding(vocab_size incl. `<unk>`, 16) -> Linear(16, 8) -> ReLU in `forward`, and call `torch.manual_seed(42)` immediately before constructing the model (not before loading data) since seed placement changes the weight initialization and thus your final numbers. Slice the first 100 tokens from the memmap, convert with `torch.tensor(..., dtype=torch.long)`, run the forward pass, then take `.mean().item()` and `.std().item()` (PyTorch's default `std` is unbiased) — `.item()` is essential so JSON serialization doesn't choke on numpy/torch scalar types.
