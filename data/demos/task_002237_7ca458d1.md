Structure the C++ program as independently testable pieces since three separate things (index/query correctness, parameterization, and concurrency) are being verified. Create the schema and a composite index on `edges` covering `(source, target)` — a single-column index on just `source` may not satisfy an index-strategy check for a "forward traversal" query, so name it exactly `idx_graph_traversal` and cover both columns used in the join predicate. Insert nodes/edges using `sqlite3_prepare_v2` + `sqlite3_bind_text` for every value — never use `sprintf`/string concatenation to build the SQL, since the verifier greps the source for `sqlite3_bind` usage. For traversal, write a recursive CTE (`WITH RECURSIVE reach(id) AS (SELECT ? UNION SELECT e.target FROM edges e JOIN reach r ON e.source = r.id) SELECT id FROM reach WHERE id != ?` roughly) bound with the start node parameter, then write all non-start reachable IDs to `traversal.log`. For the deadlock demo, open the DB in a mode that doesn't retry automatically (avoid a long `busy_timeout`, or set it near zero) so two threads doing `BEGIN IMMEDIATE`/interleaved updates with a `sleep(500ms)` between statements reliably produce `SQLITE_BUSY` on one thread; catch that specific return code and write `DEADLOCK_DETECTED` to `deadlock.log`. Compile with `-pthread -lsqlite3` and run to confirm all three output files are produced.
