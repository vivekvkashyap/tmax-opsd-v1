This is a multi-source join across sqlite, JSONL, and CSV — do it as a pipeline of CLI tools rather than one big script. Use `sqlite3 -separator $'\t' /home/user/data/users.sqlite "SELECT id, name, department FROM users;"` to dump users, use `jq -r '[.doc_id, .size_bytes] | @tsv' /home/user/data/docs.jsonl` to get doc sizes, and read `access.csv` (user_id,doc_id, no header) to map docs to users. Join access→docs to get per-doc size per user_id, sum by user_id with `awk` (accumulate into an associative array keyed on user_id), then join that sum back to the users table on id to attach name/department. Exclude any user with zero documents (don't emit them at all). Sort the final rows by total_bytes descending, ties broken by user_name ascending — `sort -t$'\t' -k3,3nr -k1,1` — and write the exact 3-column TSV to `/home/user/user_storage.tsv`. Make the script executable with `chmod +x`. For the index task, since the query filters on department and status (equality) then orders by name, the optimal composed index is `CREATE INDEX idx_dept_status_name ON users (department, status, name);` — write that exact statement to `/home/user/index_strategy.sql`.
