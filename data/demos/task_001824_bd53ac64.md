Build this as three decoupled stages you can test independently before wiring them together. Tokenize with `text.lower()`, strip everything except `[a-z0-9 ]` via regex, split on whitespace, and drop empty tokens — do this consistently for both indexing and query time. Compute TF as raw per-document term counts and IDF as `ln(N/df)` over the full corpus vocabulary; store per-document term-frequency dicts (or a sparse vector) in SQLite so the server can reconstruct TF-IDF vectors without re-reading the JSONL on every request — precomputing IDF once at load time and caching document vectors in memory (or lazily from SQLite) will keep `/similar` fast. For similarity, compute cosine similarity between the query doc's TF-IDF vector and every other doc's, sort descending by score with ties broken alphabetically by doc ID, and take the top 3. Use Python's built-in `http.server` or Flask running as a background process (`nohup`/`&` or a daemon thread) bound to `127.0.0.1:8080`, verify it's live with a manual curl before scripting the final `doc1` query, and write that raw response verbatim to the output file.
