Split this into the mTLS plumbing and the crypto classifier — solve each independently, then wire them together. For mTLS: use `/app/certs/ca.crt`/`ca.key` to generate a client key and a CSR, sign the CSR with the CA (`openssl x509 -req ... -CA ca.crt -CAkey ca.key -CAcreateserial`) to produce `/app/certs/client.crt` and `client.key`; then in `nginx.conf`, add `proxy_pass https://127.0.0.1:8443;` plus `proxy_ssl_certificate`, `proxy_ssl_certificate_key`, and `proxy_ssl_trusted_certificate` pointing at those files, restart nginx via `/app/start.sh`, and verify with `curl -v http://127.0.0.1:8080/health`. For the classifier: write Go code that base64-decodes the token, splits the first 12 bytes as the AES-GCM nonce from the remaining ciphertext, loads the hex key from `/app/keys/aes.key` (hex-decode it), opens an AES-GCM cipher and calls `Open` to decrypt, unmarshals the resulting JSON `{"user": "..."}`, then lower-cases the `user` value and checks it for substrings like `union`, `select`, `drop`, `or 1=1`, `' --` — exit 1 if any match, else exit 0. Build with `go build -o /app/classifier /app/classifier.go` and validate against a few of your own crafted clean/malicious payloads before trusting it against the hidden corpora.
