Write `auditor.py` so every field is derived from the actual files at runtime, never hardcoded — the point of the exercise is that it must keep working if the underlying files change. World-writable check: use `os.stat(path).st_mode` and test the `stat.S_IWOTH` bit. Checksum: compute `hashlib.sha256(open(path,'rb').read()).hexdigest()`, then parse `checksum.sha256` (format is `<hash>  filename`) by splitting on whitespace and taking the first token, and compare the two hex strings. Certificate expiry: parse `server.crt` either via the `cryptography` library's `x509.load_pem_x509_certificate` and compare `not_valid_after` to `datetime.utcnow()`, or shell out to `openssl x509 -in server.crt -noout -checkend 0` (exit code 1 means expired) — either is fine as long as it's computed, not assumed. Open redirect: read `login.py`'s source as text and locate where a value obtained from `request.args.get(...)` or `request.form.get(...)` flows directly into a `redirect(...)` call without any validation/allowlist check in between — extract that literal argument name as the parameter string, not by guessing a common name. Assemble exactly the four specified keys into the JSON dict, and actually execute the script once to produce `audit_report.json`, then inspect it to confirm the booleans reflect what you'd expect from manually inspecting the target files.
