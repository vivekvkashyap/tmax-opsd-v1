Don't parse each log file independently — the core bug to avoid is treating `BEGIN_ARTIFACT`/`END_ARTIFACT` blocks as file-scoped. Read all `repo.log.*` files in alphabetical (lexical) sort order, concatenate their lines into one continuous stream, then run a single stateful line-by-line parser over that stream that tracks "am I currently inside an artifact's DATA section" and which artifact name it belongs to — this correctly reassembles records that were sliced across a file boundary by the naive line-count rotation. For each artifact, buffer the base64 lines, join and `base64.b64decode` once the full payload is collected, then write it out in fixed 524,288-byte chunks (the final chunk may be smaller — don't pad it), naming them `<artifact_name>.partNNN` with zero-padded 3-digit indices starting at 000. Compute `hashlib.sha256(chunk).hexdigest()` per chunk as you write it, and build the manifest dict grouped by artifact name mapping chunk filename to hash, then dump as JSON to `/home/user/manifest.json`. Verify chunk boundaries and hash values against the files you actually wrote to disk, not against in-memory buffers, to catch write/encoding bugs.
