Build this bottom-up and verify each layer before adding the next. First, run `ffprobe -v quiet -show_entries frame=pts_time -select_streams v -of csv=p=0 /app/data/surveillance.mp4` (or similar) to get one pts_time value per line, then pipe through `awk`/`sed` to join them into a comma-separated `const double frame_timestamps[] = { ... };` in `timestamps.h` — verify the array compiles and the count of entries matches the frame count from ffprobe. Second, write `backend.c` as a minimal blocking TCP server on 127.0.0.1:9000 that parses just the `GET /api/time?frame=N` request line (no need for a full HTTP parser), looks up `frame_timestamps[N]`, bounds-checks N against `sizeof(array)/sizeof(array[0])`, and prints the double with enough precision (e.g. `%.6f`) to match ffprobe's own output format. Test it directly with `curl` before touching nginx. Third, write `nginx.conf` with explicit `pid`, `error_log`, `client_body_temp_path` etc. all under `/tmp/nginx/` (create that dir first) since you lack root, add an `upstream` or simple `proxy_pass http://127.0.0.1:9000;` block for `location /api/time`, and a `listen 8443 ssl` server block using your self-signed cert. Start backend then nginx, and curl both endpoints to confirm identical JSON before finishing.
