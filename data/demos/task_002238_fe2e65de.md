For the hang: the loop condition `while (!feof(fp))` is a classic C bug — `feof()` only becomes true *after* a failed read, so the loop body executes one extra time on EOF with stale/garbage values, and combined with `fscanf` on a line containing the literal word `END` (not two numbers), the format specifiers `%f %d` will fail to match and consume no input, causing `fscanf` to keep failing forever without advancing the file position — an infinite loop. Fix by checking the return value of `fscanf` directly (`while (fscanf(fp, "%f %d", &current_time, &current_status) == 2) { ... }`) so the loop naturally terminates when it hits the `END` line. For the precision bug: the timestamps are stored as `float`, and epoch timestamps (~1.7 billion) have far more significant digits than a 32-bit float's ~7 digits of precision can represent, so subtracting two nearly-equal large floats loses the small fractional differences that matter — switch all the time/duration variables from `float` to `double` to fix this.
