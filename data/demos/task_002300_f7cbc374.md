Start by reading the Go source to see exactly which environment variables it checks and what values it requires (locale, timezone, a port derived from the VNC display) — the program will print a FATAL message telling you which variable is wrong or missing, so treat that as your spec. Build the binary with `go build -o /home/user/app/vnc-monitor /home/user/app/vnc-monitor.go` (or `go run` first to iterate quickly). Remember VNC display `:N` maps to TCP port `5900+N`, so display `:1` is port 5901 — verify a listener is actually up on that port before assuming the app can connect (test with `nc -zv 127.0.0.1 <port>` or similar). Write `/home/user/app/monitor.env` as plain `KEY=VALUE` lines (no quotes, no `export`) since the wrapper script parses it with `grep`/`xargs` into a stripped environment — an empty environment means only variables you explicitly define exist, so include everything the binary needs, not just the obvious ones. After creating the env file, run `/home/user/app/systemd_sim.sh` and inspect its output/exit code if it fails, then check `/home/user/app/status.log` for the success line the Go program writes on a successful connection.
