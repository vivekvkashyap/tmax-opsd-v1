Design the script around three separable concerns: locking, incremental reading, and record parsing. For locking, open `/home/user/backup.state` (create if missing, default offset 0), acquire `fcntl.flock(fd, fcntl.LOCK_EX)` before reading its integer offset, and hold the lock until after you've written the new offset back and flushed — this is what the verifier statically checks for, so use the real `fcntl`/`mmap` modules, not a workaround. For reading, `mmap.mmap()` the log file and slice from the stored offset to the current end-of-file to get only new bytes; be careful that mmap requires the file to be non-empty and opened in a mode that supports mmap (open in `r+b` or `rb` depending on whether you need write access). For parsing, iterate lines of the new chunk and use a regex anchored on `^\[\d{4}-\d{2}-\d{2} \d{2}:\d{2}:\d{2}\] \w+ ` to detect the start of a new record; any line not matching that pattern is a continuation of the current record's stack trace and must be appended to it. Only records whose LEVEL is exactly `ERROR` get appended (as complete multi-line blocks, exactly as they appeared, no re-formatting) to `/home/user/error_archive.log`. After processing, write the new total byte length back to the state file before releasing the lock — this ensures a second concurrent run only processes the delta.
