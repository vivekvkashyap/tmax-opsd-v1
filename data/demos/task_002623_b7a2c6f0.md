Build this as two cooperating pieces. `ci_backend.sh` is the per-connection handler websocat execs: it must loop `while read -r line; do ... done` reading requests from stdin and writing responses to stdout immediately (no buffering) so each reply reaches the client before the next request. For rate limiting, compare the current epoch time against the timestamp stored in `/tmp/last_build_time` (create it if absent, treat missing/old file as "allow"); only update that file when a request is *accepted*, and emit exactly `RATE_LIMIT` otherwise. Evaluate the arithmetic expression with `bc` or `$(( ))`/`bc <<< "$expr"` (careful: pure bash arithmetic won't handle all cases `bc` does, but plain `+`,`-`,`*` integer math works with either). After emitting `ACCEPTED <name> <priority>`, append the INIT log line, `sleep 1`, then append SUCCESS — do the config merge (override keys win, union of both keysets, alphabetically sorted `KEY=VALUE` lines written to `config_merged.env`) during/around that same window, e.g. via `sort -t= -k1,1 -u` techniques that let later (override) entries take precedence when merging two files. `start_server.sh` just apt-get installs `websocat`/`bc`, then launches `websocat -E ws-listen:127.0.0.1:9090 exec:/home/user/ci_backend.sh &` and exits successfully leaving it backgrounded.
