Use pandas throughout and build the pipeline in verifiable stages. Read the CSV with `pd.read_csv(..., quoting=csv.QUOTE_MINIMAL)` (pandas' default C/python parser already handles quoted embedded newlines correctly as long as the field is properly quoted — don't manually split on `\n`, which is the naive mistake that drops or corrupts multi-line `Notes` rows). Hash `PatientName` with `hashlib.sha256(name.encode()).hexdigest()[:8]`, then drop `Notes`. Parse `Timestamp` with `pd.to_datetime`, then per `PatientID` group, `set_index('Timestamp').resample('D')['HeartRate'].mean()` to get daily means, `.ffill(limit=1)` for single-day gaps only, then `.rolling(3, min_periods=1).mean()` for `RollingHR` — do this per-group (`groupby('PatientID').apply(...)` or a loop) so resampling doesn't mix patients. Find the max date across the whole resampled frame, filter to that day, then `groupby('Condition').apply(lambda g: g.sample(frac=0.5, random_state=42))` for stratified 50% sampling (frac=0.5 naturally rounds down on odd group sizes). Sort by `PatientID` and write the required columns to `/home/user/audit_sample.csv`. Separately write `run_pipeline.sh` (`#!/bin/bash\npython3 /home/user/pipeline.py`, `chmod +x`) and put the exact cron line for 03:15 daily into `/home/user/cron.txt`.
