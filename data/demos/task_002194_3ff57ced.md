Model this as a tree/DAG rollup, not a flat scan. First load `employees.csv` into an adjacency structure mapping each manager_id to its list of direct reports (children), and load `projects.csv` into a per-emp_id accumulator of own hours/cost (summing multiple rows for the same emp_id, e.g. an employee can appear in more than one project row). Then compute each employee's rolled-up total via a post-order DFS/recursion from the root(s): total(emp) = own(emp) + sum(total(child) for child in children(emp)). Watch for two common pitfalls: (1) an employee's own project rows may be split across multiple lines and must be summed, not overwritten; (2) CSV field parsing must handle the CEO's empty manager_id field correctly (empty string, not "0" or missing column) so the tree root is detected properly. After computing totals for every emp_id, sort numerically by emp_id (not lexicographically, since string sort would put "10" before "2") and write the CSV with the exact header `emp_id,total_hours,total_cost` and no spaces after commas. Compile with g++ and run the binary so summary.csv is actually produced before finishing.
