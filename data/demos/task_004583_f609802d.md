Design curate_artifacts.sh around a safe concurrency pattern: use `find /home/user/repo_raw -name '*.tar.gz'` piped into `xargs -P <n> -I{} process_one {}` (or background each `&` and `wait` at the end), where `process_one` is a function/subshell that does the per-archive work. Inside that function, verify integrity first with `tar -tzf "$archive" >/dev/null 2>&1`; on failure, append the path with a locked write: `flock /home/user/corrupt.list.lock -c "echo \"$archive\" >> /home/user/corrupt.list"` (or `flock -x 200 200>>lockfile` file-descriptor style) — use a dedicated lock file per output, since two processes appending unlocked can interleave lines. On success, locate `publish.log` in the same directory as the archive, and parse its multi-line `---`-delimited records to find the block whose `Archive:` line matches this archive's basename, extracting `Team:` and `Stage:` (awk/sed with a record separator of `---`, or a small loop reading line by line resetting state at each `---`, works well — don't assume awk's default RS handles this without setting `RS="---"` or an equivalent state machine). Append the CSV line under its own flock. End the script with `wait` for all background jobs, then run it and inspect both output files for the expected line counts.
