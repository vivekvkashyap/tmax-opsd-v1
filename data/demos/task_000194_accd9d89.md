Chain three classic debugging techniques, in order. First, automate `git bisect` in `/home/user/image_processor`: write a bash test script that, for each commit, cleans any stale `img_proc`/`mem_dump.bin`, builds with `gcc -o img_proc main.c`, runs `./img_proc`, and exits 0 if the run succeeded with no `mem_dump.bin` produced, or nonzero (crash/dump present) otherwise; then run `git bisect start v2.0 v1.0`, `git bisect run <script>` to let it locate the first bad commit automatically rather than manually checking out commits one by one. Second, at that first bad commit, rebuild and run the buggy binary under `strace -o /home/user/syscalls.log ./img_proc` (or redirect strace's stderr appropriately) capturing the full trace including the abort signal. Third, run `strings mem_dump.bin | grep 'CONFIG_STR:'` to locate the corrupted-filename line, then extract only the text after the `CONFIG_STR: ` prefix (e.g. with `sed`/`awk`/`cut`) into `/home/user/corrupted_filename.txt` — don't include the prefix itself. Finally record the bisected commit's full SHA (via `git rev-parse` or `git bisect log`) into `/home/user/bad_commit.txt`.
