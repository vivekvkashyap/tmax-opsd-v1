Build the pipeline as separate, testable stages in filter_pipeline.py. First, connect to the FTP server (e.g. Python `ftplib`) and list/download every file under /data/ into memory or a temp dir — don't assume a fixed filename list, enumerate what's there. For decoding, don't hardcode one charset: try a detection library (`chardet`/`charset_normalizer`) or attempt a prioritized list of codecs (utf-8, utf-16le, shift-jis, cp1252) with strict decoding and fall back through them until one succeeds, since files are deliberately mixed-encoding; the key pitfall is decoding a UTF-16LE file as UTF-8/Latin-1 and silently getting garbage instead of the real payload, which would let malicious content slip past your substring check. After decoding to a normalized UTF-8 string, do a case-insensitive substring check for `<script`, `DROP TABLE`, and `UNION SELECT`; reject on any match. For accepted files, write the UTF-8 bytes to /home/user/processed/<same filename>, POST the UTF-8 content as the raw body to the webhook, and for every file (accepted or rejected) push the specified JSON status line onto the Redis list `pipeline:results` via a Redis client (e.g. `redis-py`). Run the script so it processes the batch already sitting on the FTP server before finishing.
