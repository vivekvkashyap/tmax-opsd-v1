First install the toolchain (`openmpi-bin`, `libopenmpi-dev`, `libhdf5-openmpi-dev`, `hdf5-tools`) so the parallel compiler wrapper and `h5dump` exist. Open `/home/user/sim/integrate.c` and locate the adaptive-timestep branch that fires `when error > tolerance`: it currently multiplies `dt` by 2.0 (growing the step when the error is too large, which is backwards) — change that multiplier to 0.5 so the step shrinks on high error, leaving the other branch alone. Immediately after that dt-adjustment logic, and before `dt` is used to advance `t`/`val`, add an `MPI_Allreduce(&dt_local, &dt_global, 1, MPI_DOUBLE, MPI_MIN, MPI_COMM_WORLD)` (or reduce in place) so every rank adopts the same globally-minimal `dt` before stepping — this keeps ranks synchronized. Compile with the HDF5 parallel wrapper (`h5pcc` or `h5cc` depending on install, or `mpicc` linked against parallel HDF5) into `/home/user/sim/integrate`, run with `mpirun -np 4 /home/user/sim/integrate`, confirm `/home/user/sim/output.h5` is produced, then pipe `h5dump /home/user/sim/output.h5` output into `/home/user/result.txt`.
