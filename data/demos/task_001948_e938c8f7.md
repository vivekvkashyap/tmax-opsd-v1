Start with the audio: install a speech-to-text tool (e.g. `whisper.cpp`/`whisper` via pip, or ffmpeg + an STT CLI) and transcribe `/app/sensor_dictation.wav` yourself — don't assume the reading list, since it contains number-words to convert plus non-numeric tokens like "unknown"/"error" that must become dropped rows, not zeros. Write the parsed numeric values to a CSV with header `temperature`, keeping malformed entries out. In the C program, compute the sample mean and standard deviation of the raw values, then drop points more than 3 standard deviations from that mean (recompute mean/CI from the remaining data), and derive the 95% CI as `mean ± 1.96 * (stddev / sqrt(n))` using the cleaned sample's stats. For the TCP server, use a simple blocking `accept()`/`recv()` loop (fork or thread per connection is fine) parsing newline-terminated commands: reply to `GET_MEAN`/`GET_CI` with the exact formatted strings, and for the `AUTH <token> GET_OUTLIER_COUNT` command validate the token before responding — drop the connection silently (no response) if the token is missing/wrong or if the command comes without `AUTH` at all. Keep the server running as a background/daemon process so the verifier's socket tests can reach it.
