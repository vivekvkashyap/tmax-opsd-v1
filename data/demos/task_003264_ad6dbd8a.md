For video frame counting, don't trust container metadata (e.g. cv2's CAP_PROP_FRAME_COUNT can be wrong/estimated) — open the file and actually decode/iterate frames to get an exact count, or use ffprobe to count frames precisely. For the archive watcher, use `watchdog`'s `on_created`/`on_closed` handlers on `/app/incoming/`, but guard against reading a file mid-write: wait until the file size stabilizes or use `on_closed` (close-write) events before opening the tarball, otherwise you'll get truncated-archive errors. When extracting, verify the tar.gz integrity first (e.g. `tarfile.open` in a try/except, checking members), then for each `.wal` file recompute MD5 over all bytes up to (and including) the trailing newline before the `END ` line — split the file content precisely at the last newline preceding `END`, not by naive string splitting, since transaction lines may themselves contain arbitrary bytes. Only rename+move files that pass both magic-line and hash checks; discard everything else silently. Run the Flask/FastAPI app and the watchdog observer in the same process (e.g. observer in a background thread) so `/archived_wals` reflects live state, and make sure the HTTP server keeps running (no dev-server auto-reload exit).
