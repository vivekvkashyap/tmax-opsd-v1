Build this pipeline step by step and verify each stage before moving to the next. (1) Write `/home/user/generate_report.py` to load `cost` from both `/home/user/aws_cost.json` and `/home/user/gcp_cost.json`, sum them, and json.dump `{"total_cost": <sum>}` to `/home/user/reports/summary.json` — create the `reports` dir first (`os.makedirs(..., exist_ok=True)`) and run the script once to produce the file. (2) Only after the file exists, `chmod 400` it — if you chmod first and then have the script overwrite it, the write will fail, so get permission-setting last in your sequence. (3) Launch `python3 -m http.server 8080 --directory /home/user/reports &`, capture its PID via `$!` (or `pgrep`) into `/home/user/server.pid` — verify with `curl localhost:8080/summary.json` that it actually serves before moving on. (4) Use `socat TCP-LISTEN:9090,fork,reuseaddr TCP:127.0.0.1:8080 &` (or an equivalent forwarder) backgrounded, save its PID to `/home/user/tunnel.pid`, and confirm end-to-end with `curl http://127.0.0.1:9090/summary.json`. Double-check both PIDs correspond to still-running processes (`kill -0 $(cat ...)`) since a race between backgrounding and reading `$!` is a common source of wrong/stale PIDs.
