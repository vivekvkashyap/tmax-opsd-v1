Never call `ZipFile.extractall()` directly on untrusted input — that's exactly the zip-slip vulnerability this task tests. Open the archive with `zipfile.ZipFile`, iterate `namelist()` (or `infolist()`, skipping entries whose name ends in `/`), and for each entry name determine if it's malicious by checking whether it's an absolute path (`os.path.isabs(name)`) or whether `os.path.normpath(os.path.join(base, name))` — after resolving `..` segments — falls outside the base directory (compare with `os.path.commonpath` or string prefix check against the resolved base). Collect malicious names into a list, sort alphabetically, and write one per line to `malicious.log`, preserving the original (unresolved) archive path string exactly as it appeared in the zip. For safe entries, read the raw bytes via `zf.read(name)`, take `os.path.basename(name)`, write bytes to `/home/user/artifacts/raw/<basename>.safe`, then create a symlink at `/home/user/artifacts/links/<basename>` pointing to the absolute path of that `.safe` file (`os.symlink(target_abs_path, link_path)`). Process entries file-by-file rather than extracting the whole archive up front.
