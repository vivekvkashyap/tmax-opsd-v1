Structure the Go program around three concerns kept cleanly separate: safe traversal, concurrent processing with locking, and atomic rename. For traversal, use `filepath.WalkDir` (or `os.Lstat`) and check `d.Type()&os.ModeSymlink != 0` (or `info.Mode()&os.ModeSymlink`) to skip symlinks explicitly — never follow them — otherwise the self-referential `loop_link -> .` symlink in `/home/user/logs` will cause infinite recursion; `WalkDir` does not follow symlinks by default but you must still explicitly detect and `continue` past symlink entries so they aren't misidentified as regular `.part` files. Collect all `.part` file paths first, then fan them out to goroutines (e.g. via a `sync.WaitGroup` or worker pool). Inside each goroutine: read the file's full text, open `full_recovery.txt` for append, take an exclusive `syscall.Flock(fd, syscall.LOCK_EX)` on its file descriptor, write the content, `syscall.Flock(fd, syscall.LOCK_UN)`, close — this prevents interleaved writes from concurrent goroutines corrupting the merged file. Only after the write+unlock succeeds should that goroutine rename `foo.part` to `foo.done` via `os.Rename`. Leave `loop_link` untouched.
