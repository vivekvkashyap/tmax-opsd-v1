This is a classic train/test data-leak bug: the fix must change the *order of operations*, not just move a function call. Read `src/processor.rs` and note that `process_data` currently computes mean/std over the full array before slicing off the test portion — that leaks test-set statistics into normalization. Restructure so the split happens first (`let (train, test) = all_data.split_at(split_index)`), compute `mean`/`std` using only `calculate_mean`/`calculate_std` on `train`, then call `normalize` on `test` using those train-derived parameters, and sum the result. Keep the public function signature the same so the existing test in the file still compiles and exercises it.
