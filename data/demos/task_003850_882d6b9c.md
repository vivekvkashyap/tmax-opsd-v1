Avoid naive line-based reads — messages contain embedded newlines inside quoted CSV fields, so use Python's `csv` module (or `csv.DictReader` over a file handle) which correctly respects quoting and multi-line fields, and iterate row-by-row rather than `readlines()`/`splitlines()` to keep it streaming-friendly for large files. Parse each timestamp with `datetime.fromisoformat` (strip/handle the trailing 'Z'), then bucket it by flooring to the nearest 5-minute boundary (e.g. `minute - minute % 5`, seconds/microseconds zeroed) and tally ERROR-level rows per bucket in a dict keyed by bucket start. After the pass, find the earliest and latest bucket present, then generate every 5-minute bucket in between in order — even ones with zero errors — defaulting missing buckets to 0 (a common bug is only emitting buckets that appeared in the data). Sort buckets chronologically, then compute the 3-bucket trailing simple moving average per bucket (using only the current and up to 2 preceding buckets — fewer at the start), formatting bucket_start as ISO8601 with a trailing 'Z' and sma_3 to exactly two decimals with `f"{value:.2f}"`. Write the CSV with the exact header and run the script, then spot-check a couple of buckets by hand against the raw log to confirm the bucketing and SMA math.
