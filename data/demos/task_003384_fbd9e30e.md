Convert the audio first with ffmpeg specifying raw PCM float, mono, 16kHz explicitly: `ffmpeg -i /app/voice_sample.wav -f f32le -ar 16000 -ac 1 /home/user/sample.bin`. For the Rust extractor, load the 64x128 projection matrix from /app/projection_matrix.bin once at startup (8192 little-endian f32 values, row-major: element [r][c] at offset r*128+c). Process the input file in a streaming loop using a buffered reader, reading 64 f32 samples at a time into a fixed-size chunk buffer; on a short final read, zero-pad the remaining slots before proceeding — never load the whole file into memory since the spec requires it to scale to 100GB+ streams. For each chunk, compute the 1x128 output vector by, for each output column j, summing chunk[i] * matrix[i][j] over i=0..64; apply ReLU (max(0, x)) elementwise, average the 128 values, and write that single f32 (little-endian) to the output. Match floating-point operation order exactly (row-vector times matrix, standard left-to-right summation) since correctness is checked via bit-exact comparison against a hidden oracle across many fuzzed inputs including edge cases like empty files and non-multiple-of-4 byte lengths — handle any trailing partial-float bytes at EOF sensibly (e.g. by treating them as not forming a full sample). Build with `cargo build --release` and run against /home/user/sample.bin producing /home/user/sample_features.bin.
