Compile with `gcc -shared -fPIC -o libalgo.so algo.c` (run from `/home/user/polyglot`). In `wrapper.py`, the returned pointer is a heap-allocated `double*` of length `len-window+1`, so set `lib.moving_average.argtypes = [ctypes.POINTER(ctypes.c_double), ctypes.c_int, ctypes.c_int]` and `restype = ctypes.POINTER(ctypes.c_double)` — leaving `restype` at its ctypes default (`c_int`) is the classic bug that truncates the pointer and crashes or garbles the result on 64-bit systems. For the rate limiter, the flaw is that `last_req` is initialized to "now" before the first call, so the very first `validate_and_limit()` always looks like it's within the 1-second window; fix it by seeding `last_req` far enough in the past (e.g. `time.time() - 2`) or adding a "first call" flag, so the first request always passes and only subsequent rapid calls are limited. Run the fixed script and redirect its stdout with `python3 wrapper.py > /home/user/polyglot/success.txt` so the exact printed line is captured.
