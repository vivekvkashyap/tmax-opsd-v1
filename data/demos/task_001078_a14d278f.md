Debug in the order symptoms appear. (1) Build: run `make`, read the linker error, and check whether every needed `.c` file (e.g. a `signal.c` implementing header-declared functions) is listed in the Makefile's sources, and whether math calls like `roundf` require adding `-lm`. (2) Crash: run under `gdb --args ./sensor_proc input.csv output.bin`, and on segfault use `bt` to find the line; a malformed CSV line with too few fields makes a chained `strtok()` call return `NULL`, and using it unchecked crashes — add a NULL check after each `strtok` call and skip the malformed line. (3) Byte-diff: `xxd output.bin | diff - <(xxd expected.bin)` reveals extra padding bytes, meaning the compiler is auto-aligning a mixed-size struct; add `__attribute__((packed))` to that struct so its on-disk layout matches the expected binary exactly.
