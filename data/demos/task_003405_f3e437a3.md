Run `/app/startup.sh` first to bring up the TCP producer (port 9000) and HTTP aggregator (port 8080). Connect to `localhost:9000`, parse each `timestamp,sensor_id,temperature` line, and discard readings outside [-50, 150]. Data arrives out of order (up to 3s late) with gaps, so buffer readings in a small sliding window keyed by timestamp before finalizing any second — don't just process lines in arrival order. For each 1-second slot, if no reading arrived, forward-fill from the last valid observed temperature (never backward-fill/interpolate). Group the filled series into 10-second tumbling windows aligned to multiples of 10; if a window has no valid prior reading to seed forward-fill at its start, drop the whole window rather than emitting a partial one. For each surviving window compute the mean temperature and POST exactly `{"window_start": <int>, "avg_temp": <float>}` to `http://localhost:8080/submit`. Run the pipeline for 60+ seconds of stream time so 5+ windows accumulate in `/tmp/results.json`. Main risks: processing before sorting/buffering for out-of-order arrival, and emitting windows that should have been dropped by the quality gate.
