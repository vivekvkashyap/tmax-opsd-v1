Approach: (1) normalize both inputs to a common delimiter (e.g. `tr '|' '\t'` on demographics or `tr '\t' '|'` on reviews) so you can `join` them on `uid` — `join` requires both inputs sorted by the join key first, so `sort -t'|' -k1,1` (or equivalent) each side before joining, and skip/strip header lines before the join then re-add the header at the end. (2) Enforce schema with `awk`: test `age` is a strictly numeric integer (e.g. `age ~ /^[0-9]+$/`) and `>=18`, `country` non-empty, `score` numeric and `>3`. (3) Tokenize `review_text` with a chain of `tr`/`sed`: lowercase via `tr '[:upper:]' '[:lower:]'`, strip everything except `[a-z0-9 ]` via `tr -cd` or `sed 's/[^a-z0-9 ]//g'`, collapse runs of spaces to a single dash, then trim leading/trailing dashes with `sed 's/^-*//; s/-*$//'` — do this after lowercasing/filtering so punctuation removal doesn't leave stray double-dashes. (4) Drop rows with empty tokenized text post-cleaning. (5) Sort final rows numerically by `uid` (`sort -t, -k1,1n`) and prepend the exact header.
