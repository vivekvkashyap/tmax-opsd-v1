Two separable subproblems: locate the anomaly frame, then build a fast graph algorithm. For the video, extract frames (e.g. `ffmpeg -i audit_dashboard.mp4 -vf select='...' frames/%04d.png` or sample every frame) and scan pixel color per frame — a pure-red frame (255,0,0) stands out from the normal green; convert the frame index to seconds using the video's fps (check with `ffprobe`) to get a timestamp to 2 decimals. For the C program, do not do naive O(n²) scans per query: read the CSV once, map tx_id/resource_id strings to small integers via your own hash table (open addressing or chaining, POSIX only), maintain per-resource "current holder" and per-transaction "waiting-for resource" state as you stream events up to max_timestamp, then build a WFG adjacency list and run iterative or recursive DFS with a visited/on-stack array to find the cycle — this keeps everything linear/near-linear so it scales to 5M+ rows within 1.5s. Sort discovered tx_ids ascending before printing, comma-separated, no trailing whitespace issues. Compile with `-O3` as the harness will.
