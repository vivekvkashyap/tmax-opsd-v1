Treat this as a pipeline with an easy first stage and a harder second stage. First, transcribe `/app/config_dictation.wav` (e.g. `pip install openai-whisper` and run it, or another available speech-to-text tool) and inspect the raw transcript text — transcription isn't perfect, so expect minor wording/punctuation variance around numbers and IPs; write your regex to be tolerant of that. Second, in the C program, use POSIX `regex.h` to match sentences describing an add/remove action with an IP (four dot-separated integer groups) and a port (integer), splitting the transcript into independent chunks for pthreads to process in parallel — but be careful that a rule spanning a chunk boundary isn't dropped, and that thread output ordering doesn't matter since you replay results in original sequence into the hash table afterward. The hash table must apply actions in dictation order: "Add" inserts/overwrites the IP+port entry, "Remove" deletes it if present (no-op otherwise); the final state is whatever remains. Sort final entries alphabetically by IP string then numerically by port, and write exactly `IP,Port` per line. Since grading is Jaccard similarity (>=0.90) against the true final state, prioritize correctly parsing action/IP/port over any other detail — a single misparsed rule can tank the score.
