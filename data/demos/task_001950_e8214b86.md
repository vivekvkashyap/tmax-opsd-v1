Diagnose the 502 by checking what Flask is actually listening on (`curl 127.0.0.1:5000/health` after starting it manually) versus what `proxy_pass` in `nginx.conf` points to — a 502 from nginx almost always means the upstream address/port in `proxy_pass` doesn't match where the backend really binds, so compare them line by line rather than assuming the config's global structure is broken. Separately, actually try running `python3 /app/app.py` and read the traceback carefully — a subtle typo in a kwarg (e.g., a misspelled `port=` argument to `app.run`) will raise a `TypeError` at call time, not a syntax error, so don't just eyeball the file. For `monitor.py`, use `subprocess.Popen` to launch both processes, keep references to the `Popen` objects, and in a loop: `proc.poll()` to detect unexpected exit and restart, plus a separate timed health check using `urllib.request.urlopen('http://127.0.0.1:8080/health', timeout=...)` — on exception or non-200, kill and respawn only the backend process (not nginx). Run monitor.py with `&` so both services persist after your session ends.
