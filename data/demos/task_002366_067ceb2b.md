Build and test each Go component standalone before combining. For the egress proxy: implement a raw `net.Listen("tcp", "127.0.0.1:8080")` HTTP server (or use `net/http` with a handler) that reads the incoming request's `Host` header exactly, and only when it equals `allowed-registry.local` does it proxy/forward the request (e.g. via `httputil.ReverseProxy` or a manual dial to `127.0.0.1:9090` and copy of the request/response) — for every other Host, write a 403 status with body `BLOCKED` and return immediately, don't attempt to connect anywhere else. For the CI runner: use `fsnotify` or a polling loop (`filepath.Glob`/`os.ReadDir` on an interval) watching the commits directory for `*.tar.gz`; on detection, extract with `archive/tar` + `compress/gzip` into a per-job workspace dir, read the first line of `ci.txt`, and launch it via `exec.Command("sh", "-c", cmd)` with `cmd.Env` set to the current environment plus `http_proxy=http://127.0.0.1:8080` appended (not just set, since you need the rest of the env too) — write the PID immediately after `cmd.Start()` (before `Wait()`), then block on `Wait()` and branch on exit status for SUCCESS/FAILED, finally removing the tarball. Test end-to-end with the sample job that curls both an allowed and blocked host to confirm the proxy logic actually gates traffic through the runner's injected env var.
