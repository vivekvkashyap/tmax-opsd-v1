Work through this as a pipeline of small, independently-testable steps rather than one big script. First create the output directory. Then write the C program: open the log file, and for each "Failed password" line extract the IP token that follows "from" (split on whitespace is sufficient — don't overengineer with regex in C). Maintain a simple growable table (array of struct{char ip[16]; int count;}) keyed by IP string, incrementing counts as you scan. After the full pass, iterate the table and write only IPs with count strictly greater than 3 to blocked_ips.txt, one per line — get the boundary right (`> 3`, not `>= 3`). Compile with gcc and actually run the binary before moving on; don't hand-write blocked_ips.txt. Next, read back blocked_ips.txt line by line to generate block.sh, emitting one `iptables -A INPUT -s <IP> -j DROP` line per IP in the exact literal format requested. Finally, copy sshd_config_old to sshd_config_secure and edit it: change any existing `PermitRootLogin`/`PasswordAuthentication` lines to `no` (or comment out `yes` variants and append fresh `no` lines) so no contradictory `yes` directive remains — a naive append without removing the old line will fail verification.
